# Claude 套餐，将不再支持自动化任务

$$$

  

Anthropic 变动了下 Claude 的套餐：**凡是能被脚本和程序放大的用量，从 6 月 15 号开始，不再从月费套餐里扣**

套餐内

不变

claude.ai 聊天、Claude Code 终端交互、Cowork，人坐在屏幕前用的场景，用量池子大小不变

拆出来的

独立月度 credit

`claude -p` 命令、Agent SDK、第三方 SDK 应用，能被脚本和 CI 自动调用的场景，走单独额度

当然了，作为套餐变更的补偿，Claude 的付费账户，每个月都可以手动的去领取一笔 Credit 作为补贴

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdgvO7kVJ4ZGvxFkia9ox4QWE5wBia99OU3Q3rrBaR6Aibcdicd6vhLy5tj9JNmDIbLoHocQkIMfoA6u7ACibqqogYuG6AJtudPicoY40/640?wx_fmt=png&from=appmsg)

注意：需要手动领一次，之后每月自动刷新，未用完的不滚存，credit 用完后走 extra usage 按 API 费率继续，或者等下月刷新  

所领到的金额，等于所支付的月费：Pro 拿 $20，Max 5x 拿 $100，Max 20x 拿 $200，Team Standard 拿 $20，Team Premium 拿 $100

  

哪些人会受影响
-------

其实，对于绝大多数的用户，并不会感受到变化....毕竟绝大多数人并不知道 \-p 指令是啥，hhhhhhhh～～ 受影响的是两种用法，分别来说：

第一类用了 `claude -p`：Claude Code 的非交互模式。就是给它一句 prompt，跑完直接输出结果，进程退出。典型场景是 CI 流水线里塞一行 `git diff | claude -p "review"`，每次提交自动跑代码审查。这个 flag 从 2025 年 2 月 Claude Code 发布第一天就有了，但大部分人不知道

第二类用了 Agent SDK，2025 年 9 月随 Sonnet 4.5 发布的 Python / TypeScript 库，把 Claude Code 的能力拆成零件给开发者编程用。开发者可以用它写一个 agent 循环跑几个小时，让 token 消耗远超古法手操 Agent

而对于其他的使用，比如常规的 Claude Code、Cowork、WebApp、以及其他 IDE，并不影响，大多数人是无感的

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdgFBTdU6IpqVqSuwGUicMqwWz2ia9aYI2eHXW5fnyOH6hUO0dZ83BHrJFzicILA2TuBc23LFHPQgAhuD3RThMVLXTGW0iaicLDmHEXE/640?wx_fmt=png&from=appmsg)

  

为什么要拆
-----

说白了，还是为了限制部分超级用户，对 token 的超预期使用

对于大多数人来说，坐在终端前使用，token 消耗有天花板。但 `claude -p` 塞进一个 for 循环，或者 Agent SDK 写一个 overnight agent，跑一夜的 token 消耗可以是人类交互的几十倍

很显然，Anthropic 最开始没有预料到 Agent 能发展到现在这个程度，固定月费的套餐池子就要靠不住了，只能对重度用户进行限制

还有个事儿值得说一下，之前 Claude 套餐限制了 OpenClaw 这类第三方应用的用量，逻辑是一样的

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdgicVzkACS5vLLOyuXyu0YxY8Rm5HFCKVJUibpWnW5YkWVJfMPKCdLc3kjic6P8f0hw33iaA7Q2Qa0q8X0H4TBb1I0Q9qOLf8oroHs/640?wx_fmt=png&from=appmsg)

对于「可被程序放大」的用量，固定月费兜不住，当时也是给用户发了一笔 Credit 作为过渡  

这种套餐切割的方法，总结起来是这样：**用户交互部分归套餐，自动化任务部分按量结算**

  

充值这件事的变化
--------

我们来回顾下 Anthropic 的收费结构

与 OpenAI 不同，Anthropic 的打法更偏向于开发者，最早这东西都是在 console.anthropic.com 这个 MaaS 平台进行服务的（甚至有中文界面），在这里，开发者需要额外注册账户、充值然后，按 token 计费

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdhrG3d6HzQvyFaUu55icHl4R2QalXNeAn2AwYlVlUKRKIAZHg3dAUxSVVvyyx3D1mSibadia4icSicNOaUNcjSGic9qS6KE94iaenCYk4/640?wx_fmt=png&from=appmsg)

而最早的 claude.ai 则是按月订阅，20美金/月，是一个有 Artifacts 的 WebAp，并不需要知道 API 是什么

  

之后的 Claude Code 出来之后，这些东西就发生了改变：既可以用 API 账户登陆，也可以用 claude.ai 的订阅账户来登陆，登陆的时候就像是下面这个...为了截图这个，我还把账户给 logout 了

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdgP6bHCO3Wt3ib7voeSf54vWHqZ5GTS8FJNY4PjgHrhcuNhWh5SibxX1yibfvxtbYs5fbZeogT0UHbibEaDYwJQYuqGKoNoOA3RId0/640?wx_fmt=png&from=appmsg)

于是，大量个人开发者使用自己的 claude.ai 账户，用 Claude Code 写东西，套餐用量不够了，需要一个按量付费的出口。于是 Anthropic 在 claude.ai 设置里加了 Extra Usage 功能，开关一开，套餐耗尽后自动按 API 费率继续，扣的是你在 claude.ai 里充的余额

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdjibicTyU2zOicViccsnmicWrf6O6jOPxZu9gUaEFXNAJ7bHH8tEiaich6ENz7BmVwwMgIianveBNTzkUKicdbqvvVEykjiaXMg9p9qRo6Sw/640?wx_fmt=png&from=appmsg)

现在这个功能，最高有 **30%** 的折扣，而且直接在 claude.ai 里充反而更方便，不用切换平台，用量监控、充值、限额设置这些功能都有

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdjcOWORyFibzy5vwlIwXhVU3r7hWE3dqVfOk4Nn5ut7duibthkD8LVwib5LrytcADItbMaFibmjTkcTEoUwNX9LecLsE3J6jNkGkMw/640?wx_fmt=png&from=appmsg)

6 月 15 号之后，Agent SDK 和 `claude -p` 的用量也会流进这个 Extra Usage 池子（credit 用完之后）。claude.ai 正在从一个聊天界面变成一个完整的消费平台

收费结构演化：

Claude Code 发布

2025.02

API 平台和 claude.ai 完全分开，各管各的

Agent SDK 随 Sonnet 4.5 发布

2025.09

Claude Code 的能力拆成库，自动化场景出现

第三方应用踢出套餐

2026.04

OpenClaw 等应用不再走订阅用量，用户拿到等额一次性 credit

Extra Usage 上线折扣

2026.05

claude.ai 设置里直接充值，打七折

Agent SDK / claude -p 拆出套餐

2026.06.15

独立月度 credit，按人头发，交互归套餐，自动化按量

  

最后
--

今天看到了很多新闻稿在谈论 Claude 的这个计费规则变更，甚至有说什么 Claude 套餐不能在 Claude Code 里面用了....反正就很离谱，但其实这事儿挺简单的：之前的计费方式确实不太可持续，没考虑到 Agent 到长程规模化调用，改一下  

其实这东西，对只用 claude 进行简易开发的人来说，其实啥也没变  

对能被脚本和程序放大的用量，从 6 月 15 号开始，走单独的计费
