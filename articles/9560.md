# Anthropic掐断“龙虾”补贴，OpenClaw创始人哭求仅换7天续命，网友吵疯：作秀还是义务

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/hLLZnAbUwNiax9B9dicdyKHDxTxPWewqGzIPBGok7erA4InsibyVy2I3icpcj4Bb4RDDn15grMaDwbfoaVH1hKO0qlMZMKjcb8KfQrCj42BfV3E/640?wx_fmt=jpeg&from=appmsg)

![](https://mmbiz.qpic.cn/mmbiz_gif/YriaiaJPb26VPQqHC66RJFpttVIMWG83T3lWHahUD4bvhxlKSayjeV2ibvC5ydqklP9QHDPD3qHJM07TV3IfHstjA/640?wx_fmt=gif)

整理｜华卫

昨晚，Anthropic 突然宣布，为更好地管理资源，自太平洋时间 4 月 4 日中午 12 点起，Claude 订阅额度将不再涵盖 OpenClaw 等第三方工具的使用。

由于 Anthropic 的新政策变更，OpenClaw 与 Claude AI 结合使用的成本或将大幅上涨。

1 不是“封杀”，就是不给“补贴”了

根据 Anthropic 发送给用户的邮件，其仍然可以使用 Claude 账户运行这些工具，但今后如果用户想要将 OpenClaw 与 Claude 结合使用，则必须选择“按需付费”选项，费用将根据您的实际使用量单独收取，不包含在您的订阅费用中。

也就是说，Claude 用户还可以用 Openclaw ，但是需要支付非补贴的 API 价格了。

邮件中强调，“订阅仍然涵盖所有 Claude 产品，包括 Claude Code 和 Claude Cowork。要继续使用第三方框架并通过 Claude 登录，请为您的帐户启用额外使用额度。此政策将于 4 月 4 日率先对 OpenClaw 生效，但适用于所有第三方框架，并将很快推广至更多框架。”

至于执行此举的原因，Anthropic 在邮件中的说法是：“我们一直在努力全面管理需求，但这些工具给我们的系统带来了过大的压力。产能是我们谨慎管理的资源，我们需要优先保障使用我们核心产品的客户。”

之后，Anthropic Claude Code 负责人 Boris Cherny 也第一时间在 X 上解释了这一通知的情况。他表示，“我们一直在努力满足用户对 Claude 日益增长的需求，但我们的订阅服务并非为应对这些第三方工具的使用模式而设计。容量是一种我们谨慎管理的资源，我们将优先满足使用我们产品和 API 的客户的需求。”

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNhcuJickFGYQHUHkgibaMvU95oHTU9OrQtelRHmriaibmqJxX7YStxqiaaSrU0MTV7eqWZQVJTrcMvZMKyBAhp5H9O34iaGicDwiaILPxo/640?wx_fmt=jpeg&from=appmsg)

并且，Cherny 称，在这些调整之后，Anthropic 将向已订阅用户提供“相当于其月费的一次性补偿”。具体如下：订阅用户将获得一次性额外抵扣，金额相当于当月的套餐费用，需在 4 月 17 日前兑换抵扣额度。如果需要更多抵扣额度，现在可以购买折扣流量包（折扣优惠最高可达 30%）。如需申请全额退款，明日查看电子邮件，邮件中将包含退款链接。

他最后说道，“我们希望有意识地管理增长，以便能够长期可持续地服务客户。此次变革正是朝着这个方向迈出的重要一步。”

2 被骂“反对开源创新”，Claude Code 负责人喊冤

据外媒报道，由于 OpenClaw 的创始人 Peter Steinberger 现在就职于 OpenAl，Anthropic 也可能在鼓励用户更多地使用其自身的工具，比如说 Claude Cowork。

而此事官宣后，OpenClaw 创始人 Peter Steinberger 在 X 上透露，“我和 OpenClaw 董事会成员 Dave Morin 曾尽力去说服 Anthropic，但最终只成功将此事推迟了一周。”同时，他吐槽道，“时间点真是巧合，他们先是将一些热门功能复制到他们的封闭框架中，然后又将开源软件拒之门外。”

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/hLLZnAbUwNiavbpCia3lHs8DHcicxia2GqASMs0Aia00Fk35vd9dPKssxrWuWdmAyLSOWxfUQ99rFlaqwqpwY5tRcZjYCveTHg2IxT9Od1Xsn9tE/640?wx_fmt=jpeg&from=appmsg)

不少网友评价道，“Anthropic 的这一举动实在令人失望，在泄露了 Claude Code 的所有源代码后，又自毁前程地疏远了开源爱好者群体。”

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNhsqETfmsp0HC4mAfolvZN5FNV0zUMvcpE50SOOhPdu3bia2YyZ0UBY9icRT6c6VFsWVy9zD2icWJwue9HPk0sOnw3pfvEqM5BD90/640?wx_fmt=jpeg&from=appmsg)

也有网友在 Cherny 的帖子质问，“真是糟糕的决定。你们为什么如此反对有史以来最伟大的开源创新？我会取消订阅。其他模型的功能也越来越强大了。”

对此，Cherny 回应称，“我们非常支持开源。事实上，我刚刚提交了一些 PR，专门用于提升 OpenClaw 的提示缓存效率。这更多是出于工程方面的限制。我们的系统针对特定类型的工作负载进行了高度优化，为了尽可能多地为用户提供最智能的模型，我们正在持续优化。即使您使用 API 密钥或超出套餐限额，系统应该仍然可以正常运行。问题仅出现在订阅服务上。如果您仍然想要取消订阅，我们将全额退款。我们知道并非所有人都意识到我们不支持这种做法，因此我们尝试明确说明这一点。”

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/hLLZnAbUwNgz5qy5pZf8hMhXX7HG7JyPzotGibeJUicQIvhEWJJEV8bd4jViarcdrKlibxEl6MRlI0BPWG5h4fA7wARXUe3nHCiauDfaiaS99d59Y/640?wx_fmt=jpeg&from=appmsg)

3 网友吵疯了：完美政策还是推广义务?

在 Steinberger 的帖文下方，却呈现出另一派讨论景象。有人认为，Anthropic“太蠢了”，在一众人工智能公司中，它有机会凭此成为新的“平台”，让其他所有东西都依赖于它运行。但 Anthropic 的做法却好像在表明：“我们无意中获得了太多的需求。我们不想搭上这艘火箭，而是选择离开高速公路。”

而另一拨网友纷纷说道，“Anthropic 主要面向商业用途。他们不关心开源，也没有义务这样做。”“运行一块显卡成本太高了。”

还有人说，“Anthropic 确实高估了他们的服务器容量，而这就是他们的回应，简单明了。他们没想到玩家会疯狂地用光所有 token，然后开设两三个甚至更多账户。这是一次容量上的失误，现在是时候承担后果了。”

亦有网友直接指出，“这其实早就是既定政策了。无论如何，产能不足必然会导致需求下降，而提高垂直整合度 / 一体化程度才是正确的举措。事实上这步棋走得非常完美，尽管人们对此感到愤怒也是可以理解的。”

这波网友持有的看法是，“这并不奇怪，也跟开源无关。问题就出在经济效益上。Claude Code 的订阅模式是为 Anthropic 生态系统量身打造的，定价也完全符合其自身定位。任何超出这个生态系统的功能都应该按 token 消耗来使用。任何不这么做的公司都是在弄虚作假。这些订阅模式已经获得了大量补贴，定价也旨在使每个用户的计算使用量分布达到一个合理的中位数。允许第三方支持固定价格订阅模式，不过是作秀罢了。”

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNhWVBldZYwPqdA3FRnOmtYiajSUuL9k7S0Y50DR7f3yx62ZfVW3vdPuRNUxG9nXoOtwhXXybJ07gKlauRicMj03AibYK0WLwMN5ick/640?wx_fmt=jpeg&from=appmsg)

Steinberger 专门回复了这条评论，他表示，“有趣的是，对于人工智能行业的其他任何一家公司来说，情况都一样，我们支持来自 MiniMax、阿里巴巴、OpenCode、GLM、OpenAI 等公司的订阅……”

而该网友进一步说道，“OpenAI 认为这对他们来说是很好的品牌推广方式。他们的共同点是：他们愿意以营销成本的形式大幅补贴你的用户。你们的产品很棒。如果我是 Minimax，我也会利用这个渠道推广我的订阅系统。但 Anthropic 完全没必要这么做。”
