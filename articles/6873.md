# AI 一键生成网站之后，最残酷的差距出现了：代码不再值钱，审美决定生死

![](https://mmbiz.qpic.cn/mmbiz_jpg/lYDricib2fsm5A5oepfeQWHz0Fx8PVZxVxhBXXtPtK6puuUFBpxjQDAAwvS81I1UsNq8FZzDMGVteKXm8t48CLpoTmVZaVbn233CJ16kdb76w/640?wx_fmt=jpeg)

作者 | RedMonk

译者 | 平川

策划 | Tina

去年底，一个网站在开发者和设计圈同时刷屏——F1 车手 Lando Norris 的个人官网上线了。不是因为内容，而是因为体验：层层叠加的 3D 动画、会“呼吸”的页面结构、随着滚动实时变化的头盔模型，以及几乎无处不在的微交互细节。

这不是那种“看起来不错”的网站，而是那种你会忍不住停下来、反复探索、甚至去研究它到底是怎么做出来的网站。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNia711yUm8LM9KyBAhTsbB2ct9qZ0CnLkaib9X73ImVY20XulIiajtZTQ8drfUF32KQ16ShPLib838XEuicicubd6M37xnhzCN8gGQJU/640?wx_fmt=jpeg&from=appmsg&tp=wxpic&wxfrom=5&wx_lazy=1#imgIndex=1)

Syntax 联合主持人、知名前端开发者 Wes Bos 也被吸引了进去。他做了一件典型的“极客操作”：直接把这个网站的 7.5 万行代码全部下载下来，一行一行地翻。结果越看越震惊——这个网站不仅仅是炫技，而是一个被精心打磨到极致的作品。

这个网站背后是多层 3D 渲染、深度图驱动的视差效果、shader 实现的动态遮罩、基于 glTF 的实时头盔渲染，以及一套主要依赖 transform 属性 的性能优化动画体系。更关键的是：这些复杂技术，并不是为了“展示技术”，而是为了服务一种非常明确的目标——让用户停下来，感受这个品牌。

Wes Bos 直言，这个网站最重要的不是技术，而是“心思”。从一个简单的文字 hover 动效，到头盔反射光的细微差异，再到页面切换时的“帷幕揭开”动画，每一个细节都明显经过反复打磨。“仅仅是一个文字悬停效果，可能就要花上一两个小时。”而这样的细节，构成了整个网站的全部。

也正是因为这种“过度用心”，这个网站产生了一个在 AI 时代越来越稀缺的效果：**你一眼就知道，这不是模板做的。**

而这，正是 Wes Bos 做出一个大胆判断的起点： 2026，将是“设计复兴之年”。

在他看来，当 UI 框架、组件库、甚至 AI 已经可以让任何人“快速生成一个还不错的网站”之后，真正的差异化不再来自“能不能做出来”，而是来自：**你有没有审美、有没有取舍、有没有能力做出让人记住的体验。**

**今年初，Wes Bos 在一期播客中进一步阐释了这个判断，以下是播客整理：**

**设计复兴：为什么**

**2026 年将是设计之年**

**Kate Holterhoff：欢迎来到 MonkCast。今天和我一起的是 Syntax 联合主持人 Wes Bos。Wes 是一位成就卓著的开发者和教育者，我非常高兴能邀请他来我们的播客。**

**现在是一月，所有的思想领袖都在发表他们的预测。你和 Scott Tolinski 做了一个预测，谈到了 CVE、过去一年中出现的一些 React 问题、RPC。其中我最感兴趣的是设计方面。你主张说设计正在复兴。我想听听更多这方面的内容。为什么这是你的第一个预测？**

**Wes Bos：** 好的。我觉得，设计领域这几年确实发生了转变。在早期，设计至关重要，因为当时涌现的许多网站和应用根本无法正常使用。设计能确保应用的可用性与友好性，让用户获得最佳的体验。同时在品牌建设层面，它也关乎你在互联网上的形象塑造。

过了几年，出现了很多 UI 框架，很多人很轻松地就搭建出了非常好看的网站。现在，我们有了 AI ，它们能够生成代码，它们正在创建可用性好而且看起来非常不错的网站。我认为这非常重要。而现在，我们差不多已经形成了这样一个基础框架，只需付出很少的努力，就能让其具备一定的可用性。通过 NPM 安装，或者引入一些 Tailwind 组件并稍作调整。看起来还算不错，相当易于使用，这很棒，因为所有内容的基础框架都相当不错了。

但我们现在面临的一个问题是，你的差异化在哪里？如果很容易就能得到一个可行的基础框架，那么我们现在该如何进行探索，这有点两面性，对吧？在可用性方面，现在它不那么难了，不用再那么费力地去做一些基本的事情，我认为我们有更多的时间可以花在风格化上了。

比如，我可能花大量的时间琢磨：这个照片选择器的上传界面该怎么设计？或者这个下拉筛选器怎么实现多选功能？过去，这类功能可能需要耗费巨大的精力，但现在变得更容易实现了。所以我开始注意到，那些更好的产品开始把更的时间投入到实际产品的 UX 可用性设计上。就像几个月前上线的网站 Lando Norris ，令人瞠目结舌。整个网站充斥着前沿的 Web 技术，那些 3D 效果堪称前所未见。

这就是它的差异化，对吧？如果 Web 上的一切从一开始就看起来完全一样，那么我可能会觉得一切都看起来完全一样也没有什么太大的问题，对吧？就像你打开 Twitter，打开 Threads，打开 Instagram。有时我甚至忘了正在使用哪个应用程序，因为它们看起来完全一样。我认为很多人喜欢这样，因为那让你觉得很熟悉，知道怎么用。

但是，从一个品牌的角度来看，我们需要有一个引人注目的亮点。你需要让人们停下滑动滚轮的手指。你需要让人们访问这个网站，体验并享受它。我认为这是人们能够彰显自身特色的一个重要方式，那就是那些精美的新一代网站。而且我觉得，我们已经开始看到越来越多这样的情况。

**Kate Holterhoff：是的，真的非常很感谢你的介绍。对于 RedMonk 的网站，我写过一些讨论 Shadcn 问题的内容，以及有很多 AI 工具让你可以快速搭建一个网站，非常棒。**

**甚至 UI 组件库都在底层使用 Shadcn。因为这个库允许让你截取代码片段，这简直是复制粘贴模式的复仇。模型可以直接调用这些片段，实现轻松导入。但我怀疑背后还有更深层的原因，绝不仅仅是 Shadcn 崛起及其成为那些库的底层技术那么简单。**

**工具让我们在很多不同的网站上都能拥有基本的 UI 体验。比如，你提到了 Syntax 网站上的 Raycast。我想问的是，除了 Shadcn 之外，是否还有其他我忽略的因素，导致这些网站逐渐趋于同质化？**

**Wes Bos：** 嗯，我认为这是开发人员不想做出改变。我们已经看到，这不是 Shadcn 出现后才有的新问题，对吧？Bootstrap 有这个问题，Foundation 有，jQuery UI 也有。很多 Tailwind 相关的东西都有，这就是为什么很多 AI 生成的东西都是紫色的。就是说，很多开发人员不一定有设计眼光。

对于设计，你做了一些事情，却把它弄得更糟了，看起来不好看。直接把东西塞进去反而简单得多。有趣的是，我们用于 UI 设计的这些工具本都是可以改变的。它们的设计初衷就是提供基础框架，并允许用户进行调整。但几乎没人愿意去对它们做微调。

有太多的网站，你几乎可以一眼看出它们使用了什么。所以回答你的问题，还有没有其他类似的事情？我认为肯定有，比如人们没有那么多时间花在这类事情上。如果能套用一个 UI，实际上看起来还相当不错，比什么开发人员自己拼凑的东西要好得多，他们就不会去做什么微调了。所以我认为这个原因很重要。

**Kate Holterhoff：** 是的，有道理。Gergely Orosz 的最新通讯已经发布，实际上谈论的就是关于设计系统。我喜欢他在其中使用的例子，如果你不用类似的东西，那么你的应用程序会看起来像一个大学的毕业设计项目。如果整个网站没有统一的设计风格，无法满足用户预期，就会产生某种违和感。

**Wes Bos：** 是的。这完全是拼凑。在播客里，我们常讨论腐化或漂移问题。如果没有设计体系之类的规范支撑，一念之间，你觉得“这个看起来不错，放这里应该也行”。然后不知不觉间，两三年过去，界面开始漂移变形，逐渐腐化。最终只剩下一团乱麻般的混搭风格。

**Kate Holterhoff：是的，真是这样。随着我们迈入 AI 时代，应用程序似乎正在以这种临时拼凑的方式被创建出来，无论未来会出现什么，这种趋势恐怕只会越来越明显。很有意思。你提到了 Lando Norris 的网站。你能不能简单说下：为什么这个网站能带来如此酷炫的浏览体验？**

**Wes Bos：** 好的，为了展示所有需要传达的信息，他们用了很多有趣的方式，对吧？比如，他们有一个图库展示了所有的头盔，当鼠标悬停在某个头盔上时，头盔周围的轮廓就会动态变化。赛道采用 3D 模型呈现，可以 360 度旋转查看。这种沉浸式体验让人忍不住想留在网站上，继续探索下去，浏览所有不同的小细节。他们完全可以只做一个非常基础的网站，但那不是 Lando Norris 的品牌，对吧？他要做到绝对的最好，他做的一切都是其品牌的延伸，对吧？

每一个小细节，从小小的卡片悬停效果、贝塞尔曲线、缓动效果、边缘处理，到鼠标悬停在链接上时文字呈现的精妙特效。所有这些细节都花费了大量的时间。仅是文字悬停特效这一项，很可能就需要花至少一两个小时反复打磨。没错。而这仅仅是整体的百分之一。显然，他们倾注了大量心血。

他们在这个网站上投入了很多心血，每个人都在谈论它。这很管用，效果立竿见影。说实话，我根本不看赛车比赛，压根不知道这家伙是谁。结果大家都在说“哦，这是 Lando Norris 啊！”他可是超级巨星，名气大得不得了。现在，不知道他是什么人那才叫傻瓜呢。但这恰恰说明他的个人品牌已经超越了他的圈子，传播得更远。现在，我知道他是谁了。前几天，我在 Costco 看到了他的玩具。我在电视上听了关于他的消息。显然，这起作用了，对吧？

**Kate Holterhoff：是的，这很有趣，我很好奇。你提到，这个网站在互联网上引起了很大的反响。我确实看到了一些东西。你是怎么追踪优秀设计趋势的？我知道有电子报之类的渠道，但就像这个网站，你是怎么发现的？**

**Wes Bos：** 我时刻关注设计圈的推特动态，关注最新动向。我发现，只要多参与设计圈的推特互动，算法就会不断推送相关的内容。大家经常在推文里 @我。对于观众来说，这或许不算个好建议。

还有一些奖项。我知道 Framer 要推出奖项。有几个网站有网页设计奖项。这在 CSS 圈曾经是件大事。但后来就变成了这样：放张人穿耐克鞋的照片，上面叠个按钮，然后就说这是个漂亮的网站。很长一段时间，都是大照片配按钮的套路。不过，计算机越来越好。如今人们正全力进军这个 3D 领域。基本上，只要有什么特别酷的东西出现，你就会听到消息。

**Kate Holterhoff：** 好的，我不知道你有没有，比如新闻通讯或者什么的推荐，让人们可以关注。

**Wes Bos：** 人们总是问我这个问题，我说，我就是源头。听 Syntax，关注我。因为我了解所有这些不同软件的更新信息，然后我在推特上关注了很多人。实际上，要跟上节奏很难，我的建议是关注那些以此为职业的人，这样你就能像被鸟妈妈照顾的小鸟般轻松应对一切。

**Kate Holterhoff：** 好的，这确实有帮助。听起来，社交媒体是实现这个目标的重要途径。明白了。这就引出了我的第二个问题，我认为这非常有意思。你提到，很多开发者对设计不感兴趣。他们似乎希望有人直接给他们一套系统，或者觉得这并非自己擅长的领域，又或者想专注于业务逻辑之类的工作。但情况并非总是如此。许多前端工程师，尤其是那些自视为设计师的人。所以 Wes，我很想知道，你是否将自己视为设计师？

**Wes Bos：** 是的，我会这么说。我喜欢设计，对于那些真正的设计师来说，他们可能不喜欢这样。但我职业生涯开始的时候就是做设计。我会做 T 恤和专辑封面。我很早就开始做印刷品，我认为自己主要是开发者，但是我构建的所有东西，我都自己做设计。当然，这并不是说像有的人那样花时间做用户体验研究等所有那些事情。他们是另一个层次，他们设计的网站看起来很酷，而且超级好用。我觉得自己还没有达到那种境界，但我确实自认为是个设计师，而且我特别享受精美的用户界面。

设计师与开发者的角色融合

**Kate Holterhoff：是的。我认为，这不仅体现在你对网站使用深色模式或浅色模式的感觉上，我觉得这非常有趣，而且还有你对网页设计的热情。我认为，这真的很令人兴奋，因为我也这么觉得。但我认为还有一件有趣的事情，当年我还是前端工程师时，如果工程团队超负荷运转，设计师们自己有时就会用 Webflow 之类的工具搭建网站。那可是五年前的事了，简直是远古历史。**

**但如今，我们身处这个 AI 编程时代，我知道这种现象只会越来越普遍，而且还在不断演变。所以我觉得，我们正目睹这些角色以令人兴奋的方式相互融合。但与此同时，我觉得更酷的是，现在，有了这些能为我们处理大量思考任务的 AI 工具，我们得以使用更复杂的工具。**

**你曾在那个预测播客里说过，为了运用一些新工具，当初曾经想成为数学家，而现在，AI 就能帮你实现。这为设计师和前端工程师开辟了前所未有的美学可能性——这是些过去根本无法触及的领域。**

**所以我觉得这真的很酷——我们不仅能看到表面的东西，比如“哇，这真漂亮”。那只是表面的装饰，而实际上背后到底发生了什么呢？”我们还能看到一些更深层次的问题，比如“这很困难，这是很棘手的事情”，不过现在，我们能够用以前无法做到的方式去解决这些问题。**

**Wes Bos：** 是啊，AI 真的让人们能跳出了自己的领域。现在，你可以直接借鉴别人的成果。设计师们对此非常满意，因为他们最讨厌和那些固执的开发者打交道，他们总是不肯实现设计师天马行空的创意。现在，设计师能更深入地参与其中。我也深有同感，毕竟我最近编写的 C# 和 C++ 代码量实在太少了。

在过去一年左右的时间里，我从事硬件项目期间开始接触 Swift 语言——这可是我从未编写过的代码。AI 确实拓宽了人们能够涉足的领域。不过我认为，很多人未必喜欢这种变化，因为这就像，在整个职业生涯中，我都在理解浏览器的内存管理是如何工作的。这是我的专长。我认为，这类人依然有存在的价值。但对于只想高效完成任务的人来说，这些工具能助你突破自身局限，实在太棒了。

**Kate Holterhoff：哇，这个比喻真妙。我完全能理解。是啊，我发现人们的角色不断演变、蜕变成新的形态，这个过程实在太有趣了。天哪，谁能想到“氛围编码”这个词会流行这么久。不过嘛，我们现在都沉浸在那种氛围里，而且还能用像 Nano Banana 这样的工具来阐明我们想要表达的很多东西。**

**我不知道，就是各种创意。它渗透到了方方面面，从 PPT 到整个工作流程。所以说，我们都在变成设计师——不管你是否自认为如此，这种能力已经，怎么说呢，变得触手可及了。**

**Wes Bos：** 是的，不过唯一缺失的就是创造力和品味。我觉得这些东西是无法靠 AI 实现的——AI 或许能帮你构思创意，或许能带你探索未曾涉足的领域。但归根结底，如果你不明白为什么它好，那它就不会真正优秀。那些拥有创造力的人——比如 Lando Norris，以及幕后的开发者与设计师，还有那些天生具备审美品味的人。无论你是在设计 API，还是在打造一些视觉呈现的作品。

**Kate Holterhoff：好的，明白了，超级有意思。我想多了解下 Norris 网站的性能问题。我觉得特别有意思的一点是，过去，当我们看到这类交互性强、体验独特的 JavaScript 密集型网站时，它们往往会非常卡顿。**

**而 Lando Norris 的网站显然并非如此。你花了不少时间讨论这个问题。我觉得有趣的是，你随口提到，这个网站没有使用框架，没有用 React，而是整合了多个库。我在想，你是否认为这是一种信号。因为我们总在讨论 React 网站——尤其是单页应用——在旧设备上有多卡顿，对吧？而在这里，我们看到的是，它完全不依赖框架、毫无延迟，这是一些完全不同的东西。你觉得呢，我不知道，这能说得通吗？这是否暗示了未来单页应用的发展方向？**

**Wes Bos：** 我不这么认为。因为像 Lando 网站就是个 SPA。而且我觉得，就算 Lando 网站是用 React 做的，速度应该也挺快。Lando 网站之所以快，关键在于——网站卡顿通常不是框架本身的问题。虽然有人喜欢往里塞大量的 JavaScript 之类的代码，但这通常不是症结所在。问题通常出在渲染环节。网站变慢的原因有很多对吧？可能是网络延迟，也可能是输入响应迟缓。但 Lando 网站的开发者们深谙其道。在他们的案例中，核心问题主要在于渲染性能，也就是如何将内容呈现到屏幕上。

为了确保速度尽可能快。举例来说，你会发现，Lando 网站的一个做法就是没有任何元素带有阴影效果。如果给所有元素都添加阴影，浏览器就需要计算这些不透明度。当元素层层叠加时，浏览器必须识别出两个不同的图层并重新渲染——既要计算透视效果，又要处理图层间的混合模式交互。

你多久一次重新渲染这些内容？缓存、资源之类的东西。Lando 网站之所以运行流畅，纯粹是因为背后的开发者非常出色。他们深谙性能优化之道，并投入大量的时间反复打磨细节。

**Kate Holterhoff：** 是的。你联系过构建这个网站的团队吗？

**Wes Bos：** 是的。视频发布后，我收到了几名开发者发来的私信。我们的确尝试过邀请他们上播客，但他们必须经过 Lando 团队的审批，所以最终没能成行。不过，他们确实向我介绍了不少关于设计决策的细节。一部分原因是他们做了一些开发限制，另一部分原因是他们花了大量的时间优化这些东西的性能。

**Kate Holterhoff：这可真够酷的。我之前没有意识到，但这么一说确实有道理。据我了解，后来有几个人也跟风尝试搭建一个像 Lando 这样的网站，看看能做到什么程度。不过话说回来，很明显你确实深入研究了具体细节，比如那些库。**

**Wes Bos：** 对，记得我和他们聊过 iOS 平台上的 WebGL 应用，就是 iOS 的 3D 功能。你可以把大量的模型直接发送到手机的图形处理芯片，其性能明显比直接用 WebGL 处理 PNG 图像文件好得多。

在这种情况下，他们有一种特殊的文件格式，能够直接发送到芯片上，你知道的，这种复杂性通常会被封装在一个框架中，它会帮你处理好这一切，比如为你完成底层的繁重工作。所以，我们可能会开始看到一些这样的框架和类似的东西，它们会为你处理好这一切，因为我认为，一般的开发者不会愿意去做那种复杂的工作的。

光是头盔动画就相当于在 3D 空间里叠加了六七层不同的动画，对吧？而且你还得准备双倍数量的动画，因为 iOS 和其它浏览器使用的格式不同。

**Kate Holterhoff：所以我想，我们也许该谈谈钱的部分。我不记得在哪里听到的，Lando Norris 网站好像是花了 5 万美元。是那么多吗？**

**Wes Bos：** 他们没有告诉我任何数字。

不过话说回来，我觉得五万块不少，又觉得有点少。我猜，他们当时大概有五个开发人员在做这个项目，对吧？就算按年薪十万美元算。就这种级别的开发者来说，这个薪酬都算低了，简直低得离谱。

这么说来，5 万美金，那只是几个星期的开发。我有点怀疑那不够，实际花的钱可能更多，但这是一个非常大的构建，而不仅仅是一个有炫酷动画的登录页面。里面有很多页面，很多不同的功能，有一个商店，你知道的，各种各样的东西。

**Kate Holterhoff：好的。我再回想下是在哪里看到的。我觉得那是一个 YouTube 视频。不管怎样，不管数字是多少，有一点很明确，他投入了很多钱，或者至少他的团队投入了很多。是的，有很多边角的东西本可以削减，但他们没有。我的意思是，那个网站是一件艺术品，对吧？它真的像，从中可以看出那种投资。**

**下一个问题是：关于你预测的 2026 年趋势——即行业将摆脱千篇一律的网站模式，转向更具差异化的定制化设计——你认为其商业价值体现在哪里？企业将如何证明这种模式的投资回报率？请详细说明一下。**

**Wes Bos：** 是的，这是个难题。因为对很多人来说，花 5 万美元做一个网站，你永远看不到回报，对吧？老实说，我总是把它归结为品牌策略。不是卖更多的 T 恤或其他什么，这只是他品牌的延伸。如果他每年花费这么多钱做广告，在电视广告、户外广告牌或其他什么上花钱，这当然是一个很好的品牌延伸途径。我不认同这种说法。这类项目的投资回报率该如何计算？这确实是个问题。

这确实是一件棘手的事情，对吧？因为设计很难有投资回报率，你完全可以把这些都放在页面上的文字里，也能传达相同的信息。如果你想这么做，我不确定这能带来什么投资回报。但如果这是品牌战略，你真想强化“这就是我们”的形象，那么我认为，这才是你该关注的方向。

**Kate Holterhoff：** 我明白你的意思。我是说，我对这件事有点拿不定主意。对了，我记起来了，是 Life of Guarez 这个 YouTube 频道说那个网站值 5 万美元，那数字后面的零可不少，不过我觉得我没弄错。总之，我们既不能证实也不能否认他们实际投入的金额是不是这个数。

我对此表示怀疑，但我认为这很有趣。2025 年，就开发工具而言，有很多投资投入到了设计方面。所以我想到了像 Figma 上市这样的事。我们也看到了很多行业动态。我的意思是，我是 Canva 的忠实用户，它增加了许多功能。然后还有 Nano Banana，我已经提到过了。因此，人们似乎在设计上投入了很多的精力。显然，这是我们所有人都想要的东西，它让我们感觉良好。

但我对此感到担忧。部分原因在于，我曾参与过交互设计团队，亲眼见证过优秀设计师的创造力——我共事过的设计师们都才华横溢，他们能构思出我从未想象过的方案，拥有海量的经验积累，打造出过真正让人惊艳的作品。如果仅由前端工程师负责，这些杰作根本无从诞生。我们工程部的主管总爱分享一个 HTML 超链接网站当范例，说那是个完美的网站。那简直是史上最丑的网站。所以绝对不能让工程师主导设计。这点大家应该都认同吧？

我想我只是听到太多类似的担忧了。这确实是个棘手的问题，Wes。我为朋友们感到紧张，你知道的，我们都在担心会失去工作。

**Wes Bos：** 谁是设计师？这确实是个棘手的问题。因为作为一个不靠设计谋生的人，我真心喜欢大家现在使用 Canva 做出来的东西——无论是老师制作的教案，还是教堂的野餐海报，每件作品都相当不错。Shadcn 组件也是同样的道理。

用 Word 打出来的东西看起来也还不错，信息传达也清楚，我觉得这比其他替代方式强多了。但问题在于，这只是最基础的要求，现在该如何更进一步，才能获得出色的设计并制作出一个漂亮的网站。是的，这是一个非常棘手的问题。特别是当你试图说服掌握资金的人。“嘿，我知道你刚在输入框里敲了几下，就蹦出个还算不错的东西，但事情可不止于此，对吧？界面设计、交互体验、品牌调性——优秀的设计远不止于表面功夫。”

**Kate Holterhoff：** 当然，对于 Canva 来说也是如此。你用过 Affinity 吗？就是去年他们收购的那个。

**Wes Bos：** 是的，我几年前用过 Affinity 软件。当时我想从 Adobe 产品转向它，但最终没能成功。不过他们现在的发展确实挺令人兴奋的。

**Kate Holterhoff：** 嗯，我还没试过。我就是个普通用户，只用 Canva 的常规功能，比如背景移除工具。说不定我该尝试新功能了。不过得说，他们的 AI 和聊天功能体验不太好，我基本放弃了这些功能。感觉这些功能可能上市太仓促了。

**Wes Bos：** 是的，我认为人们也意识到了，当每个人都在使用 Canva 时，当你看到一些不是用 Canva 制作的东西，你就会想，那是怎么做到的？那很美。我喜欢那个。这让我感觉很好。你知道，对于什么样算好、什么是你喜欢的东西，我们的基础认知在变化。所以人们总想搞些惊天动地的东西，比如 Lando Norris 的官网突然出现——哇，这太出乎意料了，简直酷毙了。懂我意思吧？哇，我没想到会这样。这太酷了。你知道吗？

**Kate Holterhoff：那么，如果我们讨论的是精美的网站，你是否认为人们还会有一种想要将 2026 年变成“奇特网络体验之年”的想法呢？毕竟大家都怀念过去，怀念旧时的 Web，对吧？我的意思是，这是否也是讨论的一部分？**

**Wes Bos：** 是的，奇怪也是一种酷，就像遇到某个设计上别出心裁的应用时，你就能感受到开发者倾注的心思。当我察觉到某个应用只是刻意营造氛围时，我无法形容我会多快把它关掉——我根本不信任这种东西。

嗯，如果页面充斥着紫色渐变、按钮这类你整天都会看到的千篇一律的东西，当你浏览这类页面时，信任感会瞬间消失。所以现在，当我打开某个界面时，只要它看起来稍微与众不同，哪怕不是交互功能，只要能吸引我的目光，我脑海里立刻就会浮现：哦，这背后有人在用心设计。这确实是个能用的应用，而不是某个人通过氛围编码随随便便搞出来的玩意儿。它不会半年后就失效，更不会让我做上传银行对账单之类的疯狂操作。

网站上的模特照片也是同样的情况。比如我看到一件衬衫，明显是用 AI 技术给模特穿上的，我觉得某种程度上可以接受。但转念一想，我立刻觉得这八成是按需印刷的诈骗公司，或者他们正准备给我寄些劣质货。如果是一家合法的公司，就在这事上用点心。所以这是一个微妙的平衡。

**Kate Holterhoff：** 是啊，说得对。今天刚看了个产品演示视频，看得一清二楚——那家冒牌公司的所有演员都是 AI 生成的，连灯光效果都假得离谱。说真的，这太让人失望了。无聊透顶，因为我早就知道套路了。

**Wes Bos：** 是的。这种新鲜感能持续几个月，直到人人都开始效仿。到那时候，我们的基线又会随之改变。

设计投资的商业价值与未来趋势

**Kate Holterhoff：实际上，我觉得你已经含蓄地提到了投资回报率的问题。比如人们更愿意在电商网站消费，因为这些网站看起来经过了精心设计——他们真的请模特试穿衬衫之类的。是的，我觉得这是在摆脱 AI 体验给人的“恐怖谷”感觉，尽管它们本身已臻完美。一种设计感与怪诞感的结合，远离完美、漂亮，但每个人都看起来都像动漫角色的感觉。**

**Wes Bos：** 是的。TikTok 做到了。对。就像 TikTok 那样。

每个人都配备了那种安装在三脚架上的精美摄像设备，还配有专业的麦克风。随后，品牌也开始采用这种拍摄方式。而 TikTok 的情况是，人们用手拿着麦克风进行拍摄，画面不够稳定。

质量比较差，甚至构图都不对。现在，甚至有人故意吃得狼吞虎咽，装作准备不足的样子——就是刻意伪装成准备不足之类的。这就像个钟摆，不断来回摆动，因为人们总在追寻更真实、更原始的东西。就像我说的，我能在 TikTok 视频的前 10 毫秒内做出反应——只要听到声音或看到动画出现在屏幕上，我就会立刻滑动屏幕，因为我感觉有人在向我推销东西，懂吗？但如果有人站在自家车库里举着手机拍摄，我就会想：这个值得一看，这才是真实的。

**Kate Holterhoff：我没有使用 TikTok，所以我不会把它作为例子，但这是一个很好的例子。所以谢谢你的分享。现在处境太艰难了，毕竟我们正身处零利率政策后的经济环境。你知道的，裁员消息不断，一切似乎都失控了。**

**那些在技术栈顶端辛勤耕耘的朋友们，此刻正感到格外脆弱。然而这些经历如此真实可感，恰恰是人们最先能看到的切入点。变化速度如此之快，你自然希望最聪慧的头脑来解决这些特定的难题。**

**不过确实，所有这些现实案例都很有启发性。我觉得有趣的是，你刚才顺带提到了 Tailwind CSS。不知道你是否注意到，Adam 发过一篇博文，说不希望大型语言模型读取文档，因为他解释说，为了关闭 PR（拉取请求），他希望人们去查阅文档，从而购买付费产品。**

**当然，不管怎样，大家对此很生气。我可以在节目笔记里放个链接。但真正让我感兴趣的是，当我初次尝试氛围编程时，每个小应用不仅采用了框架，还总会使用 Tailwind CSS 而非纯粹的 CSS、JavaScript 和 HTML。**

**最近，我发现情况并非一直如此，但我对他的说法很感兴趣，因为我感觉所有 VibeCoded 应用程序所使用的库都远远超出了必要的数量。比如说，你知道，一个简单的应用程序，完全可以只用基本元素来完成，而它默认采用的往往是那些处理简单任务时显得过于复杂的方法。所以，这里似乎存在某种脱节，但归根结底，人们关心的始终是谁在赚钱。**

**今天，特别是那些在前端领域辛勤工作的人们，那些思考视觉体验、用户界面（UI）的人们。你们对所有这些有什么看法？**

**Wes Bos：** 是啊。不过 Adam 是我的好朋友，在他写那篇文章之前，我已经和他讨论这个问题好几个月了。不过情况就是这样。他创造出了一个非常成功的东西，而且使用量还在不断飙升。而且，由于人们是通过大型语言模型来使用它，所以便不再访问文档了。而这原本是能支付开发人员薪酬的商业模式。就是这样。你向特定用户群体销售高级产品，以此盈利，然后再用这笔钱雇佣更多人参与开源项目。所幸 Syntax 赞助了五万美元。许多公司纷纷站出来表示：哇，这确实是个问题——我们的主要业务可能无法继续了，因为我们抢了它的饭碗。所以这是一件令人沮丧的事情。因为其他大多评论都在说，这种商业模式在 AI 领域根本行不通。但我认为，类似的事情有很多。

我的整个业务就是这样。多年来，我写了数百篇博文，发布了大量的内容，而 LLM 只是吞噬了所有这些，现在人们不需要那些文章了。这是一件令人沮丧的事情。我不知道所有这些问题的答案是什么。Tailwind 每生成一千个类，我们能否给它一美分？这个代币，我们能内置它吗？过去确实有人尝试过类似的方案，比如基础注意力代币（BAT），用户虽然启用了广告拦截器，但每月还得向浏览器投入 10 美元。然后根据访问的网站，我们将把这 10 美元分配给不同的人。也许这会奏效，新闻业也是一个非常类似的重要领域。你知道，如果人们在浏览这类内容时屏蔽了广告。所以也许这就是解决之道。

让我们拭目以待，但我认为，围绕这些公司窃取所有信息的问题还有很多。并非说这种行为会就此停止——我自己也在使用这些工具，它们确实很棒——但它们能肆无忌惮地吞噬所有数据，然后免费提供给用户，这实在荒谬至极。

**Kate Holterhoff：** 感觉这里有一个教训，关于那些可有可无和那些你无法避免要付费的东西，比如服务器空间，我感觉这首先会影响到那些在技术栈顶层的从业者。我对此感到担忧。我是认真的。因为我也想今年成为设计之年。我们要把它变成现实。我希望所有条件都已经准备就绪。
