# Claude Code 或将 Max 专享，Pro 套餐撑不住了

REPORT

  

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/jXSGuwJvpdiabgNt9iaZRiafnaRKQ6c8JOdm0jeGuTWk3LVzc4T6sT87ZtTibuCLtbuRM2yvORxw0ibaficzGjMjeeMXZrgGAIu3icwBdpedX3afVg/640?from=appmsg)

Gergely Orosz 把 Pro 套餐里 Claude Code 那一行被打 X 圈了出来

这周，Anthropic 一度把 Claude Code 从 Pro 套餐的功能列表里拿掉

定价页面上，Pro 那一栏的 Claude Code 行从勾变成叉。Max 5x 和 Max 20x 还留着勾。Max 5x 起步价 `$100`，Max 20x 是 `$200`

事情很快传到 Twitter 上。Anthropic 增长负责人 Amol Avasare 出来灭火，说这只是一次小规模测试

For clarity, we're running a small test on ~2% of new prosumer signups. Existing Pro and Max subscribers aren't affected.

翻译过来，就是只针对 2% 新注册的 prosumer 用户做测试，现有 Pro 和 Max 用户不受影响

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdjPtzwGlV6dvyXT91icomX5GoNwQbq9Gw8HtTjsNrgbZhkg2INtnNaicHnOSHjGdeQBWTibmqWlz05ZOBHjCZBVlnMuUjCvkjicwpg/640?wx_fmt=png&from=appmsg)

Avasare 在 X 上的官方回应

不过吧...他回避了那个 2%的问题。如果只针对 2% 用户，为什么 support 文档全量改了，定价页全量改了

非常可疑！！！

  

几个小时后，Anthropic 把页面又改了回去。现在打开 `claude.com/pricing`，Pro 套餐里的 Claude Code 又出现了，写着 `Includes Claude Code`

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdiaeKwyAIjcibufyUpAscaTSpOYmFiagx8YjWQwsmTsQYFbqkJSmFliao1NIXpicOLMvia5l97xxGhPp1IpfA9qLBjk6Z3KkSAl9T90E/640?wx_fmt=png&from=appmsg)

当前 pricing 页面，Pro 套餐 Claude Code 已恢复

  

Avasare 在同一条推下面，紧接着说了一段更有信息量的话

When we launched Max a year ago, it didn't include Claude Code, Cowork didn't exist, and agents that run for hours weren't a thing. Max was designed for heavy chat usage, that's it.

一年前 Max 发布的时候，里面没有 Claude Code，Cowork 也不存在，能跑几小时的 agent 还不是常态。Max 当时只为重度聊天用户设计

Engagement per subscriber is way up... usage has changed a lot and our current plans weren't built for this. So we're looking at different options.

单用户使用量大幅上升，使用方式跟以前不一样，现行套餐扛不住，所以我们在看其他选项

换句话说：现行套餐扛不住了，还得想办法涨价

  

Cowork 没动，Claude Code 想拿走
-------------------------

注意：Anthropic 这次没动 Cowork

Cowork 是另一款产品，给非技术从业者用，本质上也是 agent，也跑长任务，比如写文档

Claude Code 是给开发者用的，一个 session 能跑不知道多少 token，Pro 套餐的用量分分钟榨干

Anthropic 的订阅套餐定价远低于实际消耗的 token 价值，有时候差了十倍以上

  

这不孤立
----

回看一下 Anthropic 最近几个月做的事，值得玩味

三月，Anthropic 给所有付费用户加了「每周用量上限」。理由是高峰算力紧张，要让用户养成错峰习惯

之后 OpenClaw 从订阅套餐里被砍掉，要用就走 API 按 token 付钱

The Information 月初报道，企业用户的合同也在改。从原来的不限量订阅，改成按百万 token 计费。后台数据显示，很多企业的实际 token 消耗，远超订阅费能覆盖的部分

这几个事儿放在一起：**让付费跟实际 token 消耗对齐**

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdiaouCzLw2pCtIFucm7MNx8RC59TujpXoXCJbnhEnyQzfiaUoHfLhYH5NUcCvib4zibxRPWes31teARupahL43keEgrRpcSkSSxLRU/640?from=appmsg)

  

对开发者意味着什么
---------

Avasare 在推文最后留了一句

When we do land on something, if it affects existing subscribers you'll get plenty of notice before anything changes. Will hear it from us, not a screenshot on X or Reddit.

真要改的话，会提前通知现有用户，不会让人从 X 或者 Reddit 的截图里看到

这话本来是承诺，总觉得有点奇怪

毕竟，这次 support 文档和定价页同步改的事，本身就是从 Reddit 的截图里被发现的

  

总结下现在的状态：

→ 现有 Pro 用户暂时不受影响

→ 新注册的 Pro 用户里有 2% 拿不到 Claude Code

→ 定价页面已回滚

→ 内部测试还在跑

  

另外的，国内开发者多了一个维度。Claude Code 的国内访问本来就要解决网络问题。如果再叠加 5 倍涨价，会不会推动开发者迁成国内的 GLM、Kimi、MiniMax 这些国产模型，是个值得算的账

毕竟...[国人抢不到 Claude，老外抢不到 GLM](https://mp.weixin.qq.com/s?__biz=MzkzNDQxOTU2MQ==&mid=2247515520&idx=1&sn=1ad7e160a22e1c4c88213b2672e874d5&scene=21#wechat_redirect)
