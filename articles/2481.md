# 谁家更新日志那么长啊？Claude Code版本更新引围观，1096次提交一口气上线

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gW889cR13aBX42evqQIRibKlicoCrHPEpT0tQiceNphESCa2eJTqstP8G0yqMTkeMFrOGue6kOyCKdTkA/640?wx_fmt=png&from=appmsg)

编辑｜张倩

  

如果你是 Claude Code 的用户，你可能会注意到，它最近有个重要的版本更新，从节前的 2.0.76 更新到了 2.1.0。

  

而且，这次的日志，你得往下翻好几屏。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_gif/KmXPKA19gWic1JLmRXb2YuRrm3uG76sKgZgVqp6pGhzhyAKqUFI2zvibmibcia83HnTdGf540t3FsR8AxFUVoNREQA/640?wx_fmt=gif&from=appmsg)

还有人调侃说「求求了，谁去跟他们说一下什么叫 rolling release（滚动更新）吧」「照这个速度，我们周五早上就能用上新操作系统了」。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gWic1JLmRXb2YuRrm3uG76sKgQfbg0UKWE7t4XcdibrIica4gxGeBXgRsY2QAtcLiaf7l4hAYI5ZYlqsbQ/640?wx_fmt=png&from=appmsg)

根据 Claude 的总结，这次更新大概包含以下类型： 

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gWic1JLmRXb2YuRrm3uG76sKgwAjLEiaZiaLDNaT93Auo0zVic3LgSVrNxamOCGVDBa0cm7CvZzXQD1b7g/640?wx_fmt=png&from=appmsg)

  

在官宣更新的推文中，Claude Code 创建者、负责人 Boris Cherny 提到，这个版本一共合入了 1096 个 commit。同时，他也为大家拎出了一些重点，包括：

  

*   Shift+Enter 换行：开箱即用，无需任何配置
    
*   钩子支持：可直接在 agents 和 skills 的前置配置中添加钩子
    
*   Skills 增强：支持分叉上下文、热重载、自定义代理，可用 / 调用
    
*   Agent 行为优化：拒绝工具使用后 agent 不再停止，会尝试其他方案
    
*   多语言响应：可配置模型以指定语言回复（如日语、西班牙语等）
    
*   工具权限通配符：支持通配符匹配，例如 Bash (\*-h\*)
    
*   会话传送：使用 /teleport 将会话转移到 claude.ai/code
    

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gWic1JLmRXb2YuRrm3uG76sKgHicnWfDCr6xEGQib7wpicB6E3xfJuX3zaSjbVT7zibXQNxzjDTlaI90Weg/640?wx_fmt=png&from=appmsg)

  

从评论区来看，大家似乎对会话传送比较感兴趣。Boris 还在评论区科普了使用方法。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gWic1JLmRXb2YuRrm3uG76sKgX3py6AKz0oHiabEJJX2jEHn9sMFH5wxvQGmhkywnmWtKRtXEdPGX5Ww/640?wx_fmt=png&from=appmsg)

  

此外，还有人关心企业用户要怎么用这个功能。PyTorch 之父 Soumith Chintala 问到，很多企业都是通过 API Key 使用 Claude Code 的，Anthropic 是否也能为这类用户开放 /teleport 功能？目前，Boris 还没有给出回复。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gWic1JLmRXb2YuRrm3uG76sKgA7DeH8QSPUUMp1gBbttqayd1cy3WkiaRsqlRIwh3LUvE3ej1FoNWW4g/640?wx_fmt=png&from=appmsg)

  

在完成这么多更新工作后，Boris 的团队也没闲着。就在这两天，他们又推了两波更新，分别是 2.1.1 和 2.1.2，修复了一些 bug 和安全问题，还顺带加了一些小改进。这是一天一波新发布的节奏啊。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gWic1JLmRXb2YuRrm3uG76sKg86bmCjxXmIWFq2IvGbmuwBebtZ6NA3Wv5AcLRMr2icRWTvYObSwrU7A/640?wx_fmt=png&from=appmsg)

  

这种发布节奏，肯定少不了 AI 工具的功劳。毕竟 Boris 曾经提到，在之前的一个月，他提交了 259 个 PR，总共包含 497 次提交，4 万行代码增加，3.8 万行删除。每一行，都是 AI 写的。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gWic1JLmRXb2YuRrm3uG76sKgKXV9Viae77fBxCDOJorRxfDFB1RrSCQoymm6lSlk1QyBIDDTA5NTJyA/640?wx_fmt=png&from=appmsg)

  

不知道在这次的更新里，AI 编码占了多大的比例。

  

在之前的采访中，Boris 也透露过他们做 Claude Code 的一些方法论。其中比较有启发性的是，他们坚持用自己的产品作为真正的生产力工具，自己公司的人（包括研发模型的）都是 Claude Code 的重度用户，这能让他们尽快发现 bug，并在产品和模型层面做出更改。2.1.0 版本那一长串更改日志，或许就是这一理念的成果。

  

元旦假期期间，Boris Cherny 还开展了一波「线上教学」，亲自示范他自己使用这个 AI 编程工具的工作流。感兴趣的读者可以点开我们之前的文章作为参考：《[500 万人在线围观，Claude Code 创建者的 13 条独家实战秘籍爆火](https://mp.weixin.qq.com/s?__biz=MzA3MzI4MjgzMw==&mid=2651010343&idx=2&sn=440edc0a537b0cf8820a967f54574cb7&scene=21#wechat_redirect)》。

  

不过，对于这次的更新，也有开发者表示不满，指出更新多但 bug 也多，而且很多新功能不够简洁。
