# AI 都会写代码了，我们还需要一门国产编程语言吗？

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

AI 写代码已经不稀奇了。

今天，Kimi、DeepSeek 等国产大模型正在进入真实开发场景。写函数、改 bug、读文档、生成脚手架，甚至参与更长链条的开发任务，都已经不再只是演示里的能力。

但开发者都知道，代码写出来，并不等于项目完成。

一个 Demo 可以很快生成，一个真实项目却要面对更多细节：依赖能不能跑通，边界情况有没有处理，后续还能不能维护，代码能不能被别人复用。

这也是为什么，AI 编程越往真实场景走，越需要语言、工具和生态来承接。

过去一年，MoonBit 的发展正好提供了一个观察样本。它不只是在完善一门国产编程语言，也在通过包生态、AI 生成 C 编译器、AI 编程评测、外部平台接入、浏览器实验、开发者工具和形式化验证等实践，展示一门新语言如何从“能写”走向“能用”。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

MoonBit 过去一年的发展

回看过去一年，MoonBit 的发展路径比较清晰：先让开发者真正用起来，再进入更复杂的工程场景，最后补上可信代码这一环。

2025 年 6 月，MoonBit 进入 Beta 阶段。这意味着它开始从“可以尝鲜”走向“可以认真使用”。

从生态侧看，MoonBit 官方包生态 mooncakes.io 已经接近九千多个包，接近 苹果 Swift Package Index 收录的包数量，累计下载量也达到 350 万级。对于一门仍在快速成长的新语言来说，

这是一门语言从“项目”走向“生态”的重要信号。

复杂的基础软件场景

2026 年，MoonBit 团队展示了 Fastcc 案例。Fastcc 是一个用 AI agent 基于 MoonBit 从零构建的 C 编译器项目。它不是生成一个函数、一个网页或一个 Demo，而是让 AI 参与一个完整编译器项目的开发：从源代码解析，到语义处理，再到代码生成、调试和性能优化，整个过程都被纳入工程流程中。项目在 10 天内产出约 35,000 行结构化代码，并实现 self-hosting。

这件事真正值得关注的地方，是它把 AI 编程从“生成代码片段”推进到“参与复杂软件工程”。编译器本身就是典型的基础软件项目，对语言表达能力、代码组织方式和工程工具都有较高要求。Fastcc 的出现，证明 MoonBit 已经具备承接复杂工程实验的能力。

这一方向也延伸到了 AI 编程评测。2026 年 2 月发布的 SWE-AGI 基准测试选择 MoonBit 作为任务语言，要求 AI agent 按照明确规格实现 parser、interpreter、binary decoder、SAT solver 等工程模块，每个任务涉及 1000～10000 行核心逻辑。它的意义在于，把 AI 编程能力的评估从“能不能写出答案”，推进到“能不能完成一段可维护的软件工程”。

外部平台也开始把 MoonBit 纳入自己的工程体系。美国云服务商 Golem Cloud 是一个面向 WebAssembly Components 的云平台，主打 durable computing 和 agent runtime 场景。在 1.5 版本中，Golem Cloud 提供了 first-class MoonBit support，支持用 MoonBit 编写 durable agents，并编译为 WebAssembly components 运行在 Golem 上。其文档已经覆盖 HTTP endpoints、认证、OpenTelemetry、LLM/AI 能力、数据库访问、Webhooks、定时任务等 MoonBit 使用场景，相关 SDK 也已发布到 mooncakes.io。

系统软件实验

海外开发者 mizchi 长期关注前端工具、浏览器基础设施和开发者工具方向。最近，他正在用 MoonBit 推进一个自制 TUI 浏览器项目，尝试把网页阅读能力搬进终端环境。此前发布的 v0.0.2 版本已经能够显示基本可读的内容，并减少了显示错位；在最新更新中，这个自制 TUI 浏览器已经可以在终端里阅读 HackerNews，交互体验仍在继续打磨。

更底层地看，mizchi 还基于 MoonBit 开发了 Crater。这是一个用 Pure MoonBit 编写的 CSS layout engine，目标不是做完整浏览器，而是切入浏览器渲染链路中非常核心的一环：布局计算。浏览器要把 HTML 和 CSS 变成屏幕上的页面，必须先算清楚每个元素应该出现在什么位置、占多大空间。Crater 做的正是这部分基础能力。

 MoonBit 在 Web 方向的探索，已经不只是“能编译到 Wasm”或“跑一个前端示例”，而是触及了浏览器布局、终端渲染和轻量浏览器这类更底层的系统软件实验。

开发者工具方向也有新的尝试。2026 年 1 月，MoonBit 展示了静态分析实践。静态分析可以在程序运行前检查潜在问题，常见于编译器、IDE 和代码质量工具中；MCIL 则可以理解为 MoonBit 用于承接这类分析能力的一层中间表示。

在应用侧，MoonBit 也有 CRDT 实时协同编辑器这样的案例。CRDT 是一种常用于多人协作和离线同步的算法思路，比如多人同时编辑文档、断网后重新合并内容，都需要处理状态一致性和冲突合并问题。

这个案例说明，MoonBit 不只适合系统软件实验，也可以表达复杂应用逻辑。

社区贡献基础工具

国内知名开源开发者 justjavac 开源了 moonbit-case，提供字符串命名风格转换能力，比如 snake\_case、camelCase、kebab-case 等。它不是大型系统项目，但这类小工具对语言生态很重要，因为真实开发离不开这些高频、细碎、可复用的基础库。

进入 2026 年，MoonBit 的路线继续向稳定性和可信代码延伸。2 月发布的 0.8.0，被官方视为走向稳定和生产使用的重要节点；到了 4 月，0.9 版本进一步把形式化验证提升为重要能力。

在 AI 生成代码越来越普遍的背景下，真正稀缺的不只是代码本身，而是如何确认关键逻辑没有问题。MoonBit 在语言和工具链中引入形式化验证能力，正是在回应这个问题。

从 C 编译器到 AI 编程评测，从外部平台接入到 TUI 浏览器，从浏览器布局引擎到静态分析、CRDT 协同应用，再到形式化验证，MoonBit 过去一年的关键词不再是单纯的版本迭代，而是生态增长、复杂工程落地和可信代码能力的推进。

从真实项目开始，共建国产基础软件生态

中国计算机学会（CCF）（第八届 CCF 开源创新大赛）与IDEA 研究院主办：2026 MoonBit 国产基础软件开源大赛，比赛以 MoonBit 生态建设为主题，鼓励开发者围绕工具库、Wasm 组件、AI Agent、编译器、浏览器、数据处理等方向，构建具有工程价值的开源项目，赛事奖金总额达十五万元。进入决赛并获奖的优秀项目，获得 IDEA 研究院及 CCF 颁发的荣誉证书。

如果你也关注国产基础软件、新一代编程语言和 AI 编程实践，欢迎报名参赛，一起为 MoonBit 生态补上一块真实可用的拼图。
