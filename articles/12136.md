# ICLR 2026｜隐式思考模型LRT：「隐式思维链」推理，更快更强！

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gWic1GuW68DykycvknmG9tyBvLRsVGY4rRKCGuKKSkOqnGrvGwXxqqDxHlia88ZCbqyicswl2HC89BcZA/640?wx_fmt=png&from=appmsg)

  

近日，哈尔滨工业大学（深圳）联合深圳河套学院、Independent Researcher提出了隐式思考模型 LRT（Latent Reasoning Tuning），通过一个轻量级的推理网络，将大模型冗长的「思维链」压缩为紧凑的隐式向量表征，一次前向计算即可完成推理，无需逐 token 生成数千字的中间推理过程。

  

LRT 不仅实现了高效思考，还能作为一种全新的混合思考范式，在 Qwen3 系列模型上超越了其原生的非思考模式。

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqG74BmW32y41pKGckGFY3yPlHDnia4ZQgM9X1oOLeyVS2X7qD2fnIyHib7p4EFBiaeJuBBLnYEx514YOuks7NGBxHxyxSqjjLUFjU/640?wx_fmt=png&from=appmsg)

  

*   论文地址： https://openreview.net/forum?id=CbK7lYbmv8
    
*   代码开源： https://github.com/MobiusDai/LRT
    

  

出发点：大模型「自言自语」的推理过程真的全都有用吗？

  

以 OpenAI o1、DeepSeek-R1、Qwen QwQ 为代表的慢思考推理模型，通过生成详尽的逐步推理链来解决复杂问题，展现了强大的推理能力。然而，这些模型存在一个显著痛点 —— 过度思考（Overthinking）：

  

*   即使是简单的数学题，模型也可能生成数千个 token 的推理过程，其中包含大量回溯、自我验证和冗余步骤；
    
*   推理轨迹的长度往往远超最终答案，导致推理延迟高、计算开销大，严重制约了实时应用。
    

  

Question：这些冗长的推理链真的全部必要吗？

  

关键发现：推理轨迹存在大量冗余

  

为了回答上述问题，团队设计了一组实验：在 DeepSeek-R1-Distill-Qwen-7B 模型上，将推理轨迹进行不同粒度的删减 —— 随机跳过一定比例的 token 或推理步骤，然后观察模型能否仅凭残缺的推理链给出正确答案。

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqFMNHlulDAhG86gTJdajCLxqlIGTKoOib6hX3b0XnTF2Ry7EvF8kKWckEHgibld31GqHALwTvicPtPgHrQoyyRibPShyicFyK1kLJ3c/640?wx_fmt=png&from=appmsg)

  

核心洞察： 即使随机丢弃 50% 的推理轨迹，模型准确率仅下降约 2 个百分点。这可以得出两个结论：

  

1.  当前的推理轨迹中存在大量冗余信息，远超正确推理所需的信息；
    
2.  推理模型具有强大的信息过滤能力，即使面对残缺、高困惑度的推理链，依然能从中提取关键信息并总结得出正确答案。
    

  

这一发现直接启发了团队的核心思路：既然完整的逐步推理链并非必要，能否用一种更紧凑的隐式表征来替代它？

  

方法：隐式思考模型 LRT

  

基于上述洞察，团队提出了 Latent Reasoning Tuning（LRT） 框架。其核心思想可以概括为：用一个轻量级推理网络，将显式的推理链「编码」为固定长度的隐式向量，直接注入大模型即可生成最终答案。

  

技术架构

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqEBYulFyiblbmy72fEiamQOaK25CPXibxgubmIROicPpSibdRdGibDtxGWbm9648jMONxPJibHbo130ae7kDEMCup0wxnmS7OcicDfOBpA/640?wx_fmt=png&from=appmsg)

  

如上图所示，传统推理模型需要逐 token 自回归生成整条推理链（Decode → Decode → ... → Decode），而 LRT 的流程为：

  

1.  输入编码： 将问题输入大模型，提取其隐藏状态表征；
    
2.  隐式推理： 将隐藏状态送入轻量级的推理网络![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqHtnloTdJFpD1cA9XQULPfevRELKGFrblqquDjXbVmznSQQ5WAPSA85kGwmPcF0tZnSBeFETr6ab8Zykrz2hZ2OhJMtsob0HgY/640?wx_fmt=png&from=appmsg)，一次前向传播即生成固定长度的隐式推理向量；
    
3.  答案生成： 将隐式推理向量与问题 (编码向量) 拼接后送入大模型，直接解码生成最终答案。
    

  

具体方法

  

对于一个推理型大模型![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqGahb5fibj5cZrVl6HUTFkd6ic4E2JSLP1YEky90ibibia4paWm7wSR3pzdKatLgYpxZa63J7TYpRDsFPlUHDvbYFbNtLrNuicwuZUWQ/640?wx_fmt=png&from=appmsg)，将输入提示定义为：![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqGvwF6vZgLzSqxtxAdgFDQNrP7mjCrOurFz7Eia8Q2OhjnbVRMYnicFkkuqO5com0zDFkyk41FCYGTbfFPvUA0dEDQ5UGxLE1R0s/640?wx_fmt=png&from=appmsg)，推理轨迹为：![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqE68AhRqTcO5WGVxibDpjzzsicdN24SKl8icpm6G7NMTOcCK5xyDgNHOsXP89DCsIEp4wPLJP3hcapoonA0h3JZzWBSErxXmsticYo/640?wx_fmt=png&from=appmsg) ，最终答案为![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqHJOfc8I9fP5udI5N4Se9JPslpyuyG36LkgiaW6M4YsAIia5q6urbhStSibqCk2f4WRibZgibdK76DTqNUiaQiawLWfnRNribdSiavhiat90/640?wx_fmt=png&from=appmsg)。

  

通常![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqGpicvyCWaiag1BFvkIRE8MDPNswndSz1BRtaeLvia5ibUfsksyLkfoib39dm9e8dgtSo6qbH0wI4V6vSiatBZy2piaCic01ia3PuYGPLZU/640?wx_fmt=png&from=appmsg)，即推理过程所消耗的 token 数量远大于最终答案的长度 —— 这正是推理开销的根源所在。

  

模型的推理生成过程可分为两个阶段：

  

*   Prefill 阶段： 模型一次性处理输入提示 X 中的全部 token，初始化注意力状态（KV 缓存）。这一阶段是并行的，计算效率较高；
    
*   Decode 阶段： 模型逐 token 自回归地生成输出，每一步仅基于前序 token 与缓存状态进行推理。这一阶段是串行的，也是推理延迟的主要瓶颈。
    

  

在 Decode 阶段，思维链的生成过程可以形式化为：

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqFs0dPMFTVribJeiadb0qNonMY4ASyzf0DHqBrER5jQ74TwnvndCFKCp1ycYRxmhaTE5gd4IY3NOa2Uib5nTXqeaJPdsr4r0R4Dds/640?wx_fmt=png&from=appmsg)

  

其中![](https://mmbiz.qpic.cn/mmbiz_jpg/5L8bhP5dIqHs5gpdu2AM4ywycXQpuMMxnDYDcIMZKhD5t81tfpFUyVtuTwYyThSBOXN7Ynu1YyMXJhRxc8XkJdpiaehff89c4Oia3GTP9OEzg/640?wx_fmt=jpeg)表示序列拼接。

  

注意到，在给定模型参数![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqFVDth81fUQFWdyn1uhXjiaM3KjNmVReBYJaSI66hmLq0kW1fNiadibgibTmbVDcZXoZu6vTURktnxrEKjrS6s5SEyXkrwBjicQ2ib40/640?wx_fmt=png&from=appmsg)和输入 ![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqEBAlNQqAVZiaDH4fcqTbC9DOvsR65KFjYRv7DpHohwaQhqgCmXu5xiaZiblX4SuSHibtylqTPshiaYWMA3icw4l4q7VwZrOoSlY7GEc/640?wx_fmt=png&from=appmsg)的条件下，推理轨迹![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqF40XgS1PV1QGqxaRF8Q1ta3icibux8ywsfmiaHIBpyu8dy6TgIWtRC0Lw2y2tjtccVwW5kj3Fzo4sRmTx0icq0nLWcapXicXHib3MG0/640?wx_fmt=png&from=appmsg) 完全由模型的输出分布决定。

  

因此，推理轨迹可以表示为输入与模型参数的函数![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqG6WyJHaBjQicDB8Miaiaia1XR7CNVlB6BK1dl3BlSIXLPe48fBIjPfr6DQAMzXJE1RKg7UpSCtgKo3tK6czcvKpHd8Y7u1ARZN8o8/640?wx_fmt=png&from=appmsg)。相应地，LLM 先推理再总结得到答案的完整过程可以写为：

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqEt7vMap1WQKKjVoqHy1LKNDwrO7vhPIoAcbs0iaPRF2ww0erMz83IPvH4ojZRB6Sgu86LN2lMN8Be99SC1L8IrziaWC6qUpIFqc/640?wx_fmt=png&from=appmsg)

  

既然推理轨迹是输入的函数，而已经证明了它存在大量冗余，那么一个自然的想法是：使用一个更高效的函数来替代它？

  

基于这一思路，团队引入一个轻量级的推理网络![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqG1M6fapavCicbk6TLSDbIyqdoczt3ibqkanlXDg9yGuMUsQgqYUewZd7ug4XLGO4RV1Ep2nduNgD94xVBxKjfhqhtmwDcSBeGp8/640?wx_fmt=png&from=appmsg)，直接从输入映射到紧凑的隐式推理表征：![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqEdMl5th6XEDoW2ll1vQtPrbicCicLZbJBFg3e6vibJIHRW1DywrCL2aaxMRk7IodiaFmhqqGrezz7LGygEFXfMiaEUoKLbP6yMDb1g/640?wx_fmt=png&from=appmsg)

  

隐式表征![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqEvnIwWEI3iazPztbtf9eyXaEFWomvMVSRwhYxYdkftYFibIE6tH6ibFdkf4gJ2richVWlicQV20d4ehgbzZykYia4G2H8qXFfib81C9w/640?wx_fmt=png&from=appmsg)以固定长度的连续向量序列替代了原本需要逐 token 解码的冗长推理链，从而将「Decode」过程转化为高效的「Prefill」操作。

  

在训练阶段，我们采用两阶段优化策略。第一阶段通过监督微调（SFT）优化推理网络参数![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqFxLn2QbicN5X1DhYFL2ZNoL8gTsygMGccWhjSdwmJD5NeO0IRGxMB1WLJQviaiatj6VFQ93DpwVBFNNXUf5icQFC0MPrPIWF29GLw/640?wx_fmt=png&from=appmsg)，最小化负对数似然损失：

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqEzia1niajnMWpibbH6hUEMTFltAlF3NMn726cJKGp7LPGS7DzqDuxnLGwFRD2elLbMChbplsNSI1KiaEOd8ev7qGVuuTSOnOMZQ2s/640?wx_fmt=png&from=appmsg)

  

该目标鼓励![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqG1M6fapavCicbk6TLSDbIyqdoczt3ibqkanlXDg9yGuMUsQgqYUewZd7ug4XLGO4RV1Ep2nduNgD94xVBxKjfhqhtmwDcSBeGp8/640?wx_fmt=png&from=appmsg)生成的隐式表征能够引导冻结的基座模型![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqGahb5fibj5cZrVl6HUTFkd6ic4E2JSLP1YEky90ibibia4paWm7wSR3pzdKatLgYpxZa63J7TYpRDsFPlUHDvbYFbNtLrNuicwuZUWQ/640?wx_fmt=png&from=appmsg)正确预测最终答案 ![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqHMaPDo0PoUjrvqjJD6g0tibUKX2icJTrLuhVOPXicNniaHt5SSl4fVraYAYHUjYG30GHJl02SZqW4heibYdMA9Ialgib4O4FrVyCQU8/640?wx_fmt=png&from=appmsg)。第二阶段进一步采用强化学习（GRPO），以答案正确性作为奖励信号，激励推理网络在隐式空间中探索更优的推理路径，从而突破训练数据质量的瓶颈。

  

实验结果

  

1\. 高效思考 —— 在不同 Token Budget 下表现最优

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqHVx7zmiaD8uYlxrxd1wWvEtNT5Umdicu0n0fFZKfxpn2tbBibakxC4ZIgf9jDQeVfziaepKlqf4vticD0kCEqmia2oPDhFJ3QGgU4oc/640?wx_fmt=png&from=appmsg)

  

在 DeepSeek-R1-Distill-Qwen-1.5B 上，与多种高效推理方法进行对比：

  

*   在不同的 Token 预算下，LRT 在域内和域外任务上全面领先。在 512-Token 下，其平均准确率比 NoThinking 高 2.66%，比 RL 类方法 ShorterBetter、LC-R1 分别高 5.90% 和 4.74%。
    

  

2\. 混合思考 —— 超越 Qwen3 原生混合思考模式

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqGdB4NkvyGr74jZrEyletVUXWdjuiawFXL4pfgYsk2yNVAQKgh8A27RVUrwzd4Gibqebu9Z34bTwVFWiaoqwqg1dzPv1daIl6aBn4/640?wx_fmt=png&from=appmsg)

  

LRT 的模块化设计天然适合作为混合推理的新范式：面对简单问题使用隐式思考快速作答，面对困难问题切换回显式慢思考深入推理。在 Qwen3 系列模型上验证了这一能力：

  

*   在 Qwen3-4B 上，LRT 的 pass@4 平均准确率达到 71.60%，比 Qwen3 原生非思考模式高出 5.82 个百分点！在 GSM8K 上提升近 7%，在 LSAT 上提升超过 14%。这表明隐式推理向量能有效引导模型产生更多样、更高质量的解题路径。
    

  

3\. 推理效率对比

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqFe4U2SQ0rt5mnwOIicScfQtZ1olFriaUvWXlt5seJu3ib8ROYlxAAw4lrOiaAS7X1RxD5fGxolknDj9LbcWXwsLjjcLCkM6TzDTAE/640?wx_fmt=png&from=appmsg)

  

*   LRT 的推理延迟显著低于 Qwen3 的混合思考模式，甚至比非思考模式还快 —— 因为隐式推理向量引导模型生成更简洁的答案，减少了解码步数。
    

  

4\. 消融实验分析

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqFgyMqGDjqoyFfREja2eZ0KvSficjI1Ltbgg8U492u6rpcKMI1E01MXkE1yWhgFM9iaZmwDm4ZRJNgnibTzxVMpaSibnxH1Bib2ntco/640?wx_fmt=png&from=appmsg)

  

隐式推理 token 数量的影响： 随着隐式 token 数从 64 增加到 256，性能稳步提升（42.53% → 48.42%）。

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqE6aqNDa9p4rpeW9Ma4DRmfnSmSVW2WLyIzsnCWeW7Nae5lxAsEDK8Ily0FkD4mHfggIVbmwEQx9LWT5NWz9nmAKupxrKbF09U/640?wx_fmt=png&from=appmsg)

  

两阶段训练： 加入 RL 阶段后，域内任务平均提升约 9%，域外任务平均提升约 4.3%，验证了强化学习对隐式推理优化的关键作用。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqGfaGruGBAsePHDC9oUCicdwwys7cFGo0vGwEsxKYcWyVqDfy6SiatRqrXpfG5EXbBMv0HSaUibKaS11sH3lO8X6PFibLF0vQeZp8I/640?wx_fmt=png&from=appmsg)

  

更大的基础模型（如 Qwen3-8B）则能充分利用更多的隐式 token（使用 512 个隐式 token 时结果仍能提高），说明隐式推理的「容量」与基础模型能力正相关。

  

总结

  

隐式思考模型 LRT 开辟了一条全新的高效推理路径：

  

*   理论洞察： 系统性地揭示了推理轨迹的高度冗余性，证明完整的逐步推理链并非正确推理的前提；
    
*   高效思考： 通过将显式推理链压缩为隐式向量表征，用单次前向计算替代数千步自回归解码，大幅降低推理成本；
    
*   混合思考： 模块化的即插即用设计，无需修改大模型参数，可在隐式推理与显式推理之间无缝切换，为混合推理系统提供了更优的替代方案；
    
*   效果优异： 在数学、逻辑、科学等多类基准上全面超越现有高效推理方法，并在 Qwen3 系列模型上超越其原生非思考模式。
