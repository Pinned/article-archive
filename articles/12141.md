# 上周「封杀」OpenClaw，这次直接「封杀」创始人？Anthropic：误会，已解封！

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gW889cR13aBX42evqQIRibKlicoCrHPEpT0tQiceNphESCa2eJTqstP8G0yqMTkeMFrOGue6kOyCKdTkA/640?wx_fmt=png&from=appmsg)

机器之心编辑部

  

Anthropic 与 OpenClaw 的「爱恨情仇」到底要上演到何时？

  

上周，[Anthropic 刚官宣「封杀」OpenClaw](https://mp.weixin.qq.com/s?__biz=MzA3MzI4MjgzMw==&mid=2651025819&idx=1&sn=08dac87a098d07bdba1f7e578e2a8bc6&scene=21#wechat_redirect)，这周，就直接将「OpenClaw 之父」Peter Steinberger「封杀」了……

  

今天一大早，OpenClaw 创始人 Peter Steinberger 就在 X 上发文：「是的，各位，未来要确保 OpenClaw 继续与 Anthropic 模型兼容将会更加困难。」

  

此外，还附上一张来自 Anthropic 安全团队的邮件图片，邮件称，Peter Steinberger 的个人 Claude 账号因「可疑」活动被暂停。

  

虽然只有短短一句话，但也能看出 Peter Steinberger 的「怒气」与「无奈」。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqEBg6q9YeFGCwtLofibKbYdjSo7R00fWibvhOn3HHRiatNfyCbcad7MIuYzkOyEkuBVpdp4rYJ5ndkfibnaxwC5KkwmAFm42nPficV4/640?wx_fmt=png&from=appmsg)

  

以下为 Anthropic 的邮件全文：

  

> 你好，
> 
>   
> 
> 我们对与你的账户相关的可疑信号进行了内部调查，结果表明你违反了我们的使用政策。因此，我们已撤销你对 Claude 的访问权限。
> 
>   
> 
> 如果你希望申诉，请填写此表单，或在此了解更多关于申诉流程的信息。
> 
>   
> 
> 此致
> 
>   
> 
>  Anthropic 安全保障团队（Safeguards Team）

  

但「封禁」时间没有持续太久，仅仅两个小时后，Peter Steinberger 再次发文：「我的账号被恢复了，谢谢大家！」

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqHia6AJzjWnYY1nibRXb2zGcPvficFgfKiaibmC00R1MeZSVtO26E49xLauDyokDnZPY9En1eY8VjqvBvRK9znAQEKXMib1viaHYlz4uM/640?wx_fmt=png&from=appmsg)

  

不过，账号是解封了，但是事件引起的余波还在继续。自从 Peter Steinberger 发文后，热度就在不断上涨，贴文下面各种阴谋论、质疑声满天飞。

  

首先是大家觉得 Anthropic 的做法「太不厚道」。

  

刚开始的时候，有网友觉得也许是因为 Peter Steinberger 真的违反了规定才被「封禁」，但了解后，发现不是那么回事。

  

「是因为你违反了他们的服务条款、以不被允许的方式使用 API，才对你动用了 CPS（Claude Protective Services）？那当然会这么做，这就是他们的职责。按照条款，这类账户本来就不允许通过第三方工具来使用服务。」

  

但 Peter Steinberger 回答是，「这是通过 API key，并使用官方的 Claude CLI 进行的一手调用。」

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqHOWHOjkOPOE4wlDJ9LlmWUx8UdQCjNVFonzXJlYW304Ona26v2kGT707nDIUD0fMHzCrHicGye3EMqcPIYDmg7icg1lp0mXYNw4/640?wx_fmt=png&from=appmsg)

  

还有网友觉得，也许是 Anthropic 搞错了，问「这次 Anthropic 封禁的是不是 API 账号？」

  

而得到的回答是「Oh  they blocked both。」（他们两个都给封了。）

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqFRKTTgogvZibWibaGznJm8sfLOQdtib8eaPabAm67r0qnn2jnmIHD93eeh1z3yw45l3NdaXEtfODJmfyicnybiaUKAIG0whz3yU12c/640?wx_fmt=png&from=appmsg)

  

而根据 Peter Steinberger 的说法，也就是说，自从上周 Anthropic 发布新规后，他就一直在严格遵守，使用官方 API 模式，并向账户中预存了资金，从「薅羊毛的开发者」真正转变为「真金白银的付费客户」，但仍然遭到封禁，搞得他现在「连测试都没法做了。」

  

当然，也有人质疑，既然 Peter Steinberger 现在都去 Anthropic 的竞争对手 OpenAI 任职了，为何还在用 Anthropic 的产品？主要是用来做什么？

  

Peter Steinberger 也觉得冤枉，自己所做的一切还不是为了保障OpenClaw的跨模型兼容性？

  

「我们一直在跑端到端测试（e2e tests），确保对 Harness 的改动不会在主流模型上出问题。这基本就是我们使用 key 的用途。」

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqHKjFOHax7y0oCrjP70UX8zeF09KcyAZKgW9ia1WrEJ6hlM3TJEQLjYEZxovtfJuWCOe3RZf7kSaEJMpCpVDJcoWgoDGaoYTlGE/640?wx_fmt=png&from=appmsg)

  

另外，他还表示，这并没有什么冲突。

  

「你需要把两件事区分开来：我在 OpenClaw Foundation 的工作，是希望让 OpenClaw 能够很好地兼容所有的模型 / 提供商；而我在 OpenAI 的工作，则是帮助他们制定未来的产品战略。」

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqEHogV5PRFiap5aovTTwHsUrKUc8M1kfbYtOstWvibzzxdZ6no2rHglNVSWSlMYOSfXhBoicARchBgsjGStMQrwHQ1MDNBV2La2ZU/640?wx_fmt=png&from=appmsg)

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqHezrVaw0BkdRcBbykY3v7ONcqgJD8UhKBFyQnrAsuQofiaNdwB7mQbHHk7h8UgGGQfRjIhFiah9SNibLNib4mThZE2Gn4cBenTHZM/640?wx_fmt=png&from=appmsg)

  

甚至，有网友感慨，如果当初没有做出如今的选择，可能事情就不会发展到现在这个地步，「你本可以选择，但你跑错了.....」

  

Peter Steinberger 也是毫不犹豫地反击：「他们一个欢迎我，一个发法律函……」

  

没说出来的内心潜台词大概是：这下你总该知道我为什么选择 OpenAI 了吧？

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqEeaSR7L2Hth2ga5rS55vrkia38zR7eTH3CYLKRnKTobQ1TxsRIFASbS6JM3406VDf7hiasv9T7OXuAmDvBJmrwGNk3XbIz4unfQ/640?wx_fmt=png&from=appmsg)

  

随着帖子热度的高涨，之后，Anthropic 的一位工程师 Thariq Shihipar 也「入场」了。他表示，「我们并没有因为使用 OpenClaw 而封禁任何人。这很可能是触发了另一个有点过于敏感的滥用检测分类器。我会给你发私信，我们一起看看情况，帮你解除封禁。」

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqESvyosiaKndgIBDjtDZdw96HkRO9kXO7LVwZIvsUCN5UaB42ZhfcZuJYfS8xxzUSCXdA5ibk3abicib532bPOAkiaVFN78vh1dOsib8/640?wx_fmt=png&from=appmsg)

  

也不知道到底是不是 Thariq Shihipar 的帮忙，还是本身这次的「封禁」就只是个乌龙，总之，在「封禁」两个小时之后，Peter Steinberger 的账号就得到了恢复。

  

整件事情到这里也告一段落了，但是不可否认，双方之间的裂痕又变大了。

  

而之所以网友会觉得此次封禁不是误会，是有意为之，站在 Peter Steinberger 的视角看待问题，其实是存在着一个残酷的现实，在大模型时代，开源软件的「生命线」还是握在模型厂商手中。

  

不管是上次的「封杀」还是这次的「封禁」，厂商随时可以通过修改服务条款等方法来「整顿」一下……
