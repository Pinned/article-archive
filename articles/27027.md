# 刚刚，Claude进入美国版飞书，成了我的AI新同事

![](https://mmbiz.qpic.cn/sz_mmbiz_png/8Eyv4o4AcupV0oqQOESNSicmEcs0wuibbUDYlibqGiaCbXmOicvxf1dVBFSVrwI0RiakUSzQmWqNmdlxiaZr6gib5GvR9fud5xSVjw4LO3pXbUAickL8/640?from=appmsg)

**Claude进群变身“打工人”。**

**编译 |**  杨京丽  

**编辑 |**  李水青  

智东西6月24日消息，今天凌晨，Anthropic推出**面向企业团队的群聊AI协作助理****Claude Tag**，首站接入企业通讯平台Slack。用户可以在Slack中@Claude，让它**读取上下文、调用已授权工具**，并在团队协作场景中处理任务，目前Claude Tag调用的是**Claude Opus 4.8模型**。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/8Eyv4o4AcurEjQcxexSmRt35icdlHcNiamJibv2NDfuyAR6iaTrCmh1EPgF62Y9esvB5tCTg4sibQ37iaNgCAViakAYsE7qtdcowyiceyXskY1GOfT4/640?from=appmsg)

▲Anthropic推出Claude Tag功能（图源：X）

Anthropic称，Claude Tag是Claude Code功能演进的新形态，目标是让Claude从个人助手，变成**可被整个团队共同调用的协作成员**。它可以**随着时间积累频道上下文**，逐步**沉淀团队知识**，减少用户反复解释背景的成本；在获得授权后，Claude还可以**连接其他频道和数据源**，主动识别相关信息、跟进搁置任务，并在**数小时甚至数天内自主推进项目**。

目前，在Anthropic内部，@Claude已然成为其推进工作的重要方式，Anthropic产品团队**65%的代码由Claude Tag内部版本创建**。除工程场景外，Anthropic员工还会用它**追踪产品指标和数据、处理支持工单，或协助定位复杂Bug根因**。

前段时间刚加入Anthropic的“AI大神”**安德烈·卡帕西（Andrej Karpathy）**在社交平台X发文，称赞这一功能：Claude Tag代表了一种**新的Claude交互范式**，能让Claude**更深入地嵌入组织内部的日常协作中**，这是大模型用户界面与交互体验（LLM UI/UX）的**第三次重大重新设计**。

**_01_****_._**

**Claude Tag率先接入Slack**

**面向企业和团队用户开放测试**

由于Slack是团队与AI协作的自然场所，同时Anthropic很多日常工作也在这里发生，因此Anthropic选择首先在Slack上线Claude Tag。今天起，Claude Tag面向Claude Enterprise和Team客户开放测试。Anthropic后续计划进一步扩大Claude Tag的可用范围，让团队能在更多的工作场景中@Claude。

@Claude可以**读取线程、理解上下文，并实时响应**，进而让团队成员在同一频道内继续推进工作。用户可以将Claude添加到Slack中，在任意线程中@Claude，它都能基于大家已经共享的上下文开始处理任务。

![](https://mmbiz.qpic.cn/mmbiz_png/8Eyv4o4AcuomDibrVPPpgRBAiaCmNicUJgwyvqCWdOPaOx60HgVtCshicl5oYCzoztiaGgHp9zL7vaV6B2QJfOGzDgHj9fAEcyPlHVuwSZk4aN8w/640?from=appmsg)

▲Claude参与团队协作任务（图源：Anthropic）

Claude Tag将取代现有Claude in Slack应用，管理员可在30天内选择迁移。Anthropic会向符合条件的企业和团队，发放启动额度，Claude Tag目前调用的是**Opus 4.8模型**。

  

**_02_****_._**

**@Claude参与团队协作**

**随时间积累上下文、主动跟进任务**

如果用户此前使用过Claude Code或Cowork，会比较熟悉Claude Tag的交互方式。用户只需在Slack中**用自然语言@Claude提出请求**，Claude会把任务拆解为若干阶段，**调用已获授权的工具**逐步完成，并在Slack线程中回复最终产出。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/8Eyv4o4Acup3UlNeythJMP2Ps2eB9uaJzHhMichU97EvJ3ia93Kuc7pFqnb76ON3ygg3FuXUxoMSMd0X6zkZb8uP7ReFaLsG49S9fCl66pChg/640?from=appmsg)

▲用户用自然语言@Claude提出请求，Claude即可执行任务（图源：Anthropic）

与单独对话或者单次任务不同，Claude Tag更强调**“多人协作”**。在同一个Slack频道中，Claude可被所有人共同调用，频道内其他人能看到它正在处理什么，也能接续前一个人的对话继续推进。因而，它更像是整个团队的协作队友。

Claude Tag还会**随时间积累频道上下文**。它会跟随所在频道的讨论，逐步形成对团队工作的理解，用户无需反复从零解释背景。如果获得权限，Claude也可以从其他Slack频道和数据源中自动学习相关信息（不会从私聊中获取信息）。Anthropic认为，这类隐性知识能帮助Claude产出更贴近实际工作的结果。

在开启**“环境感知（ambient）”行为**后，Claude还能**主动向用户更新它认为重要的信息**。它会从已加入频道和已连接工具中识别相关内容，提醒用户注意，也能跟进那些中途搁置、尚未解决的线程或任务。

Claude Tag还支持**异步工作**。用户分配任务后，可以转去处理其他优先事项；**Claude也能为自己安排后续任务**，在**数小时甚至数天内**自主推进一个项目。Anthropic称，公司内部，大家现在习惯委派Claude处理多个并行任务。

此外，用户也可以向Claude发送私信。此时，Claude会私信回复，并使用用户个人配置的工具和连接器。

  

**_03_****_._**

**@Claude可整理线程、查询数据**

**还能修Bug、准备会议**

Claude还列出了@Claude在团队场景中的几类典型用法。它可以帮助用户快速整理长线程，提炼已做出的决定、仍待解决的问题和相关负责人；还可以查询数据，并在频道中发布指标、基准和图表，方便团队继续追问。

工程场景中，用户可以@Claude，让它**基于线程中的Bug报告修复问题**，并创建草稿PR。会议准备场景中，用户可以让它在会前**汇总CRM记录、近期线程和通话历史**。团队也可以让Claude**监控频道、整理待办、分诊告警**，并只在需要人工决策时提醒用户。

![](https://mmbiz.qpic.cn/mmbiz_png/8Eyv4o4AcuoPcDNIUiawXsTP7EOzSdeHPxzTibrzj5n0AAGKjkZQMOHJvA1uF9ag9b5ZwySBj4MNYozsfhGUFEovJLoXnticUOX2aKW9ekfyvQ/640?from=appmsg)

▲Claude跟进报错，并创建PR提醒相关负责人审查（图源：Anthropic）

此外，@Claude可以接收**长期任务或固定指令**。用户可以让它长期监控某个频道、生成周报、标记紧急事项，或提醒合适的人处理问题。Claude也能在任务推进中跟进、请求输入，在完成后能够返回结果。

![](https://mmbiz.qpic.cn/mmbiz_png/8Eyv4o4Acup9AWx8HWSehuGiarw6BwMXoibfbgtQsiaqswA9DlxhH2Fb1wzu5zS3uCCIicvIggH46TWlK7nzkrGbJt6LHUOjnDbSAC8QLaSsS3w/640?from=appmsg)

▲为Claude设置长期任务（图源：Anthropic）

  

**_04_****_._**

**管理员可按频道限定权限**

**设置工具、记忆和Token上限**

Claude Tag面向团队和组织设计，管理员可以为每个频道**配置Claude可访问的工具和信息范围**，相当于在不同的工作场景中设置独立的Claude身份。不同的Claude都有自己的记忆和权限边界。

例如，面向销售工作的Claude不会把记忆传递给工程场景中的Claude，也不会让工程师访问销售数据或销售工具。

在权限设置完成后，组织成员即可开始@Claude。管理员还可以为组织整体和单个频道，**设置Token支出上限**，并**查看@Claude已执行任务的日志**，包括每项任务由谁发起。

@Claude的上手流程包括四步：用户将Claude Tag与Slack工作区配对，为Claude授权访问工具，设置组织月度支出上限，并先在私人频道中测试Claude是否正常运行。用户完成以上步骤后，即可正常使用该功能。

  

**_05_****_._**

**结语：越来越多的AI助手**

**正在走进团队协作流**

Claude此次更新，将AI助手放进团队已有的沟通和协作链路中。它可以读取频道上下文、调用授权工具、持续跟进任务，也可以由管理员按频道设置权限和预算，让AI从个人助手，进一步走向“团队共享、组织管理”的协作场景之中。

从Claude Tag可以看出，企业AI正在从单点提效，转向更贴近真实工作流的协作形态。未来，AI能否真正融入企业日常，不只取决于模型能力，也取决于它能否理解团队上下文、遵守权限边界，并在合适的位置推动任务继续往前走。
