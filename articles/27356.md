1M 和 长程 Coding
--------------

GLM-5.2 的核心升级，大致是两件事儿：1M 上下文、长程 Coding

**1M 上下文：很多模型**标称 1M，但很多其实在 30 万 token 之后开始丢信息。GLM-5.2 在注意力结构层面做了一组改动，做到「真正可用的 1M」

长程 Coding：现在开发，基本就是任务丢给 Agent，然后等着收菜。GLM-5.2 能让 Agent 经历数千次工具调用而不崩

  

简而言之：这个模型在 Coding 上继续国产第一、开源第一

API 下周上线，模型下周正式开源，MIT 协议

Coding Plan 全量用户今天就能切过来，包括 Lite
