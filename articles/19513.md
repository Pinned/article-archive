Valkey 是什么？

如果你是第一次听说 Valkey，请允许我用最简短的语言告诉你：

> Valkey 是一个开源的、由 Linux 开源基金会托管的高性能内存型键值数据库（in-memory key-value store），完全兼容 Redis OSS 7.2.4 的协议与 API，采用 BSD 3-Clause 开源许可证。

它诞生于 2024 年 3 月——当时 Redis 调整了开源许可证（改为 SSPL v1 / RSALv2 双许可证，均为"源代码可用"类许可证），Linux 开源基金会随即托管了一个新的 fork 项目，并以 BSD 许可证继续开源。仅仅 8 天之后，Valkey 项目正式成立；3 周之后的 4 月 16 日，Valkey 在西雅图举办的北美开源峰会（Open Source Summit North America 2024） 上正式发布首个版本 7.2.5 RC。

Open Source, Forever

如果你打开 valkey.io 首页，会看到那行大大的标语：

> FAST. RELIABLE.
> 
> OPEN SOURCE, FOREVER.

紧接着的一行说明是：

> "The project is backed by the Linux Foundation, ensuring it will remain open source forever."
> 
> （本项目由 Linux 开源基金会支持，以确保它永远保持开源。）

![](https://mmbiz.qpic.cn/mmbiz_png/1who6iblk2qpeNocUibyUUa9clSPYDcBRlxSRUs1s7J8Gea2DSW1ticibicicwmIicx5lJic3ic0LYjZMpcicTiak5ChRCLIPmHjloPaf4vfsWrofRoib8o/640?wx_fmt=png&from=appmsg&watermark=1&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=2)

这不是一句口号，而是一份制度层面的承诺——Valkey 是 Linux 开源基金会旗下的开源项目，任何一家单一商业公司都无法在未来某一天把 Valkey 私有化。

这也是 Valkey 留给所有开发者的一份根本性的安心：你今天选择它，十年后它依然会是开源的。

为什么 Valkey 值得关注？

在过去两年多里，Valkey 已经走得比许多人想象得更远：

*   版本快速迭代至 9.1：从 7.2.5 起步，陆续发布 8.0（多线程 I/O）、8.1、9.0、9.1。其中 9.0 在集群能力上做了大量打磨——针对大规模部署场景显著加快了故障转移（failover）速度、实现了集群槽位的原子性搬迁（Atomic Slot Migration），让大规模集群的稳定性和运维友好度都上了一个台阶。
    
*   性能持续提升：在官方和第三方 benchmark 中，Valkey 8.0 之后的多线程 I/O 在多种场景下显著提升了吞吐能力；9.x 在延迟与内存效率上也有持续优化。
    
*   全球云厂商全面跟进：AWS ElastiCache、MemoryDB 已默认提供 Valkey 选项；Google Cloud Memorystore、Oracle Cloud 同步支持；国内方面，腾讯云分布式缓存（Tencent Cloud Distributed Cache，TCDC） 已率先上线对 Valkey 引擎的正式支持，是目前国内唯一一家产品级别支持 Valkey 的主流云厂商。
    
*   社区规模持续增长：GitHub 星标数、月度活跃 PR、Contributor 数都已稳居开源数据库第一梯队。
    
*   迁移成本极低：兼容 Redis OSS 7.2.4 协议与 API，绝大多数业务可以零代码改造完成迁移。
    

对中国开发者而言，这意味着三件事：

*   选型层面：Valkey 是一个 License 干净、性能持续提升、生态开放的键值数据库选项。
    
*   迁移层面：已有的 Redis 7.2.4 及以下版本业务，可以平滑迁移到 Valkey，无需重写代码。
    
*   贡献层面：Valkey 仍处于快速演进期，社区对中文贡献者非常友好，现在加入仍然是一个不错的时间点。
    

中国开发者在 Valkey

这是我特别想单独讲的一段。

从 Redis 时代说起

Redis 社区的最高技术决策层长期由 5 位 Core Team 成员组成；2023 年 11 月，Redis 社区为承接更多外部贡献，首次启动了 Committer 机制，新增了 5 位 Committer。也就是说，在 2024 年 3 月之前，全球被 Redis 官方正式认可的核心贡献者总共只有 10 人。

这 10 个人中，有 2 位来自中国：

*   赵钊（@soloestoy，阿里巴巴）：Redis Core Team 5 位成员之一。
    
*   朱彬彬（我本人，@enjoy-binbin，腾讯）：2023 年 11 月 Committer 机制启动时，首批 5 位 Committer 之一。

从北京到中文社区

如果说 Valkey China 微信公众号的开通是一个起点，那它其实并不是孤立发生的——它是过去几个月一系列动作里很自然的一环。

让我把这条线连起来给你看：

2025 年 12 月：  
Keyspace Beijing 峰会

2025 年 12 月，Valkey 中国首场线下技术峰会 —— Keyspace Beijing 在北京举办。这是 Valkey 全球 Keyspace 系列活动的中国首站，也是 Valkey 中国社区正式宣告成立的现场。

那天来自腾讯云、阿里云、字节跳动、张量跃迁、麒麟软件、AWS 等团队的核心贡献者与架构师齐聚北京，做了一整天面向中国开发者的深度技术分享。这是 Valkey 自创立以来，中国开发者社区第一次以"完整建制"的形式集体亮相。

![](https://mmbiz.qpic.cn/mmbiz_jpg/1who6iblk2qo3pttN0Mf3J8go5IZRGEjiceLcUl6lKcN46Y4Ay07zKcK3l8EQ0xI9ZBtXU9XKWpcL6aFRMwlhS4D2PCDnY6cIQpTAFl1HP97M/640?wx_fmt=jpeg&from=appmsg&watermark=1&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=8)

线下汇聚 + 线上沉淀，从北京 Keyspace 到 Valkey China，这是一条关于中国 Valkey 社区的完整 Roadmap。

Valkey China 会做什么？

Valkey China 是一个社区驱动、厂商中立、面向所有中文开发者的开放平台。我们承诺会做以下几件事：

*   📣 一手信息与 TSC 内部视角：Release Notes、RFC、TSC 会议纪要背后的"为什么"
    
*   🔬 深度技术解读：多线程 I/O、主从复制、集群、持久化、脚本引擎等核心特性原理剖析
    
*   🛠️ 实战内容输出：从 Redis 迁移到 Valkey 的实践、性能调优、生产经验
    
*   🎤 组织与联动中文 Meetup：线上线下 Meetup、TSC 成员 AMA（Ask Me Anything）
    
*   📅 统一的 Valkey 中文活动入口：欢迎云厂商、开源社区、技术媒体联合主办；活动预告、会后沉淀、讲师对接，都可以联系 Valkey China
    
*   🌉 搭建双向桥梁：把中文社区的声音带到 TSC 会议上，也把 TSC 的讨论带回中文世界
    
*   🤝 培育中文贡献者：帮助更多中国开发者提交第一个 PR、走通 RFC 流程、成长为社区贡献者——让你的代码，被世界使用
    

如果你正在筹备或已经举办过任何和 Valkey 相关的活动，欢迎联系 Valkey China——公众号后台直接发消息，或在文章下留言"活动合作"，Valkey China 会第一时间回应。

Valkey China：  
为什么是现在？

老实说，做和维护一个开源技术社区从来不是一件轻松的事——它需要持续的内容产出、稳定的活动组织、长期的耐心投入。在过去两年里，中文世界之所以一直缺少一个真正属于 Valkey 的社区窗口，并不是因为没人关心，而是因为每一个有能力做这件事的人，都同时在承担更紧迫、更直接的工作。

但 Valkey China 不是我一个人在维护的社区，它从第一天起，就应该是一个"大家的社区"。我只是那个先站出来按下启动键的人——真正让这个社区活下去、成长起来、长成它该有的样子的，是每一位愿意一起参与的同行者。

我期待的不是"粉丝"，而是同行者：欢迎你一起产出内容、一起组织活动、一起讨论技术、一起监督这个账号。如果哪天 Valkey China 发出了不准确的内容、做了不恰当的判断，请直接指出来——这正是一个健康开源社区的样子。

在这里也想感谢两群人

走到今天，Valkey 社区一直心怀感谢两群人：

一是 Redis 社区与 Redis 公司。 过去十几年里，Salvatore "antirez" Sanfilippo 和无数 Redis 贡献者在技术上奠定了今天的一切；Redis 公司（Redis Labs → Redis Ltd.）则在商业化推广、品牌建设、生态扶持上投入了大量资源——没有这些长期积累，就不会有今天 Redis 这个键值数据库品类的影响力，也不会有 Valkey 这个站在巨人肩膀上的项目。在 2025 年，Redis 也已经把 AGPLv3 加入了许可证组合，重新加入了 OSI 认可的开源软件行列，这是一件好事。

二是全球的云厂商们。 真正把 Redis/Valkey 这样的开源软件推到"每一个开发者伸手就能用"这个程度的，是一代又一代云厂商提供的开箱即用的托管服务、针对真实场景的性能调优、多地域的稳定运维能力。开源社区负责创造，云厂商负责普及——一个只有源代码却没人能轻松用起来的项目，不会走到今天。

Valkey 走到今天，是一整个生态共同托举的结果，不是某一方的功劳。
