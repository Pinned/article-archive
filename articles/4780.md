### 安全体系

\->`DNA 权限控制` 装悟空后必须用钉钉账号登录，远端调用时也必须身份一致，确保悟空是你的个人助理

\->`安全沙箱` Agent 在沙箱环境中运作，能执行的命令、调用的资源、访问的数据都受限制

\->`专属模型` 企业调用模型时数据不会被拿去训练，支持专属实例、专属 VPC、专属一体机

\->`专属技能` 所有 agent skill 都要经过安全验证，只有通过验证的才能使用

### RealDoc 真经：给 AI 设计的文件系统

无招先致敬了 Unix。1969 年肯·汤姆森和丹尼斯·里奇发明 Unix 时确立了几个基本原则：make it simple，everything is file，系统间松耦合

但 60 年后，**操作主体正在从人变成 AI**

人操作文件是结果导向的。打开 Word，写完文档，保存，存的是操作结果。AI 完全不同：它在推理和执行过程中不断读写、不断出错、逻辑回滚、重新再来。每一次试错都是 AI 的知识和经验的生成过程

![](https://mmbiz.qpic.cn/mmbiz_jpg/jXSGuwJvpdiagrfuiaJRmbVTIsUs6o99wo8RtEzicAFmlbdrHaKuibibWtOIRV1lTNBQRKymnicdOlQA9cHFXlXR4zuBF4Ou6ZojJ7PRj4PDKKR18/640?wx_fmt=jpeg&from=appmsg)

RealDoc 真经系统

\->`原子操作` 全面支持 CLI 命令行，对文件的每一行、每一列进行原子层面修改

\->`高速快照` 每秒上千次版本保存，帮助 AI 快速回退和重新执行

\->`全量保存` 会话内容、推理过程、思考、决策、执行上下文全面保存

\->`知识图谱` 最终生成企业级知识图谱，反哺 AI

### DingTalk CLI：从 GUI 到 CLI

无招回顾了操作系统交互界面的演进。1981-1995 是 DOS 时代，1995 年 Windows 95 开始进入 GUI 时代，2023 年大模型爆发后进入 LUI 时代

现在进入 **CLI 时代**。AI 成为操作主体，用 CLI 而不是 GUI 的原因是：如果 AI 用 GUI 操作，它在模仿人的方式，效率低下，耗费大量 Token。CLI 把所有命令进行压缩，AI 可以更准确地理解工具能力，更高效地执行

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/jXSGuwJvpdiacY1sic3AicoCnGafj66LbpiaeialfRYDkUmbWOvR3DIn3VKy0STgx2MnhqcicSWMibeICXRiayTcLbSt2zbgsNXYx93qdiaKvVgib2m4c/640?wx_fmt=jpeg&from=appmsg)

DingTalk CLI

DingTalk CLI 把钉钉所有能力原子化，沟通、协同、软件服务全部拆成上万条命令行指令

底下有八大工程体系支撑：无感认证、语义化输出、结构化体系、安全存储与原子导入等。不只跑在手机和电脑上，还会植入所有可执行体：手表、机床、IoT 设备

### 可编程企业

有了 CLI 之后，无招展示了一个关键变化

以 HR 月度考勤巡检为例：拉花名册、查排班表、逐个检查考勤记录、处理异常补卡、登记异常记录表、在部门群通报、计算薪酬。如果用 AI 用代码执行，这些操作会被转换成一个个循环体，由代码调用完成

企业正在变成 Programmable Enterprise

以前软件工程师把公司流程写成代码，代码固化成工作流，每次改进以月以年为单位。进入 AI 时代，所有代码全部 CLI 化，每次执行的成功与失败 AI 都在学习，重新编排流程，以最高效的方式执行
