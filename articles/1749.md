# MiniMax把自家“实习生”放出来了！

##### 克雷西 发自 凹非寺  
量子位 | 公众号 QbitAI

曾经那些将需求、设计与代码强行割裂的流水线，正在快速消融。  

想拥有一个得力的数字助手，你需要直接成为能独立定义最终结果的Agent Designer。

这种变化决定了Agent的进化方向——它必须深度嵌入你的工作环境，拥有你完整的职业上下文，成为一个真正懂你的长期合伙人。

MiniMax最新上线的AI原生工作台Agent 2.0就为此而来。

它通过Desktop App与Expert Agents两大载体，变成了你的“老搭档”，可以直接调用你的本地素材，熟练地遵循你的行规，帮你把活干完。

这款全新的MiniMax AI原生工作台，现已开启封闭内测。

Agent打通本地云端
-----------

这次AI-Native Workspace的更新，重点就是Desktop App与Expert Agents这两个核心组件，前者注重执行力，后者则侧重于对业务场景的理解。

Desktop App的核心逻辑，就是打通云端和本地，把云端强大的模型能力，直接连接到你的本地电脑上。

它既能顺畅地读取你硬盘里的文件夹，也能自如地操控网页完成各类任务，让AI能够经手具体业务的实际操作。

另外，它还能处理你本地的文件，无论是HR整理本地的人才库表格，法务审核硬盘里的历史合同，还是内容创作者调用深藏在文件夹里的素材……AI都能直接介入这些日常工作流。

你不再需要充当搬运工去投喂信息，Agent就能直接主动调取本地资源，实实在在地帮你把手边的工作推进下去。

为了验证这种能力到底可不可信，我专门设计了一个比较复杂但又契合日常实际的任务。

我假装自己是一位关注Micro-SaaS领域的投资人，将网络上搜集来的热门产品列了个表格，但表格里只给了产品名称和功能。

![](https://mmbiz.qpic.cn/mmbiz_png/YicUhk5aAGtDWTzrpObXlffTEo3yn7e4xGQxEjQnUuMLFJWoZWRibcadYibiauNnwRVe2sbWKOvC9ooSYZaGUhuyLw/640?wx_fmt=png&from=appmsg)

这份文件存储在本地，我不提供任何额外信息，要求Desktop App必须自己联网，去把这20个产品的底细摸透——包括它们靠什么赚钱、核心流量从哪来、以及是否存在“套壳”嫌疑。

查完之后，Desktop App需要把查到的数据填回这个本地表格，并根据这些信息直接生成一份图文并茂的深度调研报告，最后再输出一个可以直接拿去汇报的PPT。

![](https://mmbiz.qpic.cn/mmbiz_png/YicUhk5aAGtDWTzrpObXlffTEo3yn7e4xJaMnqIP0CczElicymVBE9oCv28aJibTQhpAH3YP7JjheS1MJic6Q7WrIQ/640?wx_fmt=png&from=appmsg)

这个过程当中，Desktop App需要紧锣密鼓地完成多项操作，而且这些操作环环相扣，当中涉及的产品多达20款。

来看Desktop App是怎么做的。

首先，Desktop App通过Python程序读取了表格中的内容，之后转换成了ToDo List，并逐一列出了需要检索的产品。

![](https://mmbiz.qpic.cn/mmbiz_png/YicUhk5aAGtDWTzrpObXlffTEo3yn7e4xndhmN5jL3yJU3TjxDEAuvZfRzic3HyjhHhGXiawibzAQMKF5yr4cJpRyA/640?wx_fmt=png&from=appmsg)

接下来的检索过程，Desktop App既没有一口气全部发起，也没有逐个串联，而是通过分组搜索的“串并联混合”方式，既保证了速度又避免了过高的并发。

![](https://mmbiz.qpic.cn/mmbiz_gif/YicUhk5aAGtDWTzrpObXlffTEo3yn7e4xyWjHxVObWlzkbqe64w3k9sfHKKsprlU7t9PH4swKpatljDGRld8ZYA/640?wx_fmt=gif&from=appmsg)

接下来看Desktop App生成的报告，我要求的是图文并茂，而Desktop App选择了Markdown写法，然后尽力用符号拼凑出了一些可视化的效果，也算是符合要求。

如果说Desktop App让Agent更擅长动手，那么Expert Agents则是从另一个角度出发，专注于让AI变得更内行，更能适应具体的任务场景。

这样做的原因也很好理解，通用的模型虽然博学，但在处理极具个性化或高度专业的任务时，往往难以精准对齐你的具体要求。

有了Expert Agents，你就可以把私有的知识库和经验注入Agent系统，得到完全理解你业务逻辑的垂直领域专家。

比如你可以上传一套只有资深从业者才懂的SEO流量玩法，或者一份经过实战检验的量化交易策略，Agent在执行任务时，会调用你赋予它的专属记忆和规则，给出特定标准的高质量产出。

让Agent成为长期合伙人
-------------

为啥要把Agent做成这种Desktop App+Experts的形态呢？这背后其实展现了MiniMax对Agent未来的一些思考。

MiniMax认为，Agent的终极目的是进化为能真正交付结果的长期合伙人，而成功的关键，就在于看它能否深度嵌入工作环境并完整理解职业上下文。

具体来说，这样的核心能力包括三个方面。

最基础的变化在于**记忆的连续性**。

大家都不希望自己的AI助手永远是个“初次见面者”，每次对话结束就把以前的对话一股脑给忘了。

理想的Agent，应该像个老练的同事，能逐渐理解并记住你的工作偏好，比如写报告时的语气，甚至代码缩进的特定习惯，用得越久就越懂你。

在建立默契之余，它还需要具备**对隐性经验的消化能力**。

现实中的每个岗位，其实都有大量只存在于老员工脑海中的经验与规范，这就要求AI必须能把这些行业标准和团队内部的操作流程，内化变成自己的本能。

这样才能实现——当你给它派活时，它能够直接拿出符合团队标准、并直接交付的专业成果。

更进一步的进化，则是**对业务环境的敏锐嗅觉**。

真正好用的Agent不应该是个孤立的接单员，而应该连通到你的业务系统中，时刻感知环境的变化、实时捕捉信号，不需要你专门下令，它就能像神经反射一样在关键时刻主动做出响应。

这也正是区分演示与实战工具的分水岭，只有当AI学会了自己在动态变化的业务流里主动响应时，它才算具备了通往复杂现实商业世界的入场券。

MiniMax的AI实习生，你也能用了
-------------------

硅谷最炙手可热的“AI Native”组织形态，正在定义AGI时代一种全新的工作方式。而在MiniMax，这套未来感十足的工作流早已是全员通用的日常——Agent已经成为了一名独特的实习生。

跟着它的视角走进办公室，Global Business组的Morgan正对着屏幕。她手头刚拿到一份涉及20个不同国家潜在客户的名单，必须针对每个人的背景写出风格各异的破冰邮件。

按照老办法，光是逐个背景调查就得耗掉一天。而现在，她把名单直接甩给了Agent，紧接着Agent迅速摸清了这些人的背景和职业履历，瞬间生成了20封完全定制化的邮件。

设计组的Joanna，正在演示一种很新的“Vibe Coding”。

作为设计师，她直接在编辑器里就能指挥Agent。比如觉得官网字体不合适，她不需要等待前端排期，直接告诉Agent“帮我改一下官网字体”，屏幕上的代码便自动滚动修改，网页瞬间变成了她想要的样子。

产品经理寻鹭，则是收到了用户关于“积分消耗不合理”的投诉，她没有去麻烦研发同事调取日志，而是直接让Agent介入调查。Agent像个老练的侦探潜入后台Trace系统，迅速查明真相——原来是用户上传了37个超大文件导致Token消耗飙升。

在查明原因后，Agent甚至根据公司政策，主动拟定了一个“退还20%积分”的安抚方案并起草了邮件。寻鹭全程只需要回一个“好”字，一桩棘手的纠纷就完美结案。

而在对精确性要求极高的研发部门，工程师们也在经历一场角色的重塑。

在计算平台组，Agent通读了整本晦涩的运维手册，当系统报警时能自动分析原因并给出处理建议。

研发人员现在只需列出需求列表，Agent就能像个Tech Leader一样自动拆解任务，接管绝大部分基础代码的编写与测试。

Agent 2.0的发布，正是将这套与硅谷趋势同步、且在MiniMax内部已经跑通的实战体系，进行了标准化封装，让外界用户也能直接复用这种高人效的新型生产关系。

现在，MiniMax把这位“实习生”送到了你我每个人的手边，刚刚上线的**Agent 2.0**，便是它的化身。

从今天起，你也可以把这位“全能实习生”火速招进你的团队，复刻这种属于AGI时代独有的极致生产力。
