# 离谱：Claude Code让地铁变工位，早高峰发版，打工人还笑得出来？

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gW889cR13aBX42evqQIRibKlicoCrHPEpT0tQiceNphESCa2eJTqstP8G0yqMTkeMFrOGue6kOyCKdTkA/640?wx_fmt=png&from=appmsg)

机器之心编辑部

  

AI 到底有没有彻底抢走程序员的饭碗？至少在 Spotify 看来，临界点已经到了。

  

在刚刚结束的第 4 季度财报电话会议上，Spotify 联席 CEO Gustav Söderström 扔出了一个颇具戏剧性的事实：公司里最顶尖的开发者，自去年 12 月以来就再也没有亲手写过一行代码了。

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqGUwSnBLaQ82PwjhCbGomPHlMIU7PmYE6szareZGxpx1AzIWOLEMBl9Bllx7VG9ZGkrbMHicFicwzOvrhoPDfZ4CAHWRmWYkQEVk/640?wx_fmt=png&from=appmsg)

  

地铁、手机、Slack

  

据 TechCrunch 报道，Spotify 内部目前正在使用一套名为「Honk」的系统。这套系统由生成式 AI（准确地说是 Claude Code）驱动，核心卖点是让代码部署变得像聊天一样简单。

  

Söderström 描绘了一个让任何苦逼程序员听了都想流泪（或想离职）的场景：一个 Spotify 工程师在早通勤的地铁上，掏出手机，在 Slack 里给 Claude 发条消息，让它修个 Bug 或者给 iOS 应用加个新功能。等 Claude 干完活，带着新版本代码的推送就会发回工程师的手机上。工程师只需点个合并到生产环境，打卡进公司前，活儿就已经干完了。

  

这套系统确实在提速上立了大功。Spotify 表示，2025 年全年他们推出了 50 多个新功能和改版，最近几周更是高频上线了「AI 提示播放列表」、「有声书页面匹配」等功能。

  

当然，Spotify 也明白，光靠 AI 写代码是没有护城河的，毕竟 AI 谁都能用。他们的底气在于「独家数据」。

  

Söderström 指出，与其他大模型靠抓取维基百科这种客观事实不同，Spotify 掌握着大量关于「主观偏好」的数据。比如「什么是适合健身的音乐？」，美国人普遍选嘻哈（也有几百万人硬核地选死亡金属），而很多欧洲人听 EDM，斯堪的纳维亚人则偏爱重金属。这种规模的用户偏好数据，是大模型公司拿不到的。

  

在谈到 AI 生成的音乐时，Spotify 的策略则是：允许艺术家和厂牌在元数据中标注歌曲的制作方式，但平台依然会严打垃圾内容。

  

**质疑声不断**

Claude Code 负责人 Boris Cherny 发帖引用了这篇报道，祝贺 Spotify 与 Claude Code 的成功合作。

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqH0DSCBgpXqdylwGM3DVUgDltVmmgp9Bia9U1q782McTOf8hFNUZasgLuAHT5bT1C1UbMEOahibaLPWWicWk2RicL8hy5IxRCVrM08/640?wx_fmt=png&from=appmsg)

  

不过评论区很快变成了大型「找茬」现场。

  

不少一线开发者对「几个月不写一行代码」的说法嗤之以鼻，认为这违背了基本的开发常识，闻起来全是一股包装过度的公关营销味儿。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqEia1nygAbnAycYUdPy0gQXia98asTeKrmNWmpe8eQsPwoz8QuRWNFqDXYGXKM2jBlWBqfrAibWY2XHCjfmCibO8PZeNgSdmazHunE/640?wx_fmt=png&from=appmsg)

  

还有人犀利指出，把「在早高峰地铁上用 Slack 提交代码」当成光辉业绩来炫耀，本身就很荒谬。这不仅不像是技术的进步，反而更像是糟糕工作环境下的求救信号。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqHDiaae90ZFSF4eNxCYUpJWSd4gX5haBuZ15XcAZkUcicNs6AqmssQlPWdbvCfpbshKVVsr8I6IZukFNLECwn9WX7Odze6xFj4JI/640?wx_fmt=png&from=appmsg)

  

最务实的当属 Spotify 的订阅用户：既然代码都是 AI 免费写的，开发成本都降没了，下个月的会员费是不是该打个折了？

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqFRLA86iaLNT9ibphF1ONeheBDOnv751iasMy4Aw2TUkQwga4NrWibR7WCnouH7noXQqr5R8Jafwia1ReMPc7FSsuo3aL4JnqYqbia6k/640?wx_fmt=png&from=appmsg)

  

还有不少对于 Anthropic 本身的质疑，一位网友发出了直击灵魂的拷问：既然宣称 Claude Code 已经包揽了 100% 的写代码工作，那为什么 Anthropic 自己的官网招聘页上，还挂着上百个急招的开发岗位？

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqE1icQ2wRl5o7kMJs2z5yN8c1LGMIzzibDcHHrRwWQbNPpOViasn742Twn3j4xvL7OxwL59bQQPibn8CI9M6ggBbbG6sOTyG01Dia1s/640?wx_fmt=png&from=appmsg)

  

面对这种「AI 砸饭碗但 AI 公司疯狂招人」的质疑，Boris Cherny 出面挽尊，表示人类工程师的职责正在演变，现在的重点是写提示词、跨团队沟通和拍板决策，因此「优秀的工程师比以往任何时候都重要」。

  

但网友立刻补刀：既然 AI 这么强大，它为什么不能自己给自己写提示词？

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqHxp5lMFBVvD1Zy6ECCtQO61Kh52oWp82Jw362Uoptj4Epcia0y8lxBshzF0o4oiaBn2u5weOpK90KdhhsGgmVY3kW3jmALm6V18/640?wx_fmt=png&from=appmsg)

  

更有看热闹不嫌事大的网友，直接搬出了 Anthropic CEO Dario Amodei 之前的惊人言论：「到 2027 年软件工程将被完全自动化，只剩 0.01% 的工程师」。网友质问 Boris，既然你觉得工程师如此重要，你们 CEO 为什么老爱发表这种自相矛盾的疯狂言论？

  

对此，Boris 只能无奈地甩出一句极其经典的公关式回复：「我描述的是现状，Dario 谈论的是未来。」
