# 代码不值钱之后，开源项目靠什么活下去？

开源 Agent 时代：

开源代码、贡献、商业和治理——哪些变，哪些不变

这是开源最好的时刻。一个人 2025 年 11 月发布的周末项目 OpenClaw，60 天从 9 千星涨到 18.8 万星，超过 React，登顶 GitHub 历史星标榜；智谱以最宽松的 MIT 协议开源旗舰模型 GLM-5.2，评测成绩直追最强的闭源模型；vLLM 的核心团队，靠一个开源推理引擎拿到 1.5 亿美元种子轮——史上最大的种子轮之一。

这也是开源最坏的时刻。仅 Claude Code 一家，已占 GitHub 公开提交的 4%，分析机构估计年底将到 20%——提交曲线的增长主力，正在从人变成 Agent。curl 的作者 Stenberg 今年 1 月关掉了运行六年的漏洞赏金：2025 年收到的报告，真问题不到 5%，其余是 AI 垃圾；Python 生态的 Jazzband 整个关停；GitHub 官方开帖承认危机，一线维护者的估计是，AI 生成的 PR 十个里只有一个合格。而最值钱的工程数据——人怎么提需求、Agent 怎么试错、评审怎么把关——正流进模型公司和 Coding Agent 的私有池，不经过 GitHub。

繁荣和裂缝同时出现，不是巧合——它们是同一场大变革的两面：Agent 不是来替开源写更多代码的，它在换开源三十年赖以成立的地基。

但 Agent 不会终结开源。它终结的是“以代码为中心的开源”——把开源推向一个以意图、过程和信任为中心的新阶段。

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjExT9RVDN9MY7epplZKHSZCAhib51v3q5FtodbvCzmEhWgPa2icazPiadUAskMXyPpswibvD5Owax9wN7BNBJmmIMIbD0Iyz1Sy8858/640?wx_fmt=png&from=appmsg)

> 本文核心：
> 
> 一、免费的是代码，稀缺的是可信运行。Agent 把生成成本打到近零，但验证、维护、安全、合规成本没有同比下降——项目会爆炸，可信依赖更集中。
> 
> 二、Agent 增加了贡献，增加不了维护责任。发布一个项目和维护一个项目是两回事；定义、评审、治理和长期担责，仍然只能由人来做，而且更贵。
> 
> 三、Skill 负责分发，Tool 负责执行，服务负责收入。知识藏不住，开源换分发；数据、权限和真实执行，是收费的地方；托管、验证和出事兜底是续费理由。
> 
> 四、下一代 fork 权，是记忆能带走、信誉能跟着走。过去社区不满，可以把代码 fork 走；未来代码容易复制，真正锁住的是 Agent 记忆、运行记录和信誉——这些能不能带走，才是开源自由新底线。

要看懂什么在变、什么没变，先看开源当年为什么赢。

上篇 · 开源为什么赢：四个问题的四个答案

开源从边缘的极客运动长成软件业的主导生产方式，靠的不仅仅是理想主义，而是四个核心问题曾有四个稳定的答案。

一 · 开源代码为什么值钱？

因为在 Agent 之前，代码是稀缺的人力资产。一个成熟的数据库、操作系统或框架，凝结几十人乃至几千人年的劳动。它的价值有四层，一层比一层高：

省下重复劳动——别人写好的，你不必再花几个月重写。封装工程知识——源码里藏着算法取舍、兼容性处理和无数踩坑经验，代码本身就是知识载体。形成复利资产——初始开发很贵，边际复制近零，被复用的次数越多，单位价值越大。升格为事实标准——用的人足够多，一个项目就从软件变成产业基础设施。Linux 装在全球几乎每一台服务器和安卓手机里，它的价值早已不是那几千万行代码，而是“全世界都在用”这件事本身。

旧世界的第一条公理：代码很贵，复制很便宜，最优策略是共同生产、共同复用。项目数、下载量、Star 数，因此成了价值的度量衡。

二 · 开发者为什么参与开源贡献？

没有雇佣关系，没有人发工资，全世界的工程师凭什么给同一个项目干活？三套机制叠在一起。

协作被拆到人人可参与。Issue 描述问题，Fork 独立修改，PR 提交变化，Review 把关，Maintainer 决定合并。Git 加 GitHub 把“给一个全球项目递一块砖”的成本压到几分钟，参与人口从几百个极客扩大到几千万开发者。

声誉是硬通货。开源圈排地位不看你占有什么，看你送出了什么。谁的作品被用得最广，谁就是大神；名声还能变现——好工作、咨询单、创业资源。那条链是：贡献代码，获得声誉，换来机会和收入。GitHub 的个人主页上有一张贡献日历，哪天提交过代码，哪天就亮起一个绿格子——这张“绿格子”图，曾经是全球通用的技术简历。

治理有序但不民主。开源不是无政府，是“开放参与、集中裁决”：谁贡献更多、判断更可靠，谁获得更大的技术权威。代码对所有人开放，技术路线从来握在少数核心维护者手里——这套看似不公平的结构，恰恰保证了项目几十年不散架。

三 · 开源公司靠什么赚钱？

旧商业逻辑一句话：代码免费扩大采用，围绕稀缺环节收费。在此之上，开源还有三重商业身份。

最好的获客手段。开发者不看广告、反感销售，只信同行——开源项目自带口碑传播，获客成本趋近于零。Star 曲线因此一度直接写进 VC 的尽调模型：一条陡峭的星标增速，能撬动一轮融资。MongoDB、Elastic、GitLab、HashiCorp、Databricks 这批公司，就是乘着“开源获客、云上变现”的剧本上市的。

打败闭源软件的武器。Linux 干掉了专有 Unix，Apache 用“竞争者凑份子”压住了微软 IIS。面对免费、可定制、还有全球社区迭代的对手，闭源软件的定价权节节败退。

建立生态的战略。Android 是教科书：Google 用一个免费开源的操作系统，把全球手机厂商、开发者和用户全部拉进自己的生态，再用搜索与广告收割。开源在这里既不是产品也不是获客，是构建产业同盟、定义事实标准的战略本身。

钱从五个环节收回来。

卖兜底。代码免费，承诺收费。Red Hat 是原型：Linux 谁都能免费下载，但生产系统半夜三点崩了，得有人接电话、给补丁、担责任——企业付费买的是这份承诺。这门生意最终被 IBM 以 340 亿美元收购，证明“对结果负责”本身就是产品。

卖企业版。行话叫 Open Core：社区版免费开放，企业刚需的那一层收费——权限管理、审计日志、高可用、与内部系统的对接。它成立的前提只有一个：客户自己照着免费版重做这些功能，不划算。GitLab、Elastic 都是这个路数。

卖托管。软件免费，替你运维收费。你可以自己下载部署，也可以按月付费用官方的云服务——省下的不是软件钱，是招人、值班、扩容、备份这些运维钱。MongoDB 的云服务 Atlas，收入早已超过卖软件本身。

卖双许可。同一份代码挂两张许可证：开源用途免费共享；想把它嵌进闭源商业产品，就要付费买商业授权。MySQL、Qt 靠这一招活了很多年——用我的代码赚钱，分我一份。

卖生态。前四种赚软件的钱，这一种赚整个生态的钱——就是上面 Android 的打法：系统分文不收，收的是几十亿台设备的入口、标准和流量。

四 · 开源秩序由谁定义？

一个没有老板的全球协作体，靠四种权力的平衡维持了三十年秩序。

许可证给法律边界——GPL、Apache、MIT 回答谁能用、能否改、商用如何处理，它是开源世界的宪法。维护者给技术裁决——谁握合并权，谁就有事实权力。基金会给中立担保——英特尔和 AMD、Google 和微软肯共养同一个 Linux，是因为有人担保它不属于任何一家。大公司给资源和规模——Google 推 Kubernetes、微软推 VS Code，把项目推成行业标准的最后一程。

社区还握着一件平时封存的武器：另立山头权(fork)。掌门人若变节，大家可以带着代码集体出走。这个权利像罢工权——极少动用，但必须存在。最近的一次出鞘在 2024 年：Redis 把许可证改成对云厂商收费的版本，社区几周内就带着代码另立了 Valkey，交给 Linux 基金会托管，AWS、Google 相继站队；一年后，Redis 把新版本改回了开源许可。武器用一次，所有掌门人都会记很多年。

过渡 · Agent 时代的四个新前提

下篇的判断都从这四条推出来。四条的成熟度不同——前三条已有明确方向，第四条还是产业空白。

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEwywKM1EDJC8MkqR9gSWbw0XC7eEwe1xfia8BvkVPJtlZEXNV6McibVymj3Vpwia8PTz6icDayHnUwDzBZxkJtljzsNWgsZ68224KA/640?wx_fmt=png&from=appmsg)

前提一 · 代码生成成本急剧下降。【已发生】过去要一个团队干几个月甚至几年才有可运行版本，现在 Agent 几小时到几天就给出一个能跑的。这不是效率提升，是物种变化：代码从需要攒钱购置的资产，变成随用随取的耗材。上篇四个答案共同的地基——“代码很贵”——没有了。

前提二 · 编码入口移到了 Code Agent。【正迁移】开发者的一天，不再从打开 IDE、逛 GitHub 开始，而是从对一个 Agent 说话开始。几家头部 Code Agent 已占走大部分编码入口。入口即权力：谁站在开发者和代码之间，谁就实际握着技术选型权——这项权力正在从人移交给 Agent。

前提三 · 生产线折叠，运维改道。【正迁移】旧流水线的需求、设计、编码、评审、构建、测试、发布、部署，被 Agent 折叠成“一句话进，一个跑着的服务出”——中间环节没有消失，是被自动化收进了 Agent 的工作流。CI/CD 也不会消失，但要从管住人的提交，升级为约束 Agent 高频变更的安全边界；运维盯的对象，从一段代码，变成一个 Agent 的完整行为。

前提四 · Agent 有连接协议，没有协作协议。【尚未定型】MCP、A2A 这类协议解决的是 Agent 怎么接上工具和彼此，相当于把电话线接通了；可通了话之后怎么协作——谁分工、人在哪里介入、结果怎么合并、出错找谁——还没有规矩。上一代协作协议(Issue/PR)的发明者叫 GitHub；这一代的发明者，还没有出现。

四句话：代码便宜了，入口换了，产线折叠了，协作待定。

合起来是一件事：写代码的主角，从人换成了 Agent。人的角色和职能要重排，开发流程和协作平台要按 Agent 优先重造。下篇顺着这四条推：开源生态与协作平台如何演变。

下篇 · 生态与平台如何演变：哪些变，哪些不变

一 · 开源代码还有价值吗？

> 代码便宜了（前提一），第一个被顶到台前的就是这个问题。

有，而且比看上去大——但价值的构成换了：生成价值在降，验证、运行和信任的价值在升。

先看什么在贬值。过去你引入一个工具库，是因为“别人写好了我不用写”；现在 Agent 现写一个，比搜索、评估、引入一个第三方依赖还快。占开源总量大头的中间胶水层——封装库、UI 组件、格式转换器——正在耗材化，项目数、下载量这些旧指标随之失去意义。

但就此断言“代码不值钱了”，资深工程师会立刻反问：Agent 能生成 Demo，能生成跑十年的生产系统吗？能重写代码，能替换与业务耦合了十五年的遗留系统吗？就算代码不再稀缺，公开审计、公开复现、公开追责，难道不仍是最低成本的信任机制吗？

三个反问全部成立。企业真正比较的从来不是“自己生成”和“用开源”，是生成、验证、维护、安全、合规、迁移的总成本——Agent 让第一项大降，后五项没有同比下降。升级后的公理：

软件资产的价值，不再取决于生成成本，而取决于可信运行的总成本。

由此推出两个判断。

第一，开源依赖不会消失，会分化。企业选成熟依赖，是为了社区验证、漏洞披露、版本管理和法律边界；自己现写虽快，却把验证与审计的负担全揽上身。所以：Agent 不会消灭开源依赖，它会消灭平庸的依赖。接下来会同时看到三件事——项目数量爆炸，生产依赖向少数可信项目集中，供应链的单点风险变大。

生态更繁荣，同时更脆弱。

第二，项目真正不可替代的部分，未必写在源码里。顺着“可信运行的总成本”往下拆，最稀缺的资产收敛为三层，构成一条链：

*   意图资产（需求、规约、规则、组织偏好）——决定造什么
    
*   过程资产（尝试、错误、评审、决策记录，即工程记忆）——记录为什么这样造
    
*   信任资产（验证、安全、身份、信誉、责任）——证明能不能用
    

这里说的工程记忆，不是聊天记录，而是可检索、可追溯、可迁移的需求、决策、约束、失败与验证历史。拿一个跑了十年的风控系统打比方：意图资产，是它的规则和阈值为什么定成这样；过程资产，是每一次误报之后的分析、争论和修改；信任资产，是监管的审计记录和三年零事故的运行史。代码全丢了可以让 Agent 重写，这三样丢了，系统等于从零再来。开源的价值，正从代码迁移到这三层。一个注脚：这三层资产，目前都不沉淀在 GitHub。它托管着产物，Agent 掌握着过程。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjExqiaXGVa5bGicka4eecs6XZeXIN80StBgPibXWmksIg95WFxK5z3YlkXyUhylzSeswyMaa5cJdqYgBGoibaTnWCw4BN5hglQTL0hU/640?wx_fmt=png&from=appmsg)

一句话：免费的是代码，稀缺的是可信运行。

二 · 开发者还愿意做开源贡献吗？

> 上一节说，价值迁移到了验证、维护和信任——这些活都要有人干。谁来干、图什么？拆成三问：人还愿不愿来，项目还会不会多，贡献者还值不值钱。

先答第一问：愿意的人还在，但回报在向两极分化。

顶端的回报更大了。OpenClaw 的作者靠一个周末项目，六个月走进 OpenAI；一个小团队，也能靠一套开放权重在几个月里获得全球影响力。个人英雄的上限，被 Agent 放大了。

大众的通道却在变窄。普通贡献者过去靠修小 Bug、提小 PR、攒贡献记录进入社区，这条路正在被信号通胀和垃圾贡献堵住：贡献日历失灵，提交数量通胀，“贡献换简历”这条最大众的激励链断了；curl 关闭赏金，一批项目开始限制 AI 贡献、甚至自动关闭全部外部 PR——大门在一扇扇关上。

合起来是一个哑铃：Agent 放大了顶尖个人的杠杆，也挤压了普通贡献者的声誉通道。愿意贡献的人没有消失，但老的进门方式正在失效。

再答第二问：项目会更多，活的项目占比会更低。生成成本趋零，能“开发”的人口扩大数倍——不会写代码的行业专家，也能指挥 Agent 做出内部工具，这样的新贡献者会把项目总数推向爆炸。但“发布一个项目”和“维护一个项目”是两回事：Agent 能扩大贡献的供给，却不能自动生成维护的责任。长期兼容、版本取舍、拒绝错误方向、响应安全事件——一样都替代不了。结果是项目数量爆炸，有人负责的项目占比骤降。

第三问最关键：贡献者不但还值钱，而且在涨价——但换了岗位。一位维护者说得准：如果维护者想要的只是 Agent 能生成的代码，他们自己就能生成。值钱的岗位移向三处：定义，把模糊需求写成 Agent 能执行的规约；评审，在十个 PR 只有一个合格的洪水里拦住那九个；治理，决定项目往哪走、什么不做。深度工程师则进一步收缩到高风险的底座层，相对价值上升。

过去开源缺的是贡献，未来开源缺的是拒绝贡献的能力。只有过滤（AI 披露、信誉门槛、评测前置）还不够——Agent 把生产成本降了下来，把筛选成本转嫁给了维护者。下一代制度要把这笔成本转回去：验证尽可以交给 Agent 自动做，但要在提交之前、由贡献方做完并附上证据——维护者只复核证据，不再替每个人跑全套。还有一层更深的代价：Agent 可以帮一个人给一百个项目提交代码，但他未必真正属于任何一个社区。贡献增加了，社区却变薄了。

过去开源缺的是贡献，未来开源缺的是拒绝贡献的能力。

所以贡献制度要重建的不是一条规则，是三件基础设施：可信身份、自动验证、可携带的声誉。谁把这三件事和任务派发、工程记忆整合在一起，谁就站上了 Agent 时代协作平台的空位。

一句话：Agent 解决了生产能力，没有解决维护责任和社区认同。

三 · 开源商业化的前提还成立吗？

> 入口换了、产线折叠了，钱怎么收就得重算。分两半看：老模式哪些还站得住，新模式长在哪里。

老模式：不是失效，是收费对象迁移

先逐一检验上篇的三重商业身份。

“开源是最好的获客手段”——前提半塌，形态重生。塌的是老信号：Star 能刷，贡献记录能造，VC 把星标增速写进尽调模型的时代结束了。生的是新信号：技术选型的第一读者，从开发者变成了开发者的 Agent——它不听大会演讲，它读文档、跑测试、查许可证和漏洞史。这套新分发学叫 Agent Relations——过去公司做 DevRel 经营开发者关系，未来还要经营和 Agent 的关系：让项目能被它发现、理解、验证、调用并持续信任。README 曾经写给开发者，下一代 README 首先写给 Agent；过去项目争夺开发者的注意力，未来争夺 Agent 的默认选择权。开放项目在这里反而占便宜：只有开放，Agent 才能读到源码、复现到测试，把验证走完——开源获客没有死，只是从赢得人的口碑，换成了通过机器的验证。人，仍然是治理、采购和最终责任的主体。

“开源是打败闭源的武器”——作为压低闭源定价权、建立替代生态的武器，依然成立，而且在模型层更锋利。开放权重模型正在重演 Linux 对专有 Unix 的战役，弹药从代码换成了权重。开头提到的 GLM-5.2 就是最新的战况——闭源每领先一步，几个月后就有开放权重贴上来，把这一步的溢价打掉。但闭源阵营还握着算力、数据飞轮和分发渠道，这一仗会比当年更长。

“开源是生态战略”——完全成立，而且战场翻倍。生态战这次有两场：一场在生产力层，模型、编码工具和 Skill/Tool 市场争夺开发者与企业的工作流；一场在终端层，Agent OS 争夺手机、汽车、机器人的入口——把模型焊进操作系统，比当年焊 GMS 更难拆。再叠加中美技术体系分化，更可能是两套平行生态各自上演，而不是一个全球统一的 Android 剧本——GLM-5.2 开源当天适配八家国产算力平台，信号已经很清楚。

新模式：Skill 分发，Tool 执行，服务担责

先把两个词说白：Skill 是教 Agent 做某类事的方法包，Tool 是 Agent 可以调用的真实执行接口。新机会集中在三层资产的收费上，一张表：

Skill 是知识封装，Tool 是执行能力，服务才是持续收入。Skill 易复制、藏不住，最优解是开源换分发；Tool 握着数据、权限和真实执行，有成本、有责任，是收费的地方；服务负责托管、验证和出事兜底，是续费的理由。Skill 一旦绑定持续更新的知识、评测和工作流，就已经从静态内容升级为服务。

这套结构里还有四件事没定型，值得盯着：

*   计价单位：按调用、按任务，还是按结果分成
    
*   分发渠道：Skill/Tool 的“应用商店”归模型公司还是中立平台，分成规则谁定
    
*   质量信号：用什么代替下载量——任务成功率、Token 成本、真实完成率
    
*   责任认证：谁给一个 Tool 出“敢用”的背书
    

谁先把这四件事定成标准，谁就握住新模式的收费闸门。

新一代创业剧本，vLLM（就是开头那笔 1.5 亿美元种子轮）和 SGLang 都已走通：把代码所有权和生态治理中立化，把商业控制点放到运营、服务、数据和分发上。但捐基金会不自动等于中立——商标归谁、路线图谁定、治理是否多元，要单独看。中立化的是标准，商业化的是位置。

规律只有一条——离结果越近、离责任越近，商业价值越高；离 Prompt 越近，越容易免费化。

四 · 秩序由谁定义？——老权力哪些还在，哪些变了

> 前面讲的都是资产：什么在变稀缺。最后一问讲制度：谁给稀缺立规矩。

上篇讲过，旧秩序靠四种权力加一把备用钥匙。逐项对照：

许可证——还在，但宪法出现四段空白。GPL、Apache、MIT 依然管着“谁能用、能否改、商用如何处理”，这没变。变的是四个新阶段全无条款——训练（公开代码能否用于训练）、生成（结果算不算实质性复制）、贡献（Agent 的代码谁签贡献者协议 CLA）、运行（自动组合的许可证如何合规）。“开源”一词本身也在分层——源码、权重、数据、方法分别开放与否，“开放权重”只是其中一格。

维护者——还在，裁决对象变了。合并权依然是事实权力，这没变。变的是裁决内容：从“这段代码好不好”，扩展到“这个贡献者（可能是 Agent）可不可信、这份评测过不过、这类贡献收不收”。维护者从代码裁判变成制度执行者——而配套的工具和补偿，都还没有。

基金会——还在，且从守成变成开局条件。Agent 底座的牌桌上坐着手机厂商、模型公司、超级 App 和监管，彼此戒备远超当年的 Unix 阵营——没有中立担保，牌局开不了。变的是能力要求：Agent 身份签发、技能审核、责任追溯、贡献结算，这些制度器官，现有基金会一个都没长出来。

大公司——还在，但牌桌上多了两家。Google、微软式的资源与规模依然管用，这没变。变的是名单。第一家是政府：带着主权 AI 预算入场，出资人、监管者、大买家，三个身份合一。第二家是模型公司，它的位置最特殊：训练燃料的最大消费者，过程数据的最大收割者——吃公共池吃得最多，回馈的义务也最重。回馈不该只是捐钱，出算力、出工程师、开放评测数据、参与中立治理，都算数；检验只有一条：靠公共代码练出来的本事，有没有用回公共代码的维护上。

fork 这把备用钥匙——权利还在，威慑力降了。代码可再生之后，只带走源码、历史和贡献记录，已经不足以带走一个生态；真正的锁定移到了 Agent 记忆、运行历史、评测信誉和工具调用网络上——这些带不走，代码 fork 了，社区也走不了。下一代的“fork 权”，因此是记忆的可迁移权和信誉的可携带权：它把“记忆可迁移”从企业采购条款，升格为开源自由的核心权利。

下一代的 fork 权，是记忆的可迁移权和信誉的可携带权。

五项对照合起来：旧权力一个没退场，但权力的落点整体移动了——从仓库、合并权和发行版，移向一个三层制度栈：

*   权利制度：谁拥有、谁能用、谁担责——身份、所有权、记忆归属
    
*   信任制度：Agent 是谁、结果可不可验、供应链可不可溯
    
*   经济制度：谁贡献、谁受益、失败成本谁担
    

两个补充判断：Agent 同时工业化了生产与攻击，签名、溯源、可复现构建会从安全附加项变成基础设施；Agent 任务第一次可以细粒度计量，但把“数量”换算成“价值”仍未解——这是结算平台的机会。

把这些拼在一起：“下一个 Linux”大概率不是一个代码仓库，而是这套三层制度栈的开放实现——它的内核是规则，不是代码。

五类人的行动清单

开发者

*   停止：把提交数量当能力的主要证明
    
*   开始：留下自己的规约、评审和决策记录；在定义、评审、运营里选一样深耕
    
*   看什么：你提的方案——架构、接口、规约——被多少真实项目采纳并跑着；你在评审里拦下过多少要命的问题
    

开源维护者

*   停止：无条件接收低成本的 Agent 贡献
    
*   开始：AI 参与披露、评测前置、信誉分层；把项目做成 Agent 可读
    
*   看什么：到达人工评审的 PR 里，先过了自动验证的比例；合并后被回滚的比例；一年后，核心维护者还剩几个
    

创业公司

*   停止：把 Skill 或 MCP 薄封装当壁垒
    
*   开始：开源 Skill 换分发，收费 Tool 做执行，托管验证收租；第一天就做 Agent Relations
    
*   看什么：调用量里 Agent 自动选中你的比例（分发）、任务一次做成的比例（执行）、客户第二年续费的比例（收入）
    

企业技术负责人

*   停止：只按开发效率评估 Agent，不算验证和事故的账
    
*   开始：工程记忆列入数据资产，采购合同锁定归属与迁移权；依赖设供应链门禁
    
*   看什么：Agent 写的代码，合并后有多少被回滚；它自己引入的依赖，有多少没过审批；验证花的钱，占开发总成本几成；工程记忆换一家 Agent 供应商，搬不搬得走
    

> 企业最危险的不是 Agent 写错一段代码，而是组织在效率提升中失去对依赖、决策和责任链的可见性。

平台公司与基金会（平台出资源和规模，基金会出中立和规则）

*   停止：只捐代码不捐治理权，用 Star 和下载量衡量生态
    
*   开始：平台做身份、认证与结算；基金会做维护者补偿与责任追溯
    
*   看什么：治理桌上坐着几家互不隶属的机构；关键项目里，有多少是有人拿钱负责维护的；流通的组件里，有多少能验签、能溯源
    

五类人合起来只有一句话：谁先把 Agent 身份、贡献审核、责任追溯、贡献结算做成上线跑着的系统——而不是停在白皮书里——谁就开创开源的第六种模式。

总结 · 四问的新答案，与平台的下一步

四问收成一段话：代码的价值从生成转向可信运行；贡献从提交数量转向定义、评审和治理；商业收费从功能撤向执行与责任；秩序的落点移向身份、记忆、信誉和结算。变的是形态，不变的是稀缺与责任——四个变化汇到同一个落点：开源协作平台必须重构。它不会被替代，会被再盖一层，就像 GitHub 当年没有取代 IDE，而是在 IDE 之上建了协作层。我的判断分三步：

第一步，旧平台打补丁（正在发生）：在现有流程上叠加 Agent 服务这类功能救得了急，救不了架构——身份、记忆、结算，都不在它的数据模型里。

第二步，Agent-first 协作层出现（一到三年）：源码仓库之上长出新的一层，内置任务派发、评测质量门和记忆托管——协作重心移走，这一层的竞争决定下一个“开源心脏”归谁。

第三步，制度栈标准化。身份、记忆、信誉、结算的协议进入中立组织——“下一个 Linux”的真正形态。

给建设者的路径只有一个顺序：先立信任（可信身份加自动验证，让平台上的人、Agent 和结果都可查），再攒资产（让工程记忆和声誉在平台上沉淀、可携带），最后才做结算（给贡献计量、分配收益）。顺序反了，平台上没人、没记录、没信誉，结算无从谈起——那只是个空市场。

最后留一道观察题：GitHub 会进化成 Agent 的协作平台，还是退化为 Agent 的后台存储？更原生的 Agent 协作平台会不会另起炉灶——AtomGit 已经在朝这个方向做实践探索。这道题怎么解，且看未来演化。

结语

Agent 不会终结开源。它终结的是“以代码为中心的开源”——把开源推向一个以意图、过程和信任为中心的新阶段。

问题一个没变，答案全要重写——这就是 Agent 时代开源的变与不变。

一个预告。开发者最关心的具体问题——还要不要学写代码？刷出来的贡献记录还算数吗？项目被 Agent 三周复制怎么办？Tool 和 Skill 能不能赚钱？——姊妹篇《Agent 时代，开发者最关心的十个问题》即将发表，也欢迎你把关心的问题抛过来。

代码变便宜，信任变昂贵。上一个三十年，四个答案写在代码仓库里；下一个三十年，答案写在制度里。

谁提供信任，谁保存记忆，谁制定规则，谁分配收益——回答这四个问题的人，就是 Agent 时代开源世界的新主人。
