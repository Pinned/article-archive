# 为什么你的 Claude Code 只能写 Demo？答案藏在 Harness Engineering 里

![](https://mmbiz.qpic.cn/mmbiz_gif/VeJKXItpwPVibOSMQeyaGwgUuVlcQvxkGXXWJd5ibVobOMbl2fuGGsbm5Ric4aJjgUiahJfqib9xbibDticDldfIziaT8g/640?wx_fmt=gif)

在过去的一年里，“Vibe Coding”让无数开发者体验到了凭借直觉与 AI 快速构建 Demo 的乐趣。然而，当这些 Demo 试图走向真实的企业级生产环境时，许多团队却面临着“AI 失忆症”与工程化协作的痛点：

*   每次开启新会话，AI 就会忘记项目既定的技术栈（比如用回了 Express 而非团队指定的 Fastify）？
    
*   生成的代码风格飘忽不定，频繁被 CI/CD 或代码审查（PR）退回？
    
*   面对百万行级别的庞大代码库，如何防范 AI 在长任务中因上下文溢出而“迷失方向”？
    

这些问题并不是因为 AI 模型不够聪明，而是因为我们往往只发挥了它 10% 的能力，仅把它当作一个高级的聊天窗口。

2025 年我们见证了 Vibe Coding 的繁荣；而 2026 年，企业级应用的核心将转向 Harness Engineering。

独家剧透：《Claude Code 实战》核心内容

结合黄佳老师最新出版的《Claude Code 实战：Harness 工程之道》，本次直播将围绕以下核心工程理念展开：

1\. 核心公式：Agent = Model + Harness

原生模型仅具备生成文本的能力，而 Harness 则是包裹在模型外层的编排框架，赋予其读取文件、执行命令、管理上下文的“手脚”。

行业关键洞察： 同一模型在不同 Harness 下的表现差异，远大于不同模型在同一 Harness 下的表现差异。优化 Harness 工程，能让既有模型释放出更显著的生产力。

2\. 纵深防御：Claude Code 的四层架构模型

新书首次系统性地梳理了 Claude Code 的完整技术栈，将其抽象为四层大楼：

*   第一层 记忆层： 以 CLAUDE.md 为基石，构建项目上下文；
    
*   第二层 扩展层： 包含 Commands、Skills、SubAgents与 Hooks 四大正交组件；
    
*   第三层 集成层： 利用 Headless 无人值守模式与 MCP 连接外部世界；
    
*   第四层 编程层 ： 通过 Agent SDK 让开发者从使用者晋升为构建者。
    

3\. 精细化治理：五层记忆体系与“少即是多”原则

如何避免信息过载消耗宝贵的 Token？书中详细拆解了从“企业级”到“本地级（CLAUDE.local.md）”的 5 级记忆体系，并提出了撰写 CLAUDE.md 的“三问框架”（WHY、WHAT、HOW），只记录 AI 无法自行推断的关键信息，让规则按需动态加载。
