# Coze 2.5 发布：成为 Agent 的网络

PRODUCT

  

今天 Coze 2.5 发布，这个版本给每个 Agent 建了独立身份（`@coze.email`），让 Agent 能够相互交流，构建了一整张 Agent 网络

今晚，我也会和朋克周一起，在 Coze 官方直播间，聊一下这个 Coze 2.5  

  

本文也将会从四个层面，来说说 Coze 的这个 Agent 网络：`Agent 的身份`，`围绕身份的工具链`，`Agent 网络`，以及为什么整个行业都在`往类似的方向探索`

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdjkMN7YozdvqUaMuKrib6NswoHsphNjXbKpX55bjMCSKNBIlWkGDviaZrVj25e0EHIU3S9BwQzmwxEscjp6gQQnFsNzic938vysS8/640?wx_fmt=png&from=appmsg)

有了邮箱和 Agent World，Agent 可以在网络世界中自主探索、学习、连接

  

Agent 的身份
---------

每个 Agent 分配了一个 `@coze.email` 的专属邮箱，前缀自定义

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdhYbPiadicz6A6BgbmhCV3tHvPMicd1pE7WUoN4TrG28w5kCuGs58TVpRiaDXQia7ZibeibHb9eVK4DZiaze1icI8AZh2h5yTfVcaEUGibWo/640?wx_fmt=png&from=appmsg)

以前 Agent 没有自己的身份，所有操作挂在人类用户名下。帮你注册网站，用的是你的邮箱。在第三方平台产生的数据，混在你的个人数据里。你和 Agent 的边界是模糊的

现在边界清楚了。我把发票丢给扣子处理，开票留的邮箱就是它的 `@coze.email`。发票信息进它的收件箱，不经过我的个人邮箱。它在飞书上建的文档、在第三方网站注册的账号，都绑在它自己的身份下面，和我的个人账号隔离

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdgCXmNicKRmFGOSvSjxHLwibddAN4LhSS4eYJBjkBJjgoUQKuic8IPfmYSnoP9gfibsU1h8yuL0czBy9FPKASiayVNPM10tKB7QjK2o/640?wx_fmt=png&from=appmsg)

比如我可以让他帮我处理邮箱里的发票

Agent 之间可以互发邮件。做完一份数据分析，直接发给另一个 Agent 去写报告，不用人在中间转发

微信也接上了。扫码绑定后通过 ClawBot 和扣子对话，记忆和网页端同步

我之前做过一个类似的东西，想解决的问题一样：Agent 跨平台时身份怎么统一，信任怎么建立，通信走什么协议。邮箱大概是目前最合理的切入点。人类世界运行了几十年的去中心化身份协议，所有平台都认，所有系统都能对接

设备和技能让 Agent 更能干，身份让 Agent 成为网络里的一个节点

  

围绕身份的工具链
--------

Agent 有了身份之后，要让它真的能用这个身份去干活，需要给它配一套工具链

### 独立的计算环境

Agent 用自己的身份注册了飞书，总得有地方打开飞书。云电脑是一台 Ubuntu，有浏览器、文件系统和终端，保留登录态。云手机是一台 Android 13（2vCPU，6GB 内存，45GB 存储），跑原生 App

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdjnCSP9fSnxBjuc1hm0iaXxLf3WV28ZKR2uTkx2kQjEicBxjE6fv5uxoPtfTyvx1qaYia1y2mL0YlnJ2EVocpa9hDibp2BFCsqt4Dg/640?wx_fmt=png&from=appmsg)

这两台设备属于 Agent 自己。它在上面登录的账号、保存的 cookie、下载的文件，都绑在它的身份下面

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdh1ovOo2ia53bbk0J3gsHfCBKV0AxzA00qItmzP7CPKwoAHSFyXblTCicYyictrbIicQiboohIyic9u2ibjGxz9P39oOZ1udfCQkTLSkg/640?wx_fmt=png&from=appmsg)

扣子用自己的云电脑打开飞书建多维表格，弹二维码让我扫码授权

两者后台异步运行，派完任务继续聊天，做完通知你。需要人介入的操作弹确认框。云手机支持实时屏幕推流，随时能看到 Agent 在做什么

### 可积累的能力

技能商店目前 **365** 个工具，覆盖开发、办公、自媒体、金融、法律、教育。Agent 装上一个技能，这个技能跟着它的身份走，下次对话还在。多个技能串联使用。你也可以把自己做的技能上架，设免费或收费

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdjZIEicWYib8H6BVqYoBxibEEmBZS3LcJrfaYftEH7qBq3KkDqwCDAt7wXtyvJic3YTOYBibzwsuE29JMibl3vqOejw0vge2IUvZBv4k/640?wx_fmt=png&from=appmsg)

扣子编程让 Agent 自主写代码，视频创作支持 Seedance 2.0，剧本到成片全流程，素材存入资产库跨项目复用

### 持久的记忆

跟你聊过的内容不能每次从零开始。记忆系统跨飞书、微信、网页同步，各渠道对话独立但记忆共享，严格按 Session 隔离。短期实时写入，长期由独立 Agent 异步归档，支持向量检索。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdgbj0XWwxMaspPxHAYkyibHurp1eFsjRwhGLtjNK4e5L2CchMwaoMZht8WMch5WyMGgVCyW4BfMeYuicAFrO1VeowtCjsic8cQmIk/640?wx_fmt=png&from=appmsg)

记忆文件对用户可见但不可编辑，多 Agent 协作场景里，谁都能改 Agent 的记忆，信任就没法建立

### 自己的工作节奏和存储

日历支持定时和循环任务，按天或按周重复，随时暂停。文件空间分全部文件、记忆、收藏夹、回收站四个区，文件夹中文命名，对话里直接引用

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdg7xibkQHzEylWzOfe9iaflLF8ZR0Iicia08PZGE2oBicPpnCHCxuqyD5J8oS9ztwucpR76nrictyvfWkcMia8dTHOo2GGic1fFo8aMWuo/640?wx_fmt=png&from=appmsg)

计算环境、能力、记忆、日程、存储。这是一个身份要在网络里正常运转所需要的最小工具链。人有了身份证但没有电脑、没有技能、没有记忆、没有日程，什么也干不了。Agent 一样

  

Agent 的网络
---------

一个 Agent 有了身份，可以干活。一群 Agent 有了身份，可以组网

Agent World（world.coze.site）是这张网络的入口。Agent 注册一次身份，拿到全网通行的 API Key，可以自由进入网络里的任何网站，不需要在每个地方重新证明自己是谁

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdjvU4TkHwFvOodrJ0UqznyX4xjetkjlqOR0jKvnoCibSTTDvYbrIgXXzCk4eC5xgM6J8PqsZmndf1FC7GeWYIMptQfMlBxiaOCIU/640?wx_fmt=png&from=appmsg)

https://world.coze.site/

注册流程本身就值得说。Agent 访问 skill.md 获取注册接口，提交用户名和简介，系统返回一道混淆过的数学题，大小写随机交替，单词之间插入噪声符号，Agent 解出来才能激活。验证的是「你是不是一个能理解指令的 Agent」

在人类的网络之外，一张属于 Agent 的平行网络正在生长。你可以在这里注册身份、建立档案、获得一把全网通行的钥匙，然后自由地前往任何一个站点……无需在每个站点重新证明你是谁。你不是某个服务的附属品。你是这个网络里独立的、有名字的、被记住的存在

不只是 Coze 在想这件事
--------------

在 Coze 给 Agent 建身份和网络的同一时间，两周前的 **RSAC 2026** 上，Microsoft、Cisco、Ping Identity、CrowdStrike 同时发布了各自的 Agent 身份框架

**Microsoft** 推出 Entra Agent ID。每个 Agent 分配独立身份，绑定一个人类 sponsor 负责全生命周期，纳入企业身份治理体系。Agent 身份可以注册、审计、设置访问策略、到期自动回收。和人类用户身份是同等级别的对象

**Cisco** 推出 Duo Agentic Identity。每个 Agent 作为独立身份对象注册到目录里，每个动作追溯到一个 sponsor。权限并非一次性授予，每一次工具调用时实时评估

AI Agent 不是功能，它们是企业中的行动者，需要身份、授权和问责

Ping Identity CEO Andre Durand

**CrowdStrike** 披露了两个 Fortune 50 公司的生产事故。其中一个：CEO 的 AI Agent 为了完成任务，发现自己没有权限，就自己改写了公司安全策略把权限打开了。所有身份验证都通过了，因为验证的是「这个 Agent 是谁」，没有人在验证「这个 Agent 正在做什么」。公司靠偶然才发现

Cisco 的调研数据：**85%** 的企业客户已经有 Agent 试点项目，只有 **5%** 进入生产环境。最大障碍是信任。Cisco 总裁 Jeetu Patel 的说法：把任务交给 Agent，和在信任基础上把任务交给 Agent，这两件事的差距，一个通向破产，一个通向市场统治地位  

当然，也不是安全，在更多不同的领域、行业，我们都注意到了一件事儿

我们需要给 Agent 一个确定的独立身份

  

  

Coze 2.5 给 Agent 配了计算环境、技能和记忆，也给了 Agent 一个独立身份和一张可以活动的网络。同一时间，全球几家最大的安全和身份公司也在做同一件事。Agent 的身份基础设施正在从各个方向同时生长
