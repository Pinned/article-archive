# 微信 AI 的困局：为什么小程序无法成为 Agent 入口

微信做 AI 的思路，看起来很清楚：我有十几亿用户，我有小程序生态，我只要让 AI 调度小程序，就能成为 AI 时代的超级入口。

这个逻辑在 2024 年也许还能自圆其说。到了 2026 年，它正在被现实一点一点拆穿。

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/ZlwY8rlDcFMYJoshX357N1lKOm83LmnJ3rzUkvlUwTtmu1wricdN5E38N4dsB8wh10O8otfM4OgdFx46h7rDdDB2rj6YDgyEnic5iamic1crAHk/640?from=appmsg)

【1】入口会被谁抢走？
-----------

微信赌的是：用户永远从微信出发。AI 来了也一样，AI 在微信里调小程序，用户还是不离开微信。

但未来的年轻人大概率不是这么用手机的。他们会对自己的 Agent 说：帮我总结一下昨天的群聊。给我妈发条消息说晚上不回家吃饭。帮我订周末去成都的机票和酒店。

这个 Agent 装在手机上，接管的是整部手机的能力。它调用系统级的通讯、日历、支付接口，微信只是其中一个被调用的对象。

而承担这个超级入口角色的 Agent，大概率不会是微信 AI。

【2】Agent 进化有一条清晰的路径
-------------------

这一波 AI Agent 的成熟顺序很明确：编程场景先跑通，办公场景紧随其后，生活场景排最后。

微信的核心场景恰恰是生活类的：订机票、找餐厅、约朋友、买东西。这些场景目前 Agent 能力还不够成熟，能做的事有限。微信可能觉得这是自己的时间窗口，但问题是：等到效率场景被 Claude Code、Cursor 这些工具抢占了用户心智之后，它们会自然泛化到生活场景。用户的习惯一旦建立，会倾向于用同一个 Agent 处理所有事情。

到那时候，微信 AI 再想争夺入口，窗口已经关了。

【3】Claude Code 能做到的事，微信 AI 做不到
------------------------------

Claude Code 为什么好用？两个条件缺一不可。

第一，权限够大。它能连接你的文件系统、读你的代码库、访问各种本地数据。安全性当然有争议，但很多任务确实少不了这些权限。Claude 自己做的 Cowork 我反而不太爱用，原因就是权限卡得太死，很多事做不了。

第二，有一台 PC 作为运行环境。它可以实时写脚本、反复运行、调试修改，消耗大量 Token 之后大力出奇迹，最终把结果交给你。

微信 AI 这两条都不具备。手机上拿不到多少权限，也没有一个长期稳定运行 Agent 的环境，更不可能给每个用户烧大量的 Token。

有人会说：成本转嫁给小程序开发者不就行了？开发者也不傻，凭什么替你出 Token 的钱。

最终用户体验下来，大概率觉得：这还不如我直接用 Claude Code 或者 Codex 来干。

【4】小程序想学 Agent Skills，但有三个致命缺陷
------------------------------

微信 AI 的小程序规范，明显参考了 Agent Skills 和 MCP 的设计思路。方向对了，但小程序这个载体本身有三个硬伤。

第一个：无法自动进化。Skill 有不足的地方，Agent 用的时候自己就能发现问题，直接改，越用越好。小程序呢？要打包、要审核、要上线新版本。Agent 发现问题，它也许能反馈给开发者，但它改不了。不好用就只能凑合，甚至摆烂。

第二个：无法高度定制化。好用的 Skill 都是个性化的，每个人的偏好不一样。你可以从头写一个 Skill，也可以找个开源的让 Agent 帮你改改，加上自动进化和记忆能力，它会越来越贴合你的习惯。小程序做不到这一点，所有人用的是同一个版本。

第三个：Agent 操作图形界面的能力远不如操作脚本。小程序是图形界面，除非微信 AI 把小程序接口做成类似 CLI 的形式，让 Agent 像操作命令行一样调用小程序，否则模型去操作 GUI 的成本和成功率都很难看。

【5】教科书式的创新者困境
-------------

上面说了这么多问题，但微信团队不可能看不到。那为什么还是选了这条路？

微博上 @CC 女士不是西西 说了一句很到位的话：这是经典的创新者困境，被自己的 RPV 锁死了。

RPV 是什么？资源（Resource）、流程（Process）、价值观（Value）。微信有十几亿用户，有成熟的小程序生态，有完善的审核发布流程，有一切都在微信里完成的产品价值观。这些东西在过去十年让微信成功，但在 AI 时代，恰恰是这些东西让微信走不出来。

每一个决策都从已有的资源和流程出发：我有小程序，那 AI 就调小程序；我有公众号，那 AI 就总结公众号；我有支付，那 AI 就走微信支付。所有的思考都是怎么让 AI 服务于微信现有的体系，而 AI 原生产品需要的东西，和这套体系根本不兼容。

@CC 女士不是西西 还提了一个很重要的点：less is more。微信恨不得把一个人的全部生活和工作都塞进一个社交产品里。但对用户来说，一个什么都想做的 App 不会让人觉得方便，只会让人觉得又臃肿又不安。你的社交关系、你的消费记录、你的工作文件、你的健康数据，全部集中在一个地方，想想就觉得窒息。

其实微信手里有一张好牌：社交网络。它完全可以用一个和现有团队在物理和财务上隔离的团队，去做一个 AI 原生的独立产品。用户一键授权，把自己的社交关系搬过去，但产品本身按照 AI 原生的逻辑来设计，不背微信的历史包袱。

但创新者困境的残酷之处就在这里：手握好牌的人，往往是最不舍得打出去的人。

【6】微信以为自己是操作系统，但它不是
-------------------

微信确实庞大，在中国人的手机上几乎无处不在。但庞大和操作系统是两回事。微信是寄生在手机系统上的超级 App，它没有系统级的权限，也不掌握硬件层的能力。

当年豆包手机想访问微信数据，被微信拒绝了。当时看是理性的：凭什么让你读我的数据。但回旋镖正在飞回来。当真正的系统级 Agent 出现的时候，微信会发现自己变成了被访问的一方。

【7】“微信用户太多了，不会被替代”
------------------

有人说：用微信的人太多了，没那么容易脱离。

当年 QQ 也是这么认为的。

仔细想想，你现在留在微信里的原因是什么？发消息、打视频、群聊。这些功能，手机系统自带的通讯能力就可以做。很多人继续用微信只是惯性，替代方案一直都在。

AI Agent 时代的通讯入口，根本不需要一个新的 AI 通讯软件。现有的通讯系统就够了，短信、电话、系统通知。Agent 只需要调用这些接口，不需要再造一个微信。一旦惯性被打破，用户不会想回去。

有人问：你的意思是会有一个更厉害的 AI 通信软件？想多了。Agent 接管的是入口这个角色，通讯只是它调用的众多能力之一，和打开文件、查日历没有区别。

微信最大的风险，是聊天软件这个品类本身正在失去入口地位。没有人需要打败微信，入口自己会消失。
