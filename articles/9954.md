# Linux 7.1将正式放弃37岁Intel 486处理器！

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

整理 | 屠敏  

出品 | CSDN（ID：CSDNnews）

诞生 37 年的 Intel 80486（i486），这次真的要和 Linux 内核说再见了。

据外媒 Phoronix 报道，Linux 内核维护团队已经决定，在即将到来的 Linux 7.1 中移除对这颗经典 CPU 的支持。相关补丁已进入开发分支，等合并窗口开启，这件事基本就尘埃落定了。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

一颗老 CPU 的“体面退役”

i486 处理器于 1989 年正式发布，也算是当年的明星产品。

毕竟它是 x86 架构从 16 位迈向 32 位的关键一代，内置浮点运算单元、缓存架构大幅升级，一度是高端 PC 的标配。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEzftkD5HWJBm7mJ8l4RCfKqIWsO1I07d4CiallicDL5pibT4jtcPx25k8At1ekNh2Iib16Hd0QSZoQHgGAkrOOTg0yUqUrZLALlHa4/640?wx_fmt=png&from=appmsg)

只不过后来，它在 1993 年被初代奔腾取代，但生命力其实挺顽强的——一直到 2007 年才彻底停产。

Linux 从早期版本就一直兼容 i486，这也让它成了复古玩家和嵌入式爱好者的“续命神器”。不少老机器，靠着 Linux 还能再战几年。  

但随着内核不断迭代，为一款早已退出主流市场的处理器保留兼容代码，性价比越来越低。事实上，如今仍在用 i486 运行上游 Linux 内核的用户极为罕见，也没有任何已知的 Linux 发行商还在提供支持 i486 处理器的系统版本。

这次的变化，其实也不算突然。早在 2013 年，Linux 3.8 就已经放弃了更老的 i386。现在轮到 i486，也只是时间差不多了。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QDyP5HTUOXsPlJWd79yygiasaqXicpN7ibIfqiak5WFpaxE1mGxxpfmMjiaA/640?wx_fmt=png)

Linux 内核团队的态度：不想再为 i486 多花一秒

这件事上，Linux 之父 Linus Torvalds 早在 2022 年就表过态，“也许是时候再 2022 年停止对 i486 的支持了。”

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEyCxxL1BC7QXqBR6OpTofnQJxM6ibjRk8C2l1HrHnKmwP9Pvz1r2jmPGkcCr3FtRles2EUA0MhZLywd8q3fniawY19R13H9JFcqU/640?wx_fmt=png&from=appmsg)

近期，他再次表示，“我真的觉得是时候放弃对 i486 的支持了。没有任何理由让任何人浪费一秒钟的开发精力在这种问题上。”  

而内核开发者 Ingo Molnar 则直接付诸行动，提交了一份关键补丁，率先移除了 CONFIG\_M486SX、CONFIG\_M486 和 CONFIG\_MELAN 这几个 Kconfig 编译选项。

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEzcTna0PFv1GAutzMzB0CTufru0ibSPkuaGaFLKcOdjso1r3Q120e8FITibfNlb0RnvTia7TfsKzMDxhG2NeAFEgxibM9qxxTKegJI/640?wx_fmt=png&from=appmsg)

这个补丁预计会在 Linux 7.1 中合入，届时用户将无法再编译出支持 i486 的内核镜像。在此之后的内核系列中，真正的 i486 CPU 支持代码会被彻底清理，以减轻后续维护负担。

Ingo Molnar 在提交补丁时提到，维护这种“古董级”32 位 CPU 支持，需要在 x86-32 上提供“各种复杂的硬件模拟机制”，这些兼容代码确实给内核维护者带来了持续的维护难题。所以先从删掉 Kconfig 选项开始，如果没有收到反对意见，后续就会彻底删掉 i486 相关支持。

*   M486SX：面向不带 FPU 的 486 级别 CPU，比如 AMD/Cyrix/IBM/Intel 的 SL/SLC/SLC2/SLC3/SX/SX2 以及 UMC U5S。
    
*   M486：面向标准 486 级别 CPU，比如 AMD/Cyrix/IBM/Intel 的 486DX/DX2/DX4 以及 UMC U5D。
    
*   MELAN：则是针对 AMD Elan 处理器。
    

目前，这份补丁已于本周进入 tip/tip.git 的“x86/platform”Git 分支，不出意外的话，将在本月晚些时候的 Linux 7.1 合并窗口中提交。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0QtgZAx8xsoTReptvArfwbn9MvHGVfV98Qkl5PMRS6MCt2ljwLJwoQ/640?wx_fmt=png)  

对用户的影响

如果你是普通 Linux 用户，可以完全当没这回事。

现在主流发行版（比如 Ubuntu、Fedora）早就不支持这种级别的硬件了，最低要求至少也是奔腾级别。i486 在现实使用中，早就“出局”很多年了。

从某种意义上说，如今放弃 i486 不是 Linux 抛弃兼容性传统，而是一次面向未来的理性断舍离。砍掉几乎无人使用的历史包袱，内核团队能把更多精力投入到新架构、新硬件与性能优化上，让现代 Linux 跑得更稳、更快。

随着 Linux 7.1 发布临近，这款 37 岁的经典处理器将正式与现代内核告别。它见证了 PC 与 Linux 的黄金年代，如今安静退场，也算是给一段传奇画上圆满句号。
