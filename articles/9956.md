# 套壳自家Edge？Win11新版Copilot遭网友深扒：微软让我们用不同的名字下载了7次IE

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

整理 | 苏宓  

出品 | CSDN（ID：CSDNnews）

全球第一大操作系统 Windows 里究竟该内置什么样的 AI 助手，用户才会“买账”？这个问题，微软显然还没有找到答案。

近日，Windows 11 推送了新一版 Copilot。表面上，这是一次“升级”，但实际方向却出人意料，有开发者在拆解新版 Copilot 安装包后发现，这一版本应用中内嵌了一个完整的 Microsoft Edge 浏览器，甚至只需修改 Copilot 安装目录中的关键文件与文件夹，就能直接打开 Edge 浏览器本体。

由此也有人质疑：“他们干脆把 Edge 改成 Copilot，这样就可以向股东谎称人工智能的使用率有多高……他们真是走投无路了。”

那么，这个所谓的 Copilot，到底还是不是一个独立的应用？还是说，它本质上已经变成了一个披着 AI “皮肤”的浏览器了？

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEwlEdYhFQuPoBMZUwgibibvmLMhmZJ3ibF9wRQ2iaOUIeE4zHM2zVFYTJdFdS8JibHnTcsG2xKSKfOWngLQibEYu7cmjs9I31zbfwX5c/640?wx_fmt=png&from=appmsg)

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

网友实锤：Copilot 可直接“变身”为 Edge 浏览器

这一轮讨论的导火索，是开发者 TheBobPony 在社交平台上的一次测试。

他的操作很简单：

找到 Copilot 文件夹，将“mscopilot.exe”重命名为“msedge.exe”，再把文件夹从“Copilot”改为“Edge”，就能直接启动 Microsoft Edge 浏览器。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEwALSMcS572HKlia9IGC1JEgPWtpicfhQKyKXiaGATjlbsR77k9jSVTMLhYrNic3AiaZGPsCPkGlEB305iakKPc7mJx0TXT9ENZkjtAY/640?wx_fmt=png&from=appmsg)

值得注意的是，在他的测试中，系统里原本的 Microsoft Edge 浏览器和 Edge WebView2 组件已经被卸载。

另一位网友 Yae Mica 也指出，反过来操作也可行：“事实上，这个应用（Copilot）本身已经是你正常安装的 Microsoft Edge 的一部分。你只需要重命名文件夹，或者像我做的那样，把文件夹做一个符号链接到 Copilot，然后打开‘mscopilot.exe’，就能直接使用这个应用，而无需安装第二份副本。”

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEzlJ8VjDEnGxibU4k4seUoibic54OowJH0Dco3dp6Cv12gNqibub3SjTqhCXdSGdtra9ehX2AunC5a92yib4xxhV9NTiawRREk6DxPtc/640?wx_fmt=png&from=appmsg)

这是否属于“套壳”？

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QDyP5HTUOXsPlJWd79yygiasaqXicpN7ibIfqiak5WFpaxE1mGxxpfmMjiaA/640?wx_fmt=png)

套壳 Edge？其实不然

对此，外媒 Windows Latest 对 Copilot 安装包进行了拆解分析，给出了更细致的答案。在其看来，新版 Copilot 并非单纯“套壳”Edge，但确实内嵌了一整套完整的 Edge 浏览器。

这一点，从微软商店的下载逻辑中就能看出端倪：即便电脑上已安装 Copilot，在商店搜索“Copilot”仍会显示下载按钮，点击下载后，进度几乎瞬间完成。

原因在于，用户下载的并非 Copilot 应用本身，而是一个类似 Edge 安装器的引导程序，安装过程不由商店直接处理，与当年微软 Teams 的部署方式如出一辙。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEwg3zYA57vnLOmmgcibjnHCcRKQLZFrnMqJMBmQ6yibGVbGleeJtvHAQMqf0q579hX5r6mw9tcn5ic1lXpPVeHo6ia06KNBJ45JVdk/640?wx_fmt=png&from=appmsg)

安装更新后，基于 WinUI 框架开发的旧版原生 Copilot 会从开始菜单及其他位置自动消失，由新版 Copilot 完全接管。

此外，Windows Latest 还进一步拆解 Copilot 的安装目录，发现其中有一个版本号为 146.0.3856.97 的专属文件夹，这正是完整的 Microsoft Edge 安装包，体积约 850MB，包含了 Edge 的全部核心二进制文件：msedge.exe、msedge.dll、msedge\_elf.dll、ffmpeg.dll、libGLESv2.dll、Vulkan/SwiftShader、WidevineCDM 等。

仔细看来，仅 msedge.dll 就有 315MB，坐实了其搭载完整 Chromium 引擎的事实。

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEwgN27nthJMsdM8arqV5ZGJZwMWycCn1zP9pAEakibIhAcfia7XJKj8m2f90GUjK5BdcmYP5Nc2v6WxCO9gRCQImNic1b5ib4jeLicQ/640?wx_fmt=png&from=appmsg)

要知道，如果只是普通的 WebView2 或渐进式网页应用（PWA），它会直接依赖 Windows 11 内置的 Edge，而不是自带一份独立的 Edge 分支。

但新版 Copilot 不仅内嵌了完整 Edge，还捆绑了 WebView2 运行时。在安装包中能找到 msedgewebview2.exe 以及 EmbeddedBrowserWebView.dll 等多个动态链接库，说明它除了独立 Edge，还捆绑了 WebView2 运行时。

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/S1iaf4GgGjExFbSkRdGa65iaNJibQWsuUYINNdzaUAsC99mQQh1eWOWpPgKnLgGHdltYCoPr4Gkz3wpksMqU8LPFsSMcjEIaOcaaoyKnX8dRk8/640?wx_fmt=jpeg&from=appmsg)

这也解释了为何新版 Copilot 比普通网页应用更流畅：微软在其内部打包了私有版 Edge，搭配定制启动器 mscopilot.exe，界面则通过 WebView2 渲染，相当于“双重保障”却也带来了额外负担。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0QtgZAx8xsoTReptvArfwbn9MvHGVfV98Qkl5PMRS6MCt2ljwLJwoQ/640?wx_fmt=png)

内存灾难：后台狂吃 500MB，一用就飚至 1GB

如果说内嵌浏览器环境还只是架构层面的变化，那么内存占用的飙升，会给用户带来最直观的感受。

根据 Windows Latest 的实测，新版 Copilot 的资源占用远高于旧版：

*   在不进行交互的情况下，后台运行的内存占用大约在 300MB 到 500MB 之间浮动。
    
*   一旦开始提问、生成内容或进行复杂交互，占用会进一步上升，在高负载场景下接近 1GB。
    

对比之下，早期基于 WinUI 的版本内存占用通常不到 100MB，两者差距达到一个量级。

对于高配设备而言，这种开销或许尚可接受，但对于使用 8GB 或 16GB 内存的主流设备用户来说，一个常驻后台、随时唤醒的 Copilot，很容易成为系统卡顿的诱因之一。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q3SAo8dibFicWvibnB5u4gdHLgb2AbA6UJ8VImKnyW92hibqZefwpDbPYAQ/640?wx_fmt=png)

定位反复横跳的 Copilot  

回顾 Windows 11 上 Copilot 的迭代历程，很难想象微软究竟想把它变成什么才会满意。

2023 年 2 月 7 日，Copilot 作为 Cortana 的后继产品推出，最初只是 Edge 侧边栏里的 Bing Chat，依附浏览器存在。

随后被拆分为独立窗口的 PWA 应用；之后微软高调推出原生 WinUI 版，主打轻量化和系统级深度整合，让不少用户看到希望，

可没过多久，新版又回归到以 Web 技术为核心、内嵌浏览器环境的路线。

短短两年多时间，Copilot 先后经历了侧边栏、PWA、WebView、原生 WinUI、独立 Edge 内嵌等不同形态，微软始终没能稳定在一套技术方案上，也未能打造出连贯、可靠的系统级 AI 体验。

连 Windows Latest 也辣评道：

“我无法断言微软是否真的在 AI 竞赛中落后，但目前来看，这家公司显然没能为 Windows 打造出稳定可靠的 Copilot 体验，甚至连一种技术方案都无法坚持超过三个月...即便新版 Copilot 用起来像个不错的网页应用，现阶段的 Windows 11 真的不需要更多这类东西了。Windows 11 早已被各种网页应用、PWA、Electron 程序塞得臃肿不堪。”

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0jqAkXB2hEohZkOqYzXVsmJtnNRQncYh54ZQOpcC5ZGSicYFgtBZP4w/640?wx_fmt=png)

网友热议：这是 AI 竞赛落后，还是纯粹敷衍用户？

这种频繁试错、反复推翻的做法，不仅让开发者无所适从，也让普通用户感到困惑：作为微软力推的未来功能，为何始终做不到稳定、高效、不折腾？

有网友调侃道：“微软最厉害的魔术就是让我们用不同的名字下载了 7 次 Internet Explorer。”

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEzicYarVkebYMibxOo58afliasmfAjkDUsYic4XDU4woYaiaCViaAicV9OP46E49qBQG7cg0pYwPIWazIeSL7MQ54piaxCG98tRyicE7USs/640?wx_fmt=png&from=appmsg)

另一位开发者 seemaze 认为：

「情况比这更糟。在我的公司 Mac 上，当我尝试用 Entra 登录时，唯一出现的选项只是一个超链接——“在网页版体验 Copilot”。

而且，看起来所有 Copilot 的“连接体验”实际上都只是一个聊天窗口，并没有真正与它们嵌入的应用实现整合。相比之下，Claude 在 Excel 中的表现更有用……为什么会这样？微软在 Copilot 推出的过程中一直缺乏连贯性，步调零散，这也是我觉得 Anthropic 的产品更胜一筹的原因。它至少能让我真正完成工作。

我唯一的希望是，Copilot 能利用庞大的上下文信息解决微软文档的“泥潭”，帮我快速找到问题答案。当 ChatGPT 最终能够一次性给出 Microsoft 产品查询的正确平台、版本、许可和各种注意事项时……那我才会觉得我们真正达到了 AGI。」

至此，你更新到新版 Copilot 了吗？有什么样的使用感受？
