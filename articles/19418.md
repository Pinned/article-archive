# 开源个 Agent 用的网盘，让 Skill 跨设备同步

俺做滴

  

我做了一个给 Agent 用的网盘，叫 **neuDrive.ai**，开源的

你在 Claude、Codex、Cursor 这些工具里攒下来的 skill、记忆、文件，可以通过 neuDrive 一键备份、相互同步

定位就是：**Agent 的 “OneDrive”**，你的东西跟着你，不被封

GitHub：github.com/agi-bar/neuDrive

手动备份太蠢
------

我在 Claude 上有非常多的 skill，每个都是一轮一轮调出来的。记忆也攒了不少，写作风格、技术偏好、工作习惯，Claude 越用越顺手

但 Claude 封号这件事... 其实我还是很害怕的，怕某一天我的 skill、记忆、文件，**全部被清零**

虽然目前还没有遇到过，但出于保险，我一直定期手动导出 skill 备份。后来 skill 越来越多，手动备份这件事本身就让人想骂人了

另外的，就是我最近同时在用 Claude 和 Codex，有些 skill 两边都要用。手动搬就是：导出、改格式、传过去、测试、发现不对、再改...就很烦

最后，我就自己写了个工具，发现确实好使，于是画了一个月的时间，对主流场景都进行了调试，无论是网页版的 Claude，还是 Codex，亦或者是 Cursor 之类的 IDE，都进行了支持

一句话同步
-----

以 Claude 为例，先打开这个网址：  

https://claude.ai/settings/connectors?modal=add-custom-connector  

然后在里面加一个 Connector，名称填 neuDrive

URL 填 `https://www.neudrive.ai/mcp`

就安装完毕了

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdgiaN9jujezgjpmx31Htojs1PrqW8omf83WmDlvNbkicCvhsNicyL6Dmetcm46b22bph9EhgFWmdaicFaclQ3kXa1N5XcCY17ISBNs/640?wx_fmt=png&from=appmsg)

  

同步的话，更是简单，说一句「同步我的所有东西到 neuDrive」，全部搞定

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdiayroHLT6kz9W8wbjpzQVWTcSg3PiaiarZHicyIzPs8s0HAZNIUP2DCkHLkIcOyD3En26cb3r8w23v3nZX6oI2sgciaStTbdZn3RWM/640?wx_fmt=png&from=appmsg)

  

同步完，neuDrive 后台长这样：136 个技能文件，12 条记忆，718 KiB，全在云端有一份了

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdjyWFxjGDfm8FhxLwwxB2F7P7L14WgjpC27aqX25ezY2rTf5e1cGSfNo2xiasmWJWPcibceRonGdpmMYIqueb2pKmr5JMQcDlgRM/640?wx_fmt=png&from=appmsg)

从 0 到 150，一条命令的事

  

支持的工具：Claude、ChatGPT、Codex、Cursor、Windsurf、Gemini CLI，还有浏览器扩展、通用 MCP 客户端、REST API

  

如果你喜欢命令行，也有 CLI，能够自动化同步各种东西：

git clone https://github.com/agi-bar/neuDrive.git 

cd neuDrive 

./tools/install-neudrive.sh

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdgmo4oojib4iaGZsmW1C8icSfa80VLRUKDmz2mhQnpZLv0eUHfHUcJ9Jt6k56eey9IRiaaTWupOAx6lj2nWgvLgJyNicdJQ9CthHfOk/640?wx_fmt=png&from=appmsg)
