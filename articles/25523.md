# AI 编程进入看板协作时代！Claude Code 刚刚更新：终端开发秒变可交互、实时网页

![](https://mmbiz.qpic.cn/sz_mmbiz_png/lYDricib2fsm75HzpeR3fMKqLicBuhykz9xwxxP899mT4Ca7BeG1SjzV8y8mM3biaeXJiblF9FyIzRJPGXo3cgan2dUu8lLEeoNQa6Vk4hS3zkiaw/640?wx_fmt=png&from=appmsg)

作者 | 青和

国内的端午假期，AI 编程领域又迎来一波更新。

Claude Code和 Codex 分别上线了最新功能，Artifacts 和 Record & Replay。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/lYDricib2fsm4t6WhlEylyxtcTz9aUFhFbHCMER6todVlToZGI5hXhCaedGicmfNOMaCsS3cYanN9Awib3Cp2Po9xDBshicoxlPOfkUJOCicUNs7o/640?wx_fmt=png&from=appmsg)

所谓Record & Replay，就是你把一套操作完整演示一遍，Codex 在旁边观察学习，然后把整套流程打包成一个 skill（技能）。下回要干同样的活，可直接调用这个 skill。

对比之下，还是 CC更让人兴奋！

![](https://mmbiz.qpic.cn/mmbiz_png/lYDricib2fsm6EmUnibXkFE9A7Icv1iaYND7R42eq56tTh61flicuVH9pMbCd8bGsGQfTMNj1GEL2C2HNicrYkib6qvOJAIGenjp43mBvMAyicc1dqU/640?wx_fmt=png&from=appmsg)

开通 Artifacts 能力后，Claude Code 便能够在每次开发对话的进行过程中，直接生成一个实时、可分享、可交互的网页页面。

这个页面可以是 PR 代码走查、系统架构说明、调试时间线、数据看板，也可以是发布 checklist。

事实上，Claude 3.5 Sonnet 时代（2024 年）就曾推出的 Artifacts 功能，当时主要发生在 Claude.ai 的聊天界面里。

![](https://mmbiz.qpic.cn/mmbiz_png/lYDricib2fsm6F7l7X2AGkFDTRA58CLcOYqXahgiarC2uON5LKksAReBSia0zt20FdeLcUicv4uVzT16e5JtsnW2MKzVibicPkRLZaI4o2vTX4hicCo/640?wx_fmt=png&from=appmsg)

用户可以让 Claude 生成代码片段、文档、网页设计、SVG、React 组件、流程图等内容，这些内容会出现在聊天窗口旁边的独立区域，方便查看、编辑、迭代和复用。

而这次 Claude Code 里的 Artifacts，重点在于可实时更新。

它不是单纯让 Claude Code 生成一个网页，而是把 Claude Code 在一次开发 session 里的完整工作进展（上下文），发布成一个可实时更新的视觉页面。这里的上下文包括：

*   当前代码库
    
*   Claude Code 和你的对话
    
*   已经执行过的调查过程
    
*   连接器里的数据
    
*   相关日志、监控、测试结果
    
*   代码里的函数、调用链、diff 等
    

这也意味着，编程工作的过程第一次开始以团队可见的方式沉淀下来，成为了一个“可浏览的工作台”。

从体验上看，Artifacts 的每次发布都是同一个链接下的新版本，并带有版本历史，支持回退。当你在 Claude Code 更新代码时，打开的网页会原地刷新，团队成员能立刻看到最新内容。无需接入额外数据源，链接持续有效。

调用方式也非常简单，在 session 里直接让它创建，或者提出一个明显需要可视化的需求即可。

**从“个人提效”到“团队可见”**

![](https://mmbiz.qpic.cn/sz_mmbiz_png/lYDricib2fsm7UyL9lS8nfZ0I4BJ8PDAESJCvorC71hOZMBe6LLbQ3HicaVa8knbf2RrbmPAU0oviau3tEiaqtTicjoOf0JK32pWgxMetlhoMwAHU/640?wx_fmt=png&from=appmsg)

那么，这个功能到底是为谁、为什么场景策划的呢？

Claude Code 官方贴心列出的九大使用场景，直接打破了“CC 只是程序员专属”的刻板印象。

对于研发内部，它是效率倍增器：

软件工程师：“做一个 artifact 走一遍这个 PR——diff、我的思路、我测试过什么”；

资深工程师/架构师：“把支付服务的架构从代码里画成一个 artifact”，真实的服务拓扑直接从 import 关系生成，告别凭记忆画白板；

设计师/前端工程师：“基于我们的组件库做这个注册表单的 5 个 UX 方案”；

SRE/值班工程师：“把这次事故做成 artifact——时间线、可疑提交、监控里的错误峰值——随着我排查继续重新发布”，事故页面持续生长，排查完直接变成复盘文档；

工程经理：直接生成“这周合并的 PR，按项目分组”的页面。

但最关键的信号在于最后四类角色：法务/开源合规、隐私团队、安全团队、FinOps/平台财务。要知道，这四类角色可能平时根本不会打开终端写代码。但现在：

法务：“列出所有第三方依赖和许可证，标出著作权（copyleft） 风险”；

隐私团队：“追踪所有涉及个人数据的地方，做成给隐私评审用的 artifact”；

安全团队：“把认证相关的审计发现做成 artifact，每条链接到对应代码”；

财务团队：“把 Terraform 里的云资源映射成一个 artifact，按服务分组，标出大头成本在哪”。

从这里可以看出，Claude Code 真正想要扩展的，正是这些具备专业技能和特定工作场景的非程序员群体。

我们有理由相信，代码库，将会变成公司内部越来越多职能的事实数据库，而 Claude Code 试图把它翻译成一个更容易被团队消费的界面。

这一动作，不仅是在和 Cursor、Codex 抢蛋糕，也可能会挤压 Notion、Confluence、Jira 等办公协同工具的市场空间。

**”告别“人工转述”**

将代码库翻译成可视化界面，实际上补上了团队协作的最后一块拼图。

过去，AI 编程工具最尴尬的地方在于：Agent 干了大量活——读代码、跑测试、定位 bug，但大多数时间都发生在一个人的终端里。

到了协作环节，工程师还得重新讲一遍：它看了哪些日志，怀疑哪个 commit，为什么觉得这里是根因。AI 帮你省下来的时间，相当一部分又被“人工转述 AI 工作过程”填平了。

现在，工程师不用需要口干舌燥地复述，只需要把链接丢到团队群里，大家便能看到的同一个视图、同一份上下文、同一个持续更新的开发现场。

B端优先战略

目前该功能处于 Beta 阶段，仅对 Team 和 Enterprise 用户开放，这非常符合 Anthropic 一贯的市场战略。

面向专业级场景，Claude Cowork 已发布 10 款插件，覆盖投行、财富管理、HR、私募股权等场景，并接入 Slack、DocuSign、汤森路透等数据平台，允许企业定制化配置。

Artifacts 则可视为“企业级工作流平台”思路在研发侧的延伸。

过去一年，Anthropic 的增长曲线在 SaaS 发展史上几乎是前无古人、后无来者的存在。

![](https://mmbiz.qpic.cn/mmbiz_png/lYDricib2fsm7YI4Mibzk5YWuT0Q1ehBgDCorp0SOEhsrJSxUgBqDfDpElsz3AK78LmInvbZHPm8Gbibct8s3Ett06Yc2LWxBcV4ooianiat12S3E/640?wx_fmt=png&from=appmsg)

其 ARR（年化收入）从 2025 年初的约 10 亿美元，一路冲到 2026 年 4 月官方披露的 300 亿美元。5 月 SemiAnalysis 数据，更是进一步指向 440–470 亿美元，15 个月翻了约 30 倍，并在企业级 ARR 上首次反超 OpenAI 的约 240–250 亿美元。

更关键的是结构：约 80% 的收入来自企业和开发者 API 调用，个人订阅（Claude.ai Pro/Max）占比不足 20%。

我们常说：进入团队和企业后，真正决定产品粘性的往往不是“单点能力”本身，而是它能不能嵌进组织流程：会不会留下过程？能不能被别人审？能不能回看历史版本？

现在，Artifacts 算是给 CC已经补上了团队工作场景下作为基础设施的重要一层。
