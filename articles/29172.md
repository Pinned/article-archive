# 3位工程师靠「删AI代码」创业，一周收费1万美元：10万行删到3.5万行！网友：“会Vibe Coding的，谁还花钱找你们？”

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

整理 | 郑丽媛  

出品 | CSDN（ID：CSDNnews）

2025 年 2 月，Andrej Karpathy 提出了一个叫“Vibe Coding（氛围编程）”的概念，成为了过去一年多 AI 编程领域最火的关键词之一。而最近，一家名为 Slopfix 的创业团队，则试图靠另一门生意赚钱——专门删除由 AI 写出来的代码。

本周，Slopfix 在 Hacker News 和 Reddit 上发布了一则颇具争议的帖子，标题简单粗暴：

> “我们每周收费 1 万美元，只干一件事：删除 AI 生成的代码。来吐槽一下我们的商业模式吧。”

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjExUSupb7kicFTUJ5DAdNibfW4KhdBqhYLyib653P5ygJSue7ZsDwY5NwmKV1EXnuXdVejaKgrMx3oH7prquECogIeF8ibF0ibRkNksI/640?wx_fmt=png&from=appmsg)

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEywcYzRuFC8Hrb8gUjbq5ZwC5D2w9RZS2f8P3dFANicHOSoMticFKRr292ianb1nZbWhvXz0JKGDV2Pb3j9ib0VZ4ewdBBmpcdrPuQ/640?wx_fmt=png&from=appmsg)

很快，这句话引发了开发者社区的激烈讨论。有人觉得这是 AI 编程时代的新刚需，也有人认为这是纯粹制造焦虑，还有人直言：“真正会 Vibe Coding 的人，根本不会花钱请你们收拾残局。”

但不论如何，Slopfix团队提出的这个观点，却戳中了不少开发者的痛点：AI 写代码越来越快，而维护 AI 写出来的代码，正在成为一门新生意。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

AI 写得越快，代码库越容易失控

根据官网介绍，Slopfix 的三位创始人都拥有多年软件工程经验，他们将自己的定位描述得十分直接：帮助那些由 AI 生成、已经逐渐失去可维护性的代码库，重新回到正常的轨道。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEx3ZHe7icKrHPYQKWxheb0j27ukEL1J8o5fcGLgSrspmyleWvsM4TiaITMuEwHLHKB6Vd8icmXq9cgA3pozYFsqIhDxW06WRElEnI/640?wx_fmt=png&from=appmsg)

他们发现，越来越多客户都会遇到类似情况：

*   项目刚开始时，一切都很顺利。Claude Code、Codex、Gemini CLI 等 AI Coding Agent 几乎可以一路生成代码，几周甚至几天内就能搭起一个完整产品。
    
*   然而，当项目规模逐渐扩大到数万、十几万甚至几十万行代码后，问题开始出现。刚开始，一个新功能可能十几分钟就能完成；后来却变成需要折腾几天，而且修改一个地方，还会意外影响另外两个模块。
    

为什么会这样？Slopfix 给出的解释是：随着代码规模不断增长，AI Agent 已经无法真正理解整个项目。于是，它开始不断重复造轮子，而不是寻找已有实现；开始复制相似逻辑，而不是抽象复用；为了完成眼前任务，不断增加新的代码，而不是整理旧代码。

最终，一个原本还能维护的项目，逐渐演变成了一堆越来越臃肿的“屎山代码”。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QDyP5HTUOXsPlJWd79yygiasaqXicpN7ibIfqiak5WFpaxE1mGxxpfmMjiaA/640?wx_fmt=png)

一门新生意：专门帮企业“删代码”

针对这种情况，Slopfix 提供的服务看起来十分简单。

首先，他们会免费分析客户的代码库。如果判断项目已经没有太大的优化空间，他们会直接告诉客户，不收费。如果认为能改善，他们会先承诺一个明确的优化目标，例如：在保证功能完全一致的前提下，把 10 万行代码压缩到 3.5 万行。

整个项目通常只持续一周，但真正开始修改代码之前，他们会先和客户一起，把整个应用重新梳理一遍：从每一个页面，到每一个 API 接口，再到每一个业务流程，都形成一份完整的功能清单——这份清单既是客户验收的依据，也是团队重构过程中的“安全网”，确保删除代码不会删掉功能。

到了真正的重构阶段，他们会做很多工程师熟悉的工作，例如：把项目里十几套日期格式化函数统一成一个实现；把客户自己写的“蹩脚框架”替换为成熟稳定的开源库；将大量复制粘贴的业务逻辑抽取成统一模块。如果遇到实在救不了的代码，他们就提炼核心功能后重写。

项目结束后，客户获得的不只是一个更小的代码库，还包括完整的 QA 检查清单，以及一套避免代码再次快速恶化的工程规范，例如 CLAUDE.md、Lint 规则和 CI 自动检查等。此外，Slopfix 还承诺提供两周免费质保：如果因重构导致原本正常工作的功能出现问题，他们将免费修复。

收费方式也很特别：代码删得越多，越值钱

相比传统软件咨询按照工时收费，Slopfix 设计了一套比较特殊的模式。

他们固定收费 1 万美元，对应的是：

*   3 位资深工程师；
    
*   连续 1 周的全职投入。
    

不过，客户最终支付多少，与他们具体完成目标的比例直接挂钩。比如，双方约定将代码减少 50%，但最终只减少了 20%，那么完成率就是 40%，客户只需支付 4000 美元。只有达到甚至超过最初承诺的目标，客户才需要支付全额的 1 万美元。

为了避免“删代码”而故意玩数字游戏，他们还明确规定：代码量采用 scc 工具统计，只计算非空、非注释代码，合同中还禁止使用 Code Golf（一种编程挑战，用最少的字符数完成特定任务）等方式，不会把代码压缩成难以阅读的形式，更不会通过删除注释来凑指标。

简单总结的话，Slopfix 真正销售的不仅是“代码行数减少”这个服务，而是可维护性。

“我们也用 Claude Code，但它只是工具”

有趣的是，这家公司并不反对 AI 编程——恰恰相反，他们自己也大量使用 Claude Code。

不过，Slopfix 强调了一句话：AI 只是工具，不是负责人。真正决定如何设计架构、哪些地方应该抽象、哪些地方应该删除、哪些地方必须重写的，还是拥有数十年工程经验的人。

在官方介绍中，创始人甚至调侃说：“我们和 Claude Code 的区别，在于我们拥有三十多年累计的软件工程经验，而 Agent 根本没有投票权。”

网友争论：这到底是不是一门“好生意”？

Slopfix 的帖子发布后，网友迅速分成了两派，其中不少人并不看好。

有人调侃道：“你居然觉得那些靠 Vibe Coding 写代码的人，会愿意花钱请人类工程师收拾残局？”

对此，Slopfix 回应说，他们真正面对的客户，并不是个人开发者，而是那些重新开始创业、借助 Claude Code 快速做出产品的 CTO。“AI 可以帮助他们快速验证想法，但当项目准备交给正式研发团队维护，或者准备接受安全审计时，就必须进行一次真正的软件工程重构。”

还有网友建议：“代码能跑就尽量别动。”

Slopfix 则认为，他们接触的大多数客户都还处于持续开发阶段，已经拥有真实用户甚至付费客户，因此最大的挑战不是“代码还能不能运行”，而是“还能不能继续快速开发新功能”。

除此之外，还有人发出质疑：“写到 10 万行代码的人，不可能突然开始关心代码质量。”

关于这个问题，Slopfix 的回答同样很现实：当他们发现一句“Hey Claude，帮我重构整个代码库”已不起作用、新功能越来越难加的时候，自然就会开始关心代码质量。

对于有人认为他们是在做一个“AI 修复产品”，官方也进行了澄清：“我们没有开发任何 AI 产品，本质上就是一支工程团队，只不过是专门帮助客户重构 AI 生成的代码库。”

那么，你对于 Slopfix 的商业模式又是否看好呢？如果 AI 正在把"写代码"变成一件越来越廉价的事情，那么未来软件工程师真正稀缺的能力，又是什么呢？
