# 微软服软！被骂5年的Win11将被“整改”：告别强制更新、减少Copilot、任务栏摆放自由

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

整理 | 屠敏

Windows 11 自 2021 年发布以来，因任务栏功能缩水、UI 不统一、强制网络登录以及更高的硬件门槛，成为用户集中吐槽的焦点。再加上近来微软猛推 AI 功能，Copilot 的入口几乎无处不在，让不少人直呼功能更新“过于激进”。

随着 2025 年 10 月 14 日 Windows 10 官方支持结束，不少用户不得不被迫升级到 Windows 11。然而当前一提到这款最新的操作系统，各大社交媒体平台充斥着大量负面反馈。

眼看 Windows 11 的评价持续走低，微软终于坐不住了。

近日，Windows 负责人 Pavan Davuluri 发布了一份公开声明——《我们对 Windows 质量的承诺》，并抛出一系列整改计划：从任务栏体验回归，到精简 Copilot 功能，几乎都是用户呼吁多年的改进。

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEx01L0ibmGeSibNN5t3BibJ0PN1IKpg69l2t8pfEJmoxSCYlkpccCjDawZTIuzDvZMC2fFExa0lJKYGMYZRfxRzmexYlWribpnE42E/640?wx_fmt=png&from=appmsg)

不过，这波“诚意整改”，用户是否买账，还要打个问号。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

整改第一弹：任务栏终于能自由摆放了

如果说 Windows 11 最被诟病的设计，那固定在底部的任务栏绝对能排上前列，这引得无数用户怒批它“连 Windows 10 的基础功能都不如”。

时下，微软终于松口，称将在 4 月底前向 Windows Insider 测试用户推送任务栏自定义更新，顶部、左侧、右侧的摆放位置有望全部回归。

这一改动可以说精准戳中了用户痛点，不管是大屏办公还是笔记本日常使用，宽屏能更省空间，老用户也终于能按自己的习惯来调整任务栏位置。

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEz7H6fKbqrUjvgND5aD5DWRj063f5iaAwRNPHl15layzS5ZSiciceDNqV2ZjzjMkE4SCMEsRmXpbWS2SQPOCGUNYUSBPM58cjndd8/640?wx_fmt=png&from=appmsg)

减少 Copilot 出现的频次

除了任务栏，Copilot 的“过度植入”也是 Windows 11 的一大槽点。

从记事本、截图工具到照片应用，甚至连输入法都能看到 Copilot 的入口，不少用户表示“打开个简单的软件，先被 AI 入口晃眼，完全没必要”。

这次微软明确表示要“将 AI 融入真正有价值的场景，并保持精致与专注。”

官方的核心动作就是砍掉不必要的 Copilot 入口，首批整改就瞄准了截图工具、照片、小组件和记事本这四款高频应用。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0QtgZAx8xsoTReptvArfwbn9MvHGVfV98Qkl5PMRS6MCt2ljwLJwoQ/640?wx_fmt=png)

不再强制安装更新

另一项值得关注的变化，是更新机制的松绑。微软希望用户能够更自由地掌控系统更新，而不是在没有注意的情况下被动升级。

微软表示，系统更新应该是可预测、易于安排的体验，因此会赋予用户更多控制权。譬如：

*   你可以在设备初始设置时跳过更新，更快进入桌面；
    
*   也可以在不安装更新的情况下直接重启或关机；
    
*   在需要时也可以暂停更新更长时间。
    
*   同时，还会减少自动重启和通知，让更新过程更加安静。

各方面体验升级

除了以上一些核心整改之外，微软还拿出了一系列面向日常体验的优化计划。

*   文件资源管理器是 Windows 中使用最频繁的功能之一，此次改进将重点提升启动速度、更少闪烁、更流畅的浏览体验，以及更稳定的文件操作性能。
    
*   小组件与信息流体验也将得到更多控制。微软认为，小组件应该是有帮助且相关的，而不是带来干扰或负担，因此会引入更低干扰的默认设置，让用户更自由地控制其出现方式，并优化 Discover 信息流的个性化体验。
    
*   在生态层面，Windows Insider Program 也会更加透明，包括更清晰的频道划分、更便捷的新功能获取方式、更高质量的构建版本，以及更明确地展示用户反馈如何影响产品，同时提供更多直接互动渠道。
    
*   此外，全新的反馈中心（Feedback Hub）已开始向 Insider 用户推送，新版本在设计上进行了重构，让反馈提交更快、更直观，也更方便与社区互动。
    

与此同时，微软还承诺优化系统内存占用、提升开始菜单与搜索响应速度，并对 WSL进行性能改进，兼顾普通用户与开发者需求。

画大饼还是真整改？

虽然整改清单看起来颇具诚意，但有不少网友发现，仍有不少“老问题”被刻意回避。

例如强制微软账户登录的问题，从 Windows 11 发布至今一直备受吐槽，尤其是在重装系统时，不登录账户几乎无法完成基础使用，但此次整改并未提及。

再比如系统广告与推送，微软只提到“减少通知干扰”，却没有明确是否会移除系统内嵌的广告，想要彻底摆脱“广告系统”，恐怕仍然为时过早。

也有人质疑，“更有分寸地融入 AI”，是否只是换一种方式继续植入 Copilot，而非真正减少干扰。

就连国外科技媒体 Ars Technica 也直言：“就像餐厅主厨反复强调自己重视菜品质量，反而会让人怀疑，为什么他需要说这么多次？”

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QGHX4EaTS3AknUHvnayNiaLnANlsxF3g1JR5q9HIdppdNd8c8uyuC05w/640?wx_fmt=png)

网友反应，吐槽与期待并存

对于这次整改，最终究竟会落地多少？HN 上的网友也展开了讨论，有的人质疑道：

微软在过去十多年里一直在做一些与用户利益相悖的事情，而与此同时，Linux 在桌面体验和内核性能上不断改进。现在已经到了这样一个阶段：Linux 在模拟 Windows 的完整 API 空间来运行游戏时，即便在驱动更差的情况下，性能也已经接近，同时还没有隐私侵犯和各种“反用户”的设计。对他们来说，现在才开始尝试转向真正为用户打造一个用户想要的操作系统，已经有点太晚了。那些不愿从 Windows 10 升级的用户，本应该就是一个警钟。

我不认为微软能做到这一点。随着用户认知的转变，这种趋势只会继续下去，而微软要扭转这一局面需要很长时间。目前他们只是在讨论一些相对细微的改进。再加上现在英伟达也开始认真为 Linux 开发驱动，这种转变完全有可能形成滚雪球效应，而微软所做的一切可能都不足以改变这一趋势。

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEwWgKXct7ajPJV5dPBhLvjHIsFLb34EFrX9LevA0ibg82pZnibBUNl4pZgN9wuUJXvEibjtSl0qicsTIcRMdWMg59ibsyJM2LicVwvqs/640?wx_fmt=png&from=appmsg)

也有的人觉得 Windows 还有得“救”：

Windows 在技术层面并不比 Linux 差。它之所以存在一些问题，主要是由于企业层面自上而下施加的一些“反用户”决策。但任何能够使用 Linux 的人，其实都可以绕开这些限制，获得整体上更好的体验。我自己就是同时在两台笔记本上并行使用这两套系统，并通过 KVM 切换器切换，但在很多方面，我仍然更偏爱 Windows。

一些原因如下：即使作为一个能够解决底层问题的低级程序员，我也更愿意把时间花在写程序上，而不是让操作系统“正常工作”。但 Linux 往往会要求我花几个小时去排查各种问题。Windows 在内存和交换（swap）管理方面做得更好，至少在开箱状态下是这样。Windows 拥有稳定的用户空间，并且具备长达 30 年的向后兼容性。Windows 能很好地结合图形界面（GUI）和命令行（CLI），你可以根据任务选择更高效的方式，而一些 Linux 发行版和开发者则带有某种奇怪的“理念纯洁性”，通常不愿意把 GUI 做好。

Windows 内置了一个工具，可以在系统运行时轻松创建完整的系统镜像，而且不要求镜像存储位置必须大于系统盘（包括未使用空间）。Windows 的开发者也不像某些 Linux 生态那样过度依赖动态链接库，以至于依赖管理变得非常麻烦。整体来说，Windows 的用户体验更精致，细节上的“磕碰点”要少得多。

现实来看，微软的这次整改，背后也有现实的压力。随着 Windows 10 官方支持在 2025 年 10 月结束，尽管微软额外提供了一年免费安全更新，但整体生命周期已进入尾声，用户迁移到 Windows 11 几乎不可避免。如果再不改善这些长期槽点，只会进一步削弱用户信任，甚至加速向 macOS 或 Linux 的流失。  

当前从时间来看，官方承诺，这些整改将从 4 月底开始逐步推送给测试用户，正式版发布时间仍未确定。最后，你最期待 Windows 11 的哪项整改？
