# 60岁我大爷想学OpenClaw，百度APP内就能用？！

听说百度APP，没错就是平时百度一下的百度，马上主站要接入OpenClaw了？

说实话听到这个消息，我挺惊讶的，过年前这个节骨眼上，没想到百度还能有这么大的动作。

最近OpenClaw的热度丝毫不减，最近身边非AI圈、非技术圈的朋友，还有位60岁的老领导，都来问我怎么部署了。

![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQn3ibzib86OdibwSAy3XmfyXEw6ue5Ic5aCZHLMl3SGViaZxrv0Xhhw6tIHo3BZyjOgr0y8sJnXnfzqiaRxVV7yaiaTEtG7AkM60qaoY/640?wx_fmt=png&from=appmsg)

那么现在，真的是普通玩家和小白爱好者，也都想尝试的时候了。

毕竟一个自动化的超级搭子，能直接操控电脑、调用工具、自动办公的数字员工，谁不想要呢？

![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQnqhuNuzlMSgEbCTAPDnQic0k5zD3FkTCHQECFozXm9pAeV83z3ZGYn7SVfFOZeN3icvsicu9Imib5l5m1NAbnKJaOdAjNGJSu20ks/640?wx_fmt=png&from=appmsg)

百度APP接入OpenClaw，据说是2月13号正式接。

感觉比去年很多大公司接入DeepSeek，还让我感慨——真的是把OpenClaw带到国民级的人人都能用的程度了。

我自己之前买个 Mac Mini 在本地跑 OpenClaw，不瞒大家说，花了2天时间才跑起来。

本地部署这玩意儿有多折磨人，懂的都懂。

光是调依赖、配容器、解决版本冲突，中间的卡点就有很多，一不小心半天就过去了。

前两天，看到百度智能云上线了 OpenClaw 一键部署时，我第一时间就去试了试。

![](https://mmbiz.qpic.cn/mmbiz_jpg/4KW1YfFgWQmcf2B5uTQM5gcp1HhoxQtnFNNEXuG98xpvOb4ia4Fq1lCNcYzY7J0wSwSJ0vju0bibwodqRxBtZ3iaJ8EytaJ9T1zP03UePlVUkM/640?wx_fmt=other&from=appmsg)

说实话，用之前我是有怀疑的，总觉得花钱少的方式会不会不地道。

百度智能云，居然每天开放 500 个名额，可以直接一分钱用上，效果怎么样呢？

用下来才发现，是我多虑了。

在百度主站APP全量接入之前，先具体跟大家说说怎么用百度智能云部署：

### 01. 小白都能看懂的部署操作，很省力

###   

打开百度智能云的 OpenClaw 部署页面，非常直观。

整个配置流程，核心就是三大步：选择镜像，填写购买时长，点击部署。展开以后也是几个简单动作。

首先第一步，创建一台轻量应用服务器，选择镜像和套餐

![](https://mmbiz.qpic.cn/mmbiz_jpg/4KW1YfFgWQlyIA94ftBplCbBp8hpcyUcT2uQuq1ahKEQKPUs8qYM7O7XqonwNQibRDhoheHrxCnrlmMLXfCXdFg6jJE9j8535xCUpyp9XepI/640?wx_fmt=jpeg&from=appmsg)

然后就是应用配置的一键开通。

等待实例创建完成后进入实例详情页，点击实例管理Tab，应用配置需要您开通千帆大模型、云助手、运维编排OOS产品服务。

点击“一键开通”，阅读服务协议并同意开通。

![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQm5O9a8DMylkvubziaic8sLSmjGB9LL7eibSgjict31x6WIxSuF0Vprt2D8Rr4mRQ7MHOTCxZxiaLlXdtobxyEFeBD9XDBJS0V5L6po/640?wx_fmt=png&from=appmsg)

再之后，点击“一键放行”，放行防火墙18789端口。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQk4e42xgpsSRIzjEUf6vHlV3hzfMk7jYamDSnWkibvV9IL1JqEjLck0VhXEhicAiaxWZQHm5jjgvB2uLvVcYiaBZPBF6I3yRRMNYqQ/640?wx_fmt=png&from=appmsg)

接下来，模型配置。

“模型选择“支持下拉选择模型，可以选择你想要的模型，点击“应用”后，系统会自动创建千帆APIKey进行实例内配置。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQkHGmeIqibwaiaJtJARdkTk9heWwQ7FFXZHOJDiarR05ib861EW1IsUXDX0l2XictqBiaMv4nYicbbic49gPOpyGwjmLXoksKkPxznSeE8/640?wx_fmt=png&from=appmsg)

等待执行成功后，OpenClaw服务就顺利配置完成了！

![](https://mmbiz.qpic.cn/mmbiz_jpg/4KW1YfFgWQlvK6icmtQft4bB1W5OFc6MHF56ar7vGfbxVYavIsYGvbqwx9n5JryPmSvrTf1z1ucSlPGtusrTLUvIibMyUzdhlcNkpfEiaic4SMY/640?wx_fmt=jpeg&from=appmsg)

怎么样，是不是很丝滑？

关键是省心省力还省钱，真的是用最低的成本，无痛体验最当红的AI产品。

到前面部署完成，有朋友可能会说，怎么和我看到别人的龙虾页面，不一样呢？

如果想要在OpenClaw官网网站，跟OpenClaw聊天，可以点击“获取网站地址”，点击网站地址跳转至OpenClaw官网网站。

还有朋友说，如果想把OpenClaw接入常用的聊天软件，怎么办呢？

这就需要从「应用管理」里，找到消息平台配置。

点击“消息平台配置”，选择您的目标消息平台，并输入对应的信息，点击“应用”。

信息可参考输入框右侧“参考文档”进行获取。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQlMZahxBbR4HXNVy7ksMicNYd2Q8eXKicqvmSjlicPdsW0t92VRZXzcrzHxMWvjV4wo2MUOsL62cnKGRNkIDFenWUS5HkMqHgI0ZI/640?wx_fmt=png&from=appmsg)

然后就可以了。

默认提供的就有不少配置，可以按需选择，并点击“应用”。

百度千帆已经上架了不少现成的 Skill，我们也可以集成 github 各种有趣的 Skill，如果有需要，也可以自己做。

看着后台的运行日志，OpenClaw调用了百度的深度研究Skill，先是把这个大标题拆解成了几个子方向。

然后频繁调用搜索工具去查数据、找研报，最后整合校验，甩给我一份内容相当扎实的Markdown格式研报。

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/4KW1YfFgWQlD3tRBDCL2r8buja0qUoCRscaF8JXAATgumWDy81VY3tbST5WRyxrZlq9wBKhMTAGTnWQMbMiceqcnKiaQiaXcI3ueB4hbQ0PwN4/640?wx_fmt=other&from=appmsg)

![](https://mmbiz.qpic.cn/mmbiz_jpg/4KW1YfFgWQkNRFm1DXMyKwVKsEF1yK4yqAk9pbJwddR57p6Gdr58wRw57wd5YhINAHxsOSK2akzkuDibDk5CgWyiboib3MDeVKy9m176ewvv0Y/640?wx_fmt=other&from=appmsg)

从查资料、写报告，到写代码生成HTML，最后再转成PPT，这中间跨了好几个 Skill的调用和上下文切换，**这种连续协作的能力，才是Agent真正能干活儿的关键。**

平时我看英语论文比较多，有时候真的很难啃。

能不能既帮我采集AI论文信息，同时帮我翻译成中文呢？

![](https://mmbiz.qpic.cn/mmbiz_jpg/4KW1YfFgWQnicr04hFQXG5JEXhqwXmHwRVyR4fXjWMDrVAWYbo62y1icVqaSJ0puA93PRKPQXmNm74yHUk687HSa3ZPNtBA7om6kQOWj5Kriao/640?wx_fmt=other&from=appmsg)

就等了一小会儿，一个完美的翻译内容就展现在我面前，真是太爽了。

对话完，我只觉得：如果我们把很多这种好用的 Skill，都集成到我们的百度智能云服务器的 OpenClaw，完全可以有一个独享的专家团。

04. 百度APP调用，改变的是入口逻辑
====================

  

===

我已经在期待，等到2月13日正式上线后，调用方式的测试。

据说是，在百度 App 搜索框输入 @OpenClaw 或在私信界面发送指令，就能直接唤醒已部署的智能体。

我只觉得，搜索框的角色，彻底变了。

过去搜索框负责信息查询，现在它可以成为任务控制入口。

不需要打开单独应用，也不需要进入后台面板，只需输入一句话，好比让系统整理文件、检查数据、发送通知、执行脚本，对应动作就会在后台运行并返回结果。

我们使用软件的习惯，过去20年没有本质变化，但现在不一样了。

应用入口从图标变成语言，操作路径从点击界面变成指令调用。

对普通用户来说体验会非常直观，对开发者来说也意味着新的产品设计方式，因为未来很多功能都可以通过智能体调度完成。

另一个值得注意的点是生态能力调用。

OpenClaw部署在百度生态内，可以直接访问搜索、地图、文库、网盘、小度、爱奇艺等服务接口，这种整合能力让它在执行任务时拥有更完整的数据来源。

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/4KW1YfFgWQk5NLx17ZDgiaiboK3iaZQ43Q0gqaIicR9BGiaCSfqicCInTPBh3JVPMqHiccRZ5CugVc80A77xx2xHV1AMomyzKYOZaBt3tWcvGPlFWE/640?wx_fmt=other&from=appmsg)

比如说，规划出行时能同时读取路线信息和天气情况，处理文档时能调用云端文件库，执行娱乐推荐时能读取视频资源。

这类跨应用协同能力，一直都是大型平台优势，如今被封装进个人智能体后，相当于每个用户都拥有一套可定制工作系统。

当一个拥有数亿月活的平台把智能体能力接入主入口，这件事本身就释放出非常明确的信号。

技术不再停留在实验阶段，开始进入大众使用环境。

用户日常使用频率最高的应用，往往决定技术普及速度。当最常打开的软件具备执行能力，整个使用习惯就会发生迁移。

百度这次的动作值得关注的地方，在于它有魄力、敢把先进能力放进国民级产品。

普通用户能够直接体验到自动执行任务带来的效率提升，开发者也能基于生态接口构建更多场景应用，这种双向开放结构很容易形成正循环。

从部署体验到场景测试，再到即将上线的真实调用，OpenClaw呈现出的轮廓已经很清晰：它代表一种新的软件形态，一种长期在线、随时响应、具备行动能力的个人系统助手。

当这种形态开始规模化出现，人与数字工具之间的关系也会重新定义。
