# π0.6和GEN-1谁代表未来？乾坤未定，但这条底层赛道浮出水面

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gW889cR13aBX42evqQIRibKlicoCrHPEpT0tQiceNphESCa2eJTqstP8G0yqMTkeMFrOGue6kOyCKdTkA/640?wx_fmt=png&from=appmsg)

编辑｜张倩

  

最近一段时间，相信很多具身智能公司都有点着急。

  

大洋彼岸，近期，Generalist AI 公布了一项引发行业关注的进展：据其公开披露，团队基于约 50 万小时真实世界人类行为数据训练的 GEN-1，在若干任务上的成功率超过 99%。其创始人、前 DeepMind 高级研究科学家 Pete Florence 随后发文表示：「当数据规模足够大，且团队能够完整掌控基础模型时，技术边界会以更快速度被推进。」这被视为其此前关于具身智能尺度定律（Scaling Law）判断的进一步验证。

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqGKQeKrNxvYGW5ibYhTczS7dGZoMro0los9CUJL8bVBibHe6HYnLqx7hKUO3lRl9ybBe8v0kDZMnAicIlLUIeB5G7icN229w45ZV7g/640?wx_fmt=png&from=appmsg)

  

但与此同时，更多公司其实还在 scaling 的路上。行业普遍相信，当数据规模与质量跨过某个阈值后，模型性能随规模扩大的提升规律有望更加稳定。

  

然而，在数据量激增的今天，很多公司其实困在了质量上。就像自变量 CEO 王潜所说，现在很多公司看不到 scaling law 是因为数据太糟糕，全是噪声，「以前 80% 的工作在模型算法上，现在 80% 的工作在数据上」。

  

身处行业内的人都知道，这个数字并不夸张。离身智能可以依赖人类无意识「编译」过的互联网语料快速训练，但物理世界从来没有被编译过。那些藏在操作经验里的规律，深埋在噪声里，未经组织。机器不会自己从世界里长出能力，人的经验也不会自然流入机器，中间必须有繁琐的人工采集、转译、组织这条链路，然后才能开始谈「训练」这件事。这就是为什么具身模型迭代往往以月/年计算。这种复杂性，是需要敬畏的。

  

但也恰恰是这份复杂性，在物理世界与模型、本体之间，撕开了一道巨大的机会缝隙。谁能把混沌的物理世界数据，变成结构清晰的训练输入，并构建起反馈闭环，谁就有机会参与这个缝隙的填补，并站上基础设施级的增长快车道。

  

这一层，就是一个巨大的赛道。

  

最近崭露头角的智域基石，就是从这一层长出来的公司。虽然成立才三个月，这家公司已经得到灵初、穹彻、浙江人形、智平方、小苗朗程的投资，在手订单近亿元。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqF5xKfwHqDJ4D3gibYXVjxYWPIY8YQib3MuI7ImBJ5viaQ6t7wUSl8hKTE3JqZEvVeia7xFbZDZOCxAuJj8ohGHpujgaQ6iaIRJc6lM/640?wx_fmt=png&from=appmsg)

  

和很多延续传统「标注外包」思路堆人头、租厂房、采数据的公司不同，智域基石更关注的是如何构建起一套系统化的能力，把人类在物理世界的劳动数字化，用更低的成本把原始数据转换为模型的训练输入。

  

为此，他们提出了「数据编译」的概念。所谓数据编译，就是为物理世界那些高熵、异步、多模态的原始数据，建立一套自动化「精炼管线」—— 去噪、对齐、拆解为「技能原子」，并打上可追溯的语义标签，让算法真正读懂物理规律与动作意图，最后凝结为模型。

  

![](https://mmbiz.qpic.cn/mmbiz_jpg/5L8bhP5dIqG8sHuZCwwOsr43JIuyHW4du16WGiaXricwsMvqJiceibkdgUZn2y6CrBPhj80Fx0zFMahkyh8j6LsGhWgnsKNN7icBgqhmdah9eHzs/640?wx_fmt=jpeg&from=appmsg)

  

就像操作系统为硬件和应用之间搭好了接口，这个编译层也在为物理世界和模型、本体能力之间定义一套「编译器」般的标准。这可不是简单的采集外包，而是工业体系的构建。谁能把这个体系建立起来，谁就有机会成为模型训练上游的「能力入口」，从低附加值的劳务角色，跃升为一个更有价值的基础设施层。

  

为了了解智域基石打算怎么做这件事，为什么他们能做成，我们和这家公司的创始团队聊了聊。

  

物理世界，不接受幻觉

  

在 Claude Code、龙虾成为默认的工作搭子之后，我们已经习惯了「一个指令，生成万行代码」的工作方式。但在物理世界，这样的爽文剧情（一个指令，家里被收拾得干干净净）尚未发生。

  

根本原因在于：物理世界不接受幻觉。代码可以无限回滚，大模型输出错了改个 prompt 就能重来；但机器人抓杯子，手一滑就是真实的水洒一地，每一次决策都有不可撤销的后果。

  

正因如此，物理 AI 对于数据极为挑剔。数据质量有问题，模型效果就会极其糟糕。差之毫厘，谬之千里。

  

但现实是，行业从今年开始进入到不缺原始数据的状态，但另外一个痛点开始浮现：缺的是真正能提高任务成功率的训练输入。大量采集的物理记录里，丢帧、曝光异常、动作无效、传感器不同步等问题比比皆是。这些「数据废料」直接喂给模型，不仅学不到正确的因果关系，还会白白浪费宝贵的算力，甚至让模型变成帕金森病人。

  

然而，要想把这些原始数据加工成可用的训练输入，不是简单的「清洗」或者「标注」就能解决的。

  

![](https://mmbiz.qpic.cn/mmbiz_jpg/5L8bhP5dIqG94icV7UoWv5J1oLdjzUrtGJxDNQMqiaOnBqARkwMcPodrZfZvnMdhWwWMXicpCzUeqyfZpZTMd9P4SibnzibZs3mhUF8EcvibQlmKE/640?wx_fmt=jpeg&from=appmsg)

  

首先，机器人数据通常具有多模态、强耦合、异步采样等特点。当手伸向杯子的那一瞬间，摄像头可能以 30Hz 记录画面，IMU 以 200Hz 甚至更高频率输出惯导信息，关节编码器也运行在各自的采样频率与时钟体系下。对于理想中的训练输入，在时间维度上，需要完成多传感器时间戳同步；在空间维度上，则要完成视觉、末端位姿、力觉 / 触觉与关节状态之间的坐标系标定与对齐。否则就会出现「手已经摸到杯子，视觉却显示还有五厘米」的诡异错位。将这种时空错配输入模型，往往会诱导其学习到错误的感知 — 动作映射。

  

但即便把时空对齐这些「物理层」的问题都解决了，原始数据依然离可用差得很远，因为它本质上是没有语义的。一段视频里，机器人完成了「抓住杯子、抬起、放下」这一连串动作，但对算法来说，这只是一串像素变化和关节角度曲线，它并不知道「这是在拿杯子」，更不知道「为什么要拿」「算不算成功」。如果直接用此类数据训练，模型参数优化时会难以收敛到正确的任务意图与动作结构映射，导致训练效率极低，或产生不可控的输出行为。

  

当数据量级来到万亿小时，真正的难点才刚浮现。问题不再只是技术复杂度，而是成本开始迅速失控。每一个环节 —— 质检、对齐、筛选、重编译 —— 都在吞噬资源。这种爆炸式的成本，传统的「堆人」做法根本扛不住。

  

更难的是，即使你能把这些问题都解决掉，最后的闭环也会成为障碍。谁都想把数据放进真实场景，持续得到反馈，但每一个真实的生产现场都是一座信息孤岛，有其独立的运行逻辑与准入法则。打不通这个闭环，你的数据系统就是静态的，没办法往一个确定的方向迭代。

  

于是你会发现，过去两年里，整个行业几乎把能试的方法都试了一遍：有人靠堆人去疯狂采数据，有人把希望寄托在仿真上，有人尝试统一数据标准，也有人去做资源撮合，把分散的数据拼在一起。每一条路看起来都打通了一小段，也确实在局部起作用。未来这些尝试也仍将继续，但把这些碎片放在一起看，就会发现它们之间很难真正接上：数据还是不可靠，流程还是不闭环，成本也没有被真正压下来。问题并不是某一个环节没做好，而是从原始数据到训练输入这整条链路，本身还没有被变成一件可以稳定复用、持续运转的事情。

  

当然，这事儿做不成，本身也是因为壁垒高。它需要一个跨学科的团队，既要懂机器人，也要懂大数据、云原生，还得能搞得成闭环。里面有些是技术问题，有的已经超出了技术的范畴。所以在现实里，你很难看到一个团队凑齐这些能力。而智域基石，是那个例外。

  

两位 CTO，五层编译管线

  

为什么说智域基石是个例外？看看他们的团队构成就知道了。

  

CEO 杨哲轩带着开源分布式数据库的工程经验入场；两位 CTO，一个是徐良威，走通了机器人从实验室到量产的完整周期；另一位则是杨哲轩多年的技术搭档，专攻大规模数据编译与云原生架构。而 COO 张计业生态构建能力突出，既能整合产业链上下游资源，又能精准对接政府决策逻辑与产业政策导向，擅长把技术优势转化为可落地的产业基础设施。

  

智域基石的股东们明确表示，他们也是看中了这支复合型团队「懂场景、懂数据、懂落地」的稀缺能力，认为其是具身智能产业中极具战略价值的合作伙伴。

  

这样的人才配置，目标只有一个：让数据编译从「手工作坊」升级为「自动化产线」，并最终生长为模型与原始数据之间那个不可或缺的基础设施层。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqGywJMD0UvVLNSicDHfb4nt9icvR67HbOu3s4gCAmpDkI3o7b0kLJic0XHAxzWTklzeyCXicJmicjdCTc8Not8h9GuP9D7wUiakao3ME/640?wx_fmt=png&from=appmsg)

  

整个数据编译管线分为五层。

  

第一层是质检。智域基石把它定义为「来料检测」。这里的「料」来自他们自研的一套 Ego-Centric 采集设备。这不是市面上现成的方案，而是为了把人的感知（看到了什么、听到了什么、碰到了什么）、以及环境本身的三维信息，全都同步记录下来。当然，除了 Ego-Centric 数据，他们同样采集机器人本体的数据。用他们的话说，就是要拿到「最全最原始」的数据，后面才有编译的余地。

  

有了这个前提，再回到质检本身。在这一步，大多数公司的做法是抽检，因为传统的人工质检方式支撑不了全量检测的爆炸式的成本，结果就是高达 95% 的噪声混进训练集，「任务成功率」保障不了。智域基石的做法是全量 —— 不只在数据入库时查，编译和交付的每个环节都持续在查。

  

这里面把成本压住的关键在于：他们用云原生分布式架构把质检拆解成可并行计算的最小单元，配合启发式方法、大模型与几何绑定手段，把原本需要大量人工才能完成的检查，压缩为自动化流水线。据公司测算，在相同覆盖率下，其单位质检成本显著低于传统人工流程。此外，由于与多家头部机器人厂商深度绑定，他们有机会参与质检标准的制定，并在此过程中整合上下游需求，在与客户协同中沉淀出更接近产业实际的质检规范。

  

第二层是底座。通过了质检的数据，还只是一堆散乱的传感器记录，彼此之间既没有时间同步，也没有空间定位。所以这一步要做的，就是把这些多源异构的数据流锚定到同一个时空坐标系里。他们用的是类似数据湖仓的架构，但做了针对具身智能的改造，在底层完成两件关键的事情：一是把超大规模异构数据统一纳管，不再是分散存储的孤岛；二是更重要的，把不同模态、不同频率的数据，在毫秒级做时空对齐，让一帧画面、一个动作、一次力反馈真正对应到同一个「物理瞬间」。

  

第三层是编译。时空对齐之后，数据有了结构，但仍然没有语义。编译层是整条管线技术密度最高的环节：系统会把连续的动作流拆解为带有明确意图与物理约束的「技能原子」。比如一个完整的「拿起杯子」动作，会被拆成若干带有明确起点、终点、动作意图和物理约束的片段。同时，这些片段会被打上标签、建立版本，并记录它们之间的来源关系，确保后续可以复用、追踪、甚至回滚。原始数据中无法直接表达的信息，比如在没有触觉传感器的情况下推算手与物体的接触状态，也会在这一层通过推理补全。

  

第四层是检索。编译完成后，面对的是万亿量级的数据资产。而一个具体的客户，往往只需要其中一个高度精准的切片。智域基石通过自研的检索与查询引擎，让工程师可以用类似 SQL 的方式，从海量数据里直接调用「符合条件的技能片段」。比如一句简单指令，就能调出「厨房场景下拿杯子且成功率大于 95% 的一组动作数据」，系统自动完成召回、筛选和组配，直接输出一套结构化训练数据。这本质上是把具身智能的数据问题当成了一个大规模并发检索问题来解，直接变革了交付效率与商业化规模的上限。

  

第五层是交付，解决的是怎么把动辄几十 TB、上百 TB 的数据，真正交到客户手里，并且能直接用。智域基石的做法，是把「交付」本身产品化。一方面，他们会把前面组配好的数据，自动打包成标准化的数据集，带上清晰的版本号，可以直接进入训练流程，而不是一堆需要二次加工的原始文件；另一方面，它提供弹性的交付方式，既可以通过云端直连、像调用接口一样获取数据，也可以在对安全要求很高的场景下，用高吞吐的物理介质离线交付。这一切都是体系化的，而不是一次性的项目交付。

  

看完这五层，相信大家也明白了，为什么凑齐一个能做「数据编译」这事儿的团队那么难。这里面既有具身智能行业的 know-how—— 你得知道哪些数据有用，哪些特征值得编译；也有云计算、大数据的复杂性 —— 你得知道数据怎么存、怎么切、怎么找、怎么交付、怎么把成本控住。最后成型的，应该是一套可复现、可规模化、可审计的「数据炼化体系」。

  

但说实话，光有这套「数据编译」的能力还不够，毕竟数据本身不是终点，数据能让机器人真正学会干活才是。

  

所以智域基石从一开始就不是只做数据采集、打包交付就完了。他们实际在做的，是一条更完整的闭环，这也是多家具身智能企业明确看好并投资它的重要原因。简单说就是：他们会跟地方政府合作，在真实的工业场景里建数据采集工厂；再配合那些做本体、模型的伙伴（包括股东），一起采集数据、训练模型；最后把训练好的模型和机器人一起部署到那个场景里，让它真去干活，获取宝贵的反馈。

  

这个事听起来好像顺理成章，但其实特别麻烦。要协调各方资源，进入封闭场景，同时还得跟模型、本体公司对齐需求，大家在一块儿把事跑通。这里面涉及数据的入口权和加工权，不是谁都能拿得过来，也不是谁都能把整件事情拉起来。这也能看出智域基石这家公司在技术之外的含金量。

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqEc4Kq5ON5dCH8X00eibjnWYYSh4jyYX7d6MKicGfibIPmznXSIJmWkk5we5vQRchQTVlNXgzpo8LNPPIwBxDXOrWIOn95UmXyRgk/640?wx_fmt=png&from=appmsg)

  

所以，虽然公司刚刚成立三个月，他们已经拿到了近亿元订单。

  

他们的商业模式也很清晰：早期先抓「入口」，通过自建采集能力和第一人称数据设备，获取稳定且高质量的数据来源，同时用定制化数据交付切入客户，完成冷启动和现金流积累；中期随着数据编译管线逐渐成熟，把前期沉淀下来的动作和场景，抽象成标准化数据资产，对外以订阅的方式持续供给，实现数据的复用和规模化变现；再往后，则是把整套能力开放出来，通过 API 和生态，把自己从「数据供应商」升级为「数据基础设施」，让更多开发者和企业在其之上完成数据调用、交易和构建应用。

不做 GEN-1 的信徒，也不押注 π0.6

智域基石的「路线中立」终局观

  

在采访中，其实我们还谈了谈智域基石对于未来数据路线的看法，尤其是在 GEN-1 发布、人类数据（Ego-Centric 数据）受到广泛关注的当下。

  

作为数据层的能力建设者，杨哲轩有着自己的判断。他认为，GEN-1 确实重要，因为它通过建模海量人类行为数据，探索了从真实行为中学习物理世界统计规律的可行性路径，定义了机器人该先学什么。但与之并行的其他路线 —— 如 π0.6 所代表的异构真机数据对齐与部署反馈路线 —— 同样重要，因为它们解决的是如何让世界先验附着到具体机器上，以及如何在现实里越做越好。

  

在杨哲轩看来，未来的终局不会是单一模型路线的胜利，而是三层数据、三段训练闭环的合流 —— 人类数据形成底座，异构真机完成对齐，部署闭环磨出可靠性。真正的数据终局，是能把「真实世界持续数字化」「世界先验迁移到不同本体」「部署反馈稳定回流」这三件事同时打通的基础设施。

而智域基石要做的，恰恰是把这三层数据稳定、高效、可复现地编译成每一代模型都能消化的训练输入，无论未来主流范式仍被称为 VLA，还是演化出新的后 VLA 架构。这种对于行业终局、数据服务需求的清醒认知，让智域基石与不同路线上的多家具身智能公司都达成了良好的协同关系，也会让他们的路越走越宽。

  

在物理世界、模型与机器人本体之间，搭建一个不同路线最终汇流、且难以绕开的基础设施层。这一层，就是智域基石的机会所在。
