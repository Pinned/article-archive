# Claude Code 新功能：手机远程控制

今天 Anthropic 给 Claude Code 加了 Remote Control 功能

![](https://mmbiz.qpic.cn/mmbiz_gif/jXSGuwJvpdiapBDrhy9TTYL8tNtVxmpowmNVUzvxoJDZjtlTRc7joxdd1dv7iapVa4uibyzkMseCnIBbqHJPjFUIVruG4MuPT5eKfNmFRoia3p4/640?wx_fmt=gif&from=appmsg)

你可以理解为：**Claude 版的 OpenClaw**

换句话说，在电脑里启了一个 Claude Code 项目，可以用浏览器或者手机同时连着，可以在这里查到更详细的文档：https://code.claude.com/docs/en/remote-control#from\-an-existing-session

怎么用
---

两种启动方式

一种是直接在项目目录下跑 `claude remote-control`（或简写 `claude rc`），启一个新的远程 session。终端会一直挂着，等待远程连接，同时显示连接状态和工具调用情况

另一种是你已经在 Claude Code 里干着活了，直接输入 `/rc`，当前对话历史直接带过去，不用重新开。建议先 `/rename` 给 session 起个名字，方便在手机端找到

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdhJbG2EbDWkxKuBaxQKhZHaD0T3d2m2fNAVkZmYh11dW0XP7ia9stt81mHBz1WcPm8rOmCdrnXI6oib4iapE9HcjO6gWKmicXvfeTo/640?wx_fmt=png&from=appmsg)

连接方式有三个：

*   • 扫终端里的二维码
    
*   • 直接打开终端里显示的 URL
    
*   • 或者去 claude.ai/code 的 session 列表里找（在线的 session 会有一个绿色状态点）
    

可以调的配置
------

启动的时候可以加 `--sandbox` 参数，开启文件系统和网络隔离。默认是关的

如果你不想每次都手动开远程控制，可以在 Claude Code 里 `/config`，把「Enable Remote Control for all sessions」设成 `true`

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdjbgF4mr3J088SHcicIU0DQuUKaribpiaAiaFbJClQ3d440nZFNQS8853lqoM6TgrBV7JKgAnsGrNJiaWvSKSRergPbRwGS5PzPIeYc/640?wx_fmt=png&from=appmsg)

之后每个 session 自动带，不用每次敲 `claude rc`

安全和连接机制
-------

所有通信走 Anthropic API 的 HTTPS 通道，和正常用 Claude Code 的传输安全一样。本地不开任何入站端口，只做出站请求。连接用的是多组短期凭证，各自独立过期

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdgKBHfcKkjK397x3QQRIiaJicPrWOMaibDiaC7nicF1A5FlWHPwkibcEj1iaF5jOJZaLY3IEN9fppiaOKBBia2U8JZqLtWicpQkPYE9ryX3U/640?wx_fmt=png&from=appmsg)

笔记本合盖了或者网断了，恢复后自动重连。断网超大概 10 分钟才会超时退出，重新 `claude rc` 就行

一点区别
----

有的朋友可能想到 Claude 之前发布的另一个东西，叫：**Claude Code on the Web**

这俩东西有点区别，这里说一下

**Claude Code on the Web** 是 session 直接跑在 Anthropic 的云上。你连 GitHub 仓库，Anthropic 帮你把代码 clone 到他们的云端虚拟机里，环境配置、代码执行全在云端完成。适合你没带电脑、本地没 clone 这个仓库、或者想同时并行跑好几个任务的时候

**Remote Control** 是你的电脑在跑代码，手机和浏览器只是遥控器。你的 MCP server、本地工具链、项目配置全都还在

on the Web 有个好用的地方：在终端里用 `&` 开头发消息，可以把任务甩到云端跑。比如 `& Fix the authentication bug in src/auth/login.ts`，任务就在云端跑着了，你本地该干嘛干嘛。用 `/tasks` 看进度，或者在手机上直接跟它对话、纠偏

反过来也行。云端 session 干完了或者你想接手，在终端里 `/teleport` 就能把云端 session 拉回本地继续

on the Web 的覆盖范围更广一些，Pro、Max、Team、Enterprise 都能用。目前只支持 GitHub 上的仓库

限制
--

回到 Remote Control 这边

每个 Claude Code session 同时只支持一个远程连接。终端关了 session 就断了，需要重新 `claude rc`。多开的话各管各的，互不影响

只支持 Pro 和 Max 订阅账号，API key 不能用

目前 Max 用户以 Research Preview 形式可用，Pro 用户即将开放。Team 和 Enterprise 暂不支持
