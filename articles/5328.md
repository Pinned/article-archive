# 给所有AIFUT大会购票用户的致歉声明。

今天，我们发了[关于AIFUT大会的预告。](https://mp.weixin.qq.com/s?__biz=MzIyMzA5NjEyMA==&mid=2647680939&idx=1&sn=eb6d0cd640399b39908d4e89ea6cbe75&scene=21#wechat_redirect)  
文中，向大家说，会在今晚6点开放售票。  
但是大家在6点准时进入，然后，遇到了卡认证、无法支付、卡票等等的情况。  
  
我不知道该怎么表达我的愧疚，以及该如何向大家道歉。  
也证明了我们草台班子一般的开发能力，这也让我们有了一个重要的教训，在真正面向用户、特别涉及到支付这个事情的时候，一定一定要对开发抱有敬畏，不要觉得，一个没怎么做过真实项目的团队，凭借AI就能做好一个对用户有核心体验的交付。  
  
在焦头烂额几个小时之后，还算稳定了之后，我觉得等跟大家说一下，到底发生了什么。  
  
1、认证入口崩溃  
前期对流量预估严重不足，仅按1500人容量设计认证入口，但实际30s就超过了1500人，导致我们的认证资源包无法支撑实际访问峰值，直接导致入口直接瘫痪。  
花了几万块钱扩容充值，才让认证通道目前恢复正常。  
  
2、支付异常、商户号锁定  
认证拥堵后，订单却可正常发起支付，但输入密码后出现账户异常提示，这时候，因为我们的微信商户号过去没有过交易，瞬时并发量触及微信官方阈值，触发商户号自动安全锁定，以防范异常交易。  
花了一些时间，进行后台申诉和限额调整，才进行解决。  
  
3、退款 / 出票失败投诉  
7点左右的时候，出现了少量退款未到账、出票失败问题，查了每笔订单，都是系统双向崩溃时的时间差异常订单：支付请求超时未返回，后台超时报文自动释放票号，导致订单状态不一致。  
目前财务和技术在逐笔核对支付记录，全部人工兜底退款、补票处理。  
  
4、关于小程序售票的决策反思  
当初选择将售票放在小程序内，本来是希望在同一生态内提升用户互动。但从本次高并发、强稳定性的实际需求来看，这是我们在技术选型与风险判断上的严重失误。也是我们对开发的不尊重和盲目的自大。  
  
目前，系统已经偏向稳定。  
才敢斗胆来跟大家说一声抱歉。  
  
实在实在实在是抱歉，万分抱歉。  
这个大会就像我的孩子一样，出现这样的情况，我很内疚，也觉得非常对不起所有信任我们的伙伴，还有想买票来支持我们的用户。  
  
说真的，今晚处理完这些之后，我一个人坐在电脑前，我真的觉得，我这样的草台班子，是不是根本不配办这个大会。  
  
但不配也得办下去，因为答应大家的事，就得做到。  
接下来我们能做的，就是更加的谨慎和全力。  
对得起大家愿意来的那份信任。  
  
4月8号9号，亦庄见。  
到时候，希望能用现场的体验。  
把今晚欠大家的，都还回来。
