# Linux 7.0发布，Linus突然谈“接班人”：总有一天会有人不怕大版本号

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

整理 | 郑丽媛  

出品 | CSDN（ID：CSDNnews）

长期关注 Linux 内核邮件列表的人，大概已经习惯了这样一种画风：版本发布消息里，既有技术细节，也有冷幽默，偶尔还掺杂一点“人生感悟”。

这一次，在发布 Linux Kernel 7.0 的首个候选版本（rc1）时，**Linus Torvalds** 不仅宣布了新版本，还顺带调侃了一下自己的“数字恐惧症”，甚至半开玩笑地谈到了一个敏感话题——**未来谁来接管 Linux？**

![](https://mmbiz.qpic.cn/mmbiz_jpg/Pn4Sm0RsAuhlqDjD7b0duibl7j4VLibM9GKpU29pTXPnf8xlEXE8y0CH48nEDxWnamtojdViagVyoPKVuGymK1XNg/640?wx_fmt=jpeg)

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

7.0 的诞生：大版本号，但其实没那么重要
---------------------

按照内核社区的节奏，每两周一次合并窗口（merge window）关闭之后，新的 rc 版本就会发布。这一次也不例外。

Torvalds 在邮件中写道：两周过去，合并窗口关闭，Linux 7.0 rc1 正式发布。但紧接着，他就给“7.0”泼了一盆冷水。之所以从 6.x 升到 7.0，并不是因为有什么颠覆性的技术突破，而是——

> “我对大数字有点混乱，不太擅长处理它们。”

这并不是玩笑，而是 Linux 内核长期以来的版本策略。可能很多新手会误以为：主版本号升级 = 架构大改 + 接口废弃 + 重大功能上线。但 Linux 早就不按这个逻辑走了。

内核已经很多年不再区分“稳定版/不稳定版”，也不会因为功能多少来决定是否升主版本号。版本号更多只是时间轴上的里程碑。用 Torvalds 的话来说：

> 这只是“稳步推进”的一个标记。

换句话说，7.0 不代表革命，只代表持续演进。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QDyP5HTUOXsPlJWd79yygiasaqXicpN7ibIfqiak5WFpaxE1mGxxpfmMjiaA/640?wx_fmt=png)

3.5 年一个周期：数字焦虑背后的节奏感
--------------------

简单来说，目前 Linux 的版本演进节奏大致是这样的：

*   一个大版本系列会走到 x.19
    
*   然后进入下一个 y.0
    
*   整个周期大约 3.5 年
    

这个节奏已经运行多年，也成为了社区默认的事情。但问题来了——如果一直这样下去，版本号总有一天会变得很大。

对此，Torvalds 也承认：“我并没有为主版本号变得很大这件事制定明确计划。”同时，他也补了一句：

> “到那时，我相信会有一个更有能力的人来负责 Linux ，而他不会害怕超过十几的数字。”

这段话看似玩笑，但其实透露出一个事实：Linux 社区已经开始认真考虑继任问题。就在不久前，Linux 社区制定了更明确的继任规划和治理机制，以确保在未来某一天，即便 Torvalds 退居幕后，内核项目也能平稳过渡。

按照目前每 3.5 年一次 x.0 的节奏推算，如果要走到 19.x，至少还需要 40 年。届时 56 岁的 Torvalds 虽早已退休，但 Linux 仍会继续前进——而这恰恰是成熟开源项目的标志：创始人重要，但项目不能只依赖创始人。

7.0 值得关注的几个技术点

虽然没有“史诗级更新”，但 Linux 7.0 依然包含多项值得开发者关注的改动。

### （1）Rust 支持进一步巩固。根据内核观察媒体 Phoronix 的分析，本次版本包含关键补丁，进一步稳固了 Rust 在内核中的支持。自从内核开始尝试引入 Rust 作为第二语言后，这条路线就备受争议也备受期待。C 依然是核心，但 Rust 被用于驱动和安全敏感模块的探索正在稳步推进。

（2）更快的缓存清理。新版本改进了缓存清理路径，使缓存释放更加高效。这类改动看似不起眼，但在高并发服务器、虚拟化场景下，会直接影响延迟和吞吐表现。

（3）更平滑的内核更新能力。部分代码为“非中断式内核更新”铺路，进一步提升在线更新能力。对于金融、电信、云计算等高可用场景来说，减少停机时间始终是核心目标。内核层面的支持，是实现真正“零停机”运维的基础设施之一。

（4）跟进新一代处理器特性。针对 **AMD** 与 **Intel** 最新 CPU 特性的支持也在同步推进，包括性能调优与功能适配。内核始终是硬件能力释放的第一道接口，新硬件发布后，Linux 的快速跟进往往决定了生态成熟速度。

（5）RISC-V 与 LoongArch 的持续优化。贡献者还加强了对：RISC-V 和 LoongArch 的支持与性能优化。RISC-V 作为开放指令集架构，正在服务器和嵌入式领域持续扩张；LoongArch 则代表着国产指令集生态的推进。内核层面的优化，是它们走向成熟的关键步骤。

（6）清理“上世纪遗产”。每个版本总会有一点怀旧色彩。这次被移除的是一段用于 1990 年代 ThinkPad 调制解调器的驱动代码——那还是 IBM 时代的产物。

总体而言，Linux 7.0 没有大张旗鼓的特性列表，也没有炫目的重构宣言，但它还是延续了内核项目几十年来的稳定节奏。
