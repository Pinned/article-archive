# 打不过Claude就卷“骨折价”？马斯克Grok 4.5杀疯了：数万块GB300训练，实测硬刚GPT、Claude，价格却不到一半

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

整理 | 郑丽媛  

出品 | CSDN（ID：CSDNnews）

AI 编程模型的竞争，又进入了一个新阶段。

今日凌晨，马斯克旗下 SpaceXAI 正式发布了旗下最新旗舰模型 Grok 4.5，并将其定位为目前公司“最智能的模型”，主打编程（Coding）、Agent（智能体）任务以及知识工作（Knowledge Work）三大场景，并表示在研发过程中还与 AI 编程工具 Cursor 进行了联合训练，进一步强化模型的软件工程能力。

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjExAqdEnucibPkriabSSPrZdibGELw0nvnXE6fhYDp1306Wws6ibEHJZ3emqDSIbJxxQkzh2Qibj0lKpRxU23xPhiaJo8gha1ibzyZV98g/640?wx_fmt=png&from=appmsg)

不过，相比“最强模型”这样的宣传噱头，开发者们更关心的问题其实只有两个：它到底能不能打？值不值得用？

从目前公布的各项 Benchmark、官方技术细节以及开发者的评测结果来看，Grok 4.5 的答案可以概括为一句话：绝对性能未必全面第一，但如果把速度和价格一起算进去，它可能是目前最具性价比的 AI 编程模型之一。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

数万块 GB300 GPU 训练，更强调“真实工程能力”

根据 SpaceXAI 介绍，Grok 4.5 在训练过程中使用了数万块 NVIDIA GB300 GPU，并针对超大规模训练设计了一系列稳定性优化方案。

官方表示，相比单纯扩大训练 Token 数量，他们更关注数据质量。为此，SpaceXAI团队在数据预处理阶段进行了大量工作，包括数据去重、质量评分，以及面向编程、科学、工程、数学等领域的数据筛选——与其不断增加训练 Token，不如让模型学到更多高价值的数据。

与此同时，Grok 4.5 的强化学习（RL）同样进行了大幅升级。据了解，Grok 4.5 的 RL 阶段覆盖了数十万个任务，其中绝大多数都来自真实的软件工程场景，并采用自动化评测和模型评测相结合的方式持续优化。

更有意思的是，SpaceXAI还专门搭建了一套高度异步（Highly Asynchronous）的训练架构：即便某个 Agent 连续执行数小时的复杂任务，训练过程依然可以在后台数万块 GPU 上持续进行，从而不断提升模型在真实工程环境中的推理能力。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QDyP5HTUOXsPlJWd79yygiasaqXicpN7ibIfqiak5WFpaxE1mGxxpfmMjiaA/640?wx_fmt=png)

一个 Prompt，就能完成整个应用开发

在 Grok 4.5 的能力展示方面，SpaceXAI 给出的重点几乎全部围绕软件开发展开：“Grok 4.5 不仅能处理复杂的 Rust、C/C++ 编程任务，还能根据一句  Prompt 从零开始生成完整应用。”

从官网介绍来看，他们展示的多个 Demo 均由模型一次 Prompt（One-shot）完成——用户只需要描述需求，模型便能直接生成设计完整、功能齐全、可运行的应用，而无需多轮调试。  

例如：“创建一个精美的宇宙与太阳系模拟场景，需具备可调节的时间加速功能、逼真的运动轨迹、行星轨道和恒星效果，并使用 Three.js 实现。界面元素（HUD）应设计美观，符合现代设计原则。”

除了编程之外，Grok 4.5 目前也已经成为 Grok Build 的默认模型。

在 Office 场景下，它能自动构建复杂 Excel 表格，完成联网资料检索、跨工作表公式计算，并自动留下备注方便后续修改；同时还能利用 PowerPoint 原生图形绘制流程图、架构图，并在 Word 中生成结构清晰的文档。目前，Word、PowerPoint 和 Excel 插件也已经同步上线。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0QtgZAx8xsoTReptvArfwbn9MvHGVfV98Qkl5PMRS6MCt2ljwLJwoQ/640?wx_fmt=png)

Benchmark 有惊喜，也有短板

那么，它到底有没有达到 GPT-5.5、Claude 等第一梯队模型的水平？从SpaceXAI官方公布的数据来看，答案是：部分任务已经非常接近，但整体仍存在差距。

例如，在测试复杂命令行任务能力的 Terminal Bench 2.1 中，Grok 4.5 获得 83.3%，几乎追平 GPT-5.5（83.4%），距离 Anthropic 的 Fable 5（84.3%）也仅差 1 个百分点。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEz1NKOShibiaMFHvqtAKAicue4Jjia4sKSDbh0dAA8SE2GLbwkcvgIAbt4tANOO8bC1WtoicBOX3RObd4a5s4N4ksml9b5bTx4iaricPY/640?wx_fmt=png&from=appmsg)

不过，到了更加考验真实软件工程能力的 Benchmark，差距开始拉开。

在评估模型解决真实 GitHub Issue 能力的 DeepSWE 1.1 中，Grok 4.5 得分 53%，明显低于 GPT-5.5 的 67% 和 Fable 5 的 70%。

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEwtIwl3JlbJHIicV67Y6JN9NRqdnbZMAfkiaaFluhkULjkPyw7ibDowib1LYxRFiaG35GDKvZlV8HiaoN42QkyZRe6by0sibdTjUGqnicE/640?wx_fmt=png&from=appmsg)

而在更高难度的 SWE Bench Pro 测试中，Grok 4.5 获得 64.7%，虽然已经超过部分配置下的 Claude Opus 4.8，但距离 Fable 5 的 80.4% 仍有不小差距。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEyVDWgdk8OOMu5UBT2nu8drgNj2omV9PicpBcnOUIvSZMtb9Dl2LVBZcyia2sURj6NNQRepVszLNTNpLlxR8iamR65oj7KPSFWf9E/640?wx_fmt=png&from=appmsg)

也就是说，如果只看 Benchmark，Claude 系列和 GPT-5.5 依然保持一定的领先优势。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q3SAo8dibFicWvibnB5u4gdHLgb2AbA6UJ8VImKnyW92hibqZefwpDbPYAQ/640?wx_fmt=png)

四款顶级模型同场 PK：Grok 4.5 究竟能打吗？

当然，Benchmark 只能说明一部分问题。为了更接近真实开发体验，外媒 TryAI 趁热组织了一场测评，规则很简单：

让 Grok 4.5、GPT-5.5、Claude Opus 4.8 以及 Claude Fable 5 接收完全相同的 Prompt，一次生成完整应用，不允许修改 Prompt，也不允许人工修改代码。

（1）第一关：制作一个可打乱、可自动复原的 3D 魔方

Prompt 统一为：“创建一个彩色、具有 3D 效果的魔方，包含‘打乱’和‘复原’两个按钮，并且旋转过程需要具有动画效果。”

这是整场测试中最难的一项，最终只有 Claude Opus 4.8 和 Claude Fable 5 一次成功。而 Grok 4.5 第一次生成时页面上只有标题和按钮，魔方完全没有显示，测试人员使用唯一一次允许的重试机会后，第二次才成功生成完整作品。GPT-5.5 则表现最差，只渲染出了一个深色方块，没有生成真正意义上的魔方。

（2）第二关：粒子引力沙盒

Prompt 统一为：“生成一个基于 Canvas 的粒子引力模拟器，拥有数百个粒子及运动轨迹，点击鼠标可以创建新的引力源，整体视觉效果尽可能具有观赏性。”

这一次，四款模型都成功完成了任务，因此评判标准更多偏向于视觉表现。其中：

*   GPT-5.5 的效果最惊艳，采用了霓虹发光风格，粒子轨迹丰富，整体观感极具沉浸感。 
    
*   Grok 4.5 则偏向简洁风格，粒子围绕引力中心形成规则轨道，动画流畅自然。 
    
*   Claude Fable 5 使用了柔和发光粒子，看起来更加梦幻。 
    
*   Claude Opus 4.8 的粒子数量最多，物理模拟也最复杂，不过视觉效果略逊一筹。 
    

如果只论“颜值”，TryAI 的测试人员认为 GPT-5.5 的生成效果最好。

（3）第三关：生成一款经典的打砖块游戏

Prompt 统一为：“一个可在画布上运行的打砖块游戏：球拍随鼠标移动，小球击碎彩色砖块，包含得分和生命值。”

在这场测试中，四款模型也全部一次成功。无论是挡板控制、小球碰撞、砖块消除、得分统计还是生命系统，都已经具备完整功能，可以直接游玩。其中 Grok 4.5 与 GPT-5.5 的霓虹街机风格更加突出，但整体来看，四款模型都达到了可以直接交付的水平。

在结束三轮测试后，TryAI 的测试团队还额外增加了一项趣味测试：让四款模型仅用 SVG 绘制“一匹马骑在宇航员背上漫步月球”的画面。其中，Claude Fable 5 甚至加入了对白和剧情，被评价为"不仅是在画图，更是在讲笑话"，成为这一轮最有创意的作品。

Grok 4.5 的最大“杀手锏”，其实是价格

真正让 Grok 4.5 引发讨论的，其实不是 Benchmark，而是它的价格。

目前，几个主流模型的API 定价如下：

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjExT4NXicOBl2KlgicYicNBPMgPdGvkoMhnP3LcGt0St2RWzo8icjV6iaoVP2QIicu2Fuhk7Yny9ZHn4x5bCia3ICwBmmLkmPSiazVzQYwk/640?wx_fmt=png&from=appmsg)

仅从单价来看，Grok 4.5 的输入价格只有 GPT-5.5 的 40%、Fable 5 的 20%，输出价格也仅为 GPT-5.5 的 20%。除了单价更低之外，SpaceXAI 还强调，Grok 4.5 在 SWE Bench Pro 上完成相同任务时，所消耗的 Token 数量约为 Claude Opus 4.8 的 1/4.2，同时推理速度可达到 80 Tokens/s。

关于这一点，TryAI 在测评后也得出了类似结论：Grok 4.5 能在不到 0.5 秒内输出首个 Token，整体生成速度达到约 110 Token/s，几乎是 GPT-5.5、Claude Opus 4.8 的两倍，同时也是四款模型中调用成本最低的一款。

所以，如果综合考虑 Token 单价、Token 消耗和生成速度，Grok 4.5 在性价比上的优势十分明显——从某种程度上来说，其实 Grok 4.5 这种定价策略与智谱 AI、DeepSeek 等国产大模型厂商颇为相似：性能先追到第一梯队附近，再依靠更低的价格抢占市场。

目前，Grok 4.5 已正式上线，可通过Grok Build、Cursor、SpaceXAI Console使用，开发者只需申请一个 API Key，即可通过几行代码快速接入 Grok 4.5：
