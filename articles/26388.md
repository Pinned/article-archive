# 一夜之间，ChatGPT与Codex合并了

![](https://mmbiz.qpic.cn/sz_mmbiz_png/KmXPKA19gW889cR13aBX42evqQIRibKlicoCrHPEpT0tQiceNphESCa2eJTqstP8G0yqMTkeMFrOGue6kOyCKdTkA/640?wx_fmt=png&from=appmsg)

编辑｜Youli、杨文

  

ChatGPT 与 Codex 终于要「合体」了。

  

在昨天晚上进行的 OpenAI Intelligence at Work 直播活动上，OpenAI 宣布了一项重大决定：未来几周内，将 Codex 的核心能力直接整合进 ChatGPT 应用中。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqF3WBNibXXfYg9u6eUrVX9RWljzcBVboyMYvFnnEgqibro6k4KUxWdQ8RDZKN2mibdmHiaHUD7nCV3PD111OSDuWN7iazLG5oY9RictA/640?wx_fmt=png&from=appmsg)

  

这意味着 OpenAI 可能不再希望用户把 ChatGPT 和 Codex 看成两个割裂产品，而是要让用户在 ChatGPT 这个高频入口里直接调用 Codex 的执行能力，即 ChatGPT 负责入口和对话，Codex 负责把任务真正做出来。

  

而就在昨天，OpenAI Codex 团队负责人 Tibo 还在 X 上发帖「预热」：「我们应该把『Codex』改名为『ChadGPT』吗？」

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqECpbGHcax29IOjB1yPlLQCCia8icRsu8SnXYmLmeOLYfb1R4b29rgwW9F1M0N03iaYxwm0lIiac78NS6Yu6u0WiaPJabbtnC9yHGUI/640?wx_fmt=png&from=appmsg)

  

下面有网友调侃，应该直接将 ChatGPT 改名为「Codex」。

  

![](https://mmbiz.qpic.cn/sz_mmbiz_png/5L8bhP5dIqFsXTgVCmdrNax4JI41ncZ7hYonaFdVLlwOfaKXWYibolLIQkqeia8Yon21kb4oUsL86f4wFbnnmfwRjKp08pOGHRvNQSdurIEyo/640?wx_fmt=png&from=appmsg)

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqGHZnq8IW4ahW53LTnDbXwLjDjhMPtmKD1XcNQdduNHwmzaC6cpOZFSIg5j0WicWicMKxnqTwSvucwxlqzh5WcTtAyhQwZ8khibB8/640?wx_fmt=png&from=appmsg)

  

现在来看，这一切明了了，ChatGPT 直接「吞下」了 Codex。

  

这一整合背后，是 OpenAI 对企业市场的全面提速。

  

据 OpenAI 新发布的博客显示，Codex 目前每周活跃用户已突破 500 万，自今年 2 月桌面版上线以来增长超过 6 倍。

  

用户结构的变化方面，知识工作者正在以开发者三倍以上的速度涌入这一平台，目前已占全部用户的约 20%。

  

为了承接这股需求，OpenAI 此次同步进行三大核心更新：六个角色插件、Sites、Annotations。

  

三大核心更新

  

首先来看六款面向特定岗位的专属插件，覆盖数据分析、创意制作、销售、产品设计、股权投资和投资银行六大领域。

  

每款插件均内置了针对该岗位的集成功能、操作指引和背景上下文，用户无需繁琐配置即可上手使用。

  

当然，经过个性化调整后，这些工具的效果还会进一步提升。

  

比如，Codex 数据分析插件，我们只需用自然语言提出问题，它就会自动连接数据源、生成并运行 SQL 查询、创建图表和交互式报告。

  

  

Codex 创意制作插件，则是输入创意简报，它就能自动生成 mood board、迭代高端产品摄影图片、优化不同场景版本，并快速产出展示广告，最终一键导出到 Canva 等工具中编辑。

  

  

新功能的推出并不止于此。一项名为 Sites 的能力允许 Codex 将工作成果直接输出为可托管的交互式网站。

  

OpenAI表示，Codex 可以把用户的想法、分析和计划，转化为仪表盘、规划器、评审工作区、项目看板、作品集以及轻量级工具。目前，Sites 可以通过 URL 分享给同一 workspace 的任何人，让团队拥有一个共同空间，用来探索工作内容、贡献意见、跟踪进展，并一起做出决策。

  

比如，你可以让 Codex 为即将到来的客户复盘创建一个 Site，它会生成一个交互式网页，汇总该客户相关的产品更新、待解决问题、使用趋势和下一步行动；你也可以让它基于财务模型构建一个情景规划器，让管理层直接比较不同假设，而不是在文档的多个标签页里来回查找。

  

当然，你还可以让它把发布材料转化为一个持续更新的工作中心，团队可以在里面找到最新话术、里程碑、负责人和决策记录。之后，你还可以让 Codex 随着细节变化持续更新这个 Site。

  

也就是说，团队不再需要为了适应某个单一工具或文件的限制而调整工作方式，而是可以创建真正适配工作的 Sites。而且，Sites 并不是静态页面。它们还可以帮助跟踪重大项目进展，指导客服代表开展工作，或者作为团队创意 brief 的资料库。

  

  

与此同时，新上线的 Annotations 标注功能让用户可以在文档或文件中精确圈定操作区域，使指令执行更加精准，上下文处理也更加高效。

  

  

而这正解决了 AI 工作流里最真实的一个痛点，因为 AI 生成第一版并不难，难的是后续修改。有了 Annotations 功能后，用户可以直接指向某个具体区域，让 Codex 局部修改。

  

比如选中网站导航栏，让它换字体；高亮投资 thesis 里的某个论断，让它解释来源；标记幻灯片里的图表，让它改清楚标签。这个能力原本用于代码、Markdown 和网站，现在扩展到了文档、表格和幻灯片等内容场景。

  

这使 Codex 更接近真实办公协作：不是一次性生成，而是像团队成员一样接受反馈、局部修改、持续迭代……

  

Anthropic 早走了一步

  

就在三周前，OpenAI 刚刚宣布成立专门面向企业客户的合资公司 ——OpenAI Deployment Company，并获得全球投资机构逾 40 亿美元的注资支持，目标直指将 AI 工具深度融入企业的基础设施与业务流程。

  

![](https://mmbiz.qpic.cn/mmbiz_png/5L8bhP5dIqFZ7J3ohxeHZXeUmr83mVaKUvvKtEyfwIw4gxtjYfO5xDiaXZojBTSJbAGic2F0npLoavQq9B8Op1lSlmN6t7Mb1OGH2zQf272eg/640?wx_fmt=png&from=appmsg)

  

不过，OpenAI 在企业端的这场攻势，并非没有对手。

  

Anthropic 早在今年 2 月便推出了企业智能 Agent 计划，面向金融、工程和设计等核心职能场景落地；5 月又专门上线了一批深度面向金融行业的 Agent 产品。

  

相较之下，一向专注消费者市场的 OpenAI，直到今年 3 月才为 Codex 引入插件支持，此番 ChatGPT 与 Codex 的整合，也被外界视为追赶之举。

  

两家公司的路径选择折射出各自不同的基因：Anthropic 以企业为优先、从垂直场景切入；OpenAI 则凭借 ChatGPT 积累的庞大用户基础，以规模效应向企业端渗透。

  

AI Agent 能力日趋成熟，围绕企业客户的竞争将愈演愈烈，而这场角力的胜负，或许最终将取决于谁能真正融入企业的日常工作。
