# Java 近期新闻：严格字段初始化、GlassFish、GraalVM、JReleaser、RefactorFirst

![](https://mmbiz.qpic.cn/mmbiz_gif/YriaiaJPb26VPQqHC66RJFpttVIMWG83T3lWHahUD4bvhxlKSayjeV2ibvC5ydqklP9QHDPD3qHJM07TV3IfHstjA/640?wx_fmt=gif)

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)OpenJDK

JEP 539（JVM 中的严格字段初始化预览版）已经从 JEP Draft 8350458 提升 为候选状态。该 JEP 在 Java 虚拟机中引入了严格初始化字段，这些字段在被读取之前必须先进行初始化。因此，永远不会看到诸如 0 或 null 之类的默认值。该特性可供所有生成 class 字节码文件的编译器直接使用。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)JDK 27

JDK 27 的 早期访问构建Build 29 发布，它是 Build 28 的 升级，修复了各种 问题。要了解关于这个构建的更多细节，请参阅发布说明。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)JDK 28

JDK 28 的 早期访问构建Build 5 发布，它是 Build 4 的 升级，修复了各种 问题。要了解关于这个构建的更多细节，请参阅发布说明。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)GlassFish

GlassFish 7.1.1 发布，带来了 Bug 修复、依赖项升级以及以下改进：优化 WeldUtils 类中定义的 isValidAnnotation() 方法，使其在没有有效的类型名称时立即返回失败； 此外，InjectionServicesImpl 类中定义的 isInterceptor() 方法现在已经启用缓存机制，可以避免在每次请求时都沿 CDI 注入路径遍历注解层次结构这一计算开销较大的操作。

该版本还修复了四个 CVE 漏洞，其中两个影响了 GlassFish 8.0.1、8.0.0、7.1.0 以及 7.0.0-7.0.25 版本：

*   CVE-2026-2586 是一个远程代码执行（RCE）漏洞，允许拥有管理控制台访问权限的攻击者发送经过特殊构造的恶意请求，从而以应用程序服务用户的权限执行任意操作系统命令。
    
*   CVE-2026-2587 同样是服务器端模板渲染机制中存在的 RCE 漏洞，允许攻击者控制底层主机，并实现读取 / 修改数据、执行任意命令、持久化及横向移动等操作。
    

要了解有关该版本的更多详细信息，请参阅发布说明。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)GraalVM

GraalVM 25.1 版本 发布，带来了以下值得注意的变更：生成的原生镜像大小缩减了 3%；支持在 macOS AArch64 平台的原生镜像中使用 G1 GC（仅在 Oracle GraalVM 中提供）； GraalVM 可达性元数据存储库 中提供了共计 1500 个可供 原生构建工具 使用的库。要了解有关该版本的更多详细信息，请参阅发布说明和这段 YouTube 视频。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)Micronaut

Micronaut 基金会发布了基于  Micronaut Core 5.0.4 的 Micronaut Framework 5.0.3 版本，其中包含对以下模块的补丁更新： Micronaut AWS、 Micronaut Data、 Micronaut SQL Libraries、 Micronaut R2DBC、 Micronaut Problem JSON、 Micronaut Test 以及 Micronaut Micrometer。要了解有关该版本的更多详细信息，请参阅发布说明。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)Open Liberty

Open Liberty 26.0.0.7 Beta 测试版 发布，带来了以下值得注意的变更：支持 Jakarta Data 1.1-M3，新增功能包括有状态存储库、新注解 @First ，以及在 @Find 和 @Delete 注解中将 Restriction 和 Constraint 接口用作参数； 在使用基于文件的健康检查机制时，可禁用 MicroProfile Health 4.0 中的 /health 端点。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)JReleaser

JReleaser 1.25.0 发布，带来了 Bug 修复、文档改进、依赖项升级以及一些新功能，包括：支持使用 jlink 和 native-image 处理多种归档格式，可同时生成 ZIP 和 TAR 文件；将密钥的遮蔽方式从全星号改为显示密钥的首尾两个字符。要了解有关该版本的更多详细信息，请参阅发布说明。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)Grails

Apache Grails 8.0.0 的第二个里程碑版本发布，带来了 Bug 修复、文档改进、依赖项升级以及以下值得注意的变更：当向 grails-core 构建传递了错误的目录时，Grails 转换会失败（通过 GlobalGrailsClassInjectorTransformation 类实现）；在使用 Java ThreadLocal 类的实例时，改进虚拟线程清理以及线程绑定框架状态的隔离。要了解有关该版本的更多详细信息，请参阅 发布说明。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)RefactorFirst

Improving 公司首席软件顾问 Jim Bethancourt 宣布了 RefactorFirst 0.0.0 版本。这是一个实用工具，可以用于确定应用程序的哪个部分应该优先重构。本次发布包含以下内容：改进了删除类关系的计算逻辑；参考《面向对象度量实践》一书新增了面向对象不协调检测器；从 D3 迁移至  Vizdom ，提升数据可视化效果。要了解有关该版本的更多详细信息，请参阅 发布说明。

![](https://mmbiz.qpic.cn/mmbiz_jpg/hLLZnAbUwNgTtqhUfZX04TMs3RgTvgRNEg4oro2mIk9ibIWlM5h1icS9fp5aF3ICXBAuyron8dQ9qVWcqMeKicIBdJElA3aesNF5quzm9dznO8/640?from=appmsg&tp=webp&wxfrom=5&wx_lazy=1#imgIndex=1)Java Operator SDK

Java Operator SDK  5.4.0（一个面向 Java 的 Kubernetes 操作符框架）发布，带来了以下值得注意的变更：支持分片选择器（shard selector）（这是一种 Kubernetes 风格的标签选择器，可以在普通标签选择器之外应用），以运行多个副本； 在 @ControllerConfiguration 注解中新增了 defaultFilters 参数，用于提供禁用默认过滤器事件的选项；在 Context 接口中新增了 getSecondaryResource() 和 getSecondaryResourcesAsStream() 方法，用于直接按名称查询单个辅助资源。要了解有关该版本的更多详细信息，请参阅 发布说明。
