# DeepSeek 在招“Harness 研究员”，可能是世界范围内第一次招聘这个职位

DeepSeek 挂出了一个新职位：Agent Harness 研究员。

职位本身不稀奇，稀奇的是这个名字。我翻了 Anthropic、OpenAI、Google DeepMind 的招聘页，Research Engineer 有，Agent 方向的工程师有，但把「Harness」和「研究员」放进同一个职位名的，一家都没有。这大概是全世界第一个 Harness 研究员岗位。

【1】Harness 是什么
--------------

DeepSeek 在 JD 开头给了一个公式：Model + Harness = Agent。模型负责思考，Harness 负责让思考落地：上下文管理、工具调用、文件读写、终端执行、记忆、子智能体调度，所有模型之外的工程，都算 Harness。

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/ZlwY8rlDcFNUNCuniaQ3x4Oib6EaLf5TZ1Iiag4GS81kkoyuJQQcv1posEkBJ517ErnMNavGeNMjT5V15vBsYR1Pnkn06csGKvicAJ8JfoEZAVE/640?from=appmsg)

Claude Code 好用，模型强是一方面，真正拉开差距的是 Harness 这一层：怎么压缩上下文、什么时候开子智能体、工具报错了怎么恢复。同一个模型，套上不同的 Harness，表现可以差出一个档次。

【2】为什么是「研究员」
------------

过去一年，行业普遍把 Harness 当工程问题：写好代码、调好提示词、把 Agent Loop 跑稳就行。这份 JD 把它升格成了研究问题，职责里每一条都是研究的写法。

第一条，探索 Harness 的前沿创新，点名了四个方向：上下文管理、长期记忆、Subagent 与 Multi-Agent、自进化 Agent。

第二条，与模型训练团队合作，「实现模型与 Harness 的共同进化」。Harness 收集到的真实任务反馈回流到模型训练，模型长出的新能力又要求 Harness 重新设计，这是一个双向循环，需要有人专门站在两边中间。

第三条最能说明问题：提出 Harness 领域的基准测试与评测方法，构建评测基准数据。做 benchmark 是典型的研究员工作。今天 Agent 领域连一套公认的 Harness 评测标准都没有，谁先把它建出来，谁就掌握这个方向的话语权。

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/ZlwY8rlDcFN38Wnibv4SDzYs2LicibNiaOGOicictgIfSR8SUY4FDaJdyvAb3DiaHtsRzYPiaB7hgVxHlw4rRAcGWg0KjvrN9uovnRJVzyaHSzlP72Y/640?from=appmsg)

Anthropic 内部一直在做类似的事，Claude Code 团队和模型团队互相喂数据，但他们没把这件事单独写成一个研究员职位。DeepSeek 是第一个把这层窗户纸捅破、明码标价招人的。

我之前写过 Harness Engineering，当时的判断是它会从「工程师的副业」变成一个独立工种。没想到第一个把它写进职位名的是 DeepSeek。

【3】谁适合投
-------

门槛不低。硬性要求是硕士以上、两年科研经验、计算机领域有含金量的论文发表，特别优秀的候选人可以放宽。

但光会发论文还不够。JD 同时要求你是 Agent 产品的高强度用户，深度用过代码类和通用类 Agent 产品，对模型行为有品味、有判断力。发过论文的人不少，每天泡在 Claude Code、Codex 里的人也不少，两样都占的，是个很小的圈子。如果你恰好在这个交集里，这个岗位就是为你开的。

![](https://mmbiz.qpic.cn/mmbiz_jpg/ZlwY8rlDcFPOjRw51ia4t8FspnY9NdkqXIJsRDC8mqaSs6rLkG2pk5mYbQQd8M2iaZSLYmyPcDtBSLmQq8vvOwetGQTrrIKenXZhugywxRzrE/640?from=appmsg)

另外这个职位同时招实习生，在读的同学也可以投。工作地点北京或杭州（团队主体在北京）。

【原始招聘信息】
--------

Agent Harness 研究员（实习/全职）

工作地点：北京或杭州

团队使命

Model + Harness = Agent

我们正在把 DeepSeek 的前沿模型能力，转化为领先的 Agent 产品。这其中除模型本身以外的所有工作，都属于 Harness 的范畴。

你将加入 Harness 团队，与研究员、工程师、产品经理紧密协作，探索 Harness 领域的研究前沿，定义 DeepSeek 对 Harness 的理解。

主要职责

1.  1\. 与 Harness 团队的研究员与工程师深度沟通、紧密合作，共同定义和实现 Harness 领域基于模型能力的前沿创新，包括但不限于上下文管理、长期记忆、Subagent 与 Multi-Agent、自进化 Agent 等领域。
    
2.  2\. 与模型训练团队的研究员与工程师深度沟通与合作，实现模型与 Harness 的共同进化，从 Harness 的角度实现 DeepSeek 的 Harness 与模型的深度适配。
    
3.  3\. 提出 Harness 领域的基准测试与评测方法，构建评测基准数据和制定数据标注策略，从 Harness 的角度研究并优化 Agent 在各领域的智能水平。
    
4.  4\. 以真实世界的任务做为 Harness 研究的重要反馈源，设计相关数据与实验，持续迭代 Agent 能力在真实使用场景下的表现。
    
5.  5\. 基于团队收集到的用户反馈，从 Harness 的角度研究并优化为最广大用户解决真实场景问题的能力。
    

任职要求

1.  1\. 2 年以上计算机科学或相关领域的科研经验，水平过硬，眼界广阔，有科研品味。特别优秀候选人可放宽年限。
    
2.  2\. 硕士学历及以上。特别优秀候选人可放宽。
    
3.  3\. 计算机科学领域具有含金量的论文发表。
    
4.  4\. 面对问题能够独立分析并提出自己的 idea，具备从 0 到 1 推动研究的能力。能够快速将想法转化为可运行的原型，具备高效的实验迭代能力。
    
5.  5\. 熟练使用 AI Agent 工具进行软件开发。在软件开发领域具有极强的学习能力。能够在 AI 辅助下，在没有直接经验的领域（如语言、技术、框架等）进行研究目的的编程工作。
    
6.  6\. 是 Agent 产品的高强度用户，对 Agent Harness 的开发和研究有极大的热情，对模型行为有品味有判断力。深度使用过代码类及通用类 Agent 产品，并将相关产品的使用融入到自己的工作和生活中。
    
7.  7\. 熟悉 LLM 以及 Agent 基本机制及其技术原理，包括 LLM API、KV Cache、Agent Loop、Tool Use、Reasoning、Planning、Skills、MCP、Memory、Subagent、Multi-Agent 等相关知识。对 Prompt Engineering、Context Engineering、Harness Engineering 等课题有深入的理解。
    
8.  8\. 良好的中文沟通能力。
    

加分项

1.  1\. AI 领域相关科研经验，AI 行业的研究员任职经验。
    
2.  2\. 个人开源作品及开源社区的深度贡献。
    
3.  3\. 计算机科学领域、编程领域、或数据科学领域具有含金量的比赛奖项。
    
4.  4\. 其它超乎常人的与此工作相关的才能。
