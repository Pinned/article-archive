# 阿里内部全面禁用Claude Code

7 月 3 日，据多家媒体报道，因近期 Claude Code 被曝存在植入后门的安全风险，阿里经综合评估已将其列入高风险软件名单。自 7 月 10 日起，阿里将全面禁止内部员工在办公环境使用 Anthropic 相关产品，包括 Sonnet、Opus、Fable 等大模型以及 Claude Code，并推荐使用自家发布的 Agentic 编程平台 Qoder 作为替代方案。  
  
公开资料显示，Claude Code 是 Anthropic 于 2025 年推出的 AI 编程助手，也是当前海外最受欢迎的 AI Coding 产品之一。  
  
从外部看来，此次禁用的直接导火索，始于近日持续发酵的一场安全争议。  
  
正如我们[近日报道](https://mp.weixin.qq.com/s?__biz=MzkzMDY1NDgyOQ==&mid=2247832773&idx=1&sn=3893434c7df45ddc4b739244140dc539&scene=142#wechat_redirect)，一名开发者在逆向分析 Claude Code 时偶然发现，从 Claude Code 2.1.91 版本开始，此程序二进制文件中新增了一段普通用户不可见的系统提示词。当检测到用户启用代理时，Claude Code 会收集时区、代理、中国 AI 实验室等运行环境信息，并据此判断是否属于受限制地区，或是否存在模型蒸馏、账号转售等风险场景。由于整个过程不会向用户展示，不少开发者将其称为“隐藏代码”，甚至质疑其具有“后门”性质。  
  
事件曝光后迅速引发开发者社区热议。Anthropic Claude Code 工程师 Thariq Shihipar 随后承认，这是一项于今年 3 月上线的实验功能，目的是识别未经授权的账号共享和模型蒸馏行为，并表示将在后续版本中移除相关机制。不过，不少开发者认为，即便出于安全考虑，在未公开说明的情况下于客户端加入隐藏检测逻辑，仍会削弱企业和开发者对工具的信任。  
  
值得注意的是，这场风波发生的时间点，也正值 Anthropic 与阿里围绕模型蒸馏问题公开交锋。  
  
6 月底，据彭博社援引 Anthropic 致美国国会议员及白宫官员的一封信报道，Anthropic 指控阿里非法获取 Claude 模型的能力。该公司称，阿里在今年 4 月至 6 月期间利用约 2.5 万个虚假账号，对 Claude 发起超过 2880 万次交互，试图提取模型能力，并称这是该公司迄今发现的“最大规模模型蒸馏攻击”。截至目前，阿里尚未公开回应这一指控，而相关说法也尚未得到独立验证。  
  
针对阿里此次全面禁用 Claude Code，不少人认为，更直接的原因仍是企业信息安全考量。随着 AI 编程助手逐渐接入研发流程，这类工具往往拥有读取源代码、访问 API 密钥、环境变量及内部文档的权限。任何未经充分披露的数据收集机制，都可能触发大型企业的安全审查。
