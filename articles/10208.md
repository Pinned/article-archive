##### **【新智元导读】Claude一夜翻车！4月6日的新功能刚一上线，就遭遇大规模宕机，开发者怒喷「神坛崩塌」，性能退化降智、token狂烧，prompt封杀更是惹众怒，Anthropic到底怎么了？**

Anthropic翻车了！

最近几天，全球开发者社区的愤怒，正像火山一样爆发，Claude仿佛一夜之间就跌下了神坛。

就在昨天，Anthropic推出了Claude Code v2.1.92，带来了一个听起来很酷的新功能——Ultraplan。

结果，这个新版本上线的当天，正赶上Anthropic遭遇了一次史诗级的服务中断。

![](https://mmbiz.qpic.cn/mmbiz_png/Rvq8Ow69CYVgib7iccDK8SZJaeu6nqR55Kjt6AGhW9UwVpvETmupuzOcfxOWfyGKsqPakDtQGTPJzYOIQzB9z01Jw8Bru1GiaurdEygS4ibtnMI/640?wx_fmt=png&from=appmsg)

无数满怀期待的开发者看到的不是「Ultraplan」，而是满屏的「授权失败」和「内部服务器错误」。

Claude Code疯狂弹窗，让开发者们登录，然后又通知说登不上去。大量开发者涌入reddit愤怒留言：「这家公司简直成了个笑话！」

不仅如此，Anthropic最近还被龙虾之父曝出一个损招：如果开发者尝试修改系统提示词，Anthropic后台就会直接甩出「400报错」。

或许，这就是此前针对源码泄露事件的补丁，但同样争议巨大。

![](https://mmbiz.qpic.cn/mmbiz_png/Rvq8Ow69CYVqCe3ich27QYre8Syx4JOkiafuHou3ylpwEvxs5W3MOojwIpkdA4dPHyM6sZ3CpLic1iaBoP4Ze85Vlib904PYEqZU6Am1kIe2GXjI/640?wx_fmt=png&from=appmsg)

另外，还有开发者在GitHub提交Issue，直指Claude Code在2月份更新后性能严重下降，在处理复杂工程任务时会完全「失常」，输出不稳定，频繁偏离需求，甚至不可用。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYVwFAPib0LoibVFEV8gjyuialJw2XxjuV6aW7ibIMO5vbfGZlIiaWGI3EfXUM9bEQtsnfU6WibgAIt8OOg8OYBBRzGoQ2eibatEjlJCsU/640?wx_fmt=png&from=appmsg)

短短几天，就接连三次翻车，无论是服务稳定性崩盘，还是核心能力退化，都在证明这样一件事：急于上市的Anthropic，现在已经变得太功利了。

一天一个花哨的新功能，却连基本的稳定都保证不了。

Anthropic，如今是墙倒众人推。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/UicQ7HgWiaUb3uEdSPKrwGNmZEOaaGyzVvZ8dTtE9jU1rFsda3llYbCZpmWfiazUYjWBLTGvlPpXucH8Q0lEUJN3Q/640?wx_fmt=png&from=appmsg)

**新功能一上线**

**立马大规模中断**

Claude Code v2.1.92的这个「Ultraplan」新功能，官方描述听起来让人心动。

**Ultraplan到底是什么？**

它不是新的订阅套餐，而是**Claude Code的一项新功能**，作用是将规划（planning）过程从本地终端迁移到云端。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYXMF9baTl46GpPwVxj6LoTYzgrPLVoibDUy3c5P99Ef9VheU5mEx9j9EeYnZvHAicibZWx47rlbgPC0jhfJIQqCXYZCDVE8KSz7Z0/640?wx_fmt=png&from=appmsg)

生成规划期间，**你可以释放本地终端**，不用一直等着；此外，新功能还提供一个功能更丰富的**网页界面**，可以查看规划的各个部分并添加评论。

最后，你可以选择在**云端执行**规划（需将代码放在GitHub仓库中）或者将规划**发回本地终端**执行。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYXw4ibKXDdU1LAXV8hTb98CPUFPylSWlxPbyGGk8m81TgXYFURY7704iaNVWWzUcNVmtShbtWo9qO2icjDWINeFdcDibCpP398ksYU/640?wx_fmt=png&from=appmsg)

从此，AI辅助开发将进入全新境界：AI不只是帮你补全代码，还能帮你规划整个项目的执行路径。

![](https://mmbiz.qpic.cn/mmbiz_png/Rvq8Ow69CYX5giblQHibXzfj3ceVOw9jMgicGDBx47icicZnfOibVDa9exOJjLfhicbkvx9tQZeo2PKY3Fk8dFkuia1WVdUVZwAe5hRs5zqjqjR4cu0/640?wx_fmt=png&from=appmsg)

**重大雷点：Token粉碎机**

不仅如此，这次Ultraplan还有别的大雷，比如它是一个彻头彻尾的Token粉碎机。

此前，Claude就因为更新后消耗速度莫名加快被质疑，这次就更夸张了。

原因就在于，Ultraplan是一个「云优先」的工作流，会在云端生成庞大到可怕的详细计划，每一次交互都在疯狂吞噬token。

硬核视频博主的实测数据显示，在处理复杂架构的「爆炸半径（Blast Radius）」时，云端规划的速度比本地整整快了2倍。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYVNqNSGqDHfoWia9cE0EGwcUb6b9eeZiaQsBP7boaNoam0jPvtzwGo7PWCia2Qt2NOH7dHJJINibn0kNFeoXO1iaE6MoYf8k1Y8yTag/640?wx_fmt=png&from=appmsg)

而代价，就是Token的指数级飙升！

你原本可以在本地轻量验证的代码直觉，被强制打包送上了云端流水线。每次你用Ultraplan生成一个满意的规划，套餐里的token都在狂烧。

这不是在写代码，是让开发者破产。

并且，Anthropic又被爆出了一个令极客们脊背发凉的「损招」。

**![](https://mmbiz.qpic.cn/sz_mmbiz_png/UicQ7HgWiaUb351381bTy5MO2IN89mV41M88GEiaCCibDxJoaQjYV6HfRtafnmEmfM3R1p0tmkHgBOVuXBD6UJKpsQ/640?wx_fmt=png&from=appmsg)**

龙虾之父Peter Steinberger爆料说，Anthropic开始对「自定义系统提示词」痛下杀手了！

![](https://mmbiz.qpic.cn/mmbiz_jpg/Rvq8Ow69CYWaFPYcfmUhNZc2KTaibOaHics9GJbGPJoOwzu00upU2OKy2K8NOkXEuqxooiaQBQlypchjtGNaVI3kbpW4I3KtpD04kou53VqpCk/640?wx_fmt=jpeg&from=appmsg)

有开发者在使用Claude Code或其他官方工具时，尝试通过修改`system prompt`来获得更个性化的体验。

结果，Anthropic的后台直接「400报错」了！

有人认为，这是针对此前「Claude Code源码泄露事件」的补丁。

但是用户花了昂贵的订阅费，却无法自由定义AI的行为，让开发者社区出现了极大质疑。

**2月后，Claude Code就脑雾了？**

最让核心工程党崩溃的，不是宕机，也不是贵，而是Claude Code真的变笨了！

在GitHub上，有开发者控诉：在2月更新后，Claude Code已无法胜任复杂工程任务了。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYUHicp61SBnuuPDib4Cwx33SbF7flSibnL6PHNmgk1JicGW7EmDAJQycOWe0FIa2UrelANtPbuP3Yc4e4qwsufBWPaz2ZFx0qYuE1g/640?wx_fmt=png&from=appmsg)

这位开发者并非无理取闹，他所在的团队拥有高度复杂的开发环境，并对比了数月的历史日志。

得出的结论令人心惊：从2026年2月开始，Claude在处理多文件逻辑、长上下文保持和工程推理方面出现了明显的性能退化。

![](https://mmbiz.qpic.cn/mmbiz_png/Rvq8Ow69CYUhT76RB99evZ69nuPPicvd4T9HXS3EZhIicc6J6OGhrDOGR7h59JaHeYKNUZKNeSkvXSaIfoRbOLJoiayEVrp1FB7F5DVqdkEQ3U/640?wx_fmt=png&from=appmsg)

比如忽略明确的指令，或者声称已完成，实际上代码根本没动；执行的操作与请求完全相反等等。

这种反馈在评论区引发了强烈共鸣。许多在真实生产环境中使用Claude的企业用户都有同感：AI开始表现出一种「工程懈怠」，实在令人失望。

**摸着石头过河的半成品，**

**开发者还有多少耐心？**

面对层出不穷的BUG，社区达成了一个共识：Anthropic如今太急了。

为了在激烈的AI竞速中抢占高地，他们宁愿在核心产品稳定性还没解决的情况下，全天候地堆砌新功能。

或许，这的确比修Bug容易，也更能吸引眼球。

但对于一个收费的生产力工具来说，开发者要的是稳定运行的产品，而不是什么「530253025020x 专业版超快最大模式Pro Superfast Max」，然后动不动就把整个系统搞崩。

如今，Claude靠着「更好用、更懂工程」建起的城墙，已经在挖掘。

有的团队，已经将Claude比例从80%降至20%，在核心逻辑阶段转向更稳定省钱的本地方案，直接导致API成本下降数百美元。

对AI大厂来说，这是最危险的信号。

着急冲刺上市的同时，希望Anthropic别忘了初心。
