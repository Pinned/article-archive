##### **【新智元导读】**Anthropic 将 Agent SDK 用量从订阅中剥离，按 API 零售价另给固定额度。重度用户的可用量缩水近十倍。同一周，OpenAI 向企业用户推出 Codex 两个月免费迁移。ASI 决赛圈的第一场定价战，开打了。

6 月 15 日起，claude -p、Agent SDK、GitHub Actions，以及 OpenClaw、Hermes 等第三方工具的调用，全部从订阅限额中剥离。

**为什么要拆**

  

**Anthropic 的订阅模式里有一个结构性漏洞。即将上市的 Anthropic 在 ASI 冲刺前夜，已经不能再忍这个漏洞了。**

  

**订阅的经济模型建立在一个前提上，人类打字速度有限。**

  

一个 200 美元/月的 Max 20x 用户，手动交互能消耗的 Token 量折算成 API 零售价约在 2000-5000 美元之间。

  

Anthropic 愿意补贴这个差额，因为交互数据对模型训练有价值。

  

**Agent SDK 和 claude -p 打破了这个前提。**

  

机器 24 小时不停调用，一个 for 循环几小时就能吃完一个月的额度，而且程序化调用几乎没有用户行为数据的训练价值。

  

用户基数小的时候还能忍。

  

但 SpaceX 刚租给 Anthropic 22 万块 GPU，谷歌追加了最高 400 亿美元投资，亚马逊签了 250 亿美元容量协议。

### 在您的 Claude 计划中使用 Claude Agent SDK

更新于今天

  

Claude 订阅计划现在有资格每月获得 Agent SDK 额度。此额度涵盖 Claude Agent SDK 使用、claude -p 命令和基于 Agent SDK 构建的第三方应用。本文介绍了该额度涵盖的内容、工作原理以及如何申领。

从 2026 年 6 月 15 日起，在 Pro、Max、Team 和 Enterprise 计划中可用。使用 API 密钥的 Claude 开发者平台账户不会获得额度。按使用量付费的计费方式保持不变。

  

#### 有什么变化

从 **2026 年 6 月 15 日**起，Claude Agent SDK 和 claude -p 的使用不再计入您的 Claude 计划的使用限制。您的订阅使用限制保持不变，并保留用于 Claude Code、Claude Cowork 和 Claude 的交互式使用。

  

为了支持 Agent SDK 的使用，符合条件的 Pro、Max、Team 和 Enterprise 计划用户可以申领单独的月度额度。该额度仅适用于 Agent SDK 使用，并随您的计费周期刷新。

  

#### 按计划的月度额度

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYUqzL4bwib50kr5whibek3FFZslhdKHGx97pegZWX8qjv7xyOj5X20XxoMFaqLwYG1ONqAejlZbU9A8KzldkWUcsVFEvKz47X0D8/640?wx_fmt=png&from=appmsg)

\*基于席位的 Enterprise 计划中使用标准席位的成员无资格申领 Agent SDK 月度额度。

  

#### 额度涵盖的内容

Agent SDK 月度额度适用于：

*   您自己项目中的 Claude Agent SDK 使用（Python 或 TypeScript）
    
*   Claude Code 中的 `claude -p` 命令（非交互模式）
    
*   Claude Code GitHub Actions 集成
    
*   通过 Agent SDK 使用您的 Claude 订阅进行身份验证的第三方应用
    

  

额度不适用于：

*   终端或 IDE 中的交互式 Claude Code
    
*   Web、桌面或移动应用上的 Claude 对话
    
*   Claude Cowork
    
*   从额外使用中扣除的其他功能
    

  

#### 额度如何工作

**按用户，不共享。**额度属于个人账户。它们不能在团队成员之间共享或汇总。

  

**每月刷新。**您的额度在每个计费周期开始时重置。未使用的额度不会结转到下一个计费周期。

  

**一次性选择加入。**您通过 Claude 账户申领一次额度。之后，它会在每个周期自动刷新。

  

**优先扣除。**Agent SDK 使用首先从您的月度额度中扣除，然后才是其他来源。

  

**超过额度后，使用转移到额外使用。**当您的月度额度用尽时，额外的 Agent SDK 使用按标准 API 费率流向额外使用——但仅当您启用了额外使用时。如果未启用额外使用，Agent SDK 请求将停止，直到您的额度刷新。

  

#### 保持不变的内容

此更改不影响：

*   **订阅使用限制。**作为此更新的一部分，您的计划使用限制未发生变化。
    
*   **交互式 Claude Code。**在终端或 IDE 中使用 Claude Code 继续完全按照之前的方式使用您的订阅使用限制。
    
*   **Claude 对话。**Web、桌面和移动聊天继续使用订阅使用限制。
    
*   **API** **密钥用户。**如果您使用来自 Claude 开发者平台的 API 密钥使用 Agent SDK，则没有任何变化。按使用量付费的计费方式继续进行，您不会获得 Agent SDK 月度额度。
    

  

#### 对于 Team 和 Enterprise 管理员

如果您管理 Team 或 Enterprise 账户，以下是您需要了解的内容：

  

**额度按用户计算。**您团队中的每个符合条件的用户申领自己的额度。额度不能在整个组织中汇总、转移或共享。

  

**大规模生产自动化。**Agent SDK 月度额度的规模适合个人实验和自动化。运行共享生产自动化的团队应使用 Claude 开发者平台和 API 密钥，以获得可预测的按使用量付费计费。

  

**您需要做什么。**现在无需做任何事情。您团队中符合条件的用户将在 2026 年 6 月 15 日之前收到一封电子邮件，其中包含申领额度的详细信息和说明。
