# Cloudflare 新增对 Claude 托管代理的支持

![](https://mmbiz.qpic.cn/mmbiz_jpg/lYDricib2fsm6M6ht6tAMfMBA7tmonNUvyoWyfAQmvnOHkan1fr4ibLQ8bz9gFR0iaVZecp1WMSFWp4PGbyic8G9Kku0Z8kUca1Llrteum6liaiaRc/640?wx_fmt=jpeg)

作者 ｜ Renato Losio

译者 ｜ 平川

最近，Cloudflare 新增了对 Claude 托管代理 的支持，允许开发者在 Cloudflare 平台内运行和管理 Claude 代理。开发者可以将代理连接到私有系统，选择运行时环境，并利用 Cloudflare 服务监控代理的活动。

Cloudflare 与 Claude Managed Agents 的集成，使开发者能够在 Claude 平台上运行代理逻辑，同时利用 Cloudflare 实现代码执行、安全连接和自定义工具。默认部署包含沙箱控制和日志功能，支持轻量级或基于虚拟机的执行方式，提供私有服务访问、浏览器会话监控、内置电子邮件功能以及可自定义的工具。Cloudflare 产品总监 Mike Nomitch 写道：

> 迄今为止，使用 Claude Managed Agents 意味着必须在 Anthropic 提供的基础设施上运行整个技术栈。虽然对于部分开发者来说，这非常理想，但也有开发者可能出于安全、合规或性能等原因，希望对基础设施的选择拥有更大的控制权。

![](https://mmbiz.qpic.cn/mmbiz_png/lYDricib2fsm4gQNk9wHbQRNjwQGOvDLNialVJR9wsUXzI5KHeBQTZfwtayPfLOVYibdqqKIcDOuLNJW3BBgGGPZXnul782TYtvnvqucEdzG3kA/640?wx_fmt=png&from=appmsg)

图片来源：Cloudflare 博客

正如 InfoQ 此前报道的那样，Claude Managed Agents 允许开发者在 Anthropic 平台上运行代理，并内置支持浏览、代码执行和工具使用。该服务现在已经支持自托管沙箱和 MCP 隧道，使开发者能够在自己的环境中运行代理工具，同时将编排工作保留在 Anthropic 平台上。Ampt 创始人 Jeremy Daly 指出，Cloudflare 并非唯一一家利用这一新选项的服务提供商：

> Cloudflare、Daytona、Modal 和 Vercel 都在首发阵容中，其中 Cloudflare 占据了黄金位置，提供了 Browser Run 自动化功能和快速入门模板。

Anthropic 将该模型描述为“将大脑与双手解耦”。代理逻辑和编排功能仍然保留在 Anthropic（即“大脑”）中，而执行层（即“双手”）则可以在用户选择的云基础设施上运行。

Cloudflare 开发平台 通过一些工具（Artifacts 提供的基于 Git 的代码库、使用 Workers AI 实现的边缘推理、Dynamic Workers 提供的按需应用托管）以及其他基础设施服务扩展了代理功能。Cloudflare 区域总监 Sebastian Weiss 解释 了为何这种集成对企业而言至关重要，并强调了可控基础设施和访问权限的重要性：

> 试想有一家银行。代理程序在银行自有的云环境中运行。它通过一个私有连接访问核心系统，该连接绝不接触公网。凭证是注入的，因此代理程序本身不会持有这些凭证。代理操作会被记录下来，供审计人员查阅。决策权仍然由 Claude 掌握，控制权则始终掌握在银行手中。

Cloudflare 提供了 Mesh 和 Workers VPC 服务，帮助在云端和本地环境之间连接私有服务，同时避免将其暴露在公网上。Claude 托管代理可以通过可配置的网络代理使用这些连接，让团队能够限制代理访问权限、安全地注入凭据，并且仅连接到经过批准的内部服务。在 Reddit 上，用户 boysitisover 评论 道：

> Claude 和 Cloudflare 的无服务器计算功能合二为一？听起来像是能在五分钟内把我的银行账户掏空的好办法。

Cloudflare 提供了一个可以自定义的控制平面作为开源部署模板，而开发人员则负责管理与 Claude API 使用及 Cloudflare 资源相关的费用。
