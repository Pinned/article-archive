# Claude Code 引入动态工作流，用于并行代理协调

![](https://mmbiz.qpic.cn/mmbiz_jpg/lYDricib2fsm6MKq4f6k6byVAcq9lJKy37bibuCeDgib6YEH19JbPPMwKPV60ITdEhoYzjEjiarGLia9kLsukpCibicdicnYpeDVecAWkvtVvju8fGc8/640?wx_fmt=jpeg)

作者 | Robert Krzaczyński

译者 | 平川

Anthropic 推出动态工作流（Dynamic Workflows），这是 Claude Code 的一项新功能，旨在通过在单一工作流中协调大量 AI 代理来处理复杂的软件工程任务。该功能目前处于研究预览阶段，可以让 Claude 动态生成协调脚本，将工作分解为子任务，并行执行这些任务，并在呈现最终答案前验证结果。

该版本针对单个代理难以处理的复杂任务进行了优化，例如调查广泛存在的漏洞、管理大规模迁移、进行安全审计、性能审查以及分析复杂软件项目的架构。动态工作流能够管理多个子代理，使 Claude 能够处理那些可能需要数小时甚至数天才能完成的庞大任务。

与依赖人工配置的代理团队不同，Claude 会根据用户目标按需生成工作流。该系统负责规划工作、将任务分配给专业代理、对比和验证结果，并不断迭代直至结果趋于一致。Anthropic 表示，这种方法使 Claude 能够处理那些传统上需要工程团队进行大量协调和监督的项目。

该功能可以通过明确要求 Claude 创建工作流来激活，也可通过启用名为 ultracode 的新设置来激活，该设置会自动让 Claude 判断何时适合采用基于工作流的方法。执行过程会保存进度，中断的运行可以从中断处继续执行，无需从头开始。

初步反馈表明，该功能将许多开发者此前一直手动构建的工作流程进行了规范化。一位 Reddit 用户 写道：

> 说实话，我一直在期待这样的功能。当然，它并不适用于所有场景，但我有一个项目，很久以前就想用它来测试，而我现在正在这么做。虽然还没详细检查输出结果，但速度和自主性似乎都达到了我的预期。

该公司提醒，动态工作流消耗的令牌数量可能远高于典型的 Claude Code 会话。因此，他们建议在将该功能应用于大型项目之前，先从规模较小且范围明确的任务开始。

目前，动态工作流已面向使用 Claude Code Max、Team 套餐的用户及符合条件的企业版用户开放研究预览，同时也可通过 Claude API 以及包括 Amazon Bedrock、Google Vertex AI 和 Microsoft Foundry 在内的合作伙伴平台使用。

这次发布反映了向代理协调系统转变的广泛趋势，其关注点正逐渐从单个模型的性能转向协调大量专业代理，使它们能够协同处理复杂的工程任务。
