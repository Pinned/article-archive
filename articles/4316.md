# 一次“被迫”的迁移：用了22年Windows，微软把我变成了Linux用户

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

你有没有发现，Windows 正在变得越来越“不由自己做主”？

系统用着用着突然黑屏、重启，Copilot、OneDrive 的提示和弹窗无处不在，很多时候，你甚至很难安安静静把一件事做完。

作为全球占有率高达 66.4% 的第一大桌面操作系统，Windows 依然是多数人逃不开的选择。但恰恰也是从 Windows 11 开始，越来越多老用户意识到：系统在“升级”，体验却在倒退——而且这种变化，往往还打着 AI 的旗号。

本文作者正是被这一切一步步推走的那类人。他用了 Windows 二十多年，是开发者，也是音乐人，从来不觉得自己会主动转向 Linux。直到一次又一次更新翻车、Bug 失控，工作流被彻底打断，他才不得不承认：继续忍下去，已经没有意义了。  

当他把这段经历写成博客发出来后，文章迅速冲上 Hacker News 热榜第一，也引发了大量共鸣与讨论。

我从很小的时候就一直在用 Windows。6 岁那年，我的第一台电脑就是一台 Windows 98 机器，配备了 Athlon XP 1900+（Palomino 核心）、GeForce 440 MX，以及有 256MB 充足的内存。

现在回想起来，那块显卡多少有点被坑了，但我还能怎么办？那时我只是个傻小孩。

22 年后我居然还能记得自己第一台电脑的配置，很神奇。总之，Windows 对我来说一直都很熟悉、很顺手。我知道各种使用技巧，也知道如何最大限度地提高它的效率。

二十多年里，我一直是 Windows 的忠实用户，几乎所有和电脑有关的事情，我都会首选 Windows。

哪怕后来成了软件开发者，用上了 MacBook，我有时候还是会回到 Windows。直到微软把它改造成一个我几乎认不出来、也没法再用的东西。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

一切开始崩塌

我认为一切大概是从 Windows 10 的全屏广告开始的。

你知道的，那些“贴心建议”，比如让你试试 OneDrive，或者“使用推荐的浏览器设置”（可以直接理解为：“求你了，用 Edge 和 OneDrive 吧，我们真的很急”）。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/q7kM1wTg0eiaibJmHQr0O6BwW4nUoURJRiaicVzeVA09jSuLRhv7AVibxXm89TQDKH8zaGVSkEFe3clwrpclolFhCFg/640?wx_fmt=png&from=appmsg)

不对，其实真正的开始，可能是那些未经同意的更新：

哦，你在工作啊？好可爱……

不过我们现在要更新了，所以你开的应用我们全关掉，你的电脑归我们所有。

你还有没保存的工作？那太遗憾了，没了。忍着吧。

一开始我选择无视，照常用。偶尔气得不行，也就抱怨两句。

但说到底，能把所有应用都装在一个系统里，还是挺方便的，对吧？

压垮我的最后一根稻草，是 24H2 更新。它和所有“大更新”一样，未经我同意就开始往我系统里装。我知道这次更新有问题，Reddit 上已经一堆人在骂了，于是我选择延期，一直延期。

结果只要我哪天把电脑开着，人不在旁边——BOOM。就这么一下，没人想要的重大系统更新，已经稳稳当当地装好了。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QDyP5HTUOXsPlJWd79yygiasaqXicpN7ibIfqiak5WFpaxE1mGxxpfmMjiaA/640?wx_fmt=png)

Chrome 浏览器事件

24H2 一装上，我就遇到了一个离谱到让我怀疑自己精神状态的 bug。

只要 Chrome 窗口被压在其他窗口下面，它就会开始出现一种我只能称之为“视觉癫痫”的现象。

更糟的是，这玩意有不小概率会直接把整个系统卡死。我只能疯狂拍桌子发泄，真的不是夸张。

我试过回滚系统，结果直接报错，失败了。干脆重装 Windows，问题还是原封不动。

就像得了数字版的疱疹一样，怎么都甩不掉。

那最终的“解决方案”是什么？

装 Insider 版本。

没错，微软稳定版翻车后的解法，居然是让我去用他们的“不稳定版”。

对于 Windows Defender，我也试过用 DDU 把显卡驱动彻底卸干净，再换不同版本测试，完全没用。

摆在我面前的只有两个选择：要么永远停在旧版本不升级，要么就硬着头皮忍着这个问题。至于“干脆永久关掉更新”这种建议，就别提了，真的会把人逼疯。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0QtgZAx8xsoTReptvArfwbn9MvHGVfV98Qkl5PMRS6MCt2ljwLJwoQ/640?wx_fmt=png)

我根本不想要的续集

Insider 版本……算是能用吧。但新的 bug 马上来了。Chrome 在播放视频时，会随机卡死大约 30 秒。

我的选择只有两个：要么等它自己缓过来，要么 Ctrl+Alt+Delete 再 Esc，强行把浏览器救回来。

后来我才发现，这是 NVIDIA 驱动和微软之间的不兼容问题。

进一步深挖后，我觉得这些屏幕闪烁和 Chrome 卡死的问题，很可能与 Multiplane Overlay（MPO） 管线有关。

微软指责 NVIDIA 没有在驱动中正确实现 MPO，NVIDIA 则把责任推回给微软。可以确定的是，一旦你遇到这个问题，几乎无解——两家公司只会互相甩锅，问题始终悬而未决。

需要说明的是，即便我后来退出 Insider 版本，升级到 25H2，这个 Bug 依然存在。而当我把问题发到 r/Microsoft 时，帖子直接被删除了。

显然，这个“最新、最先进”的操作系统不可能坏到无法修复的程度——那问题大概只能是出在我自己身上吧。

也正是在这个时候，我终于开始看清，作为一名 Windows 用户，你真正要面对的现实是什么：

*   随机出现、直接破坏基础功能的 Bug
    
*   未经允许就自动安装、甚至可能把系统搞坏的更新
    
*   Copilot 和 OneDrive 的广告充斥在系统各个角落
    
*   Copilot 按钮不断侵入更多应用
    
*   不借助 Rufus 等工具，甚至无法创建本地账户（终端绕过方式也已被移除）
    
*   没有真正可执行的解决方案，甚至连一次正式的失误承认都没有
    

很多人常说，Linux“太折腾人”。

这一点我并不否认。翻文档、查论坛、刷 Reddit，重新适应一套完全不同的使用逻辑，这些抱怨都很合理。

但当我回头再看上面这份清单时，突然意识到一件事：Windows 现在同样很折腾。

区别在于，在 Windows 上，你付出的所有精力，往往是在和系统本身对抗；而且下一次毫无预兆的更新，很可能就会把之前的努力全部推翻。

也许你会说：“那就关掉更新”、“装 LTSC”“跑个 GitHub 上的精简脚本不就好了”。

但问题是，为什么我要为了一个不尊重用户选择权的操作系统，去层层绕路、不断妥协？

与其如此，我更愿意把时间和精力投入到一个至少懂得什么是“选择权”、也愿意尊重用户的操作系统上。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q3SAo8dibFicWvibnB5u4gdHLgb2AbA6UJ8VImKnyW92hibqZefwpDbPYAQ/640?wx_fmt=png)

外面的世界真的更加精彩吗？

先交代一下背景：我是软件开发者，同时也是一名音乐人。

可以预料到，我一开始对 Linux 的应用生态是有顾虑的，尤其担心常用软件的支持情况，会不会直接打乱原有的工作流。

但在 Chrome 第不知道多少次崩溃之后，我终于意识到该停下来了。于是我干脆一步到位，在主力机器上安装了 CachyOS —— 一个以性能为导向、基于 Arch 的发行版（9800X3D，RTX 5080）。

这个过程并不轻松。

一开始，系统的睡眠模式就是坏的，从休眠唤醒后经常无法识别显示器。

问题还不止于此。Ableton Live 并没有 Linux 原生版本，只支持 Windows 和 macOS。这意味着我几乎没法再继续使用它——除非折腾 Wine（而且支持并不完整），或者保留一个 Windows 虚拟机，同时接受明显的音频延迟损失。

不过，与 Windows 不同的是，在 CachyOS 上，我真的能把 NVIDIA 相关的问题解决掉。

我只需要按照官方论坛里的一个帖子操作：把 NVIDIA 模块加入 mkinitcpio，修改一次配置，再重建 initramfs，问题就消失了。

此外，我还找到了 Ableton Live 的一个不错的原生替代方案——Bitwig Studio，它提供了完整的 Linux 原生版本。

得益于 PipeWire 近年来的持续改进，我现在的音频延迟已经与 macOS 持平，甚至低于 Windows。工作流本身也几乎没有发生变化——毕竟 Bitwig 出自前 Ableton 开发者之手，能看出他们对音乐制作流程非常熟悉。

至于开发工具方面，在 Windows 上你本来就默认要使用 WSL 或 Docker。某种意义上，我只是直接去掉了这个本就不稳定的中间层。

再对比前面那一连串 Windows 的混乱体验，差异已经不言自明了。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0jqAkXB2hEohZkOqYzXVsmJtnNRQncYh54ZQOpcC5ZGSicYFgtBZP4w/640?wx_fmt=png)

你真正要面对的是什么？

如果在三年前有人告诉我，微软会亲手把自己的操作系统折腾成这样，甚至在无意中帮 Linux 做了比任何资深 Linux 狂热粉还多的“宣传”，我大概只会当场笑出声，顺便觉得对方是在胡说八道。

但现实就是如此。

我已经双系统使用 CachyOS 超过一年，而在最近一个月里，我几乎已经完全切换到它上面工作和生活。

如果你也在考虑切换，我的建议是：先做一点功课。

了解一下滚动更新发行版和稳定版发行版之间的取舍，很可能能帮你省下不少麻烦。

对我来说，Cachy / Arch 的快速更新是优势，但你也必须清楚，这意味着你在用稳定性换取新特性。

那么，站在 2026 年这个时间点，从我个人的真实体验出发，Linux 现在到底处在一个什么状态？

*   网页浏览
    

主流浏览器都已经提供了 Linux 原生版本：Chrome、Firefox、Edge、Brave，一应俱全，没有明显妥协。

视频播放同样非常稳定，硬件加速也完全可用——无论是 AMD、NVIDIA，还是 Intel。

*   软件开发
    

Linux 依然是开发者最友好的平台之一。

更完善的终端体验、原生的包管理体系、Docker 无需依赖 WSL 即可直接运行，而你的生产环境，大概率本来就运行在 Linux 之上。

甚至连微软自己，也维护着一套 Linux 发行版：Azure Linux（原名 CBL-Mariner）。

*   内容创作
    

这是很多人默认 Linux 会明显落后的领域，这种判断并非毫无根据，但也并不完全准确：

*   Adobe 全家桶：可通过 Winboat 运行
    
*   体验并不理想（缺少视频加速，偶尔卡顿），但基本可用
    
*   DaVinci Resolve：Linux 原生应用
    
*   专业级视频剪辑软件，且提供免费版本
    
*   Kdenlive：原生、免费、开源
    

总体来看，内容创作在 Linux 上是可行的，但妥协点是否能接受，取决于个人需求。

*   音乐制作
    

*   Bitwig Studio：原生 Linux 的优秀 DAW
    
*   Ardour：原生、免费、开源的 DAW
    
*   音频延迟：得益于 PipeWire，Linux 的音频延迟已经低于 Windows
    

*   游戏
    

这是变化最明显、也最容易被低估的部分。长期以来，“Linux 不适合打游戏”几乎成了共识，但在 2026 年，这种说法正在逐渐失效：

*   Proton / Wine：只要不是内核级反作弊，绝大多数游戏都可以通过 Steam 的 Proton 层直接运行
    
*   性能表现：在 AMD 显卡上，整体游戏性能基本与 Windows 持平
    
*   NVIDIA：过去在 Intel + NVIDIA 组合下，确实存在 10%–30% 的性能损失。不过，近期引入的 Vulkan 扩展正在逐步解决这一问题。
    

NVIDIA 已经发布了利用这些改进的测试版驱动。一旦 Wine、DXVK、Proton 完全跟进，性能差距基本可以忽略。

真正的限制仍然来自部分带有严格反作弊机制的游戏，比如 Valorant、Call of Duty 或《英雄联盟》。

不过坦白说，无法启动《英雄联盟》在我看来反而像是个“附加特性”——也算是促使我装 Linux 的最后一个理由。

当然，也并非全是例外。比如 Arc Raiders 使用了 Easy Anti-Cheat，却依然能在 Linux 上完美运行。我自己也已经沉迷其中。这至少说明：只要开发者愿意支持，技术上并不存在不可逾越的障碍。

*   3D 建模
    

这一领域相较 Windows 和 macOS 仍有明显差距（Autodesk，我说的就是你）。

好在 Blender 提供原生版本，如果它是你的核心工具，那么在 Linux 上并不会遇到太大问题。

*   日常使用体验
    

在基础操作层面，Linux 的响应速度明显更快。

打开文件夹、启动应用、系统整体的流畅度，都有一种“机器终于开始认真干活了”的感觉。

不再需要等开始菜单“想清楚”要不要打开，也不用担心文件管理器在关键时刻卡死。

既然谈到 Linux 的进步，就绕不开一个老问题——那些总说“我想换，但还不行”的人。

“等 Linux 支持 X，我就换。”

Linux 已经支持 X。

“那 Y 呢？”

也支持。

“可 Z 还是不行……”

如果你总是在寻找下一个不切换的理由，那你可能并不是在找解决方案，而是在给自己留在原地找借口。

我以前就是这样的人，所以我很清楚。

当然，也必须说句公道话：确实存在不少场景并不适合切换，尤其是创意工作、3D 建模，以及游戏开发等领域。

如果你正好属于这种情况，完全没必要给自己压力，等生态成熟再说也无妨。

如果你压根不打算切换，那也没有问题。

我不是来评判选择的，只是单纯吐槽微软这些年的操作。

说到底，我其实也并不想切换。

毕竟，没有人愿意从零开始重新适应一台电脑。

我真正想要的，只是 Windows 好好工作。只是，这个期待，微软没有满足。

Windows 的事后回顾

就在我愉快地折腾着全新的 Linux 配置时，Windows 11 正在度过一个灾难性的年份——而现在才刚过去一个月。

根据 Windows Latest 的统计，仅 2025 年一年，Windows 就出现了 20 多个重大更新事故。而 2026 年一开局就火力全开：1 月更新直接导致黑屏和 Outlook 崩溃。

来，下面是一个速览版的 「2025 年 Windows Bug Spotify Wrapped」：  

*   USB 音频设备随机失效
    
*   摄像头无法识别
    
*   BitLocker 设置无法访问
    
*   Adobe Premiere Pro 无法在时间线上拖拽素材
    
*   光标没有原因地一直转圈
    
*   远程桌面会话随机断连
    
*   Copilot 应用被“误删”（说实话，这次反而算是个好改动）
    
*   强制安全更新触发蓝屏
    
*   Windows Hello 人脸识别失效
    
*   文件资源管理器无响应
    
*   游戏时 FPS 暴跌并伴随系统重启
    
*   任务管理器无限生成自己的副本
    
*   深色模式失效，疯狂白屏闪烁
    

那微软的回应呢？

一片沉默。

他们正忙着宣传：现在有 30% 的代码是 AI 写的。别担心，微软，我们真的看出来了。

在 2026 年剩下的时间里，微软正在憋一个大的：用 React Native 替换越来越多的系统原生应用。

但别被名字骗了，这玩意一点也不“原生”。

这些项目的设计目标就是能轻松移植到任何机器、任何架构，因为它的本质只是 JavaScript。

而且，每一个这样的应用，都会单独起一个 Chromium 进程，疯狂吞你的内存——只为了让你有幸打开一个「设置」窗口。

如果只是天气小组件，我或许还能理解。但当这套东西开始侵蚀核心系统应用时——这就是纯粹的懒惰。

我现在要开启一点阴谋论模式了：”我敢打赌，这是因为 LLM 更懂 JavaScript，而微软懒得花钱请真人去写真正的原生代码。“

与此同时，不少政府选择放弃 Windows 迁移到 Linux，“Microslop” 正在社交媒体上流行，Windows 11 的口碑跌到了历史最低点。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q3hdXZlicTdZH0gDHXnpe9sZjuIdKFqdo93tKQ8icc5U5JcMkFHiadezlA/640?wx_fmt=png)

不是我想换，是微软逼我换的

所以，我现在站在这里。已经彻底切换到 Linux。

不是因为我是什么开源原教旨主义者，也不是命令行战士，我只是个普通人。

我换，是因为 Microsoft 变成了 Microslop。
