# 每周死机3次，还疯狂偷数据？一名程序员的Linux迁徙记：“我终于把Win11删光了”

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

【CSDN 编者按】你是否也曾被 Windows 11 的强制遥测烦到崩溃？是否受够了系统更新后层出不穷的 BUG、毫无征兆的黑屏重启？当微软的傲慢让 Windows 体验持续“降级”，越来越多用户将目光投向了开源的Linux。本文作者就是一位从 Win11 彻底“叛逃”的亲历者，他不仅放弃了双系统妥协，更将主力机、工作本全部换成 Linux，还深度体验了 Mint、Debian、Artix 等多款发行版。

原文链接：https://www.notebookcheck.net/I-dumped-Windows-11-for-Linux-and-you-should-too.1190961.0.html

作者 | Sam Medley      翻译 | 郑丽媛  

出品 | CSDN（ID：CSDNnews）

这两年，越来越多开发者从 Windows 逃离到 Linux。我也终于一脚踏进了这片“开源深海”。经过几个月折腾、无数次踩坑之后，我可以负责任地说：这是我过去十多年，甚至可能是一生中，做过最正确的一次电脑相关的决定。

顺便一提，我现在用的是 **Artix Linux**。

目前，我已经在主力台式机和工作笔记本上彻底删除了 Windows 11，把所有分区都交给了 Linux——换句话说：我把每天最依赖的两台电脑，完全托付给了开源社区。

而我选择跳坑，并不是一时冲动，下面是我的真实经历。

崩溃的导火索：遥测监控 + 烂到离谱的软件稳定性
------------------------

最近为什么那么多关于“我从 Windows 转向 Linux”的文章和视频？归根结底就两个原因：

**1）疯狂的数据遥测**  
**2）越来越差的软件体验**

Windows 11 收集数据的方式，简直像十月的南瓜农场——能薅多少算多少，而且你几乎无法彻底关闭。硬件信息、使用了哪些软件、使用频率、行为趋势……通通被收集走。

更离谱的是 Copilot + Recall。微软宣称这是 AI 助手，实际上更像“合法间谍软件”。结果呢？Copilot 既没帮到多少用户，反而帮了微软和数据贩子。

**软件稳定性更是灾难。Windows 11 同一件事有多个设置入口（网络配置、设备管理等），它们之间互不沟通。每次更新不是修 Bug，而是****制造新 Bug**。

在换成 Linux 之前，我当时的硬件是：

*   AMD Ryzen 7 6800H
    
*   32GB 内存
    
*   1TB PCIe NVMe
    

配置一点都不差，但我每周仍然会遇到 2-3 次**整机死机**：卡几秒、屏幕全黑、要么重启，要么直接挂死。

忍无可忍之下，我先双系统了一阵 Linux。结果某次 Windows 自动更新，**直接把 Linux 分区抹了——**那一刻我决定：全盘删 Windows，只留 Linux。

为什么不是 macOS，而是 Linux？  

------------------------

很多人会问：既然逃离 Windows，为什么不去 macOS？

我也试过。

但我唯一的苹果设备是一台 2014 款 MacBook Air，最多只能跑 macOS 11，性能差到不行。用 OpenCore Legacy Patcher 升到 macOS 13 后，iMessage、连续互通等功能更是各种翻车，体验太差了。

而且我还要给台式机找系统，真的不想为了装个新系统再掏一两个月房贷。

于是我选择了 Linux ——高兼容 + 完全掌控权 + 老机器复活神器。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0QtgZAx8xsoTReptvArfwbn9MvHGVfV98Qkl5PMRS6MCt2ljwLJwoQ/640?wx_fmt=png)

发行版踩坑史：从 Mint 到 Artix
---------------------

Linux 的发行版就像口味各异的咖啡。

*   **Linux Mint**：新手神作，“开箱即用”，但对我来说太臃肿了。我把它装在家庭影院 PC 上，非常稳定，4、5 岁的孩子都能用。
    
*   **Debian/Fedora/Bazzite/Void**：我也都试过，这些发行版整体都还不错，但 Void 的仓库太贫瘠，驱动还各种翻车。
    
*   **最终选择：它是 Arch Linux 的衍生版本。Artix 继承了 Arch 的全部功能和可定制性，包括资源丰富的 Arch 用户仓库（AUR），能提供海量软件包和应用程序。同时它又是一款轻量化发行版，预装软件极少。**
    

Artix 和 Arch 的核心区别在于，它没有采用 SystemD 作为初始化系统。关于初始化系统的争论这里就不展开了，简而言之，Artix 的启动速度快得惊人 —— 冷启动时间不到 10 秒，而且对系统资源的占用极低。

当然，它的缺点也很明显：并非“开箱即用”的成品系统，你得自己动手。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q3SAo8dibFicWvibnB5u4gdHLgb2AbA6UJ8VImKnyW92hibqZefwpDbPYAQ/640?wx_fmt=png)

迁移路上的那些坑
--------

我的笔记本是 2014 款 MacBook Air，硬件性能其实还算扎实。但遗憾的是，2014 款 MacBook Air + Linux，经典地狱组合。

在 Air 上安装 Artix 之后，我遇到的最大问题就是**无线驱动缺失**—— 这意味着 WiFi 功能无法直接使用。解决方案其实很简单：从 Artix 的官方软件仓库下载对应的无线驱动（准确来说是博通驱动）。这个操作只需要在终端执行一条命令就能完成，但前提是电脑得联网…… 而我的笔记本当时根本没网。

最终，我只能接上 USB 转网口适配器，把笔记本直连路由器，这才成功装上了 WiFi 驱动。整个过程只花了 10 分钟左右，但那一刻真的很想摔电脑。

台式机倒是完美的开箱即用，连第二显示器的奇葩分辨率（1680×1050 竖屏）都一次点亮。后来我还折腾了 KDE、XFCE，装 KDE 后菜单白底白字、各种图形 Bug；回退 XFCE 又没了 WiFi 图标。好在折腾了 45 分钟，翻阅了大量手册和论坛帖子后，问题终于解决了，WiFi 图标恢复正常，我又能正常选择无线网络了。

除此之外，Linux 本身也存在一些固有短板。比如 Steam 平台上标注为 “Linux 兼容” 的游戏，并非真的都能正常运行。

《文明 3 完整版》就是个典型例子：启动游戏后，地图会变成全黑（不过后来我用 Lutris 这款游戏运行器，解决了这个问题）。还有一些我在 Windows 上常用的软件，Linux 平台上并没有对应的版本，比如截图工具 Greenshot、批量图片加水印的 uMark。虽然这些软件都有替代品，但要么功能不全，要么需要重新学习操作流程。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0jqAkXB2hEohZkOqYzXVsmJtnNRQncYh54ZQOpcC5ZGSicYFgtBZP4w/640?wx_fmt=png)

意外惊喜：手机管理比 Windows 还爽
---------------------

Android 天生支持 Linux，这并不稀奇。但真正让我震惊的是：连我的 **iPhone 13 Pro Max 居然也能很好用。**

本来，我这款 iPhone 13 Pro Max 和运行 XFCE4 桌面的 Artix Linux “水土不服”：手机连接电脑后，只能充电，系统完全识别不出设备。我曾尝试安装 KDE Connect 这款软件，它本该实现类似 iPhone 和 Mac 之间的联动功能，但结果却毫无作用。不过，我最终在一个意想不到的地方找到了解决方案。

之前安装 KDE 的时候，系统里留下了一个 Dolphin 文件管理器。我试着用了一下，发现它比 XFCE4 默认的 Thunar 文件管理器好用得多。更惊喜的是，当我把 iPhone 连接电脑时，Dolphin 竟然直接识别出了设备！

这让文件传输变得易如反掌，因为文件管理器能获取 iPhone 的完整文件访问权限。而且随着 iOS 文件管理功能的完善，手机里每个应用的文件都存放在对应的专属文件夹里。  

整个操作流程比在 Windows 上简单太多了 —— 毕竟再也不用依赖 iTunes 了。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QGHX4EaTS3AknUHvnayNiaLnANlsxF3g1JR5q9HIdppdNd8c8uyuC05w/640?wx_fmt=png)

用 Linux 的最大感受：我重新喜欢上电脑了
-----------------------

至于我为什么要切换到 Linux？又为什么要写这篇文章分享体验？答案只有一个：我**用得很开心**。

我重新找回了使用电脑的乐趣，整个过程充满新鲜感。而且 Linux 系统总有可探索的空间，你可以随时调整系统设置，深入学习底层运行机制。而且，相比 Windows，Linux 运行速度更快、稳定性更强，这些都是实打实的加分项。

其实还有很多细节值得一提，但说多了文章就太冗长了，这里简单补充几点。

通过 Steam 安装游戏的流程，和在其他操作系统上一样简单，只是兼容的游戏数量比 Windows 略少。不过除了前面提到的《文明 3》，我库里的所有 Steam 游戏都能流畅运行。甚至在某些情况下，游戏体验比 Windows 更好 —— 因为 Linux 后台不会运行那些占用资源的反作弊软件。

系统的可定制性是一把“双刃剑”，具体体验取决于你选择的桌面环境或窗口管理器。我用的 XFCE4 就有很强的可定制性，但要让它变得美观，需要具备一定的技术知识。它支持 Conky 这款工具，能在桌面上显示系统信息。而 Conky 的定制化程度极高，需要通过配置文件里的脚本代码来实现。

Linux 的电源管理功能则有点“一言难尽”。我的 MacBook Air 在 Linux 系统下的续航时间，和运行 macOS 时差不多（毕竟是老笔记本，电池早已老化）。但偶尔会出现一些 “流氓进程”，导致合盖后笔记本无法进入睡眠状态。不过，Artix 的轻量化特性也带来了好处：笔记本的风扇几乎很少高速运转，台式机也是如此。

Linux 的系统配置迁移也非常方便，包括桌面和应用程序的设置，只需要拷贝一个文件夹就行。只要安装了相同的软件包，我就能通过简单的拖拽操作，让 MacBook 的系统界面和台式机一模一样。而且用户对电脑界面的选择自由度是其他系统无法比拟的：如果不喜欢 XFCE4 的某个设计，要么直接修改配置文件，要么干脆换一个桌面环境（当然，就像前文提到的，这么做可能会引发新问题）。

最关键的是，系统的整体稳定性堪称一流。自从换上 Linux，我的电脑再也没有崩溃过，更不会像使用 Windows 11 那样，频繁出现黑屏之类的图形界面故障。

结语：Linux 不是银弹，但值得一试
-------------------

Linux 不会一键治愈你的所有电脑问题，但它能让你真正理解自己的系统。

和其他操作系统一样，它需要用户花时间去学习它的使用方法和独特特性。诚然，要深入钻研系统底层、充分释放它的潜力，确实需要具备一定的技术知识。但很多发行版（比如 Mint）都能做到开箱即用，用户甚至可能永远都不需要打开终端。

就我个人而言，我喜欢这种略带挑战性的体验。Linux 很好地平衡了易用性和深度探索的乐趣。如果我愿意，完全可以选择一款开箱即用的发行版，但我想要的是更强大的掌控感。这正是 Linux 的魅力所在：它的灵活性无与伦比，你可以根据自己的需求，选择一款极具挑战性的发行版，或是一款简单易用的发行版。

安装 Linux 不仅让我家的三台设备（笔记本、台式机、媒体电脑）重获新生，更让我重新找回了使用电脑的乐趣。这个过程中确实有过不少挫败感，但当你通过自己的努力解决一个难题后，那种成就感是使用 Windows 时从未有过的。

这背后的原因其实很简单：在 Linux 上遇到的问题，大部分都是我自己操作不当造成的；而在 Windows 或 macOS 上遇到的麻烦，往往源于系统本身的限制或漏洞。Linux 给了我 “折腾” 系统的自由 —— 哪怕把系统搞崩，也能自己动手修复，而这个过程本身就是一种学习。

在微软无视用户愤怒、持续“搞砸”Windows 11 的今天，转向 Linux 已不再是极客专属，而是越来越多开发者的现实选择。如果你已经纠结了很久，那么现在，就是投身 Linux 的最佳时机。
