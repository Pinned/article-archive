# Claude Code终于长出调度台：一个屏幕盯住所有AI会话，无需多开

##### **【新智元导读】Anthropic给Claude Code上了一个新功能，叫agent view。简单说，就是不用再开十个terminal窗口分别跑Claude，一个命令就能在一个界面里看到所有会话在干什么。它做的事不是给你多一个AI模型，是给你多一只手，让你能同时盯住十件事，而不是一件接一件地跑。**

  

过去一年，Claude Code从一个开发者命令行工具，变成了某种意义上的同事。

你给它一个task，它去办；办完了告诉你结果。

但有一个问题，一直没解决，就是你只能跟它一对一聊天。

![](https://mmbiz.qpic.cn/mmbiz_png/Rvq8Ow69CYWRxzkvXkibk80Rn2ujXFQcQUaEK5TxhyAictpwmpias4v62A64lYINxQyhoXHgmLHcTgLRg0a068TcbhB9mfAcD7uhGNRQSibFEe8/640?wx_fmt=png&from=appmsg)

要让它同时干5件事？那就需要开5个terminal窗口。

要切换不同任务？分屏，或者用tmux搭一个网格。

要看哪个任务卡在哪里？需要挨个窗口翻。

![](https://mmbiz.qpic.cn/sz_mmbiz_gif/Rvq8Ow69CYVzzVZJ8BibD72doSgsLgaJjrC85dBVhxiaicrFDrA2ObXSiau4WTlRGVwaxUYVDrQgXvU94lKrJFLrsf8Hxy1ibaTG8ibLAbnkoaniaQ/640?wx_fmt=gif&from=appmsg)

这就像你雇了10个实习生，但他们每个人都坐在不同的房间里，门关着。

你想知道谁在干什么，得一个一个开门。

agent view就是在这个时候来的。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYWyMutdxR3pvEkh1gIrXibca0SaRkx6ZCH7YENsibKktFcpXw2OXf6q5UI5JYQA0scx7va7eQSTlicnpNvROM1ooCLw8J6EiaVQoOY/640?wx_fmt=png&from=appmsg)

![](https://mmbiz.qpic.cn/sz_mmbiz_png/UicQ7HgWiaUb3uEdSPKrwGNmZEOaaGyzVvZ8dTtE9jU1rFsda3llYbCZpmWfiazUYjWBLTGvlPpXucH8Q0lEUJN3Q/640?wx_fmt=png&from=appmsg)

**它做的事情很简单：调度**

Claude Code正式上线了**Agent视图（agent view）**研究预览版。

简单来说，这项功能为开发者提供了一个「指挥中心」，能够在一个界面中管理所有正在进行的会话。

开发者现在可以启动多个Claude **AI****智能体**同时执行多项任务，而无需像以前那样占用多个终端窗口。

在实际操作中，每个**AI****智能体**都能在后台持续运行，开发者可以一目了然地看到哪些任务正在执行、哪些在等待反馈、哪些已经大功告成。

![](https://mmbiz.qpic.cn/sz_mmbiz_gif/Rvq8Ow69CYXtFCapkcXKwpvqtmdibysibdyKRNmJGvYQB02UWnNcBGlNiaSHqqNVY19FcHicFnEZ2LmaTrkUy9BvL8OwZ7UDm2D8zojtNVWEgMU/640?wx_fmt=gif&from=appmsg)

这种「并行处理」模式支持直接在线回复以解除阻塞，或者在不同任务间自由切换而不丢失进度。

terminal被这个界面接管。

底下一个输入框，上面一张表。每一行是一个会话。

你在底下输入框打一个prompt，按Enter，新一行加进去，那个任务在后台跑起来。

再打一个，再加一行。

![](https://mmbiz.qpic.cn/mmbiz_png/Rvq8Ow69CYU0bnXhOIg5B0GtKjGfd3QAFqgwDuHxgVe6k3azvWmUW2IT8XGpJdtichkJHkWiaZ1FjPR3uXb2pgt0tMzj2duSCsicKOkCcMJbFM/640?wx_fmt=png&from=appmsg)

每一行都有自己的状态——在工作、等你回话、已完成、失败、停了。

颜色和图标会告诉你这些。动画图标代表Claude正在动，黄色代表它在等你拍板，绿色代表搞定了，红色代表崩了。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYVd0fzlIqUfcbXMaYOSnx1Umbvic8l0wvk2icDoxcmQQ31UvjfOrVGYEnJOPG6nViaibBjOjpw73ic3sJqrK0vdRrEQRmqDXjY3heBY/640?wx_fmt=png&from=appmsg)

你不用挨个进会话看进度。表上扫一眼，就知道现状。

如果某一行需要你输入，你按Space，弹出一个peekpanel，里面是它的最新状态。

如果你想认真看完整对话，按Enter或者→，那个会话占满terminal，跟你直接claude进去一样。

完事按←，又回到表。

**![](https://mmbiz.qpic.cn/sz_mmbiz_png/UicQ7HgWiaUb351381bTy5MO2IN89mV41M88GEiaCCibDxJoaQjYV6HfRtafnmEmfM3R1p0tmkHgBOVuXBD6UJKpsQ/640?wx_fmt=png&from=appmsg)**

**一种工作姿态的切换**

过去的Claude Code，工作姿态是你和它面对面。

一个terminal、一个对话框、一来一回。它在干活的时候，你只能等。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYXiaTTq4zNW4cTd4X8mxxquOccUtjYnKYVJ9NUdMOsAq6HUmyNJobwHgtdQiaCGGptKtqChnwOQqp9550GqicehiaF9LZfj8RdgpicY/640?wx_fmt=png&from=appmsg)

agent view把这个姿态换了。

你不是在跟Claude对话，你是在调度它。

你的角色更像一个工头——派活、扫表、回答必要的问题、复核结果。

你和Claude之间不再是1:1，是1:N。

很多人第一次跑agent view时会有一种短暂的不适——好像突然不知道自己在跟谁说话。

适应过来之后，你会发现这个不适其实是一种解放。

为什么之前没人做这个

其实不是没人想过。

是之前**没必要**。

过去你给Claude Code一个task，它常常要跟你来回讨论好几轮才能跑通。这种需要紧密协作的工作方式下，开两三个并行的会话已经超出大多数人的注意力上限。

最近的变化是，Claude Code的task完成度越来越高。

你说一句修一下这个bug，它能自己读代码、定位问题、改完、跑测试、提PR——全程不用你插话。

这种独立性突破某个临界点之后，并行就有意义了。

你能同时让它做5件事，而不是5件事都需要你一直陪着。

agent view是在这个能力变化之后才合理的。

它不是早做了一个产品功能，是补上了一个能力门槛后才长出来的工具。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/UicQ7HgWiaUb3uEdSPKrwGNmZEOaaGyzVvZ8dTtE9jU1rFsda3llYbCZpmWfiazUYjWBLTGvlPpXucH8Q0lEUJN3Q/640?wx_fmt=png&from=appmsg)

**Agent View适合干什么**

三种场景值得专门列一下。

**互不依赖的并行活儿。**你脑子里有5件事可以分别让Claude干——修bug、加endpoint、写测试、查log、跑数据迁移。开5个会话扔下去，你只在blocked的时候插一句话。

**长跑的等待型任务。**PR审查、CI等结果、自动merge——这些任务跑得久，但中间不需要你介入。让它丢在后台跑，你随时来看一眼。

**临时插队不丢进度。**你正干一件事，突然来一个紧急任务。/bg把当前会话扔后台、处理新事项、做完按←回去——原来那个会话从你离开的地方继续。

它不适合什么？

**需要你深度参与的协作。**比如你和Claude一起设计一个架构、一起调一个棘手bug——这种工作你应该attach着，一来一回慢慢打磨。agent view不是替代深度对话，是替代我已经知道要做什么，你去做就行。

**![](https://mmbiz.qpic.cn/sz_mmbiz_png/UicQ7HgWiaUb351381bTy5MO2IN89mV41M88GEiaCCibDxJoaQjYV6HfRtafnmEmfM3R1p0tmkHgBOVuXBD6UJKpsQ/640?wx_fmt=png&from=appmsg)**

**在补Claude Code最缺的那一环**

Claude Code过去几个月让人惊艳的，是它写代码的能力——SWE-bench高分、Cursor/Windsurf/Claude Code三大开发者工具都用它驱动。

但它有一个隐形的天花板——**单线程**。

人脑能力的瓶颈不是写得多好，是能同时盯几件事。

一个工程师真正的产出，常常不在他写代码的速度，而在他能同时推动几条并行的工作。

Claude Code之前的形态，只能帮你跑一条线。

agent view是第一次让它能帮你跑N条。

这听起来是个UI改进，但本质是工作姿态的变化。

它把Claude Code从一个写得很快的实习生，变成了一个能管10件事的小团队负责人。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/UicQ7HgWiaUb3uEdSPKrwGNmZEOaaGyzVvZ8dTtE9jU1rFsda3llYbCZpmWfiazUYjWBLTGvlPpXucH8Q0lEUJN3Q/640?wx_fmt=png&from=appmsg)

**最后一句**

好工具的最高境界，是让你感觉不到它在那儿。

agent view现在还不是这个境界：它是一个研究预览，键盘快捷键还会变，肯定还有要打磨的地方。

但它已经做到了一件事：让你打开Claude Code的时候，不再是面对一个prompt框，而是面对一张在跑的任务表。

从一对一聊天，到管理一队Agent，这中间隔着一道门。

Agent View是这道门第一次被打开。
