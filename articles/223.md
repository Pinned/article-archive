# 突发！谷歌大规模封禁OpenClaw用户账号，开发者一夜「断粮」：“每月花250美元，被封却毫无预警”

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

整理 | 郑丽媛  

出品 | CSDN（ID：CSDNnews）

这两天，开发者圈子又被一条消息刷屏。

Google 宣布限制部分用户访问其 AI 编程平台 Antigravity，理由是这些用户通过开源 Agent 框架 OpenClaw 路由 Gemini token，构成“恶意使用（malicious usage）”，并导致平台服务质量下降。

这场风波从上周开始发酵，不少开发者在 X 和 Y Combinator 论坛上表示，自己在毫无预警的情况下被切断了 Antigravity 访问权限——更有用户担心，由于 Google 的 AI 产品与 Gmail、Workspace 使用同一套账号体系，是否会波及整个 Google 账户。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

问题的核心：Token 被“烧穿”了

这个事情的核心，其实并不复杂。

近来走红的 OpenClaw，其定位是一个开源自治 AI Agent 框架，允许用户通过替代接口接入大模型、执行 shell 命令、访问本地文件，并自动化运行复杂工作流。

因此对很多开发者来说，OpenClaw几乎是“把 AI 变成操作系统层级助手”的工具：只要接入模型，就能批量生成代码、自动改 Bug、跑 CI、处理邮件。

但问题出在“token 路由”上。

部分用户将 OpenClaw 与 Antigravity 连接，通过第三方接口方式调用 Gemini 模型，从而在现有订阅框架下消耗更多 token——在技术上，这种方式可行；但在商业规则上，它违反了平台对“合理使用”的假设。

正如 Google DeepMind 工程师、前 Windsurf CEO Varun Mohan 在 X 上所说：

> “我们最近发现 Anitgravity 后端的恶意使用大幅增加，这极大地降低了用户的服务质量。我们需要找到一种方法，快速切断这些未按预期使用产品的用户访问。”

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEw6sy718Z0kxXLqH2sXAzqDodiaw9ycZ0Gt2E0RCATh6HfuKabCzJMic4sAlRlh4ia2ibibljjjIJqZaOO9Z1n0XEOL0PeZ6WqFD0bQ/640?wx_fmt=png&from=appmsg)

换句话说——OpenClaw 的高频自动化调用，把 Antigravity 的后端“烧”到了边缘。

这样导致的结果就是：后端算力资源被大量占用，正常用户的体验下降，而平台成本迅速上升。基于此，Google 认为该行为构成了“恶意使用（malicious usage）”，从而决定直接封堵入口。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QDyP5HTUOXsPlJWd79yygiasaqXicpN7ibIfqiak5WFpaxE1mGxxpfmMjiaA/640?wx_fmt=png)

封禁是怎么发生的？

具体来说，封禁从上周开始发酵的。

多名开发者在 X 和 Y Combinator 论坛发帖称，自己在没有提前警告的情况下，突然失去了 Antigravity 访问权限。部分人甚至担心 Google 账号整体受到影响，因为 AI 产品与 Gmail、Workspace 共享统一账号体系。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEy9a4rmVKm8hySeLSuiayCZmmUcD6yuL3yT7gib222KH9APuIlKy78QniciaRsJt7L9XZ641dnJldukeVEUlicuYgT2GhSPv5icY7mU4/640?wx_fmt=png&from=appmsg)

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEwuKJSIPyhenVdodZLqIJ6scIPibjdicCzicsLCnOKP21YenVvUy1afI8pKs88yia1cBKLlIXjH7QNoKia3TV5x22paK96tWhKNibjVc/640?wx_fmt=png&from=appmsg)

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEwsVJiccgvPBg5V9UKEOTED5LiaShWwvruicF92aIte461BzQUzWJU2AxAZ5FfibIhhZHJwGVbOdJzZiaAdNXib5Apib1ThU2b8ic19v24/640?wx_fmt=png&from=appmsg)

对于用户的抱怨和质疑，Google 的回应相对克制：这不是永久封禁，他们的目标是让使用方式回归平台规则。  

Varun Mohan 也补充表示：“我们理解知道部分用户并不知道这违反了服务条款，会为他们提供恢复的途径，但我们容量有限，必须优先保证真实付费用户的体验。”

但问题在于，Google 并未公布具体的恢复时间表，而算力“容量有限”的说法，意味着未来类似的使用方式也可能长期受限。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0QtgZAx8xsoTReptvArfwbn9MvHGVfV98Qkl5PMRS6MCt2ljwLJwoQ/640?wx_fmt=png)

有些微妙的封禁时间点？

让这件事更加引人关注的，是它发生的时间节点。

大约一周前，Sam Altman 宣布，OpenClaw 创始人 Peter Steinberger 已加入 OpenAI，负责“下一代个人 Agent”方向。虽然 OpenClaw 仍由独立基金会维护，但目前已获得 OpenAI 的资金支持与战略引导。

这说明什么？一个“OpenAI 阵营”的 Agent 工具，正在调用 Google 最先进的 Gemini 模型算力。

因此从商业角度看，Google 此举不仅是在保护服务器负载，也是在阻断一条可能为竞争对手赋能的技术通道。

Google 的封禁消息发出后，Peter Steinberger 也对此快速表态：

> “Google 这手段真够狠的。用 Antigravity 的各位当心点。看来我得撤掉支持了。连 Anthropic 都会主动联系我，态度还挺友善。谷歌却直接…封禁？”

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEzJICC8U4AZFpHibbzQLgJb9xKqrN8n5KUwmDcqwGDgiaeIS1AohuHTrJpnmkyHF1W9QpF9vOetXL6d7jflZibiclib7Za9729sicKib0/640?wx_fmt=png&from=appmsg)

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q3SAo8dibFicWvibnB5u4gdHLgb2AbA6UJ8VImKnyW92hibqZefwpDbPYAQ/640?wx_fmt=png)

这不是孤例，Anthropic 早两天就动手了

如果你以为这是 Google 的个例，那就错了。

就在 Antigravity 事件前两天，Anthropic 也更新了使用条款，明确禁止通过第三方工具使用 Claude 订阅的 OAuth token（包括 OpenClaw 这类封装工具）。去年，Anthropic 也曾限制 Claude Code 的访问速率，理由是部分用户 24/7 持续运行系统，属于滥用行为。

如此看来，趋势已经非常明显：模型厂商正在收紧对“非官方入口”的访问权限，从“开放 API 平台”转向“围墙花园式生态”。

早期 LLM 爆发阶段，开放与互操作是主旋律，开发者可以自由组合模型与工具，构建自己的 Agent 体系；但现在，当 Agent 工具开始大量消耗算力、并可能影响商业模型时，平台开始重新划定边界：限制第三方封装、加强客户端识别、绑定第一方界面、强化订阅收入模型。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0jqAkXB2hEohZkOqYzXVsmJtnNRQncYh54ZQOpcC5ZGSicYFgtBZP4w/640?wx_fmt=png)

开发者为什么会紧张？

事实上，很多开发者已经把“第三方 Agent + 前沿模型”作为生产力基础设施：一旦入口被封，整个自动化工作流可能直接中断。

更关键的是，部分用户反映，在运行 OpenClaw 实例并连接 Google 产品后，出现了账号访问异常的情况——虽然 Google表示只切断了 Antigravity 访问，但由于账号体系是统一的，用户自然会担心连锁反应。

对于 Google 突发性的封禁，许多用户在 Varun Mohan 的推文下表示反对与谴责：

*   “你在没有任何警告邮件的情况下大规模封禁了付费 Ultra 订阅者，然后面无表情地走进这里骂他们是恶意用户？那些每月支付 250 美元的人只能面对违反服务条款的界面，没有任何解释，没有任何通知，什么都没有。”
    
*   “你想知道为什么开发者越来越排斥在 Google 生态系统上构建东西、选择其他实验室的模型吗？因为你们总是这样，把自己的付费用户当作一次性的韭菜，再写一段冠冕堂皇的公告糊弄过去。”
    
*   “‘恶意’这个词用得不太准确。好比他们去了自助餐，但吃得比你预期的多。你有权把他们赶出餐厅，可他们的行为是合理的，不是恶意的。”
    
*   “我没有运行 OpenClaw，也没有部署任何东西，也没有利用你的服务进行自动化滥用——我只是在文本和代码，想用 Antigravity 来起草和改进我计划在 OpenClaw 中使用的代理代码和提示——可我的账号还是被封了。你是要禁止所有包含 ‘bot’ 或 ‘OpenClaw’ 这些词的内容吗？”
    

面对一众负面评论，Varun Mohan 在第二天进行了回应：“我们非常遗憾没有提前说明，但为保障所有用户的体验质量，我们必须迅速采取行动。”——不过，显然这个回应用户并不买账。

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEyDRqWUZDf5ZIPbTz6rXutX2nHcJuENHGBeKdTh0G6U6Ul416WrLibHfQN7KKmtzX0QOS6pGs4I7YaYkRLpqTNwUZyzVv7h22HU/640?wx_fmt=png&from=appmsg)

最后，可以预见的是：随着 AI Agent 从“玩具”走向“生产力工具”，平台对算力与访问路径的控制只会越来越严格——而这次的 Antigravity 风波，可能只是一个开始。
