# 面壁智能SALA架构，正在拆掉Transformer的那堵墙

这两年，大模型圈子里真正卡住行业的点，已经从参数规模，转向了推理效率。

模型越做越大，算力和显存的压力就越像一堵墙，横在落地面前。

尤其当应用从简单对话进入长文档理解、代码仓级分析、长周期智能体任务之后，传统架构的瓶颈，越来越暴露了出来。

很多团队都在优化训练技巧、压缩权重、做蒸馏，却很少有人直接动底层注意力结构，因为那意味着要重新审视整个Transformer范式。

面壁智能，这次公开的 Linear-Sparse 混合注意力架构 SALA 和对应模型 MiniCPM-SALA，在核心结构层的技术路线上，给了不一样的答案。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQmS50L5bVibGxcrHM7LDVFIwxXLySAAcQbr8lyXE5JZSU9a2t0JKCID1jm5aJN3bnZT1hE70CLfQhpTZiaUlpJwnn74puU5WxeFE/640?wx_fmt=png&from=appmsg)

新模型能力提升现在不新鲜了，但如果是一次架构路线层面的重新定调呢？

马上过年了，比起很多公司红包的热闹和营销的手段，我觉得，面壁智能这样扎扎实实的技术推进，更值得被看见。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQmxjsGoFWKBeicibyjLbhiabefV6pJR7IB714Gdq7cWM3LcG5NbReMTTgH6gJzGTDZqwIbF9fGUOp94hurJ22ooTbx2sOGX1UKkrs/640?wx_fmt=png&from=appmsg)

01. 长上下文需求爆发，正在倒逼注意力机制进化
------------------------

  

---

大模型最早的商业场景集中在问答、写作、摘要，这类任务上下文长度通常在几千到几万 token 区间，传统全注意力机制还能承受。

随着应用形态变化，模型开始承担代码仓分析、合同审查、科研资料归纳、长链路 agent 任务规划等工作……

输入规模直接跳到几十万甚至百万 token 级别，这类需求在企业端和端侧同时增长。

企业侧希望模型一次性读取整个知识库或代码库并保持一致理解，端侧设备则存储着大量用户私有数据。

比如说，聊天记录、历史行为、位置轨迹，这些信息只有本地处理才符合隐私要求，所以端侧长上下文能力逐渐变成刚需指标。

传统 Transformer 的问题在这种场景下会被无限放大。

![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQnAA0Dslu5H2icx77aHmSVfVltCUhARrCnhUVJjg16YNFzHhc3lY6CcVCUUemg2hmuibgIdVJbqcjkbJhQic26vxUTiaVdW3Z6DGzA/640?wx_fmt=png&from=appmsg)

全注意力机制计算复杂度，随序列长度平方增长，序列翻十倍，算力需求接近百倍级上升，同时 KV Cache 也同步膨胀。

推理阶段显存占用持续上涨，哪怕模型参数规模不变，只要上下文变长就会直接炸显存，这就是很多工程团队口中的 KV Cache 困局。

![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQnM8MohGYk36JL4a9bzf5ia4khVeEHUl8oNbzECJWetdic2kK16tOCCErXX9hI5ggcNchW1ZSeIuVYjU4koLcV9faTePlyicp9LiaU/640?wx_fmt=png&from=appmsg)

放之前，这几个困局，就像一个「不可能三角」。

行业已经尝试过多条路径解决这个问题，例如线性注意力、状态空间模型、稀疏注意力结构，各自都在某些维度表现出色，却始终存在明显短板。

线性注意力与 SSM 类方法将复杂度压到线性级别，速度和资源消耗表现很理想，不过这种结构需要把历史信息压缩进固定容量状态中，序列越长，早期信息权重就越低，在复杂推理或长链条逻辑场景中容易出现记忆衰减现象。

稀疏注意力走的是另一条路线。

通过只计算关键位置注意力来降低算力需求，推理速度提升明显，不过历史 KV 必须完整保存，否则无法回溯长距离依赖。

这导致显存仍随序列线性增长，存储压力没有根本解决。

行业一直在寻找一种结构，既保留线性结构的效率，又具备稀疏结构的精准感知能力。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQn2YM7xF2aTKEib1pcelYygI1kR5iafHX5LTb9Ps4g9j7WOPRwwcpDm4175PGrqGlF40LMfL2F4DbjwMJCGQ5Y1EImCubNdLcMLs/640?wx_fmt=png&from=appmsg)

MiniCPM-SALA采用的Transformer-to-Hybrid低成本构建方法（HALO）

02. SALA的混合注意力架构
----------------

  

---

SALA 的核心设计思路很直接，将两类注意力优势拼接进同一架构体系，让不同模块负责不同任务。

整体结构中大约 75% 使用 Lightning Attention 线性模块，负责局部关键语义捕捉；

剩余 25% 使用 InfLLM v2 稀疏注意力模块，负责全局信息建模。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQkevYENQasY3VL7IxMNaIre9Is97ibEegLMR9aIibHxZO7maCiaVUFoTaSmHxkrNY69fwTkVfgbLW5pBDNxzIiaP6r4MVm9WWnIJyk/640?wx_fmt=png&from=appmsg)

这种比例经过多轮实验调优后确定，目标是找到效率与精度之间的稳定平衡点。

线性部分保证推理复杂度稳定增长，稀疏部分负责高价值信息精细建模，两者共同构成完整上下文理解路径。

这种设计真正解决的问题在于长序列任务的结构性矛盾。

纯线性模型在序列极长时容易丢失细节，纯稀疏模型在序列增长时显存压力持续扩大，SALA 把计算密度和信息密度拆分处理，让计算资源集中在重要区域，同时保证全局状态可追踪。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQmRypOHyCum0CicfbkmgRqzQic7qgibwCkRrqvopTPE8Q6BVZknb22l4hbmBpWKh11YfZCk4icYUaJdQfWiaw1nJxxJaptLzDV8vtG8/640?wx_fmt=png&from=appmsg)

换句话说，模型在处理百万 token 输入时不会同时对所有 token 做同等强度计算，而是自动分配算力权重，这使得资源利用效率明显提高。

另一个关键创新来自训练方法 HALO。

传统混合架构如果从零训练，成本会高得惊人，因为新结构需要重新学习全部语言知识与推理能力。

HALO 的策略是在已有全注意力模型基础上进行结构转换，再进行持续训练，这种方式继承原模型能力，同时让新架构逐渐适应新的注意力模式。

工程层面看，这种路线可以把算力投入降低到可接受范围，使混合注意力模型具备规模化训练可行性，对整个行业具有现实参考价值。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQmc4wtyz2ibRKpjumIzficboH2NV5RZnbWPkTYbBrqt0feEyr5tnMxl8OBkxnGpPJ7qrpx2XlpxzqQTMia057R8rKgPqqJIJuu7icA/640?wx_fmt=png&from=appmsg)

如果把当前主流架构放在同一坐标系中观察，可以看到一条明显演化路径：

全注意力模型提供稳定智能水平，线性模型提供极致效率，稀疏模型提供长序列能力，SALA 则尝试把三条路线合并成统一解。

像 Kimi KDA、DeepSeek NSA、InfLLM v2 都在不同方向推进长上下文能力，SALA 的出现让行业第一次看到统一结构实现多优势叠加的可能性。

![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQkzDQbazVqx0PTX3Wjpa3DLlObvibaTbaMY8ZkgiavibiaKPXvIGOibNZhW9U8R0Ig7Waqwda3TNrwWqm3OztY12Ekia0YNVDpQLKWUs/640?wx_fmt=png&from=appmsg)

03\. MiniCPM-SALA 实测表现，验证架构可行性
------------------------------

  

---

任何架构创新最终都要回到模型表现。

MiniCPM-SALA 作为首个基于该结构完成大规模训练的文本模型，在多个维度给出了比较清晰的数据结果。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQmffC6eTrqAhle3GvrDhIfIToia9DicpFoVcJxumFBt0u5vicvibF04n36d7nCAOTUspBib4icaEFKbjiblicIQNrMuuBvoxt1BqJJwoWE/640?wx_fmt=png&from=appmsg)![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQn8VXug5S7JqicwPO51Yu4pmXz1cK8YqiaEiaVLqjicSaUGJiabcWnlQeD0z6bWStyFMv4qUzc7sMR4wwh07Pss0dvfDG0TpxuqIVNU/640?wx_fmt=png&from=appmsg)

模型参数规模为 9B，体量处于轻量级区间，却能够支持百万 token 上下文推理。

关键在于 KV Cache 控制在 6GB 以下，这意味着普通消费级 GPU 也能完成推理任务。对于开发者来说，这直接降低部署门槛，使长上下文模型从数据中心专属能力进入个人硬件可运行阶段。

在长文本评测中，该模型展现出稳定优势，尤其在跨章节信息整合、长链推理、代码结构理解等任务中表现突出。

![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQlbVxib4nKGqNNgqxCoNgrcndiap4Hch4UO9sib6l4fVggyawwLWuemeBmy6E0z0hIicBJdC2o0wEnhqnD7NnDkkqhdg5Ra3O2N34A/640?wx_fmt=png&from=appmsg)![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQkBATibv8nQSbk3oerEE840PJSia3YZq98zexPmoGGy4JepprhzNCp60lescKVtcubLDKVxJ97GY9Wx1VjkW3ZFADle1ia0iclw7Yw/640?wx_fmt=png&from=appmsg)

更值得关注的是，它在知识问答、数学推理、代码生成等常规能力测试中仍保持与同规模全注意力模型相近水平，没有出现性能折损现象，这说明混合结构并没有牺牲通用智能能力。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/4KW1YfFgWQkzbeodcBa6GicpBTrFdq3Tet449u7oH9moY2KIyMdIs84vpCrzKA5T0YgnIjD5jW1pEpibxNmGiad99dBwGIMpdrPtlb9Vka3qOM/640?wx_fmt=png&from=appmsg)

模型同时引入 HyPE 混合位置编码机制，使短文本和长文本处理能力保持一致，不会出现短输入性能下降的问题。

推理速度方面的数据，对工程落地很有价值。

在云端推理芯片测试中，当上下文长度达到 256K token 时，MiniCPM-SALA 推理速度达到同规模全注意力模型约 3.5 倍，而且测试没有使用投机采样或额外加速技巧，结果完全来自架构本身。

对于企业部署来说，这类性能提升意味着成本直接下降，因为单位时间可处理请求数明显增加。

端侧的表现，同样不错。

当前很多模型在 8B 规模下运行 256K 上下文就会触及显存极限，而 MiniCPM-SALA 在消费级 GPU 上完成百万上下文推理，这为手机、车载系统、机器人等终端设备运行通用模型打开空间。

端侧模型一旦具备长上下文能力，个人助理类产品就能持续读取用户历史数据并维持长期记忆，体验会发生质变，这也是行业普遍认为下一阶段智能终端竞争的关键指标。

![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQnyuhiaf4mErJicGINeUFvvo9sravj0KH2TcjtwwOEBpV85Yugmb0xfCnficjBiadxJoJQos6u6xRtNwICvywe6S3XYRPqwgZR0JwA/640?wx_fmt=png&from=appmsg)

04. 架构路线之争，正在成为大模型竞争核心变量
------------------------

  

---

大模型行业早期竞争集中在参数规模和训练数据量，随后转向推理成本和部署效率，现在焦点逐渐落在底层结构设计上。

谁能在架构层找到更优解，谁就能在同等算力条件下跑出更高性能。

SALA 的出现，也让我看到一种新的可能性：未来模型竞争不只看谁的模型更大，还要看谁的结构更合理。

![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQluZDtaYnBPkLvzAgewwOibCbZgDSMc42yxDcDfvZcic3hEC5TIrOcqOGIliaJj9EhpAiaia4ibxeSeA6tJ6LBic0XrRib0wy4QGcgp0Dk/640?wx_fmt=png&from=appmsg)

我在想，这种变化对行业格局，也许会产生多米诺骨牌一样的连锁反应。

有硬件适配逻辑的改变：

当模型显存需求下降，部署环境选择范围会扩大，GPU 不再是唯一选择，边缘计算设备也能承担更多任务。

进一步，也会带来应用生态会被重新洗牌：

长上下文能力成熟后，大量原本依赖数据库或检索系统的应用可能直接由模型完成，因为模型可以一次读取全部资料并生成结果。

以及说，训练策略会发生变化，HALO 这类迁移训练方法降低新架构实验成本，使更多团队具备尝试底层创新的能力。

从技术趋势观察，注意力机制很可能进入混合化阶段。

单一路线难以同时满足效率、精度、可扩展性三项指标，多结构协同会成为主流设计方向。

未来模型可能根据任务动态切换注意力模式，复杂推理使用高精度模块，大规模扫描使用高效率模块，这类自适应结构将成为研究重点。

![](https://mmbiz.qpic.cn/mmbiz_png/4KW1YfFgWQk3EC9DRibK5CRZb0Ng414PXkaQ3BdTQ45b92wgH59AQC7iae3Vc5JpTHzYCqesHg4qB1FJibUJR2hIvibWBfgJcHWtJ8l4aUSeFTg/640?wx_fmt=png&from=appmsg)
