# “连关机都做不到了！”Windows 11新年首个补丁大翻车，微软紧急追加更新“止血”

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

整理 | 郑丽媛  

出品 | CSDN（ID：CSDNnews）

对大多数用户来说，操作系统最基本的能力至少有这几件：能开机、能关机、能登录。而在 2026 年刚开年不久，微软却在 Windows 11 的一次例行安全更新中，把这三件事中的两件直接搞出了问题：

● 今年 1 月的 Patch Tuesday 刚推送没几天，就有用户发现：电脑关不掉了——不是卡顿，也不是慢，而是系统明确“接收了关机指令”，却拒绝真正执行。

● 与此同时，另一批用户则在远程办公时发现，Remote Desktop 登录彻底失效。

问题严重到什么程度呢？微软罕见地绕过了正常更新节奏，而是直接发布了带外（Out-of-Band）紧急更新，试图在事故进一步扩大前“止血”。

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

本该修 Bug 的补丁，反而制造了系统级故障

这次事故的源头，是 1 月例行发布的 Windows 累积更新。按照惯例，这类 Patch Tuesday 更新主要用于修补安全Bug，而这一次，更新规模尤其庞大：

（1）覆盖 Windows 及多项底层组件

（2）修复数量超过 100 个Bug

（3）其中至少 一个漏洞已被确认在野利用

对企业 IT 和安全团队来说，这类更新几乎没有“观望”的余地，只能尽快部署。然而问题恰恰出在这里——补丁装上去之后，系统开始出现基础功能失效。

在部分 Windows 11 设备上，不少用户发现：关机、重启、休眠全部失效，系统看似进入了关机流程，实际上仍在运行，甚至关机后还会“自行复活”。

显然，这已经不是体验问题，而是操作系统的行为逻辑被破坏。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QDyP5HTUOXsPlJWd79yygiasaqXicpN7ibIfqiak5WFpaxE1mGxxpfmMjiaA/640?wx_fmt=png)

真凶曝光：System Guard Secure Launch

随后，微软确认问题与 System Guard Secure Launch 有关。这是 Windows 在启动阶段引入的一项安全加固机制，用于防止引导链被篡改，通常在企业版、IoT 版本以及部分新设备上默认启用。

问题在于，这项机制与 1 月补丁产生了严重兼容性问题，直接导致关机流程异常：在 Secure Launch 启用的系统中，关机流程无法被正确完成，操作系统对关机指令“视而不见”。

这样导致的直接后果非常现实：

● 笔记本电脑看似已关机，实际上整夜耗电

● 办公室台式机下班后仍在持续运行

● 运维和 IT 团队无法远程确认系统状态

在问题持续发酵后，微软于 1 月 17 日 紧急发布了带外更新 KB5077797，主要针对Windows 11 23H2、Enterprise / IoT 等 Secure Launch 常见环境。官方说明显示，该补丁已恢复系统正常的关机和休眠行为，并且不再等待下一个 Patch Tuesday，直接推送给用户。

从这一动作本身就能看出，这次问题已经被微软内部认定为“不可接受级别”。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0QtgZAx8xsoTReptvArfwbn9MvHGVfV98Qkl5PMRS6MCt2ljwLJwoQ/640?wx_fmt=png)

远程桌面也“阵亡”了，连 Outlook 也被殃及

如果说关机失败已经足够糟糕，那么远程桌面的问题则进一步放大了影响面。

在 1 月补丁安装后，不少用户发现：Remote Desktop 登录失败，凭据验证窗口反复弹出，身份认证也进入了死循环。该问题同时影响客户端和服务器环境，对远程办公、运维管理、云桌面场景的打击尤为明显。

好在，KB5077797 同样修复了这一问题。因此微软方面建议：所有受影响用户尽快安装 KB5077797，以解决关机异常和远程桌面登录问题。

可雪上加霜的是，微软随后又承认，1 月 Patch Tuesday 还引入了另一个 Bug：经典版 Outlook 的 POP 账户配置文件可能卡死或冻结——而截至目前，这一问题仍未发布修复补丁。

换句话说，此次 1 月更新不仅制造了系统级事故，还留下了“未清理干净的后遗症”。  

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q3SAo8dibFicWvibnB5u4gdHLgb2AbA6UJ8VImKnyW92hibqZefwpDbPYAQ/640?wx_fmt=png)

更新，正在成为 Windows 11 的风险源？

按理说，其实 Windows 的每一次正式更新，都会先经过 Windows Insider Program 的测试。但这次更新后，却发生了关机失效、远程登录崩溃——这些都不是边缘问题，而是用户第一时间就能遇到的严重缺陷。

为此，越来越多开发者和运维人员开始质疑：Insider 测试是否已经无法覆盖真实使用场景？还是更新节奏已经快到连微软自己都兜不住质量？

当前，由于一味加大 AI 功能，Windows 11 的公众形象本来就不太好，频繁出现的低级、致命更新问题，无疑是在不断消耗用户信任——此次“关机”和“登录”两个最基础的能力被同时破坏，无疑是一个新下限。  

而对微软来说，如果不能在更新质量和测试机制上真正刹车，那么 Windows 11 的口碑恐怕只会继续下滑。
