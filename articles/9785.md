# 突发！Claude封杀OpenClaw

![](https://mmbiz.qpic.cn/mmbiz_png/z7ZD1WagSLgsRsrdDyQwMQFeic5uAcFg7nUPJuxZKtOOpVA2qEx77mXgLIZRcOIDgFRTT28ymhVVnUyawribP28g/640)![](https://mmbiz.qpic.cn/mmbiz_jpg/8Eyv4o4Acuqldanm83mv1JPEc6Nc2qibY0omqkmHqWof8iaJRhWCBlrib5SmzRqHDSOvZ6v7mNOLbTQsT127yTjd4pIbvx5GqKGCrIjP8vgDOc/640?wx_fmt=jpeg&from=appmsg)

**龙虾之父：哭求只换来7天续命。**

**作者 |**  许丽思  

**编辑 |**  漠影  

智东西4月4日报道，刚刚，Anthropic宣布，从美国东部时间4月4日下午3点（北京时间4月5日凌晨3点）起，**Claude禁用OpenClaw等第三方工具，用户只能通过额外套餐或Claude API Key使用这些工具。**

Claude官方发送的邮件还提到，这项政策从4月4日起先在OpenClaw上强制执行，但适用于所有第三方工具链，并将很快推广至更多工具。

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/8Eyv4o4AcurfRZ3ljibyG7k5oSwNqy6v2ttxtXZE0wqiaklO503HZ3eVrNS3ibxLlAetrIptR2cKbXt9icBIhXNv8V3yicj3oXGXWXn1Zicl6cmOM/640?from=appmsg)

封锁核心原因就是，Anthropic要优先保障使用自家产品和API的客户。

看到这个消息，普通开发者可以说是“天塌了”：

过去每个月花20-200不等美元的Claude订阅费，再加上OpenClaw的自动化工作流，就可以拥有能够自动回复消息、管理日程、读写文件、执行代码，更像一名不停运转的AI员工。

但现在，不只是成本上涨，而是预算从可预测变成失控，原本按月封顶，如今随调用量波动。

OpenClaw创始人Peter Steinberger提到，自己也试图劝说Anthropic，但没用，只换来了7天续命。

  

**_01_****_._**

**Anthropic：订阅服务**

**不是为了第三方工具使用模式而设计**

为什么要封OpenClaw？Anthropic的Claude Code负责人Boris Cherny连发了好几条推文解释。

他提到，Anthropic一直在努力应对Claude需求的增长，而其**订阅服务，本来就不是为第三方工具的使用模式设计的。**算力是需要审慎管理的资源，Anthropic要优先保障使用自家产品和API的客户。

不过，Anthropic也提出了补偿措施，订阅用户将获得一次性等额月费积分。如果需要更多抵扣额度，现在可以购买折扣使用包。如果想要全额退款，明天的邮件就会包含申请全额退款的链接。

另一个Anthropic持续封杀OpenClaw的原因是，或许是因为Peter Steinberger已加入死对头OpenAI。

今年2月，**Peter Steinberger宣布正式加入OpenAI，参与推动其下一代个人Agent方向的研发**。他还强调，OpenClaw将继续作为一个“开放、独立、仅刚刚起步的项目”存在。

Peter Steinberger当时还解释，他看重的是将Agent能力尽快带给更广泛的用户群体，而与OpenAI的合作，是当前实现这一目标的最快路径。

针对这一封锁事件，Peter Steinberger在X上回应，他和Dave Morin（OpenClaw基金会的董事会成员）**已经尽力去说服Anthropic，但最终只把将这件事推迟了一周。**

他感慨，时间点真是巧合，Anthropic先是将一些流行的功能复制到他们的封闭框架中，然后又把开源软件拒之门外。

**_02_****_._**

**一边复刻能力，一边收紧入口，**

**Anthropic全面围堵OpenClaw**

Anthropic对OpenClaw的围堵，其实是一步步收紧的。

2025年11月，OpenClaw诞生时还叫做Clawdbot，后来被Anthropic指控侵权，于是项目先短暂改成Moltbot，再在2026年1月底定名为OpenClaw。

之后，Anthropic又转向产品层面，复刻OpenClaw的核心能力：

OpenClaw原先可以借助WhatsApp在手机端发指令、让桌面端持续执行任务，Anthropic便推出Dispatch，把手机发起、桌面执行的链路做成官方能力；

OpenClaw曾把Discord、Telegram变成远程操控入口，Anthropic就上线Claude Code Channels，并通过MCP协议打通协作通道；

OpenClaw能够深入操作系统，控制浏览器和各类应用，Anthropic就拿出Computer Use和Claude Code，把这类能力纳入自家产品体系。

Anthropic并不是单纯限制OpenClaw，而是在通过品牌切割、能力内置和官方替代，逐步OpenClaw的生存空间。

据Semafor今日消息，**Anthropic正在打造自己的OpenClaw版本。**当被问及客户是否要求Anthropic开发自己的OpenClaw版本时，首席商务官Paul Smith承认，确实有这件事，产品发展得相当快。

不止OpenClaw，Anthropic长期以来也对各种第三方工具有所限制。

2025年6月，Anthropic限制了Windsurf访问Claude模型。

2025年8月，Anthropic切断了OpenAI对Claude API的访问，理由是OpenAI在用Claude做竞品基准测试。

2026年1月8日，Anthropic被曝出切断了Cursor对Claude模型的访问权限。

2026年1月9日，Anthropic把禁令正式写进了文档：OAuth认证仅限于Claude Code和Claude.ai使用。通过Claude Free、Pro或Max账户获取的OAuth token，不得用于任何其他产品、工具或服务——包括Agent SDK。

除了Anthropic，也有其他AI大厂早已经对OpenClaw展开封锁行动。今年2月，Google就禁止OpenClaw调用Gemini AI Ultra、Antigravity。

  

**_03_****_._**

**结语：开放红利退潮，**

**AI加速进入平台主权时代**

随着大模型厂商开始亲自下场做Agent、做入口、做工作流，第三方工具赖以生存的空间和低成本接入红利正在迅速消失，随时可能被替代，被一纸规则直接清场。

Anthropic这记大招，某种程度上宣告了AI产业正从开放繁荣走向平台主权时代，留给开发者的，或许只有尽快摆脱对单一巨头的依赖。
