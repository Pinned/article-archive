##### **【新智元导读】谷歌突击封杀使用开源智能体 OpenClaw 的开发者账号。这表面是打击违规算力调用，实为谷歌对 OpenAI 阵营的生态围剿。巨头筑起高墙，AI 跨平台开源红利终结，企业面临严峻的断供风险。**

昨天，一批海外开发者的谷歌账号毫无预兆地被大面积封禁。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYWv8qNibib5sZSHSfDIwa2R36PN809nbeIDRr6ZlPqibef30oNVmYkiawGpJpUutbyVwhb0Uyic04AJLicvEqKVX2GAKVPiaZMnvjYPMU/640?wx_fmt=png&from=appmsg)

![](https://mmbiz.qpic.cn/mmbiz_png/Rvq8Ow69CYUH2oNz2aQ9xG7KesN2SB7IMvNMmT4qxBaqUuLfzIhOnicx6JAWTcapic9FHGUL9HPNhGTXRia9fq6g3MI7n178rcnG56Y2qYe520/640?wx_fmt=png&from=appmsg)

![](https://mmbiz.qpic.cn/mmbiz_png/Rvq8Ow69CYVjzFGFBfDatgmWZwS9HAODqX8cPgYibIm6ehHibjQjk9lYHd3j96EicNibRFdUMH3aRibiamme8ibk8LpjKSPVjEFenozU0sI7Cz6MeY/640?wx_fmt=png&from=appmsg)

没有提前警告，也没有申诉通道。

部分用户发现自己不仅被踢出了谷歌 AI IDE Antigravity，连带着绑定的 Gmail 等核心服务也一并被锁。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYXCXSfFjxMTFmYapUlJ2ibln4Oj5y0hQia5KDW0qvFlICY4WTnaicCvDnOZSQBC5LRHqrq0ramlBa64Y7rDmyQuIjQLGVSyF6iczaA/640?wx_fmt=png&from=appmsg)

哪怕是每月支付 250 美元订阅费的 Ultra 级别企业客户，在这场清洗中也没能拿到免死金牌。

这起封禁事件的起因，指向了 OpenClaw。

过去几个月里，大量用户习惯了用 OpenClaw 作为中间件，接入 Antigravity 并调用背后的 Gemini 模型来处理日常工作。

前 Windsurf 创始人、现任谷歌 DeepMind 工程师 Varun Mohan 出面定调，称后台监测到激增的恶意调用，部分用户通过第三方外壳超量挤占算力资源，导致整个服务的稳定性遭到破坏。为了保证绝大多数人的体验，官方必须立刻拔掉他们的网线。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYVWpsnP8q6ibnSNha74PctoTQ64XdqCA3SRxp2Sxib8wqmHBeKQF91QoiaT7RLSRk8WKyibXicBrlNb8IrPmIYCByd0UP2pSJHhLr6A/640?wx_fmt=png&from=appmsg)

但这套技术合规层面的说辞，难以掩盖背后已经摆到台面上的大厂利益搏杀。

冲突爆发的时机极其微妙。

就在上周，OpenClaw 之父 Peter Steinberger 刚被 OpenAI CEO 奥特曼招致麾下，负责牵头打造下一代个人智能体。

拓展阅读：[突发！OpenClaw创始人加入OpenAI：智能体革命，真的来了](https://mp.weixin.qq.com/s?__biz=MzI3MTA0MTk1MA==&mid=2652676116&idx=1&sn=ecbe7af8ec5838e835e0234032461112&scene=21#wechat_redirect)

这个原本中立且备受欢迎的开源项目，转眼间拿到了谷歌最大竞争对手的资金支持和战略倾斜。

另一边的动作同样充满火药味。

执行此次封禁的 Varun Mohan 此前带着自己的团队并入谷歌 DeepMind，一手搭起了今天的 Antigravity 平台。

两边的新锐力量，如今在各自新主人的阵营里迎头相撞。

谷歌掐断 OpenClaw 的接口，本质上是堵死了一条让 OpenAI 阵营工具低成本调用自家最强模型的暗道。

面对这种级别的封锁，Peter Steinberger 的回应同样决绝，他立刻宣布 OpenClaw 将在此后的版本中彻底剔除对谷歌架构的支持。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYUVHhRp33zRqeciclBqZVF0YK7hicJLSvASJtcQskruPBzIeWLeZcuaIov4dRibJIM5iaBHyfMgcpPjoKzlskm5JQUvcHNIXe7LZJ4/640?wx_fmt=png&from=appmsg)

OpenClaw 官宣支持谷歌 Gemini 3.1 Pro，仅仅在 3 天前。

拓展阅读：[20天狂更超10次，OpenClaw接入Gemini 3.1和Apple Watch](https://mp.weixin.qq.com/s?__biz=MzI3MTA0MTk1MA==&mid=2652677175&idx=1&sn=e815f7ea11ae0c8e9b413661a63befe3&scene=21#wechat_redirect)

这种彻底的决裂正在成为行业常态。

OpenClaw 原本代表了一种极具潜力的工作模式，即个人开发者可以写一个脚本，让智能体在本地设备和各个大厂的应用之间自由穿梭，自动发邮件、跑代码、理数据。

这曾是 AI 爆发初期最吸引人的图景。

但大模型厂商已经受够了只做底层的算力苦力。

去年，Anthropic 就对全天候高频调用 Claude Code 的用户实施了限流。

今年初，他们更是直接引入客户端指纹识别技术，把所有第三方套壳工具挡在门外。

巨头们的算盘打得很清楚，他们要把用户彻底圈在自己垂直整合的生态里，把控每一条交互数据，并把每一次底层运算都变成真金白银的订阅收入。

跨平台互通的开源红利期正在加速消亡。

这场风波给所有指望靠 AI 提效的企业敲响了警钟。

把公司的核心业务逻辑绑在第三方智能体外壳上，已经成了一项随时可能被断供的高危操作。

当底层模型供应商随意修改合规条款的解释权时，企业几乎没有谈判筹码。

账号体系的深度绑定更是暴露了致命的系统性风险。

一次由于调用智能体引发的违规，就能让整个开发团队的邮箱和通讯系统瞬间瘫痪，这种不确定性是任何商业公司都无法承受的。

企业技术主管们被迫重新审视现有的架构。

曾经被 OpenClaw 巧妙利用的低价调用通道被悉数堵死，未来想要在企业内部大规模部署智能体，要么转向本地私有化部署，要么老老实实签下极其昂贵的官方 API 合同。

大厂在收网，生态在封闭。

当技术的旷野被巨头们砌上高墙，开发者们才恍然大悟：从来就没有什么开源的乌托邦，只有尚未划定边界的生意场。
