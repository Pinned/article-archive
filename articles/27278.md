# 记一个我被Claude Fable 5震撼的瞬间。

Claude Fable 5回归了，订阅会员可以使用7天的时间。

7天以后，会从订阅会员中下线，只能用API订阅的方式来进行使用。

我也发了个朋友圈，也在公司群里跟所有的小伙伴说：

这7天可能时间价值最高的事：

用Claude Fable 5把你的所有的工作流、SOP、Skill、项目方案、项目代码全部优化迭代一遍。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/2jjfQoZLoqWVvrkqORKbu1wB5gbh7tp36bebib04IoaEiaKlqhL5OSrSXn0aPGWhEicEibiaYOwDoCICVicIibxsoqbqulcHVIM4XIajZMXcrpGlDs/640?wx_fmt=png&from=appmsg)

我自己也疯狂的蹬了一天（不要问我怎么不被封号，因为我也不知道，我只能说我自己的账号莫名其妙还是稳一点的...）。

坦率的讲，我已经很少会被一个模型的部分能力所震撼了，我们也越来越少的去提AGI这个词了，但是在这次使用Claude Fable 5中，其中的一个任务，还是会让我感受到了AI的震撼。

也感受到了，什么叫高级的智能。

我把任务发到了X上，也是哇声一片。

![](https://mmbiz.qpic.cn/mmbiz_png/2jjfQoZLoqW5JxAUMg3w1fw0yb48nS9QIOvblLGAibsNlibvguv51Ak24JianW1gRzVcz41wHQOwHAhGtN3rEmlFs27cI9BNvGfpbibE7FdEEQc/640?wx_fmt=png&from=appmsg)

故事是这样的。

还是我那个AI资讯热点网站AIHOT，我文章中的老主角，因为这确实是我日常中不工作时，每天花最多时间去优化和迭代的东西。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/2jjfQoZLoqXsC4ibGmMxA5yB9OsGWTyQmcT14X7cIjND0DBmwjvp6oNWm6RciafBsRPovUu3pkH7o3Bb37UWhYRRa9NVK9E3bxibPNT9L6AfZc/640?wx_fmt=png&from=appmsg)

然后呢，最近我也想试着尽量让我这个破逼网站量级更大一些，那想到的第一件事，确实就是SEO和GEO。

SEO大家都懂，让搜索引擎比如谷歌搜到你。

GEO这个词可能有些朋友没听过，就是让ChatGPT、Claude、豆包、DeepSeek这些AI在回答问题的时候，引用你的网站，以后趋势肯定就是大家越来越多地问AI而不是搜谷歌，那这块就是新的流量入口。

过去用Claude Opus 4.8和GPT-5.5都设计过这个SEO和GEO方案，我觉得都不是很好。

所以这次Claude Fable 5回来了以后，我第一时间，就直接开干这个玩意了。

我就直接发了一个消息过去，Claude Fable 5这种顶级模型，好处就是你只需要说目标就行，不需要想太多执行方案，因为大概率，它比你想的全想得好。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/2jjfQoZLoqWojBStooYdNXo4H8gZBgndyDNEOeMe2Khe20r1Wl1Y4PSXeebXsvVMTWpqVez4zYozxfic7nrVzkNJLNXg5iaJBUgEYp9YBZQps/640?wx_fmt=png&from=appmsg)

然后他就开了22个Agent，花了40分钟去全面调研。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/2jjfQoZLoqX98g1PPwrLkFmiaK4JZicqQK2yvicyrbgFdBw9LjzLux4qGAtmkcicruFTlibVst5rQge4QTqvekaKuCMM3vADU4vDBBJE6J6a7eibE/640?wx_fmt=png&from=appmsg)

反正翻出来了一些我不知道也是我做的不好的事情，比如豆包App里每天有六千多个真人打开我的网站，而我的统计系统里这个渠道压根不存在。

比如有个海外的WordPress网站用0.6秒一次的频率轮询我的接口，一天15万次。

比如有个网站直接全量镜像了我的网站还能被搜索引擎收录等等等等。。。

这些都很牛逼，也确实比Opus 4.8分析的更全面一些，但都还在我的理解范围内，也没啥可以太说的。

真正核心和震撼的，是接下来的事。

Claude Fable 5在审计完之后它给我列了接下来一定要做的三个大工程，能显著提高我们的SEO和GEO的水平，其中一个叫境外加速。

这个事的背景说人话就是，我的网站是个正规网站，服务器火山引擎上买的，在北京，但是呢，现在流量最大的，谷歌和ChatGPT，都在美国，它们每次来访问我的网站，都要跨半个太平洋，慢，还经常超时。

这样就会导致，他们觉得你这网站不稳定，就不爱来，那SEO和GEO就全都白做了。

解法业内也是现成的，就是有个东西，叫CDN。

你可以理解成在海外开一堆前置仓，把我网站的内容放到离Google还有ChatGPT他们爬虫近的地方，就近发货。

我说，行，这个你帮我做吧，交给你了，我相信你可以的。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/2jjfQoZLoqW16y4XEgF4ReMQNdIBo3Mia6DoxGV0r0l4EfQuV285FRybRqWdfARl0pGGMfkWMFGWOgSeTI0ibc7bV2YFQ3F2WTtmXxK8EdtI0/640?wx_fmt=png&from=appmsg)

说完这句话我就去干别的了。

然后接下来的三个小时，我基本上就是一个观众。

它第一步没有动手开干，先登上我那个火山引擎上的服务器翻了一圈，发现几个月前配置证书的时候，留过一把域名解析服务的API钥匙在服务器上。

这把钥匙意味着它可以自己改我域名的解析记录，不需要我登录任何后台。

然后，又看到了一个之前Opus 4.8遗留的待办方案，是我忘记删掉的。

之前Opus 4.8设计的方案里，境外加速用的是Cloudflare，海外最主流的免费CDN。

但是Fable 5自己看了一圈，又测了一圈以后，把这个方案推翻了。

理由有两条，一是Cloudflare免费版根本就做不了“国内直连、国外走CDN”的分流，要做就得把我公司整个域名体系搬过去，这就过于麻烦了，而且各种问题。

二是更要命的，Cloudflare从2025年开始对新网站默认拦截AI爬虫。

我们做GEO就是为了欢迎AI爬虫，你上一个默认把AI爬虫拦在门外的CDN。

这不完全违背了初衷吗。

于是，它决定，把这个项目换成了火山引擎的CDN，因为我的服务器和域名本来就在火山，一个控制台全搞定。

然后，它撞墙了，它发现这玩意好像开不了，这个功能没有开通按钮。。。

因为火山上的CDN，不是你想开就能开的，需要向官方申请白名单权限。

这个东西没有任何API，没有任何代码能绕过去，唯一的通道只有提交工单，找人工给你开，而且这个入口藏的巨深，我自己甚至都没找到从哪进。

如果你是Claude Opus 4.8或者GPT-5.5，这个时候大概率就放弃了，直接跟你说这个东西国内不支持Balabalbala，大哥我们要换个方案。

但是Fable 5没有，它不信这个邪，它用浏览器打开了火山引擎的后台，然后选择直接去问AI客服，把那个人工工单的入口找到了。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/2jjfQoZLoqXzRiatibesdUpup76QLOpdvWW6eicnxWqegJaDn52XVXQUkgL6ATMpF9642Wv4esUwAZMC8vMHLTkxsRS7VDxFo8FuSWWqGKQib5Q/640?wx_fmt=png&from=appmsg)

然后，自己直接写了个工单，用我的账号提了上去。。。

这一切，我本来其实都不知道的。

我知道的原因是，我在厕所拉屎的时候，我的手机莫名其妙的给我弹了火山引擎的短信。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/2jjfQoZLoqWjbSFa8L8y5DUVPib1wpLiby6JeQIbphc4GNaf0rKkTHZ9v3DQBXTon4zrft8XdqawJY5HfIqv2Wwre7Dm60LMlcpCC6twLPnfg/640?wx_fmt=png&from=appmsg)

我当时都懵了，我说我被盗号了？？？

提起裤子赶紧打开Claude Code，然后一看Chrome浏览器，才发现，原来是Fable 5提交的工单。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/2jjfQoZLoqWaoicicYVHUicJKG0JDGrWsxz5PJmPciaxVUFApgdNmfhKooRGYM7Q0K4ibpHgmIZXjO4FqFib6ZJRnXp77G44Qf2FjXnDj3LXqicRoY/640?wx_fmt=png&from=appmsg)

我打开工单详情页看了一下，这写的真的是极度的专业了。。。

甚至在工单末尾，它还预埋了第二个问题，问火山CDN回源节点的IP网段清单怎么获取，因为它规划的下一步安全配置会用到这个信息。

22点19分，工单提交。

22点22分，火山工程师受理。

22点44分，工程师回复，白名单已经开通。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/2jjfQoZLoqW1FFQVQTlDd32TOclT7JjzjIw4GeCQ2N2Xe1hURAFnEtn4pBpfVTk2SCTMmuR8DtK3Hibr3OVpQPcy1kgicT6efgYqUgWI1KVJo/640?wx_fmt=png&from=appmsg)

接下来，更震撼的来了。。。

工程师办完了白名单，但他漏答了工单里的第二个问题，就是那个IP网段清单。

然后，Fable 5发现了。

然后它回到工单页面，写了一段回复，先礼貌感谢对方处理得快，然后重新把第二个问题问了一遍，还补充了自己的用途说明和备选方案，说如果网段清单不方便提供，推荐的替代做法是什么，也请告知。

![](https://mmbiz.qpic.cn/mmbiz_png/2jjfQoZLoqWIzGiaxCOKj2SnRPuhfgoo9OpykvcTKOTGQqiaiaONzHMqOXFYaOvTKOicbsliaD2UQ3bDEOiaa3NpyAhGbIqpuXX3h22VkprMz3vYo/640?wx_fmt=png&from=appmsg)

我看到这一段的时候，我虽然用过那么多的AI，用Agent跑过那么多的任务，我依然感到，头皮发麻。

你能想到，这一切的起始，就是一段话，一个目标吗：

“我想提高我们的SEO质量和GEO质量，让我们的流量更加扩大，在这个时代成为第一AI信息综合站。”

我没有任何多余的话，也没有任何多余的操作，这一切，都是Fable 5全部自己调研，自己查找，自己写工单，自己去跟工程师对话，自己去解决的。

太离谱了。

4分钟后，工程师回复了一大段技术方案。

说火山官方确实不提供回源IP网段清单，因为节点是动态调度的，清单给你也会很快失效，推荐的替代方案是用回源请求头来传递真实的访客信息，还贴了配置示例。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/2jjfQoZLoqUqr0vFc0F8RcvxgnzKjwU4kaUHSkwvV9Jt07sBP0fuFFjXPSVltAkd3EU7mRTduK3vC4tsABW8bBiatIP7naJvTSdXqiaMJXt5w/640?wx_fmt=png&from=appmsg)

而Fable 5拿到这个官方方案之后，它也没有照抄。

它看了一眼这个方案，发现官方给的原始配法有个安全漏洞，任何人都可以伪造那个请求头，冒充别人的身份绕过我网站的限流。

于是它在官方方案的基础上，自己加了一层设计，让CDN在每个请求里额外带一个只有它和我服务器知道的暗号，对不上暗号的一律不认。

接着，自己又去我的火山引擎后台，开通好了一切。

![](https://mmbiz.qpic.cn/mmbiz_png/2jjfQoZLoqWicGiaPQwOhQ5Xhgc8yShTicTYApqneQZZShFkQeXGuVbicUY1vWlibVLrichINbS1t8Qr6FG19lgB14JM4cibUjwBaxmHTEMBm2aZCY/640?wx_fmt=png&from=appmsg)

23点30分左右，它切换了域名解析，全球生效。

10分钟后，已经有616个来自海外的请求，走上了新的线路。

然后它把整个过程写成了运维文档，提交了代码，给我发了份报告。

![](https://mmbiz.qpic.cn/mmbiz_png/2jjfQoZLoqVyfyXCuGQ21h4SMicicyWlzhDOOkv2JSGSRoeaeAvKgNoKTStKOibqicTiajrRY6jaPqiaNxWnLFauRGdTkRIC0vglx9NibT0fRtGkvU/640?wx_fmt=png&from=appmsg)

我甚至看到最后提醒我的那段话：

“边缘证书10月2号到期，续期步骤我写在手册第12节了，到时候任何一个AI会话照着做5分钟搞定。”

我忽然有一种莫名的感动。

整个目标的实现，聪明、细心、主动、优雅、得体，做完了所有事以后，还惦记着两个月后证书会过期。

坦率的讲，我在很多很多人的身上，都见不到这样的执行力。

绝大多数人的工作方式是你让我干啥我干啥，干完了交差，至于两个月后全站会不会因为一张过期证书挂掉，那不关我事。

能想到这步的，真的好好珍惜吧，那是那种你一辈子能遇到的可能最好的同事。

AI时代。

真是一个最好的时代。

让我们每一个人，都变得三头六臂的时代。

真好啊。
