# 灵光：日抛「APP」，用「Feed流」分发

PRODUCT

  

2017 年，微信刚出小程序的时候，张小龙如此说到

我分享一个时钟的小程序到群里面，那么群里面每个人看到这个小程序，不用点进去就可以看到已经有一个时钟在那里运转

很显然，这个想法并未被实现。毕竟，以当时的适配压力来说，吃不消的

现在，AI 则是把这条路重铺了遍，需求也变得可以能被当场满足、用完即扔。我们也看到了新的问题：日抛的软件，该如何分发?

参考一下，新闻、短视频、小红书...日抛的内容可以用信息流分发。那么日抛的软件，会不会也类似？

**对此，灵光给出了尝试：让日抛的「APP」，用「Feed流」分发**

就像这样，将 App 像内容一样分发

  

在 Feed 流里，应用也是内容
----------------

举个例子，你看到应用里的这些游戏，即便你并不点开，也能去玩，不需要点进详情页

游戏在 feed 里，也可以用的卡片形态发布

开篇说的那个「不用点进去，就能看到运转」，张小龙 17 年的讲过，但迟迟没有落地

第二年，Facebook 开发者大会上，Facebook 推出 Playable Ads。广告以一段 lead-in 视频在 News Feed 里出现，点击后进入全屏 HTML5 demo，试玩后再决定是否下载

2018 年 Facebook F8 大会上发布 Playable Ads 的宣传材料，能看到 lead-in 视频和全屏试玩 demo 的流程

代表案例是 Angry Birds。信息流里先出现 15 秒游戏画面，点击后在全屏里玩。Facebook 想要用户在安装前先试一把，降低下载后的流失

张小龙想要的是不点进去就能看到小程序在运转。Facebook 想要的是 feed 里先玩再下载

九年后，灵光圈把这两件事做到了一起

  

灵光圈的由来
------

去年 11 月，蚂蚁推出灵光。两种主要交互：对话和**闪应用**。对话框里说一句话，AI 就给出一个可以跑起来的小应用

![](https://mmbiz.qpic.cn/sz_mmbiz_gif/jXSGuwJvpdjoku9ibxZdTbmfpOQV2jv4Sc1B1Sic9J4JrkS1uwdyOFCDX6UXqmkdYUrIQgniaPibPXLTAo5dlFpflcXOibJ5QLyicLOUwPUr8BANs/640?wx_fmt=gif&from=appmsg)

灵光圈 feed 页

截至今天，灵光用户累计创建的闪应用超过 **3000 万个**

3000 万个闪应用需要一个地方流通。灵光圈就是这个地方。做完一个闪应用，一键发到灵光圈，别人能浏览、点赞、评论、收藏

两个动作比较重要。**Fork**：看到别人做的闪应用想改，用自然语言说一句「加个匿名模式」或「改成儿童版，字大一点」，灵光会重新生成一个版本。**共享**：一个人做的闪应用(投票器、排行榜、签到表)，分享给一群人，所有人看到的是同一份实时同步的数据

  

闪应用能调手机硬件，所以能被真的玩
-----------------

在 feed 里能「刷」应用，前提是应用真的能被玩

这次升级的核心，是打通了手机原生能力。AI 生成的应用能直接调：

**相机**扫描、识别、实时取景

**麦克风**语音识别

**LBS**定位、地图、路线

**陀螺仪**倾斜感应、体感交互

**震动马达**触感反馈

**本地存储、系统通知**常驻式的数据和提醒

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdiaBJVmTrwgKXVnRcicucAXicicdqHN95OicUEKD5xGoEZK5YqOE0s0DnXaYUA0eiaN7xdvNGkVKAAKhEaRTxuY3w941QeQLE37vUgZg/640?wx_fmt=png&from=appmsg)

调用传感器能力的闪应用提示界面，底部灰字「轻触允许，立即体验」

说「做一个拍照识花的应用」，它真能打开摄像头识别；

说「做一个带震感的番茄钟」，震动马达真响；

调硬件时界面会弹一行小字：「此闪应用需调用传感器能力 · 轻触允许，立即体验」

AI 生成的应用，第一次能跟物理世界发生关系

  

闪应用是一种新物种
---------

拿闪应用跟传统 App 比，它功能不深、性能不够、复杂交互做不到。但这种比较跑偏了

短视频改变世界，原因不在于它比电影更好。两者根本不是一回事。电影是作品，短视频是一种新的表达媒介，创作门槛极低，人人可参与，每个人的表达都有机会被看见

闪应用也一样。它是一种新物种。可以被刷、被分享、即用即生成，满足长尾需求，不值得为它开发专门 App 的那种

![](https://mmbiz.qpic.cn/sz_mmbiz_gif/jXSGuwJvpdiaamMaUxeHMxTWOviaoVkvydoMWk5RYx7xSUN6w9scW8Cb1L8mQib0ILo9WgkNtiagiaLQT9IvBZGjvVtYHwEiciaN23tKA8oWoQNmb0/640?wx_fmt=gif&from=appmsg)

比如这个：心跳密码

生成成本足够低的时候，软件的属性会从「工具」迁移到「表达」。红楼梦人物测试、抓大鹅小游戏、飞镖逃生游戏，本身就是创作，可以发给朋友玩

**软件也可以是娱乐**

  

应用被刷，还能被 Fork
-------------

刷是一面，Fork 是另一面

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdgzIWWBiaXt7WQP21VBU6kS4m8Cic2wQiaLiaejjiakzxLLBP6o8JiaABDa17OwtXlHa3uteIuS9yXu6ODkObqq8PdJRwLbe7zDcGO8w/640?wx_fmt=png&from=appmsg)

在闪应用详情页，能看到作者、点赞评论和「再创作」按钮

灵光圈上的应用没有下载、安装、设置... 但它们被刷到、被用、被 Fork、被接力

**在灵光圈上，每一个应用都可以是下一次创作的起点**

路径大概是：一个人做了骑行路线工具，另一个人 Fork 过来加坡度显示。第三个人 Fork 这个版本，改成带娃散步版，加上公厕和便利店标记

每一步都是一句自然语言

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdjlj8RjnJ2npwzzGYnq4ru4dR6KLm8H918ichMSNibcQ4w2hIJXtPjXY9lkXorAUrxlHIariagjOmaib0A9h6J5Vxg2S4u2m5M0feU/640?wx_fmt=png&from=appmsg)

可以在输入框里写着自然语言修改意图

GitHub Fork 代码需要你先会读代码，灵光圈 Fork 意图只需要你会说话

**GitHub 让程序员 Fork 代码，灵光圈让所有人 Fork 意图**

  

区别于 VibeCoding
--------------

Cursor、Claude Code 让知道怎么开发程序的人，通过借用 AI 进行更快速的开发，叫 **Vibe Coding**

灵光面向的是不会写代码、也没打算学的人，通过「许愿」来获得应用，叫 **Wish Coding**(意图编程)

**让最普通的人，能够制作软件**

  

说起来，灵光是我最高频用的国产手机 AI App

准确说，我不怎么用国产 ChatBot，除了灵光...甚至每周都会开一两次，多半是拍照问东西。出门旅行见到不认识的植物，菜市场遇到不知道是什么的菜，打开就拍一下、问一下

至于原因...就是他的交互最简单，最舒服，也觉得他们的产品感是很有东西的

然后今天灵光圈上线了，着眼于 APP 日抛、Feed 分发

那么，这件事究竟会成长成什么，就交给时间了
