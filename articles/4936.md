# 开源贾维斯Moltbot，被阿里云全栈打包了？

最近AI圈很神奇。

一群人不讨论哪家模型更强了，却在认真研究该买哪一款Mac mini来跑自己的AI助手。

看GitHub热榜，Star数几天时间飙到八万多，讨论区比OpenAI官方群还热闹……

大家在折腾的，就是那个最近大热的开源Agent——Clawdbot，刚刚改了个新名字叫Moltbot。

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404R98Mt3ibJMjiaxDgGj7yXdGecLOeiahDqIPe0bN5SjRzjjC708Tlib00Wvw/640?wx_fmt=png&from=appmsg)

很让我兴奋的信号是，阿里云下场了。

还不是象征性的支持，是直接把一整套云端基础设施、模型能力和消息通道全部打通，上线了完整的 Moltbot 云服务方案。

包括了轻量云服务器、百炼平台上的千问模型、钉钉消息链路……

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RXOxKZsu2kh7khhWU8dhQEJKQjjKgEeiclrHPZwt2wcDH7hPjzUE3PJA/640?wx_fmt=png&from=appmsg)

当国内外各类博主，还在介绍简单的部署教程时，中国云厂商直接送上「全栈打包」。

身边开发者朋友都很开心，像是过年前就收到了AI大礼包。

细细一想，这也是Agent时代，第一次出现真正意义上的“通讯即入口，云端即大脑”的基础设施落地。

下面具体说说是怎么一回事。

01. 人人一个云端Agent，风向变了
====================

  

===

Moltbot（Clawdbot）爆红，我觉得是因为直观， 因为方便。

它也没做什么全新的模型突破，但是让人不管在哪，一句话就可以调用Agent干活了。

订机票、查资料、跑脚本、自动化操作电脑、本地文件处理……很像个真的全时助理、工作生活主理人了。

这也是为啥Mac mini被当成Agent专用主机，接着就卖爆。

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RpsOMnsJxvyccqh5S3aulvb2ZVa71EpngichFQMLxhEEWghEbwQ5KaLg/640?wx_fmt=png&from=appmsg)

可真正用过以后，很快发现几个问题：

首先一个，本地机器的算力和并发能力非常有限，多个任务一跑就卡；

再就是，Agent本身高度依赖稳定网络和长时间在线，本地电脑一旦关机、休眠、断网，AI助手就人间蒸发；

还有，维护环境、更新依赖、排查问题，对非技术用户极不友好。

和当初的Stable DIffusion情况类似，本地部署变成瓶颈，云端托管成了刚需。

在国内，这正好是阿里云最熟悉的战场。

就在1月28日，阿里云上线全套Moltbot云服务，用户可在阿里云轻量应用服务器快速启用Moltbot，并按需调用阿里云百炼上一百多款千问系列模型。

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RAD3yLmbCfTnGyMrcmJyIjqZib5hlicB0yqS89omWiaayiaLTAibRoG0iaxpg/640?wx_fmt=png&from=appmsg)

在消息通道上，支持iMessage消息应用，还能基于阿里云计算巢实现钉钉消息互动。

而且，阿里云后续将继续打通更多应用工具，满足我们这些用户更丰富的七七八八的各种任务处理需求。

接下来跟朋友们说说具体怎么做。

03. 小白也能上手，AI助手随时有
==================

  

===

阿里云官方的文档其实很详细，比较官方说明书，大家可以照着一步一步来。

链接：https://help.aliyun.com/zh/simple-application-server/use-cases/quickly-deploy-and-use-moltbot

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RKichgqSRLb85qicM2oyDXEADs9N2dibBcmYkX5UOSfN5szL36q6cdttpw/640?wx_fmt=png&from=appmsg)

如果看着觉得字太多容易晕，可以参考我这个，按自己亲身的「真实操作顺序 + 人话教程风格」来试试，照着慢慢操作就能跑起来。

看起来步骤比较多，有点吓人，但其实就是做三件事：

*   在阿里云服务器上把 Moltbot 跑起来 
    
*   让它能调用百炼大模型API 
    
*   再把它接到钉钉机器人里，让自己和朋友们可以在群里直接问 AI
    
      
    

我们从头开始说。

一、先把 Moltbot 部署到服务器
-------------------

  

---

如果已经买过阿里云轻量服务器并装过 Moltbot，可以跳过。但如果打算“重置系统重新装”，系统盘数据会被清空，记得先备份。

### 第 1 步：购买轻量应用服务器

###   

### 1\. 打开阿里云控制台 → 进入 轻量应用服务器

### 2\. 点击 创建服务器

###   

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RrD0sfGJ1vnvhF4O8ZVkT3MGq6DIudyL4gxS9ZJMNECHt7cX5CkYjFA/640?wx_fmt=png&from=appmsg)

https://swasnext.console.aliyun.com/buy?regionId=us-east-1&planId=swas.s.c2m2s40b1.linux&imageId=bd45493af84846deb5dcd3fca6c1a1d9&amount=1&duration=12&autoRenew=false%23/#/

配置时注意这几项：

*   镜像：一定要选。 应用镜像 → 找到 Moltbot 镜像
    
*   地域：选离你或公司网络近的（影响访问速度）
    
*   套餐规格：新手测试选基础款即可，后期不够再升级
    
*   带宽：默认一般够用，访问量大再加
    
*   系统盘：默认即可
    
      
    

确认后付款，不需要买太多太贵；等 1～2 分钟服务器创建完成。

### 第 2 步：进入服务器

###   

服务器创建好后：

1.  在轻量服务器列表里找到你的实例
    
2.  点击 实例 ID
    
3.  进入服务器详情页
    
      
    

到这一步，Moltbot 已经在服务器里装好了，我们开始给它接模型。

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RxXP6D4s1O37s5stgUXrn6Rcj3X6smJic42BbIriaES1kMp91xtpM2FBg/640?wx_fmt=png&from=appmsg)

二、给 Moltbot 接入百炼大模型
-------------------

  

---

Moltbot 自己只是个机器人壳子，真正回答问题的是大模型，所以要先配 API Key。

### 第 1 步：获取百炼 API Key

###   

1.  打开 阿里云百炼大模型服务平台
    
2.  进入 API Key 管理
    
3.  点击 创建 API Key
    
4.  复制保存好这串 Key
    
      
    

⚠️ 这玩意等于你的“模型钱包密码”，泄露了别人能刷你 Token 费用。

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RGWiavuduIOzicUwKWSmusV7NPCHJDAmcepcYpTOp1vUMo9a9ibjCTiaIQw/640?wx_fmt=png&from=appmsg)

### 第 2 步：回到服务器配置 Moltbot

回到刚才的服务器详情页：

1.  点上方的 应用详情
    
2.  你会看到 Moltbot 的配置面板
    
      
    

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RJe1ZYtAKpe48LCTtbKwwNc4WJ19icgBeRpnf8OichjDk01sPYqiaCE77Q/640?wx_fmt=png&from=appmsg)

按顺序点下面这些按钮：

#### ① 端口放通

点击 一键放通。这一步让外网能访问你的机器人，不然打不开网页

#### ② 写入百炼 API Key

这一步相当于把“大脑接口”插上

#### ③ 生成 Moltbot 访问 Token

系统会生成一个带 Token 的访问地址。这个 URL = 管理员免登录入口，很关键，谁拿到都能控制你的机器人，绝对不要外传。

### ④ 打开机器人页面

点击 打开网站

你会进入 Moltbot（Clawdbot）的对话后台页面

看到聊天界面就说明部署成功了 🎉

三、开启 API 访问能力（给钉钉用）
-------------------

  

---

后面钉钉要通过接口调用 Moltbot，所以要打开接口开关。

在 Moltbot 页面里操作：

1.  左侧菜单 → Setting
    
2.  进入 Config
    
3.  左侧再点 Gateway
    
4.  切换到 HTTP
    
5.  找到 Responses
    
6.  把 Enabled 打开
    
7.  点 Save
    
      
    

现在 Moltbot 已经可以被外部系统调用了

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RyIicuCGlPfv77zU2BKxtXxyRyAcS9aIsk1xxVcEgWm4CNV0giaFQ1xZw/640?wx_fmt=png&from=appmsg)

下面开始接钉钉。

1.  打开 钉钉开放平台
    
2.  进入 应用开发

后面创建连接要用。

### 第 3 步：创建消息卡片模板

###   

钉钉机器人要用“卡片”才能流式输出 AI 回复。

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RXibzjLPJ668K0r7qH3iax9bNOqL9G6KcpJj7tTs5ia5sibZcPLDBXqSn6Q/640?wx_fmt=png&from=appmsg)

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RC5zqJUnd88Cia2vboZRv9lgZLPNcdeBqRVbibLMKsMjxh2xuBMw0lSkg/640?wx_fmt=png&from=appmsg)

### 第 4 步：给应用开卡片权限

回到钉钉应用后台：

4.  勾选互动卡片实例
    
5.  申请权限
    
      
    

不然机器人发不了卡片消息。

04. 说说阿里云
=========

  

===

看到“阿里云支持 Moltbot”，很多人第一反应可能是：给个服务器镜像，大家自己装。

真不是这么简单。

这次上线的，是一整套围绕Agent运行逻辑搭起来的云端工作流环境。

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RE8l4up1xgqKjVxlibUwZI5Bwzaf2SiczEW4mQddVNSkibn3LKckgF3BmA/640?wx_fmt=png&from=appmsg)

阿里云轻量应用服务器，已经预置了 Moltbot 及完整运行环境，不需要再折腾依赖、拉代码、配环境变量，开机就是一个已经满血版的Agent。

背后直接对接的，是阿里云百炼平台上超过一百款千问系列模型，天然具备多模型调度能力，可以根据任务类型调用不同能力层级的模型。

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404RumNbqdS0bzBDzPVcXxBLAkWz9Z8NeDibicCibCdDPFXVgwML28aeIsx0Q/640?wx_fmt=png&from=appmsg)

轻任务走轻模型，复杂推理走大模型，这种算力与成本之间的弹性调度，只有放在云端才能真正玩得转。

这就把Moltbot从一个好玩的开源项目，直接抬到了企业级可用的Agent运行底座。

在手机或电脑端，钉钉发一句指令，云端的Moltbot就开始执行任务，跑脚本、调接口、处理文档，然后再把结果回传到聊天窗口。

![](https://mmbiz.qpic.cn/mmbiz_png/9CRQL8peAG1Krw7utpTYCOog5bhZ404Rw5Uu2hDX1NdaaHUFvVKsRiaiaRY3Tv4c2iccbSGYPlzC5Qz85UEedRE4g/640?wx_fmt=png&from=appmsg)

我们过去一年谈Agent，大多是开发者视角，讨论工具调用、记忆机制、任务规划，但对普通用户来说，门槛始终在那儿：你要么去网站，要么装App，要么学一点复杂操作。

Moltbot原本的爆红，就是因为它绕开了这些门槛，而阿里云这次做的，是把这种“无感使用”能力，变成稳定、长期、可扩展的基础设施服务。

再叠加钉钉这种高频工作入口，很多原本需要人手点来点去的软件操作，开始可以被一句话调度。

比如让Agent整理会议纪要、批量处理表格、生成日报、抓取行业数据、跑简单自动化流程，这些事情过去分散在不同软件里，现在开始被收敛到一个“对话式任务入口”。

05. Agent开始进入规模化时代
==================

  

===

一个很有意思的对比是，对于一个爆红的“个人助理”式Agent，海外用户选择为Moltbot买Mac mini，核心是想把Agent当成个人设备；

而国内以阿里云为代表的云厂商，他们所提供方案，却是把Agent当成“云上长期在线的服务实例”。

两种思路，背后代表的是两个阶段。

前者是尝鲜期，强调自由度和可控性，适合技术玩家；

后者是普及期，强调稳定性、并发能力和持续在线，适合真正要把Agent当生产力工具的人。

云端部署意味着Agent可以7×24小时在线，可以同时处理多个任务，可以在不同终端被随时唤醒，而不再受限于某一台物理设备的状态。

再加上阿里云轻量服务器这次给到的价格门槛，新用户一年低至几十块的体验成本，实际上是在做一件很关键的事：

把「拥有一个专属AI Agent」这件事，从技术行为变成更多人的消费行为。

就像当年个人网站从自建服务器走向虚拟主机，后来小程序从定制开发走向模板化生成，当使用门槛降到足够低，形态就会从小众极客文化，转向大众基础能力。

大模型解决了“能理解和生成”的问题，Agent在解决“能规划和执行”的问题，而云厂商此刻面对的新命题是：如何让这些具备行动力的AI，拥有稳定的生存环境和随处可达的入口。

阿里云这次围绕Moltbot做的整合，似乎给了答案：

算力我来托管，模型我来调度，消息入口我来打通，部署复杂度我来抹平。

开发者不用从零拼积木，普通用户也不用理解底层架构，就能直接拥有一个“常驻云端、随叫随到”的Agent。

我们看到的已经不是单点创新，已经是一整套围绕AI行动能力重构的云端生态雏形。
