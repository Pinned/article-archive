# 服务，即软件

你没看错，我写的不是 SaaS（软件即服务），是「服务即软件」  
  
经常有朋友拿着一款 Agent 产品给我看，说：“你看这个产品能不能火？能不能赚钱？”  
我通常会让他 360 度转一圈，然后说：做不了  
  
在中国，通过 Agent 产品去赚钱，确实有点难度：  
\- AI 的受众群体没那么广，想通过类似于 SaaS 收租的方式很难能够 cover 住成本  
\- 如果产品是面向 Pro-C，用 token 消耗来收费...但很多时候，那些 Pro-C 会把 Claude Code 或者其他的工具魔改成自己喜欢的样子  
  
那在中国什么东西能赚钱呢？  
答：有钱的企业的服务  
  
在当前时间节点，把一个通用 Agent 产品推给企业，然后每个员工收 100 块/月的订阅费，大概率很难跑出规模  
但如果把他包装成一个「解决确定问题的服务外包」，就可以直接收他 10 万块钱的开发费+后续的费用  
  
比如，一家企业每个月都要做大量会展物料：主视觉、海报、社交媒体配图，每种材料都需要不同的尺寸，以前这件事靠设计师手改；而现在就可以使用 Agent 进行逐月自动生成，打包交付了

而是先找到企业已经在花钱的服务，再用 Agent 重构交付  
  
之所以现在能做，我觉得是依托于两个点：  
1\. Agent 能干很多活了  
2\. 甲方的 Agent 是能够被乙方的 Agent 所生产的  
  
在这个过程中，其实还有一个有趣的现象：  
以前做 to B 服务的公司，它的扩张是有上限的，因为服务成本需要靠人头来堆；而现在这些所谓的专家是能够被“分身化”的，如果能够成功的进行 Agent 交付，它的上限可能会比以往都要更高  
  
以及，对于原本的“软件即服务”的 SaaS，是需要把软件做好，等客户来用；  
而对于“服务即软件”的 SaaS，需要先锁定一个企业真实愿意花钱的服务场景，然后再用 Agent 把成本打下来，并且持续交付
