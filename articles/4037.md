# 终于，苹果支持Claude Agent SDK了！

![](https://mmbiz.qpic.cn/mmbiz_png/z7ZD1WagSLgsRsrdDyQwMQFeic5uAcFg7nUPJuxZKtOOpVA2qEx77mXgLIZRcOIDgFRTT28ymhVVnUyawribP28g/640)![](https://mmbiz.qpic.cn/mmbiz_jpg/z7ZD1WagSLjsmF5Umgx2XLTmqcicvxyqJIVOM5ZKdESjPwDyrdg7ibque3SCx4dSQII8e1con6rfRL2ibPcA5iavYw/640?from=appmsg)

**苹果Xcode支持智能体编程，集成Claude和Codex。**

**作者 |**  江宇  

**编辑 |**  心缘  

智东西2月4日报道，今日凌晨，苹果与Anthropic联合宣布，面向苹果平台开发者的官方编程工具**Xcode**推出26.3候选版本，并首次原生集成**Claude Agent**，支持以**Agentic Coding**（智能体编程）方式开发。

除了Claude Agent外，Xcode 26.3还支持接入OpenAI的**Codex**代码Agent。

![](https://mmbiz.qpic.cn/mmbiz_jpg/z7ZD1WagSLjsmF5Umgx2XLTmqcicvxyqJWu3KhC773ZSMYohPO3HxJoW09Q4YttNPKLPZybpicno1nvHcS5PLUPA/640?from=appmsg)

▲（图源：苹果官网与Anthropic官网）

在这一模式下，开发者可将目标直接交由Agent执行，由其自主完成任务拆解、代码修改、界面调试以及开发文档检索等开放工作。

相较于此前Xcode 26引入的ChatGPT与Claude模型的代码补能力，此次升级不仅开放了对**Claude Agent SDK的原生支持**，还通过Model Context Protocol**（MCP）标准实现**与更多Agent系统的兼容。

这将让开发者可以在IDE中直接调用包括**子Agent、后台任务、插件**等一整套Agent生态能力，推动开发流程走向“目标驱动型AI”。

目前，所有苹果开发者计划成员均可获取Xcode 26.3的候选版本，该版本也即将登陆App Store正式上线。

  

**_01_****_._**

**Claude Agent与Codex同步接入Xcode**

Xcode 26.3中，开发者可选择接入Anthropic的Claude Agent或OpenAI的Codex，两者作为“Agentic Coding”首发支持模型，能深入参与整个App开发生命周期。

这些Agent不仅可以处理单步请求，还能：

1、**理解整个项目结构**：自动探索代码文件体系，识别架构依赖与上下文逻辑，先计划再执行；

2、**执行复杂目标任务**：开发者只需描述目标，无需逐步拆解，Claude Agent或Codex会自主判断修改文件、查找文档、更新设置并迭代调试；

3、**进行视觉验证与迭代**：Claude Agent还支持通过Xcode Previews查看UI实际渲染效果，结合视觉结果进行优化，尤其适用于SwiftUI界面开发；

4、**调用原生API与文档搜索**：Agent可直接访问苹果开发文档与Xcode工具集，确保使用最新API并遵循最佳实践。

![](https://mmbiz.qpic.cn/mmbiz_jpg/z7ZD1WagSLjsmF5Umgx2XLTmqcicvxyqJCCwRUdmHMfh0SBwGoLPXDJLLwKkM4eP9lUPictI291omzyCSXUSKzsQ/640?from=appmsg)

▲iPhone APP开发界面（图源：苹果官网）

整个流程中，Xcode为agent创建的**每次更改都会生成一个可回退的“节点”**，确保开发安全。

  

**_02_****_._**

**Agent“看得见”界面**

**开启AI自主调试**

得益于对Claude Agent SDK的深度集成，**Xcode 26.3首次开放了“视觉闭环”的能力**。

**Claude Agent能截取实时的Xcode Previews界面**，理解其UI设计是否与开发者意图一致，并基于视觉反馈主动修改代码、重新构建。这使得Claude不仅“能写”，更“能看”，显著提升UI代码的一次生成质量。

![](https://mmbiz.qpic.cn/mmbiz_jpg/z7ZD1WagSLjsmF5Umgx2XLTmqcicvxyqJf4xPib9PCJve7icjyY4iaVGqOyMPNGic1lE99uoRRk7DO8chB2eGQCrkvg/640?from=appmsg)

▲iPhone APP开发界面（图源：苹果官网）

此外，Xcode的Agent系统也支持“**异步长时任务**”，即Agent可在后台持续执行目标任务（如完成复杂模块开发、统一架构重构），并在必要时才唤起开发者确认或修正。

  

**_03_****_._**

**MCP打通AI接入标准**

**开放更多Agent可能性**

Xcode 26.3的另一大亮点，是**正式开放了Model Context Protocol（MCP）标准接口**。

这一机制允许开发者将Xcode的功能暴露给**任意兼容MCP的外部Agent系统**，不局限于Claude或Codex。

通过MCP，Agent不仅能实现项目**结构探索、文件读写修改、视图预览与文档调用**，还可在CLI（命令行）环境下与Xcode交互，例如在远程终端中触发界面生成或Agent编排。

这一开放标准将为开发生态引入更广泛的AI系统，并为未来Agent插件化、子Agent协同、跨模态开发等能力铺路。

  

**_04_****_._**

**结语：Agent化IDE初步落地**

**苹果迈出Agentic Coding关键一步**

苹果在本次更新中特别提到，他们与Anthropic、OpenAI密切合作，优化了Token使用和工具调用机制，确保Agent运行效率与结果可控。

结合其强大的IDE生态、苹果自家硬件平台的深度融合，Xcode正成为AI agent部署的前沿试验场。

未来是否会有更多Agent进入IDE生态、是否会出现“多Agent协作开发”的新玩法，值得持续关注。
