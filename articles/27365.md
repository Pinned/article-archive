# Kimi Work：「赛博禅心，共发稿 887 篇」

Kimi

  

刚刚，[Kimi 发布了桌面端的 Kimi Work](https://mp.weixin.qq.com/s?__biz=Mzk0NDU1MDkyNg==&mid=2247488492&idx=1&sn=f7a0013c095a20f7aaeac00eacec1c16&scene=21#wechat_redirect)，定位「面向知识工作者的桌面 Agent 产品」，支持 300 个 Agent 并行工作

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdg8TzRHIouyIXClABk5cGk8qcjNdzvChbLHWvgNvAUNbGatlcL0rC5eQdbWu490lXEaKX4O6VFhSiaMO4GDzWLD9bwRcaDoBJHs/640?wx_fmt=png&from=appmsg)

核心能力包括：WebBridge 浏览器操控、同花顺/天眼查/iFind 等专业数据源接入、超长文本处理、设计感 PPT 生成、Skill 系统  

下载地址⬇️ Mac 版（需 Apple 芯片，MacOS 12 及以上系统）

https://appsupport.moonshot.cn/api/app/pkg/latest/macos/download

WIndows 听说已经在路上了，很快

第一时间试了下，就是：**极好的数据抓取、分析、呈现 Agent**

Kimi Work，也是我用到的、唯一能抓到公众号后台数据的 Agent

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdj6UwAMzGDx5OcricmSmMIY9PyIurI2YEGpBYTEPP6zEMSYfNsdTx5jsIzHK1XEY03fvsSXxW069GQpbmmkDgHoB3D3417ibgdOs/640?wx_fmt=png&from=appmsg)

  

公众号的后台数据，是不太好导出的。所有信息也都是动态加载，试了 Claude、Codex、Atlas 等等一堆东西，都读不了，但 Kimi Work 可以

  

不仅是公众号数据，各种数据它都能搞定，比如同花顺、天眼查、iFind、Yahoo Finance 这些金融数据源，抓完数据直接做分析和可视化，比如让他去做一份金融看板

  

这是一个极好的数据抓取、分析、呈现 Agent，目前鲜有对手

Kimi Work 强在哪
-------------

我觉得大概是两点吧：

*   其一、这套 Agent 搭的还是很完整的，能够有效的知道怎么去定位元素
    
*   其二、这个浏览器插件很不错，能被调用的很顺畅
    

  

作为对比，Claude、Codex 以及“最好的 AI 浏览器” Altas 都不行

比如这个，用 Claude in Chrome 打开公众号后台，右侧边栏会直接报错「Can't access this page」

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/jXSGuwJvpdjOSMmkApVTAPJGxiau32RKCCdVHKicC7OAjP1DiaVmG7JyyNvPaIvNWcTiaQojXk383E5v6jZudWttaqWNEAKsjq4ZI2zA4ycaq48/640?from=appmsg)

  

Codex 的 Chrome 插件也没读出来。可能是页面渲染方式的问题，微信公众号后台对这些工具来说都读不了

![](https://mmbiz.qpic.cn/mmbiz_jpg/jXSGuwJvpdg6EobrfibwGRrXKFUdISgGj5lVqEM3ZzLiat4Ax4NdibbnbqPmUDibXXtQX8RGG6MxXV5g2cjMeovBbibdqI2yCyliaIEtVQSd9A8tQ/640?from=appmsg)

  

OpenAI 的浏览器 Altas 的也试了，返回「ChatGPT does not have access to the page content」，然后建议我把数据导出成 CSV 再发给它...哎...

![](https://mmbiz.qpic.cn/mmbiz_jpg/jXSGuwJvpdhQnItUHQH2y3e1UJhUkBUNjGG9dNiaZSzDf4lql8mr094ajLB7dLUMic10gMytUHTgJXPOd7DTaEeTjsoWcvu91Y26Of9trvictU/640?from=appmsg)

  

具体原因，可能是页面渲染方式的问题或者其他，但总之，微信公众号后台对这些工具来说都读不了

  

而另一方面，Kimi Work 能读，成功的读到了，在过程中我是看着他思考的。在接到「帮我抓取公众号数据」这个任务后，就进行了以下的思考和操作

*   觉得自己应该使用浏览器，看看这个页面是个什么情况
    
*   发现这里的数据是动态加载的，尝试写代码解决
    
*   发现通过接口的方式，获取的信息不全，于是模拟人去挨个打开页面
    
*   定位到页面的每个元素，然后抓取里面的信息
    

  

这些数据抓完了之后，它还做了做一套数据表格，可以互动的：

怎么用  

------

下载地址⬇️ Mac 版（需 Apple 芯片，MacOS 12 及以上系统） 

https://appsupport.moonshot.cn/api/app/pkg/latest/macos/download

  

进去之后，可以切到 Work 模式，勾选右下角「K2.6 Agent 集群」

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdiaMsU7w3TuZUSeWEy5qsibkXVibXGMydSxLgYlIoGsjYPacMAYg8fDCGtF69cibfsdomsXuTY9zy4F32OWOC56audatgAGt5xOd5w/640?wx_fmt=png&from=appmsg)

  

如果你需要进行网页数据抓取，也可以安装 WebBridge

地址：`kimi.com/zh-cn/features/webbridge`，装完重启桌面端就能用

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdjBOhdYL1dNXpnGxUlIkiaJGPHTlbR80OX15fkQyWUzb8Bpheq8K6WyuzBLHnnDJiabgw8z1O2EQTgE01ffKFm0wlO28NFbohX0k/640?wx_fmt=png&from=appmsg)
