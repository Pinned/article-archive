# 因用GitHub经常“崩”，OpenAI被曝正在开发自家代码托管平台！

![](https://mmbiz.qpic.cn/mmbiz_gif/Pn4Sm0RsAujqPrtpN5QqqOuSG0AdUUx8wRg3vvr7aQJicdC0cHDBqZrJBZajQUXx97BwNaAE6L6x7OGlsTIpWjw/640?wx_fmt=gif)

整理 | 屠敏  

出品 | CSDN（ID：CSDNnews）

3 月 4 日，据外媒 The Information 报道，人工智能巨头 OpenAI 正在悄悄研发一款全新的代码托管平台，直接将目标对准微软旗下的 GitHub。

消息一出，引发科技行业关注，而促使 OpenAI 迈出这一步的直接原因，竟然是 GitHub 近期频繁出现的服务宕机问题，其稳定性不足已严重影响到  OpenAI 内部的研发工作。

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/S1iaf4GgGjEx8sC30bDn8pcl6Z2JNDZqlkG0cxY0mgm33FyRqt6fRojPvDkSjEsIZzdC7Pibha52ic6dBB2MgFHY9t3m08Mt0nIqZFzGTJvGEs/640?wx_fmt=jpeg)

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QuwjkSP1G6wEJHaJCLTONqlcQexqRgJcIICxofIOJs6B6tWBfibb7now/640?wx_fmt=png)

近来频繁宕机的 GitHub

作为全球最主流的代码协作与托管平台，GitHub 一直是众多科技企业和开发者的核心工具，OpenAI 也不例外。

但从 2025 年下半年开始，GitHub 的服务稳定性似乎大幅下滑，各类宕机问题接连出现：

*   2025 年 8 月，GitHub 因数据库基础设施调整引发全球大规模宕机，影响了 GitHub 网站及多项服务，包括 pull requests、GitHub Pages 和 GitHub API 等。
    
*   2025 年 11 月–12 月，GitHub 又多次出现服务中断，包括 Copilot 模型暂不可用、拉取请求评审失败以及部分功能降级，明显影响了开发者体验。
    
*   2026 年 1 月，Copilot 在一次模型更新过程中发生故障，部分请求错误率一度达到 100%。故障的直接原因是 GitHub 内部配置问题，但恢复过程受上游 GPT‑4.1 模型可用性波动影响，使得服务完全恢复的时间延长。
    

这些服务中断时长从几分钟到数小时不等，对 OpenAI 的研发工作造成了一定的影响。有消息人士称，OpenAI 的工程师们对 GitHub 频繁宕机十分不满。

也因此，为了打造一个他们更好掌控的产品，OpenAI 最终决定启动自主研发代码托管平台的项目，打造属于自己的 GitHub 替代方案。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06QDyP5HTUOXsPlJWd79yygiasaqXicpN7ibIfqiak5WFpaxE1mGxxpfmMjiaA/640?wx_fmt=png)

项目尚处早期阶段

知情人士透露，目前 OpenAI 这款代码托管平台的研发工作仍处于早期阶段，按照现有进度，预计还需要数月时间才能完成开发并落地。尽管尚未成型，但该项目的商业化可能性已经被内部团队提上日程。

参与项目的员工透露，曾初步探讨过将该代码库作为独立产品向 OpenAI 现有客户群体开放的可能性。

不过截至目前，这一项目的相关细节尚未有官方披露，OpenAI 也未对研发代码托管平台的传闻作出任何回应，其最终是仅作为内部工具使用，还是推向市场实现商业化，仍有待进一步观察。

  

![](https://mmbiz.qpic.cn/mmbiz_png/Pn4Sm0RsAujX5kS5KQ6BaBUsy1RqR06Q0QtgZAx8xsoTReptvArfwbn9MvHGVfV98Qkl5PMRS6MCt2ljwLJwoQ/640?wx_fmt=png)

股东变对手？OpenAI 与微软竞合关系迎考验

OpenAI 此次研发 GitHub 替代平台的动作如果为真，最受关注的一点，在于其与微软之间微妙且复杂的关系。

微软不仅是全球最大的软件公司之一，也是 OpenAI 的重要合作伙伴。双方自 2019 年起便建立深度合作，微软通过 Azure 云服务为 OpenAI 提供算力支持，同时还享有其相关技术的 IP 授权，而 GitHub 的 AI 编码工具也正依托 OpenAI 的模型驱动。

长期以来，双方保持着互利共赢的合作模式，但此次 OpenAI 布局代码托管领域将与微软之间出现直接竞争。

有外媒分析道，OpenAI 此次的举动，标志着其正从单纯的 AI 模型提供商，向涵盖底层开发工具的全栈技术巨头转型。而凭借其在 AI 领域的技术优势，若最终推出的代码托管平台能实现 AI 技术与代码托管的深度融合，将形成差异化竞争优势，有望重塑全球代码托管市场的现有格局。
