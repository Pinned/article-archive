  **新智元报道**  

编辑：艾伦

##### **【新智元导读】接下来的两周时间，Claude 将为美国时间的非高峰时段提供双倍配额，而这些时间正好是国内开发者的高峰时段。Anthropic 无意间为国内开发者献上了大礼。**

北京时间 3 月 14 号到 28 号这两周，Claude 搞了个限时「错峰双倍配额」活动。

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/Rvq8Ow69CYXBBlRic6vJpNB64KzA3Clmicf7sXptUb7TJUCrwiaIAiaExNRicqYEtL1vmIbyHibPtFVsIS5LFh5ByIlb2p4D0dP7lI7wVMXna61r0/640?wx_fmt=jpeg&from=appmsg)

名义上叫「感谢回馈」，实际上就是服务器扛不住尖峰流量了，拿额度诱惑你错峰干活。

具体数据和规则：

*   **时间窗口**：只要避开美东 8:00 AM—2:00 PM（美西 5:00 AM—11:00 AM），你的五小时额度直接翻倍。
    
*   **覆盖范围**：网页端、桌面端、App、Cowork，以及重头戏——Claude Code 和各类 Office 插件，全算在内。
    
*   **适用人群**：Free、Pro、Max、Team 都能吃到新红利。Enterprise 没份（这帮大客户本来也有按量计费或者专属合同，不需要掺和民用级促销）。
    
*   **关键点**：这部分赠送的额度，**不计入每周使用上限**。
    

![](https://mmbiz.qpic.cn/mmbiz_png/Rvq8Ow69CYW8Jjx6Tk5VWhzgu6q20hwnOLqpX7fx86Zib5cTkDQ0BIibqlT7evZGWaSHW5hQupQDC7Hl07CXRx67R4OTfNeib0cGVIalQ7p1cg/640?wx_fmt=png&from=appmsg)

众所周知，Claude 和 Claude Code 共享同一个额度池。

官方定义的「五小时窗口」一直是个黑盒，它不仅看在线时长，还盯着你的上下文长度、任务复杂度和模型规格。

平时拿 Claude Code 跑一波代码审查、重构个老模块、再过两遍测试，配额报警灯大概率就亮了，只能被迫下机。

现在平峰期额度翻倍，意味着你能在一个连续的工作流里，把一个积灰的 PR 从头撸到尾直接 Merge，不用再尴尬地等 CD。

最有意思的是时差。

美国刚切了夏令时，Anthropic 划定的「高峰时段」，换算成北京时间差不多是晚上 8 点到次日凌晨 2 点。

**反过来，能吃到双倍配额的「平峰时段」，刚好是北京时间凌晨 2 点到晚上 8 点。**

**这简直是精准投放给国内开发者的「时差****红利****」。**

Anthropic 的潜台词很明显：白天随你怎么折腾，晚上美国老乡要上班了，你们省着点用。

也已经有网友意识到了这一点，整理出了全球各大时区受该政策普惠的时间段：

![](https://mmbiz.qpic.cn/mmbiz_jpg/Rvq8Ow69CYU2AngSD9d6FibMxd3QtMmFcxzMib6BicmZZ4LhvsMrFxMqERUhHaiboRSNBALC6U7zhIPj52uJuctdggV71ZXPJtHWicAX6JrCgJ7c/640?wx_fmt=jpeg&from=appmsg)

这种利用时区差做流量隔离的手段，比单纯的降价聪明得多。

为什么突然搞这一出？

半个月前，Claude 刚崩过一次大的。

![](https://mmbiz.qpic.cn/mmbiz_png/Rvq8Ow69CYXDqg5jkF09aCMohticTM08kBuBO4tibbyx2iaMwiadsrPT1qicntyAyDbGkVZIhTPGVMTZspzcjljDG9zFpmtYQr9xRMewt4LduicOE/640?wx_fmt=png&from=appmsg)

状态页拉满红灯，网页端和 Claude Code 统统连不上，响应慢得令人发指。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/Rvq8Ow69CYUK1xsyxzcK7USEJKxsiboypSFEzicj1rwWH9U1ickS5CSaCxWFtI91HFMjcXGQmlFr7slicH5aSfdm3Wh504expmwZbpE6GHF7LVw/640?wx_fmt=png&from=appmsg)

底层的模型推理没挂，而是订阅端的并发请求、超长上下文和 Agent 工具调用，把前置网关和队列撑爆了。

面对这种尖峰流量，基础设施公司的常规解法就是削峰填谷——堵不如疏，我给你加倍，你晚点再来。

同时这也是一场大规模的运营压测。

**Anthropic 大概想跑通两个逻辑：**

**一是验证「时间窗口」能不能作为新的流量调度杆，把高峰期的****并发****拉下来；**

**二是倒逼这帮重度用户养成「分时段调用」的习惯，为以后顺理成章地卖「额外超额流量包」和更贵的高级席位铺路。**

先限流，再分层，最后把平峰变香、高峰变贵，这是所有算力稀缺平台殊途同归的剧本。

对国内开发者来说，未来两周的策略很简单：把那些吃上下文的重活（大仓库重构、长链路调试、批量改文件）全挪到北京时间的白天干；晚上的宝贵额度，留给必须即时响应的紧急 Debug。
