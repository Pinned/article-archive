CORE PRODUCT

飞书 aily：飞书原生的「龙虾」
-----------------

谢欣说，龙虾太火了，总有人问飞书什么时候出自己的龙虾。他自己也深入用了龙虾，发现确实厉害，但也确实不适合大众使用：上手门槛太高，配置繁琐，而且不安全

所以飞书的回应不是做一个独立的龙虾，而是**升级 aily，让它成为飞书原生的智能伙伴**

### 和龙虾的关键区别

谢欣着重强调了一点。OpenClaw 不管是在飞书还是其他 IM 里，只是把 IM 当入口。它并不知道你在 IM 里的聊天情况，除非你把消息转发给它，或者把它拉到群里（这又增加了安全风险）

**aily 不一样。它真的可以知道你在飞书里的工作信息**（前提是你授权），文档、日历、聊天记录，它都能读到

在这个过程中，不需要手动导出 PDF 上传，不需要整理上下文。而且不需要把 Agent 加进群暴露风险，它自动以用户身份安全获取上下文

周洲说：

以前做数字分身，大家做的其实是一个「模仿你说话」的 Chatbot，因为没有权限和工具，很多事做不了

今天给了它飞书的工具和上下文，它真的能干活。**关键是它能不能懂你，知不知道你最近关注什么、跟谁在聊天，这对一个合格的伙伴来说是最重要的事**

### 好上手、超能干、更放心

**好上手**

无需安装配置，1 分钟激活。内置海量官方技能库（经过安全扫描和开源认证），用户可自定义安装技能，aily 也可以自主发现和安装需要的技能

**超能干**

自带飞书各类功能，能调用飞书中的工具和上下文。日报、PPT、数据报告直接让它做。可以创建定时任务（比如每天晚上 8 点帮你巡群整理信息），7×24 小时工作。系统每日自动抽取工作记忆，沉淀为长期资产

还有一个首创能力：**结合飞书内的事件主动触发工作**。比如你可以设置：收到重要同事私聊消息时，自动查找相关知识，提前写好回复

后台任务能力也值得说一下。aily 做比较复杂的事情时会启动后台任务，你不需要等它完成，可以继续给它发消息做别的事，它相当于后台有一个小分身在工作

**更放心**

运行在飞书内部，企业信任环境，自研架构。数据所有权和飞书账号一致，数据不出企业。权限体系和用户本人完全一致，操作全程可追溯，敏感操作需人工确认。Skill Hub 所有接入的技能均经过官方审核和开源认证

### aily 能帮你干什么？

→直属上级发消息，自动分析意图、搜集背景资料、起草回复草稿

→持续关注某个体验群，出现用户反馈的 badcase 自动整理到收集文档

→把产研部的产品反馈和市场部的用户需求汇总，生成一份可以直接用的对客 PPT

→跟助理说「我写文案从不用感叹号，汇报喜欢先结论后细节」，偏好会被记住并同步到个性化配置

→跟助理说「帮我检索品牌营销相关的热门技能」，自动检索并一键安装

→定期总结你最近的关注事项，主动检索和跟踪企业内外的相关信息

### Case 1：产品助理的访谈分析

飞书产品助理孙昊天上场演示。他先说了一个感受：

用了 aily 之后，不需要再帮 AI 整理上下文了。以前要把各种信息导出 PDF 上传，现在只要告诉它问题是什么，它就能结合飞书里的文档和聊天记录帮你做。原来每天用两三次 AI，现在几乎所有工作都先让 AI 看一看。

他的案例背景：上个月访谈了 8 个 OpenClaw 深度用户，半个月后做了回访，想把所有信息整理出来做产品策略分析。三个让他惊艳的点：

**第一**，让 aily 把 8 篇访谈文档整理成多维表格。没有告诉 aily 表格该长什么样，它自己读完文档后创建了结构，用户类型、访谈日期、各问题回答全部填好

**飞书 aily 智能伙伴 vs aily 专业版**

  

产品形态

智能伙伴：飞书 IM 内的 Agent 联系人｜专业版：独立 Web 端 + 移动端

核心定位

智能伙伴：开箱即用、零门槛的个人智能助理｜专业版：承载高阶专业能力的生产力工具

交互方式

智能伙伴：对话式交互，融入飞书 IM｜专业版：GUI 辅助界面，精细化输入与调整

典型场景

智能伙伴：日常问答、上下文理解、基础任务｜专业版：深度文档创作、PPT 生成、数据分析可视化

### 数据分析可视化

把 Excel 或多维表格的数据丢给 aily 专业版，自动生成可视化报告：核心指标、转化漏斗、按月趋势。给 AI 的指令非常简单：「分析这个业务数据，生成可视化报告，给我建议和洞察」，没有其他额外要求

更关键的是 aily 专业版能给出业务洞察，而且不只基于当前数据，还能查到企业内其他数据（比如最近的业务策略），做交叉对比，给出贴合业务的建议。**与飞书上所有数据资产完全打通，不用导出不用上传**

为什么是飞书
------

发布会内容讲完了，回头看一下今天飞书做的这些事情背后的逻辑

飞书内使用 Agent，Agent 获得的不仅是通信的入口，而是安全地获得了全部工作的上下文和工具，拥有了完整的工作环境。这和把龙虾装在个人电脑上是完全不同的

**超能干**

区别于外部 Agent 只能处理孤立文本，飞书 Agent 拥有完整上下文和业务执行入口。飞书沉淀了国内企业最完整的上下文资产，Agent 能实时调用消息、文档、多维表格等数据。得益于 All-in-One 体系，Agent 不只给建议，更能直接动手：改表格、回消息、搭系统，方案生成后自动执行

**更安全**

权限体系和用户本人完全一致，操作全程可追溯，敏感操作需人工确认。所有交互产物沉淀为企业数字资产，不是漂浮在外部平台的零散碎片。无需 Agent 入群暴露风险，自动以用户身份安全获取上下文

**好上手**

将复杂的 AI 技术封装成开箱即用的产品。每项 AI 功能在交付前已完成合规审核和多场景 AI 成熟度评估，用户拿到的不是「待调优的草稿」，而是生产力工具
