# 字节版龙虾架构火爆GitHub！开源获35k+ Star，内置Skill全家桶，原生适配飞书

##### 闻乐 发自 凹非寺  
量子位 | 公众号 QbitAI

原来字节也有龙虾——

**Deer-Flow2**超级智能体管理框架。

开源发布后迅速登上了GitHub Trending榜首，已经收获了35.3k Star。

![](https://mmbiz.qpic.cn/mmbiz_png/A6fTew8FFGGjLTO341TicXPEXGrM1VjoEd4365ZWB01aootrQhSYu7GHGGPKRibL9iagjqvvsrYJiaqjtaSwJpXZDWcL6YJrknDN9GPp5owvVeQ/640?wx_fmt=png&from=appmsg)

Deer-Flow2采用模块化多智能体架构，这些智能体通过LangGraph实现协同合作。

主打**开箱即用**，内置了Tavily、Brave Search、DuckDuckGo等多种搜索引擎，还集成了Jina等爬虫工具，基本把信息收集的十八般兵器都给配齐了。

当然，扩展性也没落下，自定义API或模型随意接。

核心能力上，多智能体协同、沙箱安全执行、一键部署全都有，Docker快速部署和本地开发任你挑，主流大模型统统兼容。

不过最贴心的还得是IM渠道支持——

原生适配飞书、Telegram、Slack，没有公网IP也能跑。

![](https://mmbiz.qpic.cn/sz_mmbiz_gif/A6fTew8FFGEFDGfr64qpBKVbjGunNCyGgnXUhhSxOz1suMvkf6mhySoPL3qtfUO24mFibU6BO5RDGsdhSk6wTgCu4pLIE67Z1iaLjKTWHfRss/640?wx_fmt=gif&from=appmsg)

核心能力与技术亮点
---------

DeerFlow在迭代过程中完成了一次彻底的架构升级。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/A6fTew8FFGFRYthzRDytO7pbfAEoXPZrYdDgswGESEMv6rbeymvoy4fKQuiaGlW7KDXPwjOJcsZSGD35dr1MicpMFBSh51bia4mek6IrwAAGl4/640?wx_fmt=png&from=appmsg)

1.0版本采用固定5节点多智能体架构，能力边界相对明确，主要聚焦在**深度研究**场景。

而2.0版本则对整体结构进行了全面重构，从底层骨架到上层能力都实现了飞跃。

新版本采用**单一主智能体+11 层中间件链+动态子智能体**的全新架构，**将核心能力收敛到工具集与中间件链中**，让整个系统更轻量、更灵活、更易扩展。

相比1.0需要调整整体结构才能新增能力，2.0只需添加新技能就能完成拓展，无需改动底层框架。

原本作为核心的深度研究，也从唯一主打能力转变为框架内置的一项基础能力。

在框架层面，DeerFlow 2.0已经整合**子智能体调度、长期记忆、隔离沙箱执行环境、可扩展技能与工具等关键模块**，形成了一套完整、成熟的智能体运行能力体系。

![](https://mmbiz.qpic.cn/mmbiz_png/A6fTew8FFGFEUlJxHvxnZtpbkLTyWiaFMbicVrPJicu2niaiaqGZWmjNV0MoiaxXC3ICIJictoU99grQ1zic3qRsCezTQZuGIuj3hDM5cKklibrriaibTA/640?wx_fmt=png&from=appmsg)

#### 可插拔Skill体系

为了让智能体快速适配不同场景，DeerFlow 2.0搭建了一套可插拔的技能体系。

出厂自带深度研究、数据分析、图表生成、音视频创作等十余种常用技能，系统会根据任务需求渐进式加载控制token消耗，这样就避免了上下文被过度占用而导致的效率下降。

![](https://mmbiz.qpic.cn/mmbiz_png/A6fTew8FFGFtFwFjqMicBj8ib6h9rRSR2mJ6woNxzW9Hic1OCW7Dwypiam5dPIJc5LxfVBPNSr2eHXZspBxBn68tcWBpzKialXwnYBJiaY8gdmoias/640?wx_fmt=png&from=appmsg)

如果内置能力无法满足需求，用户还可以自行封装专属技能。

配合官方提供的skill-creator工具，几分钟就能为智能体扩展新能力。

同时系统提供MCP与Python接口，支持自定义工具的深度集成，甚至**可接入Claude Code**，让用户在终端就能完成工具的下发、查看与管理操作

#### 隔离沙箱执行环境

DeerFlow 2.0还配备了独立隔离沙箱。每个任务都在专属沙箱中运行，拥有完整文件系统与Bash执行权限，支持文件读写、脚本运行、命令操作等。

系统提供本地、Docker、Kubernetes三种运行模式。

其中Docker模式采用字节开源的AIO Sandbox，隔离级别更高、运行更稳定。

同时自动完成虚拟路径与物理路径的映射，确保开发环境与部署环境保持一致。

#### 子智能体调度+上下文工程

面对复杂长时任务，DeerFlow 2.0通过调度机制与上下文工程双管齐下。

主智能体会先对任务进行结构化拆解，再按需调度最多3个子智能体并行执行子智能体可选用通用能力或命令行专家型。

每个子智能体都拥有独立上下文，互不干扰、互不污染

在此基础上，框架还通过多层中间件链、上下文自动摘要压缩、外部文件存储、子任务限流等设计，系统性解决长时任务中上下文窗口不足的问题。

如何部署
----

DeerFlow提供了Docker和本地这两种主要的部署方式。

Docker部署是最简单快捷的方式，只需几个命令，就能在本地启动完整的DeerFlow服务。

首先克隆仓库：

> git clone https://github.com/bytedance/deer-flow.git

等待仓库下载完成后，进入项目根目录：

> cd deer-flow

![](https://mmbiz.qpic.cn/sz_mmbiz_png/A6fTew8FFGHiaIb7IknfF49SWKRdFIffj9orzEGzicLkz2iauBdiapQxbHIT84Quf6IMMKLg5yNGp55Jia8SpxWicpRGA0ZEBVQ1ThWCia8BSkkTnU/640?wx_fmt=png&from=appmsg)

生成本地配置，输入：

> make config

系统会自动生成config.yaml配置文件和.env文件（如果没有make命令，Windows可安装MinGW）。

然后找到项目目录下的Config.yaml文件，填入模型相关配置。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/A6fTew8FFGEv7AZKL0r35aibyRMZrOLBNSXnl1YwyCJuG9ldpwicax0rwibtYgFPlrQxuPjOzvvlicicrduz8YRV6So8puC0gAGWsicMCgr1HOJqo/640?wx_fmt=png&from=appmsg)

设置完成后，输入：

> make docker-init

自动拉取字节开源的AIO Sandbox沙箱镜像，首次拉取可能需要几分钟。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/A6fTew8FFGEm76mRP62hxJzKDYgI59AJPezxLJyZydYplNH4s7nicKu3nkP7F7YibCjGSs8H0ibSYB7GS5zzeaLVX3Aso2SlkMibicNnasKzmicjI/640?wx_fmt=png&from=appmsg)

镜像拉取完成后，启动服务，输入：

> docker-start

服务启动后，访问 **http://localhost:2026**即可进入Web界面。

![](https://mmbiz.qpic.cn/mmbiz_jpg/A6fTew8FFGH9wKWs3VoZcic1HA5vN3U5IdnBP32KjktE1wWWpk1HSMC9k69cy0fyzdiawNW5ap0IzS8icPbYib6JsHeIqQNXXTd9ApdGVyicaqvo/640?wx_fmt=jpeg&from=appmsg)

如果需要进行深度定制或二次开发，可以选择本地部署方式。

本地部署需要满足一定的前置条件，包括**Python 3.12+、Node.js 22+、pnpm、uv包管理器以及nginx**。

满足前置环境后检查依赖，打开终端进入deer-flow根目录，输入：

> make check

系统会自动校验上述依赖是否齐全、缺少的会提示补充。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/A6fTew8FFGGKfDCWdvIKS7Oz7QUUNOpqkRwHXSdWtrYE5HYrgRSiczDY9AJTBcNYhc2jPrA2p8WECaVVqKmZuYvu2Z8kCrJwm1TPOibk2qS60/640?wx_fmt=png&from=appmsg)

这时候你可以输入：

> make install

系统会自动安装python和node相关依赖包。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/A6fTew8FFGEjVfqFfUmoSsf8KBOtFVQa5xKKdtqvYPk1dvo3ZksibhCAjusFvtokmUM3bfWJpZwWnF7Od67lsep2OPxLCYkZicoJAzLGBfXrs/640?wx_fmt=png&from=appmsg)

接下来可以输入**make setup-sandbox**（按需）预拉取沙箱镜像，避免后续首次使用时等待。

然后启动服务：

> make dev

![](https://mmbiz.qpic.cn/mmbiz_png/A6fTew8FFGGfPN7pqIESnju38xOcAAFicCXaSylHDoAU48BRw8pfo1yiaZqia8xPDFWt6g5BOl4YtUR5cW0NJuX4xV43vXR9q1qibciaL3rDT9ls/640?wx_fmt=png&from=appmsg)

这种方式适合需要修改源码、调试功能或贡献代码的开发者。

DeerFlow原生支持从即时通讯应用接收任务，目前支持Telegram、Slack和飞书/Lark三个渠道，且都不需要公网IP。
