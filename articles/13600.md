# Claude 进了 Word，Pro 和 Max 都能用

PRODUCT

**Anthropic** 4 月 10 日发布 Claude for Word 公测版，第一批只开给 Team 和 Enterprise 用户

一周后的现在，Pro 和 Max 用户也被放进来，可以搭配刚发的 **Opus 4.7** 用

算时间线的话：Claude for Excel 去年 10 月发，PowerPoint 今年 2 月发，Word 这次补齐。Anthropic 的 Office 三件套凑齐了

![](https://mmbiz.qpic.cn/sz_mmbiz_jpg/jXSGuwJvpdia6m0rLMpeRgoyrKVLoD2fWShicQiaqgcvna1a6lkjRueq3ItmtMiaWL9u9VI0gFuaDnt4icg7JvOUMzjymhwDFblckTva1tEQ8q6w/640?from=appmsg)

4 月 17 日 Pro/Max 开放时 Anthropic 官方推文配的图。Word 里打开 NDA 合同，右边 Claude 一键给出 6 条开放批注的总结，跑的是 Sonnet 4.6

Claude for Word 是什么
-------------------

打开 Word，在 Home 或 Insert 的插件栏里找到 Claude 图标，点开之后它在右边展开一个侧边栏，位置和微软自家的 Copilot 一样

登录用 Claude 账号就行。企业如果已经有 LLM 网关（Bedrock / Vertex AI / Azure），可以走自家通路，不用单独登录 Claude

支持的文件格式是 `.docx` 和 `.docm`，`.doc`、`.rtf` 这类旧格式要先另存为 `.docx`

支持的 Word 版本清单：

→ 网页版 Word

→ Windows 版 Word（Microsoft 365 订阅，版本 2205 / Build 15202.10000 及以上）

→ Mac 版 Word（16.61 / Build 22040100 及以上）

不支持 Word 2016/2019 永久授权版、iPad 版、Android 版

六件事情它能帮你做
---------

### 一、问文档问题，答案带跳转锚点

选中一段、或者直接问`第 3 节的营收预测基于什么假设`，Claude 给答案，同时给引用的小标签。点标签，Word 跳到对应段落

### 二、选段改写，只改这段

选一段文字，告诉 Claude 怎么改（`压短`、`换语气`、`去掉被动句`），它只改这段，四周的标题编号和样式不动

### 三、修订模式（Tracked Changes）

Claude 的改动以 Word 原生修订模式呈现：原文划掉，新文标为插入，全部出现在 Word 自带的审阅窗格里。可以逐条接受，也可以逐条拒绝

**这条是整个产品最关键的设计**

### 四、评论驱动的编辑

在文档里用 Word 批注写修改意见，Claude 读取所有批注，逐条改对应段落，并在批注串里回复它做了什么

![](https://mmbiz.qpic.cn/mmbiz_png/jXSGuwJvpdj4wia3lj08rdF4WfVSdNFd37IMl1YibzL5UlsaFmicMojuvYwCa9C3vVWZFwrjA8qI6Sl1hMLID8CPF4Uu0a7IQLC3Vk82Whxe24/640?from=appmsg)

Claude 把修改直接挂在批注上，每条改动有说明

### 五、模板填充

打开你自己的模板，告诉 Claude 要填什么。它按模板已有的标题级别、段落样式、表格格式来写，排版不乱

### 六、语义导航

问`这份合同里哪些条款涉及数据保留`，Claude 返回所有语义相关的段落。匹配的是主题，不只是关键词

一个对话贯穿 Word / Excel / PowerPoint
--------------------------------

Claude for Word 和已经发布的 Claude for Excel、Claude for PowerPoint 共享上下文

三个文件同时开着，一条对话里，让 Claude 从 Excel 拉数据进 Word 备忘录，或者把 Word 文档缩成 PowerPoint 幻灯片

省掉复制粘贴，也省掉三个窗口来回切

为什么要走修订模式
---------

这条值得单独说

AI 改文档最常见的做法：直接生成新版本，让用户对比前后差异。Claude for Word 选了另一条路

每一处改动都以 Word 原生的 tracked changes 呈现。原文留着不动，新内容以插入形式出现，用户逐条审阅

这个选择隐含两层意思。第一，AI 的输出不覆盖原文，留下完整的审计轨迹。第二，AI 的角色被定义成协作者，最终决定权留在人手里

![](https://mmbiz.qpic.cn/sz_mmbiz_png/jXSGuwJvpdjicjlhA26sSAib2g4dWKZN1gOBJQ0dZvkYp81eYOMDmibib8CxJCFlHx5lEXU2Nt8vP1BO3jAB1wyZiaWibJGsYZ7QUNnndq2jPDtOk/640?from=appmsg)

对手方把文档改了一遍发回来，Claude 把改动按章节逐条列出，点 §1 Definition 这类标签可以直接跳到正文位置

在法律、金融、合规这些对`谁在什么时候改了什么`有硬性追溯要求的行业，这个设计是必须的

安全和数据处理
-------

官方说法：Claude for Word 在你现有的安全框架内运行

如果企业已经用 Amazon Bedrock、Google Vertex AI 或 Microsoft Azure 的内部 LLM 网关接 Claude，可以不用单独登录 Claude 账号，走同一条通路

数据保留：输入和输出在后端 30 天内自动删除。缓存保留几小时，方便用户找回刚关闭的文档上下文

Beta 阶段有几个限制需要提一下

→ 对话记录不跨会话保留，每次打开都是新对话

→ 目前不支持 Enterprise 的审计日志和 Compliance API

→ 不继承企业自定义的数据保留设置

官方明确不建议用于：未经人工审阅的客户交付稿、诉讼文件、审计关键文档、未加访问控制的高度敏感数据

一个需要警惕的风险：prompt injection
--------------------------

官方专门用一段话提醒：提示词注入攻击

假如用 Claude for Word 打开一份外部拿到的文档（对手方的合同草案、邮件收到的模板、网上下载的文件），文档里可能被埋了隐藏指令，让 Claude 做你没授权的事情

Anthropic 自己测试里出现的三种情形：

→ 通过网页搜索把敏感信息泄出去

→ 悄悄改合同条款或财务数字

→ 执行破坏性动作（前提是你允许 Claude 自主执行）

官方原话是 **Only use Claude for Word with trusted documents**

只用于可信来源的文档
