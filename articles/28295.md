Rust 终于冲进 Top 10，TIOBE 曾担心它“涨不动了”

如果只看今年上半年，Rust 在 TIOBE 编程语言排行榜上的表现堪称一部“反转剧”。

今年 4 月，TIOBE 官方还曾公开表示，Rust 的增长势头似乎正在放缓，进入 Top 10 的目标可能比预想中更加遥远。然而，仅仅几个月后，最新发布的 7 月 TIOBE 指数就带来了一个令人意外的消息：Rust 首次跻身TIOBE 榜单 Top 10，创下自诞生以来的最好成绩。 

![](https://mmbiz.qpic.cn/mmbiz_png/S1iaf4GgGjEyRjK25UEicezUNHic0ibSamLNIHq6va4R1icBysiaib3tPd0s6jpSILWdB7xltGyx4T7Lu2nicUWSUVRrgTx2QWD2bIN0lPcpXBfho2c/640?wx_fmt=png&from=appmsg)

事实上，Rust 的排名一直在稳步上升。今年 6 月，它刚刚刷新历史纪录，来到第 12 名；进入 7 月后，又一举冲到第 10 位，正式进入第一梯队：本月最新榜单中，前五名依然由 Python、C、C++、Java 和 C# 占据，而 Rust 则成为 Top 10 中最年轻的成员之一。

Rust 的崛起并非偶然。过去几年，内存安全问题受到行业关注，越来越多开发者开始寻找既能保证性能，又能避免传统内存错误的新选择，而 Rust 恰恰踩中了这一趋势。

与 C、C++ 一样，Rust 可以生成接近原生性能的机器代码，因此非常适合操作系统、浏览器、数据库、嵌入式设备等性能敏感场景。但不同的是，它通过所有权（Ownership）、借用（Borrowing）以及生命周期（Lifetime）等机制，在编译阶段就能消除大量空指针、悬垂指针、缓冲区越界等内存安全问题，无需依赖垃圾回收器即可实现高性能与高安全性的平衡。

正因为如此，Rust 被普遍认为是 C 和 C++ 最有力的挑战者。近年来，无论是 Linux 内核、Firefox 浏览器，还是微软、谷歌、亚马逊、Cloudflare 等大型企业，都在持续扩大 Rust 的应用范围，希望借此降低系统级软件中的内存安全漏洞。

当然，这并不意味着 C 和 C++ 已经“落伍”。TIOBE CEO 特别指出，C/C++ 社区同样意识到了内存安全的重要性，目前正通过新的语言标准、静态分析工具以及更加严格的开发规范，不断提升两门老牌语言的安全性。

![](https://mmbiz.qpic.cn/sz_mmbiz_png/S1iaf4GgGjEzqtTTcZQ6e2I5pkYk5eLI57mWtf3aibkaB6IiciaElUWjfH08khOf1icGXFMe68ZNGUjw5Mfic8AGeETXqdRKW9C5KRF7ZnXrm6sVo/640?wx_fmt=png&from=appmsg)

除了 Rust 的突破，本月榜单还有一些编程语言的变化值得关注：

*   相较于上个月，由于 Rust 的名次上升，Delphi/Object Pascal 和 Scratch 分别下滑了 1 名，位列第 11 名和第 12 名；
    
*   经历几个月出走 Top 20 榜单，Ruby 的本月排名终于上升，从上个月的 33 名一举跃升至第 20 名。
    

说完了编程语言排名的变化，再来看看 TIOBE 指数本身——今年恰好是 TIOBE 指数正式发布的 25 周年。

TIOBE CEO 坦言，过去 25 年里，他们收到过大量正面和负面的反馈，而其中最常见、也是最合理的一条意见就是：TIOBE 衡量的是语言的“流行度”，不是“哪门语言最好”。同一个项目，用 Python 原型快、用 Rust 性能稳、用 Java 生态全——没有标准答案，只有最适合的方案。

为了帮开发者解决“到底选哪个语言”的实际问题，在迎来 25 周年之际，TIOBE 也宣布将在下周推出一项全新的工具——Programming Language Flowchart（编程语言选择流程图）。与单纯按照热度排名不同，这份流程图希望帮助开发者根据项目需求、应用场景以及实际开发目标，选择真正适合自己的编程语言，而不是简单地“哪个排名高就学哪个”。
